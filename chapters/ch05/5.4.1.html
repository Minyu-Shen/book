5.4.1 SAC算法

为什么需要SAC

回顾前几节：5.1节的DQN将Q-learning与深度神经网络结合，实现了离策略学习，能通过经验回放缓冲区（5.1.1节）高效利用历史数据，但它只能处理离散动作空间。5.2节的PPO采用策略梯度方法，天然支持连续动作空间，训练也比较稳定，但它是on-policy算法，每批数据用完后就必须丢弃，样本效率较低。5.3节的DDPG和TD3虽然支持连续动作空间且是off-policy的，但它们使用确定性策略，探索能力依赖外部噪声，训练稳定性仍有不足。

在管理科学的实际应用中，许多决策变量天然是连续的：物流中的车辆调度速度、库存管理中的补货数量、金融中的投资组合权重、交通信号的绿灯时长等。对于这类问题，我们希望算法同时具备以下三个特性：

  • 支持连续动作空间，不需要将决策变量人为离散化；
  • 离策略学习，能够重复利用历史经验，提高样本效率；
  • 训练稳定，不容易因超参数设置不当而崩溃。

Soft Actor-Critic（SAC，Haarnoja et al., 2018）正是为了同时满足这三个需求而提出的。它的核心思想是在标准的强化学习目标中加入一个熵正则化项：不仅要最大化累积奖励，还要让策略尽可能保持随机性。这个看似简单的改动带来了深远的影响：它自然地鼓励探索，显著提升了训练的稳定性和鲁棒性。

SAC在管理科学中的优势：许多管理科学问题的仿真器运行成本高昂（例如，一次完整的交通仿真可能需要数分钟）。SAC的离策略特性使其能够充分利用每一条仿真数据，大幅减少所需的仿真次数。同时，它对超参数不敏感的特性，减少了研究人员的调参负担。

最大熵强化学习

标准强化学习的目标是最大化期望累积折扣奖励：

$$J_{\text{std}}(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t R_t\right]$$

SAC在此基础上增加了一个策略熵（policy entropy）奖励项，形成最大熵目标：

$$J(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t \Big(R_t + \alpha \,\mathcal{H}\big(\pi(\cdot \mid S_t)\big)\Big)\right] \tag{5.4-1}$$

其中 $\mathcal{H}\big(\pi(\cdot \mid s)\big)$ 是策略在状态 $s$ 下的熵。对连续动作空间下的策略分布 $\pi(\cdot \mid s)$，熵定义为：

$$\mathcal{H}\big(\pi(\cdot \mid s)\big) = -\int \pi(a \mid s) \log \pi(a \mid s) \, da = \mathbb{E}_{a \sim \pi(\cdot|s)}\big[-\log \pi(a \mid s)\big] \tag{5.4-2}$$

参数 $\alpha > 0$ 称为温度系数（temperature），它控制了熵项的权重，也就是探索与利用之间的平衡：

  • $\alpha$ 较大时，算法更倾向于保持策略的随机性（多探索）；
  • $\alpha$ 较小时，算法更倾向于追求高奖励（多利用）；
  • $\alpha \to 0$ 时，退化为标准的强化学习目标。
最大熵目标的直觉

想象一位基金经理需要在多个资产之间分配投资比例（连续动作）。最大熵目标要求他：在保证预期收益的前提下，尽可能分散投资。这样做的好处是：（1）避免过早押注于某个单一方案，在信息不充分时保持灵活性；（2）即使环境发生变化，分散的策略也更具鲁棒性；（3）学习过程中能持续探索不同的投资组合，不会因为偶然发现一个还不错的方案就停止尝试。

公式(5.4-1)可以等价地写成更紧凑的形式。将熵的定义代入，每一步的奖励变为 $R_t - \alpha \log \pi(A_t \mid S_t)$，即：

$$J(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t \Big(R_t - \alpha \log \pi(A_t \mid S_t)\Big)\right] \tag{5.4-3}$$

这个形式更便于后续推导：可以把 $-\alpha \log \pi(a \mid s)$ 看作一个与动作概率相关的额外奖励。策略越随机（概率越分散），$-\log \pi$ 越大，获得的额外奖励越多。

软贝尔曼方程

回顾2.1.3节的标准贝尔曼期望方程，状态值函数与动作值函数之间满足：

$$V^\pi(s) = \mathbb{E}_{a \sim \pi}\big[Q^\pi(s, a)\big], \quad Q^\pi(s, a) = R(s,a) + \gamma \, \mathbb{E}_{s' \sim P}\big[V^\pi(s')\big]$$

在最大熵框架下，这些方程被修改为软贝尔曼方程（soft Bellman equation）。关键变化在于：状态值函数不再是Q值的简单期望，而是要减去策略的对数概率（即加上熵奖励）：

$$V(s) = \mathbb{E}_{a \sim \pi}\Big[Q(s, a) - \alpha \log \pi(a \mid s)\Big] \tag{5.4-4}$$
$$Q(s, a) = R(s, a) + \gamma \, \mathbb{E}_{s' \sim P}\Big[V(s')\Big] \tag{5.4-5}$$

将公式(5.4-4)代入公式(5.4-5)，可以消去 $V$ 得到：

$$Q(s, a) = R(s,a) + \gamma \, \mathbb{E}_{s'}\Big[\mathbb{E}_{a' \sim \pi}\big[Q(s', a') - \alpha \log \pi(a' \mid s')\big]\Big] \tag{5.4-6}$$

对比标准贝尔曼方程，软贝尔曼方程唯一的不同是多了 $-\alpha \log \pi(a \mid s)$ 项。这使得在固定点处，Q值不仅反映了累积奖励，还反映了未来策略的随机程度。

软贝尔曼方程的含义

在标准贝尔曼方程中，$V(s)$ 等于在状态 $s$ 下按照策略行动所获得的期望Q值。在软贝尔曼方程中，$V(s)$ 还包含了策略的熵奖励。这意味着：如果两个策略获得相同的Q值，但其中一个更随机（熵更高），那么更随机的策略对应更高的 $V(s)$。由此可以导出最大熵框架下的最优策略形式：

$$\pi^*(a \mid s) \propto \exp\!\Big(\frac{1}{\alpha} Q^*(s, a)\Big) \tag{5.4-7}$$

这是一个玻尔兹曼分布（Boltzmann distribution），温度系数 $\alpha$ 控制分布的尖锐程度。$\alpha$ 越小，分布越集中于Q值最大的动作；$\alpha$ 越大，分布越均匀。

SAC的网络架构

SAC使用三组神经网络协同工作：

1. 两个Q网络 $Q_{\theta_1}$ 和 $Q_{\theta_2}$

输入为状态-动作对 $(s, a)$，输出一个标量Q值。使用两个Q网络并取较小值是为了缓解Q值过高估计问题，与5.3.1节TD3中的twin Q-networks思想一致（TD3首先提出了这一技巧，SAC借鉴并沿用）。在计算目标值和更新策略时，总是取两者的最小值：

$$Q_{\min}(s, a) = \min\big(Q_{\theta_1}(s, a),\; Q_{\theta_2}(s, a)\big) \tag{5.4-8}$$

2. 策略网络 $\pi_\phi$

输入为状态 $s$，输出一个高斯分布的均值 $\mu_\phi(s)$ 和标准差 $\sigma_\phi(s)$。与5.2.1节中PPO使用的高斯策略参数化方式类似，SAC也采用对角高斯分布，但额外加入了 $\tanh$ 压缩以保证动作有界。动作通过以下方式采样：

$$a = \tanh\!\big(\mu_\phi(s) + \sigma_\phi(s) \odot \epsilon\big), \quad \epsilon \sim \mathcal{N}(0, I) \tag{5.4-9}$$

这里的 $\tanh$ 将动作压缩到 $[-1, 1]$ 范围内，$\odot$ 表示逐元素相乘。这种采样方式使用了5.2.2节介绍的重参数化技巧：将随机性分离到外部噪声 $\epsilon$ 中，使得 $a$ 成为 $\phi$ 的确定性可微函数。为什么 SAC 必须这样做，下文将详细解释。

3. 目标Q网络 $Q_{\bar{\theta}_1}$ 和 $Q_{\bar{\theta}_2}$

这是两个Q网络的缓慢更新副本，用于计算TD目标值，以稳定训练过程。这与5.1.1节DQN中引入的目标网络思想相同：

$$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau) \bar{\theta}_i, \quad i = 1, 2 \tag{5.4-10}$$

其中 $\tau$ 通常取很小的值（如 $0.005$），使得目标网络缓慢跟踪主网络。

图 5.4-3　SAC 的网络架构。策略网络（绿色）输出高斯分布并采样动作；两个 Q 网络（黄色）评估动作价值；目标 Q 网络（红色虚线）通过软更新缓慢跟踪主网络，其最小值用于计算稳定的 TD 目标。

SAC的策略优化：为什么用重参数化

5.2.2节介绍了重参数化技巧的数学原理。现在来看 SAC 为什么需要它。

PPO 的梯度计算完全不涉及从 $\pi_\theta$ 采样，反向传播畅通无阻（参见5.2.2节）。5.3节的 DDPG 使用确定性策略 $a = \mu_\theta(s)$，优化目标 $\max_\theta \mathbb{E}_s[Q(s, \mu_\theta(s))]$，梯度由链式法则给出 $\nabla_a Q|_{a=\mu_\theta} \cdot \nabla_\theta \mu_\theta$，同样全程可微（探索噪声只在数据采集时加，不参与梯度计算）。

但 SAC 不同。SAC 和 DDPG 一样使用经验回放缓冲区，从中采样 $(s, a_{\text{buf}}, r, s')$ 来训练。Critic 的更新用的是缓冲区里的旧动作 $a_{\text{buf}}$：计算 TD 目标，最小化 $Q_\psi(s, a_{\text{buf}})$ 与目标值的误差。这在操作上和 DQN 的 Q 网络更新类似（都是用缓冲区数据做 TD 学习），但目标不同：DQN 学的是最优值函数 $Q^*$（用 $\max_{a'}$），SAC 的 Critic 学的是当前策略下的 $Q^\pi$（用 $\mathbb{E}_{a' \sim \pi_\theta}$，即策略评估）。无论如何，Critic 的更新不涉及对采样操作求导，梯度计算没有问题。

但Actor 的更新不同。SAC 的 Actor 优化目标是：

$$\max_\theta \;\mathbb{E}_{a \sim \pi_\theta(\cdot|s)}\!\left[Q_\psi(s, a) - \alpha \log \pi_\theta(a|s)\right]$$

这里的 $s$ 从缓冲区取，但 $a$ 不能用缓冲区里的旧动作——它必须是从当前策略 $\pi_\theta(\cdot|s)$ 重新采样的。原因是：我们要评估的是"当前策略在状态 $s$ 下的表现"，而不是"当初采集数据时旧策略选的那个动作的表现"。每次 Actor 梯度更新时，都要对每个状态 $s$ 重新采样一个新动作 $a \sim \pi_\theta(\cdot|s)$，送入 $Q_\psi(s, a)$ 评估。

问题就出在这里：梯度需要从 $Q_\psi(s, a)$ 经过 $a$ 回传到 $\theta$，而 $a$ 是从 $\pi_\theta$ 随机采出来的。采样操作本质上是随机数生成器，$\frac{\partial a}{\partial \theta}$ 没有定义，反向传播在这一步被阻断。

面对这个问题，有两种方案：

方案一：对数导数技巧（绕开问题）

SAC 的 Actor 目标中，核心困难在 $Q_\psi(s,a)$ 这一项：$Q_\psi$ 是一个独立的网络，不含参数 $\theta$，但它的输入 $a$ 是从 $\pi_\theta$ 中采样的。对数导数技巧可以绕开这个问题。以离散动作为例展示推导过程（连续情况将求和换成积分）：

$$\nabla_\theta \mathbb{E}_{a \sim \pi_\theta}\!\left[Q_\psi(s, a)\right] = \nabla_\theta \sum_a \pi_\theta(a|s)\, Q_\psi(s, a) = \sum_a \nabla_\theta \pi_\theta(a|s) \cdot Q_\psi(s, a)$$

关键一步：利用恒等式 $\nabla_\theta \pi_\theta(a|s) = \pi_\theta(a|s) \cdot \nabla_\theta \log \pi_\theta(a|s)$，乘以 $\frac{\pi_\theta}{\pi_\theta} = 1$：

$$= \sum_a \pi_\theta(a|s) \cdot Q_\psi(s, a) \cdot \nabla_\theta \log \pi_\theta(a|s) = \mathbb{E}_{a \sim \pi_\theta}\!\left[Q_\psi(s, a) \cdot \nabla_\theta \log \pi_\theta(a \mid s)\right]$$

结果中，$\nabla_\theta$ 只作用于 $\log \pi_\theta(a|s)$（在已采样的 $a$ 处对 $\theta$ 求导，确定性可微），$Q_\psi(s,a)$ 只作为标量系数乘在外面。这个恒等式正是策略梯度定理（3.3.1节）的数学基础，所有策略梯度方法都可以用它，包括 SAC。

至于 SAC 目标中的熵项 $-\alpha \log\pi_\theta(a|s)$：虽然它也在期望 $\mathbb{E}_{a \sim \pi_\theta}$ 里面，但 $\mathbb{E}_{a \sim \pi_\theta}[-\alpha \log\pi_\theta(a|s)] = \alpha H(\pi_\theta(\cdot|s))$ 就是策略的熵。对于高斯策略，熵有解析公式 $H = \frac{d}{2}(1+\log 2\pi) + \sum_i \log\sigma_{\theta,i}(s)$，只依赖 $\sigma_\theta$，不需要采样就能精确计算，$\nabla_\theta H$ 也可以直接求。而 $\mathbb{E}_{a \sim \pi_\theta}[Q_\psi(s,a)]$ 就没这么幸运——$Q_\psi$ 是一个任意的神经网络，这个期望没有解析形式，必须通过采样来估计。所以核心瓶颈确实在 $Q_\psi$ 项。

对数导数技巧 vs 重要性采样。这两者解决的是不同问题。对数导数技巧让梯度公式本身不需要穿过采样操作——即使是最原始的 REINFORCE（从 $\pi_\theta$ 在线采样、不复用数据）也不需要重参数化。PPO 的重要性采样是在此基础上额外做的一步，目的是让旧数据可以复用多轮更新，与避免对采样求导无关。

那么 SAC 为什么不直接用对数导数技巧？因为方差高。$Q_\psi(s,a)$ 只提供了一个标量评分，梯度信号只知道"这个 $a$ 的 $Q$ 值大还是小"，但不知道"$a$ 往哪个方向调整能让 $Q$ 更大"——$Q_\psi$ 对 $a$ 的导数 $\nabla_a Q_\psi$ 的信息被完全浪费了。既然 $Q_\psi$ 是一个可微的神经网络，$\nabla_a Q_\psi$ 可以免费计算，不利用它太可惜了。这就是方案二的动机。

方案二：重参数化技巧（解决问题）

重参数化提供了一种更干净的方案。把采样写成 $a = \mu_\theta(s) + \sigma_\theta(s) \cdot \varepsilon$，其中 $\varepsilon \sim \mathcal{N}(0,1)$ 与 $\theta$ 无关，SAC 的整个目标变成：

$$J(\theta) = \mathbb{E}_{\varepsilon \sim \mathcal{N}(0,1)}\!\left[Q_\psi\!\left(s,\, \mu_\theta + \sigma_\theta \varepsilon\right) - \alpha \log \pi_\theta\!\left(\mu_\theta + \sigma_\theta \varepsilon \mid s\right)\right]$$

现在期望是对 $\varepsilon$ 取的（不含 $\theta$），里面所有项——无论是 $Q_\psi$ 还是 $\log\pi_\theta$ 还是别的什么——都是 $\theta$ 的确定性可微函数。不需要逐项分析哪个有解析形式、哪个没有；统一采样 $\varepsilon$，计算整个表达式，一路反向传播即可。梯度沿 $Q_\psi(s,a) \to a \to \mu_\theta, \sigma_\theta \to \theta$ 回传，充分利用了 $\nabla_a Q_\psi$ 的信息，方差远低于对数导数技巧。这正是上一节介绍的重参数化技巧在 SAC 中的具体应用。

SAC算法

SAC的训练过程包含三个交替进行的更新步骤。每次从经验回放缓冲区中采样一个小批量 $(s, a, r, s')$，依次更新Q网络、策略网络和目标网络。

第一步：更新Q网络。首先计算TD目标值。用当前策略网络在下一状态 $s'$ 采样一个动作 $\tilde{a}'$，利用目标Q网络计算软Q值：

$$y = r + \gamma \Big(\min_{i=1,2} Q_{\bar{\theta}_i}(s', \tilde{a}') - \alpha \log \pi_\phi(\tilde{a}' \mid s')\Big), \quad \tilde{a}' \sim \pi_\phi(\cdot \mid s') \tag{5.4-11}$$

然后最小化每个Q网络的预测值与TD目标之间的均方误差：

$$L(\theta_i) = \mathbb{E}_{(s,a,r,s')}\left[\Big(Q_{\theta_i}(s, a) - y\Big)^2\right], \quad i = 1, 2 \tag{5.4-12}$$

第二步：更新策略网络。策略网络的目标是最大化期望的软Q值，即让策略选出的动作既有高Q值，又保持足够的随机性：

$$L(\phi) = \mathbb{E}_{s}\left[\mathbb{E}_{\tilde{a} \sim \pi_\phi}\Big[\alpha \log \pi_\phi(\tilde{a} \mid s) - Q_{\min}(s, \tilde{a})\Big]\right] \tag{5.4-13}$$

这里 $Q_{\min}(s, \tilde{a}) = \min(Q_{\theta_1}(s, \tilde{a}),\; Q_{\theta_2}(s, \tilde{a}))$。利用公式(5.4-9)的重参数化技巧，$\tilde{a}$ 关于 $\phi$ 可微，因此可以用标准的梯度下降来最小化 $L(\phi)$。

第三步：软更新目标网络。按照公式(5.4-10)缓慢更新目标网络参数。

Soft Actor-Critic (SAC) 算法
  1. 初始化策略网络参数 $\phi$，Q网络参数 $\theta_1, \theta_2$，令目标网络 $\bar{\theta}_i \leftarrow \theta_i$
  2. 初始化经验回放缓冲区 $\mathcal{D} \leftarrow \emptyset$
  3. 对每个时间步 $t = 0, 1, 2, \ldots$：
    1. 观察状态 $s_t$，从策略 $a_t \sim \pi_\phi(\cdot \mid s_t)$ 采样动作
    2. 执行 $a_t$，观察奖励 $r_t$ 和下一状态 $s_{t+1}$，将 $(s_t, a_t, r_t, s_{t+1})$ 存入 $\mathcal{D}$
    3. 从 $\mathcal{D}$ 中随机采样小批量 $\{(s, a, r, s')\}$
    4. 更新Q网络：对每个样本，用公式(5.4-11)计算TD目标 $y$，用公式(5.4-12)更新 $\theta_1, \theta_2$
    5. 更新策略网络：用公式(5.4-13)更新 $\phi$
    6. 软更新目标网络：$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau)\bar{\theta}_i$，$i = 1, 2$
SAC是离策略算法：步骤(c)中从缓冲区采样的数据可以来自任何历史策略。步骤(a)中的采样使用重参数化技巧（公式5.4-9）。

自动温度调节

温度系数 $\alpha$ 的选择对SAC的性能影响很大。手动调节 $\alpha$ 既费时又不可靠，因为最优的 $\alpha$ 值会随训练进程而变化（初期需要大量探索，后期应减少随机性）。SAC的改进版本（Haarnoja et al., 2018b）提出了一种自动温度调节机制：将 $\alpha$ 也作为一个可学习的参数，通过优化以下目标来自动调整：

$$L(\alpha) = \mathbb{E}_{a \sim \pi_\phi}\Big[-\alpha \big(\log \pi_\phi(a \mid s) + \bar{\mathcal{H}}\big)\Big] \tag{5.4-14}$$

其中 $\bar{\mathcal{H}}$ 是目标熵，通常设为 $-\dim(\mathcal{A})$（即负的动作空间维度）。这个约束的含义是：当策略的熵低于目标值时，自动增大 $\alpha$ 以鼓励更多探索；当熵高于目标值时，自动减小 $\alpha$ 以更多地利用已有知识。

在实践中，自动温度调节几乎是标配。它使得SAC成为一个超参数设置非常方便的算法：研究人员只需指定目标熵（通常用默认值即可），无需手动调节 $\alpha$。
