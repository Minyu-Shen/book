5.1 Deep Q-Network

在第一章中，我们学习了Q-learning算法（3.1.2节）：通过反复与环境交互，逐步更新Q值表格，最终收敛到最优动作值函数 $Q^*(s,a)$。在网约车司机的例子中，3个状态、每个状态2个动作，总共只需要维护6个Q值，用一张小表格就能轻松存储。然而，实际的管理科学问题远非如此简单。本节将介绍如何用神经网络替代Q表格，引出深度强化学习的第一个里程碑算法：Deep Q-Network（DQN）。

从Q表到Q网络

回顾3.1.2节中的Q-learning：我们维护一张Q表（Q-table），表的行是状态 $s$，列是动作 $a$，每个单元格存储 $Q(s,a)$ 的估计值。每次经历一个转移 $(S_t, A_t, R_t, S_{t+1})$，就更新对应单元格的值。这种方法在状态空间 $\mathcal{S}$ 和动作空间 $\mathcal{A}$ 都很小时非常有效。

但在许多管理科学问题中，状态空间可能非常庞大，甚至是连续的：

  • 网约车调度：司机的位置（经纬度）、当前时刻、周边订单分布、道路拥堵状况……状态维度可达数十甚至上百维
  • 库存管理：多产品库存水平、在途订单数量、需求预测值、供应商交货时间……状态空间随产品数量指数增长
  • 交通信号控制：各车道排队长度、车辆速度分布、上下游路口的信号相位……连续且高维

以网约车为例。第一章中我们将问题简化为3个区域（市中心、郊区、机场）。但如果将城市划分为100个区域，同时考虑24个小时的时段，状态空间就扩展到 $100 \times 24 = 2400$ 个状态。若进一步加入需求水平（高/中/低3档），状态数变为 $100 \times 24 \times 3 = 7200$。每增加一个维度，状态数量就会成倍增长。这就是所谓的维数灾难（curse of dimensionality）：Q表格的大小随状态维度指数增长，存储和学习都变得不可行。

解决方案是用一个参数化的函数来近似 $Q^*(s,a)$，而不是逐个存储。在第四章中，我们已经知道神经网络是一种强大的函数近似器。自然的想法是：用一个参数为 $\theta$ 的神经网络 $Q(s,a;\theta)$ 来代替Q表格。

Q网络（Q-Network）

用神经网络 $Q(s,a;\theta)$ 近似最优动作值函数 $Q^*(s,a)$。网络的输入是状态 $s$，输出是每个动作对应的Q值估计。对于有 $|\mathcal{A}|$ 个动作的问题，网络的输出层有 $|\mathcal{A}|$ 个节点，第 $i$ 个输出对应动作 $a_i$ 的Q值。

这种设计的优势在于：只需一次前向传播，就能同时得到所有动作的Q值，从而高效地计算 $\arg\max_a Q(s,a;\theta)$。

图 5.1-1   Q网络结构示意：输入状态向量 $s$，输出所有动作的Q值估计。网络参数 $\theta$ 通过训练学习。

对比Q表和Q网络：

Q表（表格方法）Q网络（函数近似）
存储方式$|\mathcal{S}| \times |\mathcal{A}|$ 大小的表格参数量为 $|\theta|$ 的神经网络
泛化能力没有泛化：不同状态的Q值完全独立相似状态共享参数，自动泛化
适用范围小型离散状态空间大型离散或连续状态空间
更新方式直接修改表格中一个单元格通过梯度下降更新所有参数 $\theta$

泛化能力是Q网络的关键优势。在表格方法中，在状态 $s_1$ 处获得的经验完全不会帮助估计状态 $s_2$ 的Q值，即使 $s_1$ 和 $s_2$ 非常相似。而神经网络的参数是所有状态共享的，在某个状态处的训练会影响相似状态的输出，从而加速学习。例如在网约车调度中，如果网络学到了"高峰时段在需求密集区域接单Q值高"这一模式，那么对所有高峰时段的密集区域，网络都会给出较高的接单Q值，即使其中某些具体的（区域，时段）组合从未被访问过。

损失函数与训练目标

有了Q网络 $Q(s,a;\theta)$，下一个问题是：如何训练它？回顾3.1.2节的Q-learning更新规则：

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_t + \gamma \max_{a} Q(S_{t+1}, a) - Q(S_t, A_t) \right]$$

表格Q-learning直接修改表格中的一个单元格。对于神经网络，我们不能直接修改某个输出值，而是需要定义一个损失函数，然后通过梯度下降来更新参数 $\theta$。

从贝尔曼最优方程出发（参见2.2.2节），最优动作值函数满足：

$$Q^*(s, a) = \mathbb{E}\left[R + \gamma \max_{a'} Q^*(S', a') \;\middle|\; S=s, A=a\right] \tag{5.1-1}$$

如果我们的Q网络已经完美逼近了 $Q^*$，那么对任意 $(s,a)$，将 $Q(s,a;\theta)$ 代入上式两端应该相等。因此，可以将训练目标定义为最小化两端之差的平方：

DQN 损失函数（初步版本）
$$L(\theta) = \mathbb{E}_{(S,A,R,S')}\left[\left(\underbrace{R + \gamma \max_{a'} Q(S', a'; \theta)}_{\text{TD目标（target）}} - \underbrace{Q(S, A; \theta)}_{\text{当前估计}}\right)^2\right] \tag{5.1-2}$$

其中期望是对经验转移 $(S,A,R,S')$ 取的。TD目标 $R + \gamma \max_{a'} Q(S', a'; \theta)$ 是我们希望Q网络输出逼近的值。

对每个样本 $(s_t, a_t, r_t, s_{t+1})$，该损失的梯度为：

$$\nabla_\theta L = -2\left(r_t + \gamma \max_{a'} Q(s_{t+1}, a'; \theta) - Q(s_t, a_t; \theta)\right) \nabla_\theta Q(s_t, a_t; \theta) \tag{5.1-3}$$

看起来很简单：用梯度下降不断减小TD误差的平方即可。然而，如果直接这样做，训练往往会不稳定甚至发散。原因主要有两个：

问题一：目标不稳定。在监督学习中，训练标签（如图像的类别）是固定的。但在公式(5.1-2)中，TD目标 $R + \gamma \max_{a'} Q(S', a'; \theta)$ 本身依赖于参数 $\theta$。每当 $\theta$ 更新一次，目标也随之变化。这就像射击一个不断移动的靶子，参数可能在追逐不断变化的目标中来回振荡，无法收敛。

问题二：样本相关性。智能体在环境中依次采集的数据 $(s_0, a_0, r_0, s_1), (s_1, a_1, r_1, s_2), \ldots$ 在时间上高度相关。前后相邻的样本共享状态 $s_t$，且来自相似的环境区域。而神经网络的随机梯度下降要求训练样本之间近似独立。如果用相关样本训练，网络容易过度拟合最近的经验，忘记之前学到的知识。

为了解决这两个问题，Mnih等人（2015）在发表于 Nature 的经典论文中提出了两个关键技巧：经验回放和目标网络。

经验回放

经验回放（experience replay）的思想非常直观：把智能体与环境交互产生的转移数据存入一个缓冲区（replay buffer），训练时从缓冲区中随机抽样一小批（mini-batch）数据来计算梯度，而不是用最新的一条数据。

经验回放缓冲区（Replay Buffer）

维护一个固定容量为 $N$ 的缓冲区 $\mathcal{D}$，存储智能体的历史转移：

$$\mathcal{D} = \{(s_i, a_i, r_i, s_i')\}_{i=1}^{N} \tag{5.1-4}$$

每当智能体产生一条新的转移，将其存入缓冲区（如果缓冲区已满，则覆盖最旧的数据）。训练时，从 $\mathcal{D}$ 中均匀随机抽取大小为 $B$ 的mini-batch来计算梯度。

下面的表格展示了经验回放的工作过程。假设缓冲区容量 $N=6$，mini-batch大小 $B=3$：

1$s_0$$a_2$$+3$$s_1$第1步

图 5.1-2   经验回放示意。缓冲区存储了6条转移数据，黄色高亮行为随机抽取的mini-batch（编号2、4、6），用于一次梯度更新。

经验回放解决了前面提到的样本相关性问题。由于mini-batch是从不同时间步的数据中随机抽取的，样本之间的时序相关性被大大打破。此外，同一条经验可以被多次抽到、用于多次更新，提高了数据利用效率。

经验回放还有一个不太显然的好处：由于缓冲区中保存了较早时期（可能来自不同策略）的数据，Q网络实际上是在多种行为策略产生的混合数据上训练的。这与Q-learning的离策略（off-policy）特性天然契合。如果用的是SARSA这样的在策略算法，经验回放就不太适用了，因为旧数据不是由当前策略产生的。

目标网络

目标网络（target network）解决的是目标不稳定问题。思路也很简单：用一个独立的网络来计算TD目标，使目标在一段时间内保持不变。

目标网络（Target Network）

维护两个结构相同的网络：

  • 在线网络（online network）$Q(s,a;\theta)$：用于选择动作，参数每步更新
  • 目标网络（target network）$Q(s,a;\theta^-)$：用于计算TD目标，参数每隔 $C$ 步才从在线网络复制一次

损失函数变为：

$$L(\theta) = \mathbb{E}_{(S,A,R,S') \sim \mathcal{D}}\left[\left(R + \gamma \max_{a'} Q(S', a'; \theta^-) - Q(S, A; \theta)\right)^2\right] \tag{5.1-5}$$

其中 $\theta^-$ 在两次同步之间保持不变，因此TD目标不会随每次梯度更新而变化。

具体做法是：每隔 $C$ 步（如 $C=1000$），执行一次 $\theta^- \leftarrow \theta$，将在线网络的参数完整复制给目标网络。在两次同步之间的1000步中，目标网络的参数 $\theta^-$ 保持冻结，TD目标也因此保持稳定。这就像把移动的靶子暂时固定住，让射手有足够的时间瞄准。当射手（在线网络）已经对准当前靶子位置，再把靶子移动到新位置（同步参数），如此循环。

图 5.1-3   目标网络的参数同步机制。每隔 $C$ 步，将在线网络参数 $\theta$ 复制给目标网络 $\theta^-$，其余时间 $\theta^-$ 保持不变。
补充阅读：软更新（Soft Update）

除了每隔 $C$ 步硬复制参数，另一种常见做法是软更新（soft update），也称为 Polyak 平均：每一步都用一个很小的比例 $\tau$（如 $\tau=0.005$）将在线网络参数混入目标网络：

$$\theta^- \leftarrow \tau\,\theta + (1-\tau)\,\theta^- \tag{5.1-6}$$

软更新使目标网络参数平滑地跟随在线网络，避免了硬复制时的突变。这种方式在后续算法（如5.3节的SAC）中被广泛采用。

DQN算法

将Q网络、经验回放和目标网络三者结合起来，就得到了完整的DQN算法。下面给出算法的伪代码：

算法：Deep Q-Network（DQN）
  1. 初始化在线网络参数 $\theta$（随机初始化），目标网络参数 $\theta^- \leftarrow \theta$
  2. 初始化经验回放缓冲区 $\mathcal{D}$，容量为 $N$
  3. 对每个回合 $e = 1, 2, \ldots$：
    1. 初始化状态 $S_0$
    2. 对每一步 $t = 0, 1, 2, \ldots$：
      1. 用 $\varepsilon$-greedy 选择动作：以 $\varepsilon$ 的概率随机选择，否则 $A_t = \arg\max_a Q(S_t, a; \theta)$
      2. 执行 $A_t$，观察奖励 $R_t$ 和下一状态 $S_{t+1}$
      3. 将转移 $(S_t, A_t, R_t, S_{t+1})$ 存入 $\mathcal{D}$
      4. 从 $\mathcal{D}$ 中随机抽取大小为 $B$ 的mini-batch $\{(s_j, a_j, r_j, s_j')\}_{j=1}^{B}$
      5. 计算TD目标：$y_j = r_j + \gamma \max_{a'} Q(s_j', a'; \theta^-)$
      6. 对损失 $\frac{1}{B}\sum_{j=1}^{B}(y_j - Q(s_j, a_j; \theta))^2$ 执行一步梯度下降，更新 $\theta$
      7. 每隔 $C$ 步：$\theta^- \leftarrow \theta$
典型超参数：缓冲区容量 $N=10^5 \sim 10^6$，mini-batch大小 $B=32$，目标网络更新频率 $C=1000$，折扣因子 $\gamma=0.99$，探索率 $\varepsilon$ 从1.0线性衰减至0.01。

与表格Q-learning（3.1.2节）对比，DQN的核心结构可以简洁地概括为：

DQN = Q-learning + 神经网络 + 经验回放 + 目标网络

例子：网约车调度的DQN

让我们将网约车司机的例子扩展到一个稍大的场景，来看DQN如何工作。

在第一章中，网约车司机只需考虑3个区域和1种时段。现在假设城市仍然分为3个区域（市中心 $c$、郊区 $b$、机场 $a$），但引入4个时段（早高峰、白天、晚高峰、夜间），用整数 $0,1,2,3$ 表示。状态定义为（区域, 时段）的组合，状态空间大小为 $|\mathcal{S}| = 3 \times 4 = 12$。动作仍为2个：接单或空驶到另一个区域。

不同时段下，接单奖励和转移概率都不同。例如：

  • 早高峰（时段0）：市中心需求旺盛，接单奖励高（$r=12$）；机场需求一般（$r=8$）
  • 夜间（时段3）：市中心接单奖励降至 $r=4$，但机场有红眼航班，奖励升至 $r=16$

奖励公式可写为：

$$R(s, a) = R\big((\text{zone}, \text{period}),\; a\big) \tag{5.1-7}$$

其中具体的奖励值由区域和时段共同决定，详见下表中的部分示例：

区域时段接单奖励空驶奖励接单后转移概率
市中心 $c$0（早高峰）$12$$-4$$c$: 0.5, $b$: 0.3, $a$: 0.2
市中心 $c$3（夜间）$4$$-4$$c$: 0.3, $b$: 0.5, $a$: 0.2
机场 $a$0（早高峰）$8$$-5$$c$: 0.6, $b$: 0.4
机场 $a$3（夜间）$16$$-5$$c$: 0.7, $b$: 0.3

虽然12个状态仍然可以用表格方法处理，但这个例子帮助我们理解DQN的工作方式。状态被表示为一个特征向量：例如用独热编码（one-hot encoding）表示区域（3维）和时段（4维），拼接后得到7维输入向量。Q网络接收这个7维向量，输出2个动作的Q值。

当状态维度进一步增加时（如100个区域 + 24个时段 + 需求水平），表格方法需要 $100 \times 24 \times 3 \times 2 = 14400$ 个Q值条目，而Q网络的参数量可以保持在合理范围，并且能够在相似状态之间泛化。

从DQN到Double DQN

DQN虽然效果显著，但存在一个系统性的问题：Q值过高估计（overestimation）。

问题出在TD目标中的 $\max$ 运算。考虑公式(5.1-5)中的TD目标：

$$y = R + \gamma \max_{a'} Q(S', a'; \theta^-)$$

$\max$ 运算符同时承担了两个角色：选择哪个动作最好，以及估计该动作的Q值。当Q值的估计存在噪声时（训练初期必然如此），$\max$ 倾向于选出那个被高估的动作。这就像从一组有误差的成绩中选最高分，选出来的往往比真实最高分偏高。这种偏差会在训练过程中不断积累，导致所有Q值被系统性地高估。

推导：为什么 max 会导致过高估计

假设真实Q值为 $Q^*(s',a_1) = Q^*(s',a_2) = V$，但估计值含有均值为零的噪声：$\hat{Q}(s',a_i) = V + \epsilon_i$，其中 $\epsilon_i$ 独立且 $\mathbb{E}[\epsilon_i]=0$。则：

$$\mathbb{E}\left[\max(\hat{Q}(s',a_1), \hat{Q}(s',a_2))\right] = \mathbb{E}\left[\max(V+\epsilon_1, V+\epsilon_2)\right] = V + \mathbb{E}\left[\max(\epsilon_1, \epsilon_2)\right] \geq V$$

最后一个不等式严格成立（只要 $\epsilon_i$ 不恒为零），因为两个均值为零的随机变量的最大值的期望严格大于零。动作越多，过高估计越严重。

Double DQN（van Hasselt et al., 2016）的解决方案很巧妙：将动作选择和Q值估计分开，用两个不同的网络分别完成。具体地，用在线网络 $\theta$ 来选择动作，用目标网络 $\theta^-$ 来估计该动作的Q值：

Double DQN 的 TD 目标
$$y^{\text{DDQN}} = R + \gamma\, Q\!\left(S',\; \underbrace{\arg\max_{a'} Q(S', a'; \theta)}_{\text{在线网络选动作}}\;;\; \underbrace{\theta^-}_{\text{目标网络估值}}\right) \tag{5.1-8}$$

对比DQN的TD目标 $y = R + \gamma \max_{a'} Q(S', a'; \theta^-)$，Double DQN的改动很小（只需将动作选择从 $\theta^-$ 改为 $\theta$），但能显著减少过高估计。直觉上，即使在线网络由于噪声选择了一个被高估的动作，目标网络对该动作的估值通常不会同样偏高（因为两个网络的噪声不同），从而纠正了偏差。

在实现层面，Double DQN只需修改DQN算法第2(b)(v)步中计算TD目标的方式，其余完全相同。这种微小的修改带来了稳定的性能提升，因此在实践中，Double DQN已经基本取代了原始DQN。

Dueling DQN：分离状态价值与动作优势

标准DQN的网络直接输出每个动作的 $Q(s,a)$。但在很多状态下，不同动作的Q值差异其实很小——真正重要的是"这个状态本身好不好"（状态价值 $V(s)$），而非"这个动作比平均水平好多少"（优势 $A(s,a)$）。

Dueling DQN（Wang et al., 2016）将网络的输出层拆分为两个分支：

Dueling 架构
$$Q(s, a;\, \theta) = V(s;\, \theta_V) + \left[A(s, a;\, \theta_A) - \frac{1}{|\mathcal{A}|}\sum_{a'} A(s, a';\, \theta_A)\right] \tag{5.1-9}$$

$V$ 分支输出一个标量（状态价值），$A$ 分支输出 $|\mathcal{A}|$ 个值（每个动作的优势）。减去均值是为了可辨识性：保证 $V$ 分支确实学到状态价值，而非被 $A$ 分支吸收。

两个分支共享前面的特征提取层，只在最后一两层分叉。这个架构的好处是：在很多状态下（特别是所有动作的Q值都差不多时），网络可以先把 $V(s)$ 学准，不必通过每个动作单独学习。这在动作空间较大时尤其有效。

Dueling DQN 只改变了网络结构，不改变训练流程——仍然使用经验回放、目标网络和（通常搭配）Double DQN 的 TD 目标。在实现上只需将网络最后一层从直接输出Q值改为分支输出再合并。

优先经验回放（Prioritized Experience Replay）

标准DQN的经验回放从缓冲区中均匀随机抽取样本。但直觉上，有些经验比其他经验更"值得学习"——那些TD误差较大的样本意味着当前Q网络的预测与实际差距很大，包含更多可学习的信息。

优先经验回放（Schaul et al., 2016）正是基于这个直觉：按TD误差的大小为每个样本分配采样优先级。TD误差越大，被采样到的概率越高：

优先采样概率
$$P(i) = \frac{p_i^\alpha}{\sum_k p_k^\alpha} \tag{5.1-10}$$

其中 $p_i = |\delta_i| + \epsilon$（TD误差的绝对值加一个小常数 $\epsilon$ 防止优先级为零），$\alpha \in [0, 1]$ 控制优先化的程度：$\alpha = 0$ 退化为均匀采样，$\alpha = 1$ 完全按优先级采样。

优先采样改变了样本的分布，可能引入偏差。为此需要用重要性采样权重来修正：

$$w_i = \left(\frac{1}{N \cdot P(i)}\right)^\beta \tag{5.1-11}$$

其中 $\beta$ 从一个较小的值线性递增到 $1$（训练结束时完全修正偏差）。每个样本的梯度乘以 $w_i$ 后再更新网络。

在实现上，优先经验回放通常用 sum-tree 数据结构高效地按优先级采样，时间复杂度为 $O(\log N)$。

实践中的组合

上述三项改进（Double DQN、Dueling 架构、优先经验回放）相互独立，可以自由组合。实践中通常同时启用全部三项，这个组合有时被称为 "D3QN"。Rainbow（Hessel et al., 2018）则进一步整合了六项 DQN 改进，证明这些改进具有互补性。对于管理科学中的离散决策问题，至少启用 Double DQN 和 Dueling 架构是推荐做法，因为代码改动极小但收益显著。

例子：库存管理的DQN实现

为了展示 DQN 在管理科学问题中的完整实现流程，这里用一个简单的库存管理问题作为例子。一个零售商每天需要决定订货量，面对随机需求，目标是最小化持有成本和缺货损失。

  • 状态：当前库存水平 $s \in \{0, 1, \ldots, 30\}$
  • 动作：订货量 $a \in \{0, 5, 10, 15, 20\}$（5个离散档位）
  • 需求：每天随机需求 $D \sim \text{Poisson}(8)$
  • 状态转移：$s' = \min(s + a - D,\; 30)^+$（库存上限30，不允许为负）
  • 奖励：$R = \text{销售收入} - \text{持有成本} - \text{缺货惩罚} - \text{订货成本}$
$$R = 10 \cdot \min(s, D) - 1 \cdot (s + a - D)^+ - 5 \cdot (D - s)^+ - 2 \cdot a \tag{5.1-12}$$ 
import torch
import torch.nn as nn
import numpy as np
from collections import deque

# --- 环境 ---
class InventoryEnv:
    def __init__(self, max_inv=30, mean_demand=8):
        self.max_inv = max_inv
        self.mean_demand = mean_demand
        self.state = max_inv // 2

    def reset(self):
        self.state = np.random.randint(0, self.max_inv + 1)
        return self.state

    def step(self, action_idx):
        order = [0, 5, 10, 15, 20][action_idx]
        demand = np.random.poisson(self.mean_demand)
        sold = min(self.state, demand)
        revenue = 10 * sold
        holding = 1 * max(self.state + order - demand, 0)
        stockout = 5 * max(demand - self.state, 0)
        order_cost = 2 * order
        reward = revenue - holding - stockout - order_cost
        self.state = min(max(self.state + order - demand, 0), self.max_inv)
        return self.state, reward

# --- Q网络（Dueling架构）---
class DuelingQNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.shared = nn.Sequential(nn.Linear(1, 64), nn.ReLU(), nn.Linear(64, 64), nn.ReLU())
        self.val = nn.Linear(64, 1)         # V(s) 分支
        self.adv = nn.Linear(64, 5)         # A(s,a) 分支

    def forward(self, x):
        h = self.shared(x)
        v = self.val(h)                     # (batch, 1)
        a = self.adv(h)                     # (batch, 5)
        return v + a - a.mean(dim=1, keepdim=True)

# --- 训练 ---
env = InventoryEnv()
q_net = DuelingQNet()
target_net = DuelingQNet()
target_net.load_state_dict(q_net.state_dict())
optimizer = torch.optim.Adam(q_net.parameters(), lr=1e-3)
buf = deque(maxlen=5000)
gamma, eps, batch_size = 0.99, 1.0, 64

for episode in range(500):
    s = env.reset()
    ep_reward = 0
    for _ in range(200):
        # ε-greedy
        if np.random.random() < eps:
            a = np.random.randint(5)
        else:
            with torch.no_grad():
                q = q_net(torch.tensor([[s / 30.0]], dtype=torch.float32))
            a = q.argmax().item()
        s2, r = env.step(a)
        buf.append((s, a, r, s2))
        ep_reward += r
        s = s2

        # 训练
        if len(buf) >= batch_size:
            batch = [buf[i] for i in np.random.choice(len(buf), batch_size)]
            ss = torch.tensor([[b[0]/30.0] for b in batch], dtype=torch.float32)
            aa = torch.tensor([b[1] for b in batch], dtype=torch.long)
            rr = torch.tensor([b[2] for b in batch], dtype=torch.float32)
            ss2 = torch.tensor([[b[3]/30.0] for b in batch], dtype=torch.float32)
            # Double DQN目标
            with torch.no_grad():
                best_a = q_net(ss2).argmax(dim=1)           # 在线网络选动作
                q_target = target_net(ss2).gather(1, best_a.unsqueeze(1)).squeeze()
            target = rr + gamma * q_target
            q_pred = q_net(ss).gather(1, aa.unsqueeze(1)).squeeze()
            loss = nn.functional.mse_loss(q_pred, target)
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

    eps = max(0.05, eps * 0.99)
    if episode % 10 == 0:
        target_net.load_state_dict(q_net.state_dict())
    if episode % 50 == 0:
        print(f"Episode {episode}, reward={ep_reward:.0f}, ε={eps:.2f}")

# 查看学到的策略
print("\n学到的补货策略：")
for s in [0, 5, 10, 15, 20, 25, 30]:
    with torch.no_grad():
        q = q_net(torch.tensor([[s / 30.0]], dtype=torch.float32))
    action = [0, 5, 10, 15, 20][q.argmax().item()]
    print(f"  库存={s:2d} → 订货 {action}")

代码中同时使用了 Dueling 架构和 Double DQN 目标。可以观察到，学到的策略符合直觉：库存低时大量订货，库存充足时少订或不订。

小结

本节从表格Q-learning出发，逐步引入了神经网络函数近似、经验回放和目标网络三个核心组件，构建了DQN算法，并介绍了三项关键改进：

改进解决的问题核心思想
Double DQN$\max$ 导致的Q值过高估计用在线网络选动作、目标网络估值，分离两个角色
Dueling DQN无法高效学习状态本身的价值将Q网络拆分为 $V$ 和 $A$ 两个分支
优先经验回放均匀采样浪费低信息量的样本按TD误差大小分配采样优先级

这三项改进相互独立，可以自由组合。在管理科学的离散决策问题中（如库存管理、调度决策），推荐至少启用 Double DQN + Dueling 架构。

DQN 的局限性在于：它只适用于离散动作空间，因为需要对所有动作取 $\max$。对于连续动作空间（如调整价格、分配资源比例），需要其他方法——5.3节的 TD3 和5.4节的 SAC 正是为此而生。
