5.1.1 DQN基本算法

在第三章中，我们学习了Q-learning算法（3.2.2节）：通过反复与环境交互，逐步更新Q值表格，最终收敛到最优动作值函数 $Q^*(s,a)$。在网约车司机的例子中，3个状态、每个状态2个动作，总共只需要维护6个Q值，用一张小表格就能轻松存储。然而，实际的管理科学问题远非如此简单。本节将介绍如何用神经网络替代Q表格，引出深度强化学习的第一个里程碑算法：Deep Q-Network（DQN）。

从Q表到Q网络

回顾3.2.2节中的Q-learning：我们维护一张Q表（Q-table），表的行是状态 $s$，列是动作 $a$，每个单元格存储 $Q(s,a)$ 的估计值。每次经历一个转移 $(S_t, A_t, R_t, S_{t+1})$，就更新对应单元格的值。这种方法在状态空间 $\mathcal{S}$ 和动作空间 $\mathcal{A}$ 都很小时非常有效。

但在许多管理科学问题中，状态空间可能非常庞大，甚至是连续的：

  • 网约车调度：司机的位置（经纬度）、当前时刻、周边订单分布、道路拥堵状况……状态维度可达数十甚至上百维
  • 库存管理：多产品库存水平、在途订单数量、需求预测值、供应商交货时间……状态空间随产品数量指数增长
  • 交通信号控制：各车道排队长度、车辆速度分布、上下游路口的信号相位……连续且高维

以网约车为例。第一章中我们将问题简化为3个区域（市中心、郊区、机场）。但如果将城市划分为100个区域，同时考虑24个小时的时段，状态空间就扩展到 $100 \times 24 = 2400$ 个状态。若进一步加入需求水平（高/中/低3档），状态数变为 $100 \times 24 \times 3 = 7200$。每增加一个维度，状态数量就会成倍增长。这就是所谓的维数灾难（curse of dimensionality）：Q表格的大小随状态维度指数增长，存储和学习都变得不可行。

解决方案是用一个参数化的函数来近似 $Q^*(s,a)$，而不是逐个存储。在第四章中，我们已经知道神经网络是一种强大的函数近似器。自然的想法是：用一个参数为 $\theta$ 的神经网络来代替Q表格。

从数学上看，$Q^*(s,a)$ 是关于 $s$ 和 $a$ 的函数，用神经网络来近似它有两种自然的架构（见图 5.1-1）：

  • 架构一：把 $(s, a)$ 一起作为网络输入，输出一个标量 $Q$ 值。每次想比较不同动作的优劣，需要对每个候选动作分别做一次前向传播。
  • 架构二：只输入状态 $s$，让网络一次性输出所有动作的 $Q$ 值。只需一次前向传播就能得到 $\arg\max_a Q(s,a;\theta)$，效率高得多。DQN 采用这种架构。
Q网络（Q-Network）

用神经网络近似最优动作值函数 $Q^*(s,a)$。DQN 采用架构二：网络输入是状态 $s$，输出层有 $|\mathcal{A}|$ 个节点，第 $i$ 个输出对应动作 $a_i$ 的Q值估计。只需一次前向传播，就能同时得到所有动作的Q值，从而高效地计算 $\arg\max_a Q(s,a;\theta)$。

架构一：输入 $(s, a)$，输出标量 $Q$
架构二：输入 $s$，输出 $|\mathcal{A}|$ 个 $Q$ 值
图 5.1-1   Q网络的两种架构。架构一将状态 $s$ 和动作 $a$ 拼接输入，每次只输出一个 $Q$ 值；架构二只输入 $s$，一次输出所有 $|\mathcal{A}|$ 个动作的Q值。动画展示前向传播过程：信号从输入层逐层传递到输出层。DQN 通常采用架构二，因为只需一次前向传播即可比较所有动作。

对比Q表和Q网络：

Q表（表格方法）Q网络（函数近似）
存储方式$|\mathcal{S}| \times |\mathcal{A}|$ 大小的表格参数量为 $|\theta|$ 的神经网络
泛化能力没有泛化：不同状态的Q值完全独立相似状态共享参数，自动泛化
适用范围小型离散状态空间大型离散或连续状态空间
更新方式直接修改表格中一个单元格通过梯度下降更新所有参数 $\theta$

泛化能力是Q网络的关键优势。在表格方法中，在状态 $s_1$ 处获得的经验完全不会帮助估计状态 $s_2$ 的Q值，即使 $s_1$ 和 $s_2$ 非常相似。而神经网络的参数是所有状态共享的，在某个状态处的训练会影响相似状态的输出，从而加速学习。例如在网约车调度中，如果网络学到了"高峰时段在需求密集区域接单Q值高"这一模式，那么对所有高峰时段的密集区域，网络都会给出较高的接单Q值，即使其中某些具体的（区域，时段）组合从未被访问过。

探索与数据采集：$\varepsilon$-greedy

有了Q网络之后，智能体如何用它来选择动作？最直接的做法是每次都选Q值最高的动作（贪心策略），但这样可能永远不会尝试那些暂时Q值较低但实际上更优的动作。这正是3.2.1节讨论的探索-利用困境。DQN 沿用了表格Q-learning中的 $\varepsilon$-greedy 策略（3.2.1节式(3.2-2)）来平衡二者：

$$A_t = \begin{cases} \text{随机动作}, & \text{以概率 } \varepsilon \\ \arg\max_a Q(S_t, a; \theta), & \text{以概率 } 1 - \varepsilon \end{cases} \tag{5.1-1}$$

与表格方法不同的是，DQN 通常让 $\varepsilon$ 随训练逐步衰减：训练初期 $\varepsilon$ 较大（如1.0），智能体大量随机探索以积累经验；随着Q网络逐渐学到好的估计，$\varepsilon$ 线性衰减到一个较小的值（如0.01），转为以利用为主。在网约车的例子中，这意味着一开始司机对各区域各时段的收益规律不了解，需要到处跑来积累经验；等熟悉之后，就主要按经验选择最优路线，只偶尔做少量探索。

注意，$\varepsilon$-greedy 使得智能体的行为策略（用于收集数据的策略）和目标策略（Q网络正在逼近的贪心策略 $\arg\max_a Q^*$）并不相同。这正是 Q-learning 的 off-policy 特性（3.2.1节）：智能体用一种策略探索，学到的却是另一种（最优）策略的Q值。

智能体在 $\varepsilon$-greedy 策略下与环境交互，每步产生一条转移数据 $(S_t, A_t, R_t, S_{t+1})$。接下来的问题是：如何用这些数据来训练Q网络？

损失函数与训练目标

回顾3.2.2节的Q-learning更新规则：

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_t + \gamma \max_{a} Q(S_{t+1}, a) - Q(S_t, A_t) \right]$$

表格Q-learning直接修改表格中的一个单元格。对于神经网络，我们不能直接修改某个输出值，而是需要定义一个损失函数，然后通过梯度下降来更新参数 $\theta$。

从贝尔曼最优方程出发（2.2.2节式(2.2-10)），最优动作值函数满足：

$$Q^*(s, a) = \mathbb{E}\left[R + \gamma \max_{a'} Q^*(S', a') \;\middle|\; S=s, A=a\right] \tag{5.1-2}$$

其中 $R$ 和 $S'$ 都是随机变量——在网约车例子中，司机接单后乘客去哪里（$S'$）以及对应的车费（$R$）是随机的。如果 Q 网络已经完美逼近了 $Q^*$，那么上式两端应该相等。因此，训练目标就是最小化两端之差的平方：

DQN 损失函数（初步版本）
$$L(\theta) = \mathbb{E}_{(S,A,R,S')}\left[\left(\underbrace{R + \gamma \max_{a'} Q(S', a'; \theta)}_{\text{TD目标（target）}} - \underbrace{Q(S, A; \theta)}_{\text{当前估计}}\right)^2\right] \tag{5.1-3}$$

期望是对经验转移 $(S,A,R,S')$ 取的。但我们不可能穷举所有转移来精确计算这个期望。DQN 的做法是用采样来逼近：让智能体在环境中不断交互，收集真实的转移样本 $(s_t, a_t, r_t, s_{t+1})$，每个样本提供期望的一次无偏估计。大量样本的平均趋近真实期望——这正是蒙特卡洛思想在损失函数上的应用。

对每个样本 $(s_t, a_t, r_t, s_{t+1})$，该损失的梯度为：

$$\nabla_\theta L = -2\left(r_t + \gamma \max_{a'} Q(s_{t+1}, a'; \theta) - Q(s_t, a_t; \theta)\right) \nabla_\theta Q(s_t, a_t; \theta) \tag{5.1-4}$$

看起来很简单：用梯度下降不断减小TD误差的平方即可。然而，如果直接这样做，训练往往会不稳定甚至发散。原因主要有两个：

问题一：目标不稳定。在监督学习中，训练标签（如图像的类别）是固定的。但在公式(5.1-3)中，TD目标 $R + \gamma \max_{a'} Q(S', a'; \theta)$ 本身依赖于参数 $\theta$。每当 $\theta$ 更新一次，目标也随之变化。这就像射击一个不断移动的靶子，参数可能在追逐不断变化的目标中来回振荡，无法收敛。

问题二：样本相关性。智能体在环境中依次采集的数据 $(s_0, a_0, r_0, s_1), (s_1, a_1, r_1, s_2), \ldots$ 在时间上高度相关。前后相邻的样本共享状态 $s_t$，且来自相似的环境区域。而神经网络的随机梯度下降要求训练样本之间近似独立。如果用相关样本训练，网络容易过度拟合最近的经验，忘记之前学到的知识。

为了解决这两个问题，Mnih等人（2015）在发表于 Nature 的经典论文中提出了两个关键技巧：经验回放和目标网络。

经验回放

经验回放（experience replay）的思想非常直观：把智能体与环境交互产生的转移数据存入一个缓冲区（replay buffer），训练时从缓冲区中随机抽样一小批（mini-batch）数据来计算梯度，而不是用最新的一条数据。

经验回放缓冲区（Replay Buffer）

维护一个固定容量为 $N$ 的缓冲区 $\mathcal{D}$，存储智能体的历史转移：

$$\mathcal{D} = \{(s_i, a_i, r_i, s_i')\}_{i=1}^{N} \tag{5.1-5}$$

每当智能体产生一条新的转移，将其存入缓冲区（如果缓冲区已满，则覆盖最旧的数据）。训练时，从 $\mathcal{D}$ 中均匀随机抽取大小为 $B$ 的mini-batch来计算梯度。

下面的表格用一个简化的例子展示经验回放的工作过程。假设缓冲区容量 $N=6$，mini-batch大小 $B=3$，智能体已经交互了6步，产生了6条转移数据：

编号状态 $s$动作 $a$奖励 $r$下一状态 $s'$存入时间
1$s_0$$a_2$$+3$$s_1$第1步
2$s_1$$a_0$$-1$$s_2$第2步
3$s_2$$a_1$$+5$$s_3$第3步

图 5.1-2   经验回放示意。缓冲区存储了6条转移数据，黄色高亮行为随机抽取的mini-batch（编号2、4、6），用于一次梯度更新。

经验回放解决了前面提到的样本相关性问题。由于mini-batch是从不同时间步的数据中随机抽取的，样本之间的时序相关性被大大打破。此外，同一条经验可以被多次抽到、用于多次更新，提高了数据利用效率。

经验回放还有一个不太显然的好处：由于缓冲区中保存了较早时期（可能来自不同策略）的数据，Q网络实际上是在多种行为策略产生的混合数据上训练的。这与Q-learning的离策略（off-policy）特性天然契合。如果用的是SARSA这样的在策略算法，经验回放就不太适用了，因为旧数据不是由当前策略产生的。

目标网络

目标网络（target network）解决的是目标不稳定问题。思路也很简单：用一个独立的网络来计算TD目标，使目标在一段时间内保持不变。

目标网络（Target Network）

维护两个结构相同的网络：

  • 在线网络（online network）$Q(s,a;\theta)$：用于选择动作，参数每步更新
  • 目标网络（target network）$Q(s,a;\theta^-)$：用于计算TD目标，参数每隔 $C$ 步才从在线网络复制一次

损失函数变为：

$$L(\theta) = \mathbb{E}_{(S,A,R,S') \sim \mathcal{D}}\left[\left(R + \gamma \max_{a'} Q(S', a'; \theta^-) - Q(S, A; \theta)\right)^2\right] \tag{5.1-6}$$

其中 $\theta^-$ 在两次同步之间保持不变，因此TD目标不会随每次梯度更新而变化。

具体做法是：每隔 $C$ 步（如 $C=1000$），执行一次 $\theta^- \leftarrow \theta$，将在线网络的参数完整复制给目标网络。在两次同步之间的1000步中，目标网络的参数 $\theta^-$ 保持冻结，TD目标也因此保持稳定。这就像把移动的靶子暂时固定住，让射手有足够的时间瞄准。当射手（在线网络）已经对准当前靶子位置，再把靶子移动到新位置（同步参数），如此循环。

图 5.1-3   目标网络的参数同步机制。蓝色曲线是在线网络 $\theta$，每步梯度更新使其持续变化；黄色阶梯线是目标网络 $\theta^-$，在两次同步之间保持冻结。每隔 $C$ 步（红色箭头），将 $\theta$ 复制给 $\theta^-$，使阶梯跳到当前 $\theta$ 的值。冻结期间 TD 目标保持稳定，避免了"追逐移动靶子"的问题。
补充阅读：软更新（Soft Update）

除了每隔 $C$ 步硬复制参数，另一种常见做法是软更新（soft update），也称为 Polyak 平均：每一步都用一个很小的比例 $\tau$（如 $\tau=0.005$）将在线网络参数混入目标网络：

$$\theta^- \leftarrow \tau\,\theta + (1-\tau)\,\theta^- \tag{5.1-7}$$

软更新使目标网络参数平滑地跟随在线网络，避免了硬复制时的突变。这种方式在后续算法（如5.3节的SAC）中被广泛采用。

DQN算法

将Q网络、经验回放、目标网络和 $\varepsilon$-greedy 探索结合起来，就得到了完整的DQN算法。下面给出算法的伪代码：

算法：Deep Q-Network（DQN）
  1. 初始化在线网络参数 $\theta$（随机初始化），目标网络参数 $\theta^- \leftarrow \theta$
  2. 初始化经验回放缓冲区 $\mathcal{D}$，容量为 $N$
  3. 对每个回合 $e = 1, 2, \ldots$：
    1. 初始化状态 $S_0$
    2. 对每一步 $t = 0, 1, 2, \ldots$：
      1. 用 $\varepsilon$-greedy 选择动作：以 $\varepsilon$ 的概率随机选择，否则 $A_t = \arg\max_a Q(S_t, a; \theta)$
      2. 执行 $A_t$，观察奖励 $R_t$ 和下一状态 $S_{t+1}$
      3. 将转移 $(S_t, A_t, R_t, S_{t+1})$ 存入 $\mathcal{D}$
      4. 从 $\mathcal{D}$ 中随机抽取大小为 $B$ 的mini-batch $\{(s_j, a_j, r_j, s_j')\}_{j=1}^{B}$
      5. 计算TD目标：$y_j = r_j + \gamma \max_{a'} Q(s_j', a'; \theta^-)$
      6. 对损失 $\frac{1}{B}\sum_{j=1}^{B}(y_j - Q(s_j, a_j; \theta))^2$ 执行一步梯度下降，更新 $\theta$
      7. 每隔 $C$ 步：$\theta^- \leftarrow \theta$
典型超参数：缓冲区容量 $N=10^5 \sim 10^6$，mini-batch大小 $B=32$，目标网络更新频率 $C=1000$，折扣因子 $\gamma=0.99$，探索率 $\varepsilon$ 从1.0线性衰减至0.01。

与表格Q-learning（3.2.2节）对比，DQN的核心结构可以简洁地概括为：

DQN = Q-learning + 神经网络 + 经验回放 + 目标网络

例子：网约车调度的DQN

让我们将网约车司机的例子扩展到一个稍大的场景，来看DQN如何工作。

在第一章中，网约车司机只需考虑3个区域和1种时段。现在假设城市仍然分为3个区域（市中心 $c$、郊区 $b$、机场 $a$），但引入4个时段（早高峰、白天、晚高峰、夜间），用整数 $0,1,2,3$ 表示。状态定义为（区域, 时段）的组合，状态空间大小为 $|\mathcal{S}| = 3 \times 4 = 12$。动作仍为2个：接单或空驶到另一个区域。

与第三章一样，接单后的车费取决于乘客的目的地（即下一个状态 $s'$），奖励是 $R(s, a, s')$ 而非 $R(s, a)$（参见2.2.2节式(2.2-9)的讨论）。但现在引入了时段维度，同一区域的同一动作在不同时段下的表现会不同。这很符合直觉：

  • 转移概率随时段变化。早高峰时市中心接单，乘客更可能去郊区（通勤需求）；夜间则更可能是短途，留在市中心的概率更高。
  • 车费随时段变化。高峰时段打车需求旺盛，平台溢价使车费上浮（例如早高峰的车费是基准的 $1.2$ 倍）；夜间需求少，车费相应下降（基准的 $0.8$ 倍）。

因此，即使是同一个状态-动作对（如"市中心接单"），在不同时段下的即时奖励 $R$ 和转移概率 $P(s'|s,a)$ 都不同。下表给出了几个代表性的场景：

区域时段动作可能的转移
市中心 $c$0（早高峰）接单→ $c$（概率0.3，车费6）、→ $b$（概率0.5，车费10）、→ $a$（概率0.2，车费14）
空驶去机场→ $a$（确定，油费 $-4$）

对比表中两个时段下市中心接单的情况：早高峰通勤需求旺盛，乘客去郊区的概率最高（0.5），平台溢价使车费上浮（送到郊区10元 vs 基准8元）；夜间短途为主，留在市中心的概率升至0.6，车费则因需求低迷而下降（送到郊区仅6元）。机场夜间的情况也很符合直觉：红眼航班旅客多数前往市中心（概率0.8），但夜间车费偏低（11元）。DQN 不需要事先知道这些规律，它通过反复与环境交互，自动从采样数据中学到各时段各区域的最优决策。

虽然12个状态仍然可以用表格方法处理，但这个例子帮助我们理解DQN的工作方式。状态被表示为一个特征向量：例如用独热编码（one-hot encoding）表示区域（3维）和时段（4维），拼接后得到7维输入向量。Q网络接收这个7维向量，输出2个动作的Q值。

当状态维度进一步增加时（如100个区域 + 24个时段 + 需求水平），表格方法需要 $100 \times 24 \times 3 \times 2 = 14400$ 个Q值条目，而Q网络的参数量可以保持在合理范围，并且能够在相似状态之间泛化。

交互演示

下面的交互演示在12状态的网约车环境（3个区域 $\times$ 4个时段）中逐步运行DQN。网络结构为 $7 \to 8 \to 2$：输入层7个节点（3维区域独热 + 4维时段独热），隐藏层8个 ReLU 节点，输出层2个节点分别对应接单和空驶的Q值。注意所有8个隐藏节点在每次前向传播中同时参与计算，每个节点接收全部7个输入的加权和，不存在"某个节点只负责某个状态"的分工。正是这种全连接结构使得不同状态可以共享参数，实现泛化。输出层的两个节点分别对应两个动作：接单和空驶。虽然"空驶"在不同区域的目的地不同（市中心空驶去机场，机场空驶去市中心），但作为动作编码它们共享同一个输出节点，具体去向由环境决定。

训练初期缓冲区中样本不足（少于 mini-batch 大小 $B=4$），此时只收集数据不做梯度更新。缓冲区积累到足够样本后才开始训练。你可以逐子步观察DQN算法的每一个环节，也可以快进到收敛观察最终策略。

DQN 的基本框架——Q网络 + 经验回放 + 目标网络——已经足以处理许多问题。但在实践中，还有三项重要改进可以显著提升 DQN 的性能：Double DQN（减少Q值过估计）、Dueling 架构（分离状态价值和动作优势）和优先经验回放（让有信息量的样本被更多地学习）。下一节将逐一介绍这三项改进。

完整 Python 实现

下面的代码完整实现了本节交互演示中的 DQN 算法，包括12状态网约车环境（3区域 $\times$ 4时段）、Q网络（$7 \to 8 \to 2$）、经验回放和目标网络。运行后可观察Q值收敛过程和最终策略。

import torch
import torch.nn as nn
import numpy as np
from collections import deque
import random

# ==================== 环境：12状态网约车 MDP ====================
class TaxiEnv:
    """3区域 × 4时段的网约车环境，与5.1.1节交互演示一致。"""
    ZONES = ['c', 'b', 'a']  # 市中心、郊区、机场
    ZONE_NAMES = {'c': '市中心', 'b': '郊区', 'a': '机场'}
    PERIOD_NAMES = ['早高峰', '白天', '晚高峰', '夜间']
    ACTIONS = ['接单', '空驶']

    # 基础车费（按目的地）
    PICKUP_BASE = {
        'c': [('c', 5), ('b', 8), ('a', 12)],
        'b': [('c', 5), ('b', 1), ('a', 8)],
        'a': [('c', 14), ('b', 10)],
    }
    # 转移概率随时段变化
    PICKUP_PROBS = {
        'c': [[0.3, 0.5, 0.2], [0.5, 0.3, 0.2], [0.4, 0.4, 0.2], [0.6, 0.3, 0.1]],
        'b': [[0.2, 0.7, 0.1], [0.1, 0.8, 0.1], [0.15, 0.75, 0.1], [0.1, 0.85, 0.05]],
        'a': [[0.5, 0.5], [0.6, 0.4], [0.55, 0.45], [0.8, 0.2]],
    }
    # 车费缩放因子（高峰溢价，夜间降价）
    PERIOD_SCALE = [1.2, 1.0, 1.1, 0.8]
    # 空驶目的地和油费
    DEADHEAD = {'c': ('a', -4), 'b': ('c', -3), 'a': ('c', -5)}

    def __init__(self):
        self.zone = 'c'
        self.period = 0

    def reset(self):
        self.zone = random.choice(self.ZONES)
        self.period = 0
        return self._state_vec()

    def _state_vec(self):
        """返回7维 one-hot 向量: [c, b, a, P0, P1, P2, P3]"""
        v = [0.0] * 7
        v[self.ZONES.index(self.zone)] = 1.0
        v[3 + self.period] = 1.0
        return v

    def step(self, action):
        if action == 0:  # 接单
            base = self.PICKUP_BASE[self.zone]
            probs = self.PICKUP_PROBS[self.zone][self.period]
            scale = self.PERIOD_SCALE[self.period]
            idx = np.random.choice(len(base), p=probs)
            next_zone, base_fare = base[idx]
            reward = round(base_fare * scale, 1)
        else:  # 空驶
            next_zone, reward = self.DEADHEAD[self.zone]

        self.zone = next_zone
        self.period = (self.period + 1) % 4
        return self._state_vec(), reward

# ==================== Q 网络 (7 → 8 → 2) ====================
class QNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(7, 8), nn.ReLU(),
            nn.Linear(8, 2)
        )

    def forward(self, x):
        return self.net(x)

# ==================== DQN 训练 ====================
env = TaxiEnv()
q_net = QNet()
target_net = QNet()
target_net.load_state_dict(q_net.state_dict())
optimizer = torch.optim.Adam(q_net.parameters(), lr=0.008)

gamma = 0.9
buf_size, batch_size, target_freq = 200, 4, 50
eps_start, eps_end, eps_decay = 1.0, 0.05, 500
replay_buffer = deque(maxlen=buf_size)

total_steps = 0
for episode in range(100):
    s = env.reset()
    ep_reward = 0
    for t in range(50):
        eps = eps_end + (eps_start - eps_end) * np.exp(-total_steps / eps_decay)
        if random.random() < eps:
            a = random.randint(0, 1)
        else:
            with torch.no_grad():
                q = q_net(torch.tensor([s], dtype=torch.float32))
            a = q.argmax().item()

        s_next, reward = env.step(a)
        replay_buffer.append((s, a, reward, s_next))
        ep_reward += reward
        s = s_next

        # 训练
        if len(replay_buffer) >= batch_size:
            batch = random.sample(replay_buffer, batch_size)
            ss = torch.tensor([b[0] for b in batch], dtype=torch.float32)
            aa = torch.tensor([b[1] for b in batch], dtype=torch.long)
            rr = torch.tensor([b[2] for b in batch], dtype=torch.float32)
            ss2 = torch.tensor([b[3] for b in batch], dtype=torch.float32)

            with torch.no_grad():
                q_target = target_net(ss2).max(dim=1).values
            target = rr + gamma * q_target
            q_pred = q_net(ss).gather(1, aa.unsqueeze(1)).squeeze()
            loss = nn.functional.mse_loss(q_pred, target)

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

        total_steps += 1
        if total_steps % target_freq == 0:
            target_net.load_state_dict(q_net.state_dict())

    if episode % 20 == 0:
        print(f"Episode {episode:3d}, eps={eps:.2f}, reward={ep_reward:.0f}")

# ==================== 输出策略 ====================
print("\n学到的策略：")
for zone in TaxiEnv.ZONES:
    for period in range(4):
        env.zone, env.period = zone, period
        sv = env._state_vec()
        with torch.no_grad():
            q = q_net(torch.tensor([sv], dtype=torch.float32))[0]
        best = TaxiEnv.ACTIONS[q.argmax().item()]
        print(f"  {TaxiEnv.ZONE_NAMES[zone]}·{TaxiEnv.PERIOD_NAMES[period]}: "
              f"Q(接单)={q[0]:.1f}, Q(空驶)={q[1]:.1f} → {best}")
