5.4 Soft Actor-Critic

为什么需要SAC

回顾前几节：5.1节的DQN将Q-learning与深度神经网络结合，实现了离策略学习，能通过经验回放缓冲区高效利用历史数据，但它只能处理离散动作空间。5.2节的PPO采用策略梯度方法，天然支持连续动作空间，训练也比较稳定，但它是on-policy算法，每批数据用完后就必须丢弃，样本效率较低。5.3节的DDPG和TD3虽然支持连续动作空间且是off-policy的，但它们使用确定性策略，探索能力依赖外部噪声，训练稳定性仍有不足。

在管理科学的实际应用中，许多决策变量天然是连续的：物流中的车辆调度速度、库存管理中的补货数量、金融中的投资组合权重、交通信号的绿灯时长等。对于这类问题，我们希望算法同时具备以下三个特性：

  • 支持连续动作空间，不需要将决策变量人为离散化；
  • 离策略学习，能够重复利用历史经验，提高样本效率；
  • 训练稳定，不容易因超参数设置不当而崩溃。

Soft Actor-Critic（SAC，Haarnoja et al., 2018）正是为了同时满足这三个需求而提出的。它的核心思想是在标准的强化学习目标中加入一个熵正则化项：不仅要最大化累积奖励，还要让策略尽可能保持随机性。这个看似简单的改动带来了深远的影响：它自然地鼓励探索，显著提升了训练的稳定性和鲁棒性。

SAC在管理科学中的优势：许多管理科学问题的仿真器运行成本高昂（例如，一次完整的交通仿真可能需要数分钟）。SAC的离策略特性使其能够充分利用每一条仿真数据，大幅减少所需的仿真次数。同时，它对超参数不敏感的特性，减少了研究人员的调参负担。

最大熵强化学习

标准强化学习的目标是最大化期望累积折扣奖励：

$$J_{\text{std}}(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t R_t\right]$$

SAC在此基础上增加了一个策略熵（policy entropy）奖励项，形成最大熵目标：

$$J(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t \Big(R_t + \alpha \,\mathcal{H}\big(\pi(\cdot \mid S_t)\big)\Big)\right] \tag{5.4-1}$$

其中 $\mathcal{H}\big(\pi(\cdot \mid s)\big)$ 是策略在状态 $s$ 下的熵。对连续动作空间下的策略分布 $\pi(\cdot \mid s)$，熵定义为：

$$\mathcal{H}\big(\pi(\cdot \mid s)\big) = -\int \pi(a \mid s) \log \pi(a \mid s) \, da = \mathbb{E}_{a \sim \pi(\cdot|s)}\big[-\log \pi(a \mid s)\big] \tag{5.4-2}$$

参数 $\alpha > 0$ 称为温度系数（temperature），它控制了熵项的权重，也就是探索与利用之间的平衡：

  • $\alpha$ 较大时，算法更倾向于保持策略的随机性（多探索）；
  • $\alpha$ 较小时，算法更倾向于追求高奖励（多利用）；
  • $\alpha \to 0$ 时，退化为标准的强化学习目标。
最大熵目标的直觉

想象一位基金经理需要在多个资产之间分配投资比例（连续动作）。最大熵目标要求他：在保证预期收益的前提下，尽可能分散投资。这样做的好处是：（1）避免过早押注于某个单一方案，在信息不充分时保持灵活性；（2）即使环境发生变化，分散的策略也更具鲁棒性；（3）学习过程中能持续探索不同的投资组合，不会因为偶然发现一个还不错的方案就停止尝试。

公式(5.4-1)可以等价地写成更紧凑的形式。将熵的定义代入，每一步的奖励变为 $R_t - \alpha \log \pi(A_t \mid S_t)$，即：

$$J(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t \Big(R_t - \alpha \log \pi(A_t \mid S_t)\Big)\right] \tag{5.4-3}$$

这个形式更便于后续推导：可以把 $-\alpha \log \pi(a \mid s)$ 看作一个与动作概率相关的额外奖励。策略越随机（概率越分散），$-\log \pi$ 越大，获得的额外奖励越多。

软贝尔曼方程

回顾2.1.3节的标准贝尔曼期望方程，状态值函数与动作值函数之间满足：

$$V^\pi(s) = \mathbb{E}_{a \sim \pi}\big[Q^\pi(s, a)\big], \quad Q^\pi(s, a) = R(s,a) + \gamma \, \mathbb{E}_{s' \sim P}\big[V^\pi(s')\big]$$

在最大熵框架下，这些方程被修改为软贝尔曼方程（soft Bellman equation）。关键变化在于：状态值函数不再是Q值的简单期望，而是要减去策略的对数概率（即加上熵奖励）：

$$V(s) = \mathbb{E}_{a \sim \pi}\Big[Q(s, a) - \alpha \log \pi(a \mid s)\Big] \tag{5.4-4}$$
$$Q(s, a) = R(s, a) + \gamma \, \mathbb{E}_{s' \sim P}\Big[V(s')\Big] \tag{5.4-5}$$

将公式(5.4-4)代入公式(5.4-5)，可以消去 $V$ 得到：

$$Q(s, a) = R(s,a) + \gamma \, \mathbb{E}_{s'}\Big[\mathbb{E}_{a' \sim \pi}\big[Q(s', a') - \alpha \log \pi(a' \mid s')\big]\Big] \tag{5.4-6}$$

对比标准贝尔曼方程，软贝尔曼方程唯一的不同是多了 $-\alpha \log \pi(a \mid s)$ 项。这使得在固定点处，Q值不仅反映了累积奖励，还反映了未来策略的随机程度。

软贝尔曼方程的含义

在标准贝尔曼方程中，$V(s)$ 等于在状态 $s$ 下按照策略行动所获得的期望Q值。在软贝尔曼方程中，$V(s)$ 还包含了策略的熵奖励。这意味着：如果两个策略获得相同的Q值，但其中一个更随机（熵更高），那么更随机的策略对应更高的 $V(s)$。由此可以导出最大熵框架下的最优策略形式：

$$\pi^*(a \mid s) \propto \exp\!\Big(\frac{1}{\alpha} Q^*(s, a)\Big) \tag{5.4-7}$$

这是一个玻尔兹曼分布（Boltzmann distribution），温度系数 $\alpha$ 控制分布的尖锐程度。$\alpha$ 越小，分布越集中于Q值最大的动作；$\alpha$ 越大，分布越均匀。

SAC的网络架构

SAC使用三组神经网络协同工作：

1. 两个Q网络 $Q_{\theta_1}$ 和 $Q_{\theta_2}$

输入为状态-动作对 $(s, a)$，输出一个标量Q值。使用两个Q网络并取较小值是为了缓解Q值过高估计问题（与5.1节DQN中的Double DQN思想一致）。在计算目标值和更新策略时，总是取两者的最小值：

$$Q_{\min}(s, a) = \min\big(Q_{\theta_1}(s, a),\; Q_{\theta_2}(s, a)\big) \tag{5.4-8}$$

2. 策略网络 $\pi_\phi$

输入为状态 $s$，输出一个高斯分布的均值 $\mu_\phi(s)$ 和标准差 $\sigma_\phi(s)$。动作通过以下方式采样：

$$a = \tanh\!\big(\mu_\phi(s) + \sigma_\phi(s) \odot \epsilon\big), \quad \epsilon \sim \mathcal{N}(0, I) \tag{5.4-9}$$

这里的 $\tanh$ 将动作压缩到 $[-1, 1]$ 范围内，$\odot$ 表示逐元素相乘。这种采样方式称为重参数化技巧（reparameterization trick）：将随机性从策略网络中分离出来（体现在外部噪声 $\epsilon$ 中），使得梯度可以通过采样的动作反向传播到策略网络参数 $\phi$。

3. 目标Q网络 $Q_{\bar{\theta}_1}$ 和 $Q_{\bar{\theta}_2}$

这是两个Q网络的缓慢更新副本，用于计算TD目标值，以稳定训练过程（与5.1节DQN中的目标网络思想相同）：

$$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau) \bar{\theta}_i, \quad i = 1, 2 \tag{5.4-10}$$

其中 $\tau$ 通常取很小的值（如 $0.005$），使得目标网络缓慢跟踪主网络。

图 5.4-1　SAC 的网络架构。策略网络（绿色）输出高斯分布并采样动作；两个 Q 网络（黄色）评估动作价值；目标 Q 网络（红色虚线）通过软更新缓慢跟踪主网络，其最小值用于计算稳定的 TD 目标。

SAC算法

SAC的训练过程包含三个交替进行的更新步骤。每次从经验回放缓冲区中采样一个小批量 $(s, a, r, s')$，依次更新Q网络、策略网络和目标网络。

第一步：更新Q网络。首先计算TD目标值。用当前策略网络在下一状态 $s'$ 采样一个动作 $\tilde{a}'$，利用目标Q网络计算软Q值：

$$y = r + \gamma \Big(\min_{i=1,2} Q_{\bar{\theta}_i}(s', \tilde{a}') - \alpha \log \pi_\phi(\tilde{a}' \mid s')\Big), \quad \tilde{a}' \sim \pi_\phi(\cdot \mid s') \tag{5.4-11}$$

然后最小化每个Q网络的预测值与TD目标之间的均方误差：

$$L(\theta_i) = \mathbb{E}_{(s,a,r,s')}\left[\Big(Q_{\theta_i}(s, a) - y\Big)^2\right], \quad i = 1, 2 \tag{5.4-12}$$

第二步：更新策略网络。策略网络的目标是最大化期望的软Q值，即让策略选出的动作既有高Q值，又保持足够的随机性：

$$L(\phi) = \mathbb{E}_{s}\left[\mathbb{E}_{\tilde{a} \sim \pi_\phi}\Big[\alpha \log \pi_\phi(\tilde{a} \mid s) - Q_{\min}(s, \tilde{a})\Big]\right] \tag{5.4-13}$$

这里 $Q_{\min}(s, \tilde{a}) = \min(Q_{\theta_1}(s, \tilde{a}),\; Q_{\theta_2}(s, \tilde{a}))$。利用公式(5.4-9)的重参数化技巧，$\tilde{a}$ 关于 $\phi$ 可微，因此可以用标准的梯度下降来最小化 $L(\phi)$。

第三步：软更新目标网络。按照公式(5.4-10)缓慢更新目标网络参数。

Soft Actor-Critic (SAC) 算法
  1. 初始化策略网络参数 $\phi$，Q网络参数 $\theta_1, \theta_2$，令目标网络 $\bar{\theta}_i \leftarrow \theta_i$
  2. 初始化经验回放缓冲区 $\mathcal{D} \leftarrow \emptyset$
  3. 对每个时间步 $t = 0, 1, 2, \ldots$：
    1. 观察状态 $s_t$，从策略 $a_t \sim \pi_\phi(\cdot \mid s_t)$ 采样动作
    2. 执行 $a_t$，观察奖励 $r_t$ 和下一状态 $s_{t+1}$，将 $(s_t, a_t, r_t, s_{t+1})$ 存入 $\mathcal{D}$
    3. 从 $\mathcal{D}$ 中随机采样小批量 $\{(s, a, r, s')\}$
    4. 更新Q网络：对每个样本，用公式(5.4-11)计算TD目标 $y$，用公式(5.4-12)更新 $\theta_1, \theta_2$
    5. 更新策略网络：用公式(5.4-13)更新 $\phi$
    6. 软更新目标网络：$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau)\bar{\theta}_i$，$i = 1, 2$
SAC是离策略算法：步骤(c)中从缓冲区采样的数据可以来自任何历史策略。步骤(a)中的采样使用重参数化技巧（公式5.4-9）。

自动温度调节

温度系数 $\alpha$ 的选择对SAC的性能影响很大。手动调节 $\alpha$ 既费时又不可靠，因为最优的 $\alpha$ 值会随训练进程而变化（初期需要大量探索，后期应减少随机性）。SAC的改进版本（Haarnoja et al., 2018b）提出了一种自动温度调节机制：将 $\alpha$ 也作为一个可学习的参数，通过优化以下目标来自动调整：

$$L(\alpha) = \mathbb{E}_{a \sim \pi_\phi}\Big[-\alpha \big(\log \pi_\phi(a \mid s) + \bar{\mathcal{H}}\big)\Big] \tag{5.4-14}$$

其中 $\bar{\mathcal{H}}$ 是目标熵，通常设为 $-\dim(\mathcal{A})$（即负的动作空间维度）。这个约束的含义是：当策略的熵低于目标值时，自动增大 $\alpha$ 以鼓励更多探索；当熵高于目标值时，自动减小 $\alpha$ 以更多地利用已有知识。

在实践中，自动温度调节几乎是标配。它使得SAC成为一个超参数设置非常方便的算法：研究人员只需指定目标熵（通常用默认值即可），无需手动调节 $\alpha$。

例子：连续动作空间的投资组合配置

考虑一个简化的投资组合配置问题。某投资者每个交易日需要决定如何将资金分配到三种资产：股票、债券和现金。

状态：6维连续向量，各分量归一化到 $[0, 1]$：

$$s_t = \big(w_{\text{stock}},\; w_{\text{bond}},\; w_{\text{cash}},\; r_{\text{stock}}^{\text{recent}},\; r_{\text{bond}}^{\text{recent}},\; \text{vol}\big) \tag{5.4-15}$$

其中 $w_{\text{stock}}, w_{\text{bond}}, w_{\text{cash}}$ 分别是当前三种资产的持仓比例，$r_{\text{stock}}^{\text{recent}}, r_{\text{bond}}^{\text{recent}}$ 是最近若干日的平均收益率（归一化后），$\text{vol}$ 是近期市场波动率的归一化值。

动作：策略网络输出 $a = (a_1, a_2) \in [-1, 1]^2$，然后通过 softmax 变换得到目标配置权重：

$$\big(w_{\text{stock}}',\; w_{\text{bond}}',\; w_{\text{cash}}'\big) = \text{softmax}\big(a_1,\; a_2,\; 0\big) \tag{5.4-16}$$

这种参数化方式保证了三个权重非负且恰好求和为1。以 $0$ 为第三个分量的参考点，当 $a_1$ 和 $a_2$ 都很小时，三种资产接近等权配置。

奖励：每天的奖励等于组合收益减去交易成本和风险惩罚：

$$R_t = \underbrace{r_{\text{portfolio},t}}_{\text{组合收益}} - \underbrace{c \cdot \lVert \Delta w_t \rVert_1}_{\text{交易成本}} - \underbrace{\beta \cdot r_{\text{portfolio},t}^2}_{\text{风险惩罚}} \tag{5.4-17}$$

其中 $r_{\text{portfolio},t} = w_{\text{stock}}' \cdot r_{\text{stock},t} + w_{\text{bond}}' \cdot r_{\text{bond},t} + w_{\text{cash}}' \cdot r_f$ 是当日组合收益率，$\lVert \Delta w_t \rVert_1$ 是调仓前后权重变化的绝对值之和，$c = 0.001$ 为交易成本系数，$\beta = 5.0$ 为风险厌恶系数。二次惩罚项 $\beta \cdot r_{\text{portfolio},t}^2$ 使得智能体倾向于避免收益率的大幅波动，体现了风险厌恶。

为什么最大熵适合投资组合？

SAC的熵正则化与投资组合理论中的分散化原则有天然的契合。在标准RL中，策略可能收敛到将全部资金押注在某一种资产上（例如历史收益最高的股票）。这种集中配置虽然在训练数据中表现优异，却极度脆弱：一旦市场环境变化，收益可能急剧下降。SAC的熵奖励 $-\alpha \log \pi(a \mid s)$ 自动惩罚这种过度集中的行为。当策略试图将所有权重集中到单一资产时，$\log \pi$ 会很大（概率集中），熵奖励很低；反之，分散化的配置对应更高的策略熵，获得额外的熵奖励。温度系数 $\alpha$ 的大小则控制了分散化的程度：$\alpha$ 越大，配置越均匀；$\alpha$ 较小时，策略可以更多地倾向高收益资产，同时仍保持一定程度的分散化。

下面给出使用 SAC 训练投资组合配置策略的完整 PyTorch 代码。环境使用简单的几何布朗运动（GBM）模拟股票价格，均值回归过程模拟债券收益率。

import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
from collections import deque
import random

# ---- 简易市场模拟器 ----
class MarketEnv:
    """三资产市场环境：股票(GBM)、债券(均值回归)、现金(无风险)"""
    def __init__(self, episode_len=252):
        self.episode_len = episode_len  # 一年约252个交易日
        self.rf = 0.0002               # 日无风险利率 ≈ 年化5%
        self.stock_mu = 0.0005          # 股票日漂移 ≈ 年化12%
        self.stock_sigma = 0.02         # 股票日波动率 ≈ 年化32%
        self.bond_mu = 0.0003           # 债券长期均值
        self.bond_kappa = 0.05          # 均值回归速度
        self.bond_sigma = 0.005         # 债券波动率
        self.cost = 0.001               # 交易成本系数
        self.risk_beta = 5.0            # 风险惩罚系数
        self.lookback = 10
        self.reset()

    def reset(self):
        self.t = 0
        self.weights = np.array([1/3, 1/3, 1/3])  # 等权初始化
        self.stock_returns = deque(maxlen=self.lookback)
        self.bond_returns = deque(maxlen=self.lookback)
        for _ in range(self.lookback):
            self.stock_returns.append(np.random.normal(self.stock_mu, self.stock_sigma))
            self.bond_returns.append(np.random.normal(self.bond_mu, self.bond_sigma))
        return self._get_state()

    def _get_state(self):
        recent_stock = np.mean(self.stock_returns)
        recent_bond = np.mean(self.bond_returns)
        vol = np.std(list(self.stock_returns))
        # 归一化到 [0, 1] 范围
        s = np.array([
            self.weights[0], self.weights[1], self.weights[2],
            np.clip((recent_stock + 0.05) / 0.1, 0, 1),
            np.clip((recent_bond + 0.02) / 0.04, 0, 1),
            np.clip(vol / 0.05, 0, 1)
        ], dtype=np.float32)
        return s

    def step(self, action):
        # action: (a1, a2) in [-1,1]^2 -> softmax -> 目标权重
        logits = np.array([action[0], action[1], 0.0])
        new_weights = np.exp(logits) / np.exp(logits).sum()
        # 模拟资产收益
        r_stock = np.random.normal(self.stock_mu, self.stock_sigma)
        r_bond = self.bond_kappa * (self.bond_mu - np.mean(self.bond_returns)) \
                 + np.random.normal(0, self.bond_sigma)
        self.stock_returns.append(r_stock)
        self.bond_returns.append(r_bond)
        # 组合收益
        r_port = new_weights[0]*r_stock + new_weights[1]*r_bond + new_weights[2]*self.rf
        # 交易成本
        trade_cost = self.cost * np.abs(new_weights - self.weights).sum()
        # 奖励 = 收益 - 成本 - 风险惩罚
        reward = r_port - trade_cost - self.risk_beta * r_port**2
        self.weights = new_weights
        self.t += 1
        done = (self.t >= self.episode_len)
        return self._get_state(), float(reward), done

# ---- SAC 网络定义 ----
class QNetwork(nn.Module):
    def __init__(self, state_dim, action_dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, 1))

    def forward(self, s, a):
        return self.net(torch.cat([s, a], dim=-1)).squeeze(-1)

class PolicyNetwork(nn.Module):
    def __init__(self, state_dim, action_dim, hidden=128):
        super().__init__()
        self.shared = nn.Sequential(
            nn.Linear(state_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU())
        self.mu_head = nn.Linear(hidden, action_dim)
        self.log_std_head = nn.Linear(hidden, action_dim)

    def forward(self, s):
        h = self.shared(s)
        mu = self.mu_head(h)
        log_std = self.log_std_head(h).clamp(-5, 2)
        return mu, log_std

    def sample(self, s):
        mu, log_std = self.forward(s)
        std = log_std.exp()
        eps = torch.randn_like(std)
        a_pre = mu + std * eps                # 重参数化
        a = torch.tanh(a_pre)                 # 压缩到 [-1,1]
        # 计算 log_prob（需要 tanh 修正）
        log_prob = (-0.5 * eps.pow(2) - log_std - 0.5*np.log(2*np.pi)).sum(-1)
        log_prob -= (2*(np.log(2) - a_pre - torch.nn.functional.softplus(-2*a_pre))).sum(-1)
        return a, log_prob

# ---- 训练循环 ----
def train_sac(n_episodes=200, batch_size=128, gamma=0.99, tau=0.005, lr=3e-4):
    env = MarketEnv()
    s_dim, a_dim = 6, 2
    pi = PolicyNetwork(s_dim, a_dim)
    q1, q2 = QNetwork(s_dim, a_dim), QNetwork(s_dim, a_dim)
    q1_tgt = QNetwork(s_dim, a_dim); q1_tgt.load_state_dict(q1.state_dict())
    q2_tgt = QNetwork(s_dim, a_dim); q2_tgt.load_state_dict(q2.state_dict())
    log_alpha = torch.tensor(0.0, requires_grad=True)
    target_entropy = -a_dim
    opt_pi = optim.Adam(pi.parameters(), lr=lr)
    opt_q = optim.Adam(list(q1.parameters())+list(q2.parameters()), lr=lr)
    opt_a = optim.Adam([log_alpha], lr=lr)
    buf = deque(maxlen=100_000)

    for ep in range(n_episodes):
        s, ep_ret = env.reset(), 0.0
        done = False
        while not done:
            with torch.no_grad():
                a, _ = pi.sample(torch.tensor(s).unsqueeze(0))
                a_np = a.squeeze(0).numpy()
            s2, r, done = env.step(a_np)
            buf.append((s, a_np, r, s2, float(done)))
            s, ep_ret = s2, ep_ret + r
            if len(buf) < batch_size:
                continue
            batch = random.sample(buf, batch_size)
            sb = torch.tensor(np.array([x[0] for x in batch]))
            ab = torch.tensor(np.array([x[1] for x in batch]))
            rb = torch.tensor(np.array([x[2] for x in batch]), dtype=torch.float32)
            s2b = torch.tensor(np.array([x[3] for x in batch]))
            db = torch.tensor(np.array([x[4] for x in batch]), dtype=torch.float32)
            alpha = log_alpha.exp().detach()
            # Q 网络更新
            with torch.no_grad():
                a2, lp2 = pi.sample(s2b)
                q_tgt = rb + gamma*(1-db)*(torch.min(q1_tgt(s2b,a2), q2_tgt(s2b,a2)) - alpha*lp2)
            loss_q = ((q1(sb,ab)-q_tgt)**2 + (q2(sb,ab)-q_tgt)**2).mean()
            opt_q.zero_grad(); loss_q.backward(); opt_q.step()
            # 策略网络更新
            a_new, lp_new = pi.sample(sb)
            loss_pi = (alpha*lp_new - torch.min(q1(sb,a_new), q2(sb,a_new))).mean()
            opt_pi.zero_grad(); loss_pi.backward(); opt_pi.step()
            # 自动温度调节
            loss_a = -(log_alpha.exp() * (lp_new.detach() + target_entropy)).mean()
            opt_a.zero_grad(); loss_a.backward(); opt_a.step()
            # 目标网络软更新
            for p, pt in zip(q1.parameters(), q1_tgt.parameters()):
                pt.data.copy_(tau*p.data + (1-tau)*pt.data)
            for p, pt in zip(q2.parameters(), q2_tgt.parameters()):
                pt.data.copy_(tau*p.data + (1-tau)*pt.data)
        if (ep+1) % 20 == 0:
            print(f"Episode {ep+1:4d} | Return: {ep_ret:.4f} | alpha: {log_alpha.exp().item():.4f}")
    return pi

if __name__ == "__main__":
    train_sac()

上述代码中有几个值得注意的实现细节：

  • 市场模拟器中，股票价格服从几何布朗运动（日收益率为正态分布），债券收益率服从均值回归过程（Ornstein-Uhlenbeck过程的离散近似），现金提供固定的无风险利率。
  • 动作参数化：策略网络输出2维向量 $(a_1, a_2) \in [-1,1]^2$，通过 $\text{softmax}(a_1, a_2, 0)$ 映射为三资产权重。这种设计将2维连续动作映射到单纯形上，比直接输出3个权重再归一化更适合SAC的高斯策略。
  • 对数概率的 tanh 修正：由于动作经过了 $\tanh$ 压缩，计算 $\log \pi(a|s)$ 时需要加上雅可比修正项 $-\sum_i \log(1 - \tanh^2(u_i))$，其中 $u_i$ 是 $\tanh$ 之前的值。代码中使用数值稳定的等价形式实现。

小结

SAC将离策略学习、Actor-Critic架构和最大熵正则化三者结合，在连续动作空间问题上表现出色。下面将本章介绍的几种算法从多个维度进行对比：

算法选择的经验法则：
动作空间离散且较小 → 优先考虑 DQN；
希望快速上手、追求训练稳定 → 优先考虑 PPO；
动作空间连续且仿真成本高 → 优先考虑 SAC。

实际研究中，建议先用PPO验证问题建模是否合理（因为PPO最容易调通），再根据需要切换到SAC以提高样本效率。下一节将介绍一种与前面截然不同的范式：基于模型的深度强化学习，通过学习环境动态模型来大幅提高样本效率。
