5.3.2 例子：广告实时竞价

上一节介绍了 DDPG 和 TD3 的算法原理。本节通过一个简化的广告实时竞价（Real-Time Bidding, RTB）问题，展示 TD3 的完整实现，并在交互演示中直观观察双 Critic、延迟更新和目标平滑的工作过程。

问题描述

在程序化广告市场中，广告主需要在预算约束下，对每次广告展示机会决定出价金额，以最大化获得的点击总价值。每次竞价在毫秒级别完成，广告主提交出价后，平台通过拍卖决定赢家。这个场景的核心挑战在于：出价太高会快速耗尽预算，后期无法参与竞价；出价太低则赢不到好的广告位。智能体需要根据剩余预算、时间和当前广告位质量来动态调整出价策略。

出价金额是一个连续值，正好适合用 TD3 的确定性策略来处理。

MDP 建模

我们将 RTB 问题建模为一个 episodic MDP。一个 episode 代表一天的竞价周期，共 $T = 100$ 个时间步，每步对应一次竞价机会。

广告实时竞价 MDP

状态 $s_t \in \mathbb{R}^4$：

  • $s_t^{(1)}$：剩余预算比例 $\in [0, 1]$
  • $s_t^{(2)}$：剩余时间比例 $\in [0, 1]$
  • $s_t^{(3)}$：当前广告位的预估点击率 $\in [0, 1]$
  • $s_t^{(4)}$：竞争强度（市场基础出价）$\in [0, 1]$

动作 $a_t \in [0, 1]$：出价金额，表示为最大出价的比例。

奖励：

$$r_t = \begin{cases} v \cdot \text{click}_t - \text{cost}_t & \text{if } a_t \geq \text{threshold}_t \text{ and budget} > 0 \\ 0 & \text{otherwise} \end{cases} \tag{5.3-11}$$

其中 $v$ 是每次点击的价值，$\text{click}_t \sim \text{Bernoulli}(\text{ctr}_t)$ 表示是否被点击，$\text{cost}_t$ 是赢得竞价的费用（第二价格拍卖：等于市场出价），$\text{threshold}_t$ 是竞争对手的最高出价。

奖励函数（式(5.3-11)）体现了广告竞价的经济逻辑：只有出价高于竞争对手（赢得竞价）且预算充足时才有收益；收益等于点击价值减去竞价费用。这鼓励智能体在高质量广告位（高点击率）上适度出价，在低质量广告位上节省预算。

TD3 实现

下面给出完整的 PyTorch 实现代码，包含环境模拟器和 TD3 算法。代码中标注了 TD3 三项核心改进的位置。

import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np
from collections import deque
import random

# ========== 简易 RTB 环境 ==========
class RTBEnv:
    """简化的实时竞价环境"""
    def __init__(self, T=100, budget=50.0, click_value=5.0):
        self.T = T
        self.init_budget = budget
        self.click_value = click_value

    def reset(self):
        self.t = 0
        self.budget = self.init_budget
        self.done = False
        return self._obs()

    def _obs(self):
        ctr = np.clip(np.random.beta(2, 5), 0.05, 0.95)  # 点击率
        competition = np.clip(np.random.beta(3, 3), 0.1, 0.9)  # 竞争强度
        return np.array([
            self.budget / self.init_budget,    # 剩余预算比例
            1.0 - self.t / self.T,             # 剩余时间比例
            ctr,                               # 预估点击率
            competition                        # 竞争强度
        ], dtype=np.float32)

    def step(self, action):
        bid = float(np.clip(action, 0, 1))  # 出价 ∈ [0,1]
        obs = self._obs()
        ctr, competition = obs[2], obs[3]
        # 市场出价 = 竞争强度 + 噪声
        market_price = competition * 0.6 + np.random.uniform(0, 0.3)

        reward = 0.0
        if bid >= market_price and self.budget > 0:
            cost = market_price  # 第二价格
            self.budget -= cost
            clicked = np.random.random() < ctr
            reward = self.click_value * clicked - cost

        self.t += 1
        self.done = (self.t >= self.T) or (self.budget <= 0)
        next_obs = self._obs() if not self.done else np.zeros(4, dtype=np.float32)
        return next_obs, reward, self.done

# ========== 网络定义 ==========
class Actor(nn.Module):
    def __init__(self, state_dim, action_dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, action_dim), nn.Sigmoid()  # 输出 ∈ [0,1]
        )
    def forward(self, s):
        return self.net(s)

class Critic(nn.Module):
    def __init__(self, state_dim, action_dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, 1)
        )
    def forward(self, s, a):
        return self.net(torch.cat([s, a], dim=-1))

# ========== TD3 算法 ==========
class TD3:
    def __init__(self, state_dim=4, action_dim=1, gamma=0.99, tau=0.005,
                 policy_noise=0.2, noise_clip=0.5, policy_delay=2,
                 lr=3e-4, buffer_size=100000, batch_size=128):
        self.gamma = gamma
        self.tau = tau
        self.policy_noise = policy_noise
        self.noise_clip = noise_clip
        self.policy_delay = policy_delay
        self.batch_size = batch_size

        # 主网络
        self.actor = Actor(state_dim, action_dim)
        self.critic1 = Critic(state_dim, action_dim)
        self.critic2 = Critic(state_dim, action_dim)
        # 目标网络
        self.actor_target = Actor(state_dim, action_dim)
        self.critic1_target = Critic(state_dim, action_dim)
        self.critic2_target = Critic(state_dim, action_dim)
        # 初始化目标网络 = 主网络
        self.actor_target.load_state_dict(self.actor.state_dict())
        self.critic1_target.load_state_dict(self.critic1.state_dict())
        self.critic2_target.load_state_dict(self.critic2.state_dict())
        # 优化器
        self.actor_opt = torch.optim.Adam(self.actor.parameters(), lr=lr)
        self.critic1_opt = torch.optim.Adam(self.critic1.parameters(), lr=lr)
        self.critic2_opt = torch.optim.Adam(self.critic2.parameters(), lr=lr)
        # 经验回放
        self.buffer = deque(maxlen=buffer_size)
        self.total_updates = 0

    def select_action(self, state, explore_noise=0.1):
        s = torch.FloatTensor(state).unsqueeze(0)
        with torch.no_grad():
            a = self.actor(s).squeeze(0).numpy()
        a += np.random.normal(0, explore_noise, size=a.shape)
        return np.clip(a, 0, 1)

    def store(self, s, a, r, s_next, done):
        self.buffer.append((s, a, r, s_next, float(done)))

    def update(self):
        if len(self.buffer) < self.batch_size:
            return
        batch = random.sample(self.buffer, self.batch_size)
        s, a, r, s2, d = zip(*batch)
        s  = torch.FloatTensor(np.array(s))
        a  = torch.FloatTensor(np.array(a))
        r  = torch.FloatTensor(r).unsqueeze(1)
        s2 = torch.FloatTensor(np.array(s2))
        d  = torch.FloatTensor(d).unsqueeze(1)

        with torch.no_grad():
            # ---- 改进三：目标策略平滑 ----
            noise = (torch.randn_like(a) * self.policy_noise
                     ).clamp(-self.noise_clip, self.noise_clip)
            a2 = (self.actor_target(s2) + noise).clamp(0, 1)
            # ---- 改进一：Twin Critics 取最小值 ----
            target_q = torch.min(
                self.critic1_target(s2, a2),
                self.critic2_target(s2, a2)
            )
            y = r + (1 - d) * self.gamma * target_q

        # 更新两个 Critic
        loss1 = F.mse_loss(self.critic1(s, a), y)
        self.critic1_opt.zero_grad(); loss1.backward(); self.critic1_opt.step()
        loss2 = F.mse_loss(self.critic2(s, a), y)
        self.critic2_opt.zero_grad(); loss2.backward(); self.critic2_opt.step()

        self.total_updates += 1
        # ---- 改进二：延迟更新 Actor 和目标网络 ----
        if self.total_updates % self.policy_delay == 0:
            actor_loss = -self.critic1(s, self.actor(s)).mean()
            self.actor_opt.zero_grad(); actor_loss.backward(); self.actor_opt.step()
            # 软更新目标网络
            for p, pt in zip(self.actor.parameters(), self.actor_target.parameters()):
                pt.data.copy_(self.tau * p.data + (1 - self.tau) * pt.data)
            for p, pt in zip(self.critic1.parameters(), self.critic1_target.parameters()):
                pt.data.copy_(self.tau * p.data + (1 - self.tau) * pt.data)
            for p, pt in zip(self.critic2.parameters(), self.critic2_target.parameters()):
                pt.data.copy_(self.tau * p.data + (1 - self.tau) * pt.data)

# ========== 训练 ==========
env = RTBEnv()
agent = TD3()
episode_rewards = []

for ep in range(300):
    obs = env.reset()
    total_reward = 0
    while not env.done:
        action = agent.select_action(obs, explore_noise=0.1)
        next_obs, reward, done = env.step(action[0])
        agent.store(obs, action, reward, next_obs, done)
        agent.update()
        obs = next_obs
        total_reward += reward
    episode_rewards.append(total_reward)
    if (ep + 1) % 50 == 0:
        avg = np.mean(episode_rewards[-50:])
        print(f"Episode {ep+1}, 近50轮平均回报: {avg:.2f}")

实现要点

代码中标注了 TD3 的三项核心改进所在：update 方法中的 Twin Critics（两个 Critic 分别更新，TD 目标取 $\min$）、目标策略平滑（noise 和 a2 的计算）以及延迟更新（policy_delay 条件判断）。值得注意以下几个细节：

  • 动作范围。Actor 的最后一层使用 $\text{Sigmoid}$ 激活函数，确保输出在 $[0, 1]$ 范围内。更一般的情况下（如动作范围为 $[a_{\text{low}}, a_{\text{high}}]$），可以用 $\text{Tanh}$ 激活后做线性变换。
  • Actor 损失。actor_loss = -self.critic1(s, self.actor(s)).mean() 对应公式(5.3-8)的负值（因为 PyTorch 做梯度下降，取负号变成梯度上升）。注意只使用 $Q_{\phi_1}$，不使用 $Q_{\phi_2}$。
  • 探索噪声与目标噪声。探索噪声（explore_noise=0.1）用于数据采集时的探索，目标噪声（policy_noise=0.2）用于计算 TD 目标时的平滑。两者的作用不同。

实现注记

在将上述代码应用于实际的 RTB 问题时，有几点需要注意：

  • 状态设计。实际的 RTB 系统中状态会更加丰富，可能包括用户画像特征、广告创意特征、时段特征、历史竞价统计等。状态维度从 4 维扩展到数十维，但 TD3 的算法结构不需要改变。
  • 动作范围。这里出价被归一化到 $[0, 1]$，实际中需要根据业务场景将其映射回实际出价金额。如果动作范围不对称或存在约束（如出价不能低于底价），可以在 Actor 输出后做相应的变换。
  • 奖励稀疏性。RTB 中大部分展示不会产生点击，导致奖励信号稀疏。实践中常使用奖励塑形（reward shaping），例如将竞价胜出本身也赋予一个小的正奖励，帮助算法更快学习。
  • 预算约束。这里通过状态中的剩余预算比例和环境的自然终止来隐式处理预算约束。更复杂的做法可以引入 Lagrangian 方法或约束优化来显式处理预算限制。
核心要点：广告实时竞价是一个典型的连续动作 off-policy 问题：出价是连续值，历史竞价数据可以重复利用。TD3 的确定性策略直接输出出价金额，双 Critic 防止策略学到"虚假高价值"的出价策略，延迟更新和目标平滑进一步保证训练稳定性。
