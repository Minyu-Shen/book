5.3 DDPG与TD3

5.2节介绍的PPO适用于离散和连续动作空间，但它属于 on-policy 算法：每次策略更新后，旧数据就不再使用（虽然PPO对同一批数据做多轮更新，但批与批之间的数据仍然被丢弃）。在许多管理科学场景中，与环境交互的代价很高，例如仿真器运行一次需要数分钟，或者历史数据无法重新采集。此时，能够反复利用历史数据的 off-policy 算法就更加高效。

本节将介绍两种面向连续动作空间的 off-policy 算法：DDPG（Deep Deterministic Policy Gradient）和 TD3（Twin Delayed DDPG）。它们的核心思想是使用一个确定性策略直接输出连续动作值，而不是输出概率分布。

从Actor-Critic到确定性策略

在5.2节的连续动作设置中，策略被参数化为高斯分布 $\pi_\theta(a \mid s) = \mathcal{N}\bigl(\mu_\theta(s),\, \sigma_\theta(s)^2\bigr)$。执行动作时需要从分布中采样，计算策略梯度时需要求对数概率 $\log \pi_\theta(a \mid s)$。这种随机策略的好处是天然包含探索，但也带来了额外的计算开销和方差。

一个自然的问题是：能不能跳过概率分布，让策略网络直接输出一个动作值？即定义一个确定性策略：

$$a = \mu_\theta(s) \tag{5.3-1}$$

给定状态 $s$，策略 $\mu_\theta$ 确定性地输出一个动作 $a \in \mathbb{R}^d$，没有随机性。这样做有几个好处：

  • 无需采样。推理时直接前向传播一次就得到动作，不需要从分布中抽样。
  • 无需对数概率。随机策略梯度中的 $\nabla_\theta \log \pi_\theta(a \mid s)$ 项不再出现，梯度计算更加直接。
  • 更适合高维连续动作。当动作维度 $d$ 很大时，从高维高斯分布中采样并计算对数概率的代价随 $d$ 增长；确定性策略则不受此影响。
确定性策略 vs 随机策略

随机策略 $\pi_\theta(a \mid s)$：输出一个概率分布，动作通过采样得到。天然包含探索，但需要计算 $\log \pi$，且有采样方差。

确定性策略 $\mu_\theta(s)$：输出一个确定的动作向量。计算简洁，但需要额外的探索机制（如添加噪声）。

当然，确定性策略也有一个明显的代价：它本身不包含探索。如果智能体始终执行 $\mu_\theta(s)$，它就永远不会尝试其他动作，无法发现更好的策略。解决方法是在训练时向动作中添加噪声（后文将详细讨论）。

确定性策略梯度

确定性策略的优化目标与随机策略相同：最大化期望累计回报 $J(\theta) = \mathbb{E}[G_0]$。问题是如何计算 $\nabla_\theta J(\theta)$。

回顾3.3.1节的随机策略梯度定理：

$$\nabla_\theta J(\theta) = \mathbb{E}_{s \sim d^{\pi_\theta},\, a \sim \pi_\theta}\!\left[\nabla_\theta \log \pi_\theta(a \mid s)\, Q^{\pi_\theta}(s, a)\right]$$

其中 $d^{\pi_\theta}(s)$ 是策略 $\pi_\theta$ 下的状态分布。这个公式之所以有效，是因为它将对策略参数的梯度转化为了一个期望，可以用采样来估计。

Silver et al.（2014）证明了一个类似的定理，适用于确定性策略。

确定性策略梯度定理（Deterministic Policy Gradient Theorem）

设 $\mu_\theta(s)$ 是一个确定性策略，$d^{\mu_\theta}(s)$ 是其下的状态分布。假设 $Q^{\mu_\theta}(s,a)$ 关于 $a$ 可微，则：

$$\nabla_\theta J(\theta) = \mathbb{E}_{s \sim d^{\mu_\theta}}\!\left[\nabla_\theta \mu_\theta(s) \cdot \nabla_a Q^{\mu_\theta}(s, a)\Big|_{a = \mu_\theta(s)}\right] \tag{5.3-2}$$

这个公式的含义非常直观。将它拆解为两个部分：

  • $\nabla_a Q^{\mu_\theta}(s, a)\big|_{a = \mu_\theta(s)}$：在当前动作处，$Q$ 值关于动作的梯度。它指出了动作应该朝哪个方向调整才能增大 $Q$ 值。
  • $\nabla_\theta \mu_\theta(s)$：策略网络的参数如何影响输出动作。它是从动作空间到参数空间的映射。

两者相乘（通过链式法则），就得到了参数 $\theta$ 应该如何调整才能增大 $Q$ 值。整个过程就是一次链式求导：先问"动作往哪个方向变会更好"，再问"参数往哪个方向变会让动作朝那个方向移动"。

与随机策略梯度的对比

随机策略梯度与确定性策略梯度在形式上有着深刻的联系。Silver et al.（2014）证明，当随机策略的方差趋于零时，随机策略梯度收敛到确定性策略梯度。

具体来说，考虑高斯策略 $\pi_\theta(a \mid s) = \mathcal{N}(\mu_\theta(s), \sigma^2 I)$。随机策略梯度为：

$$\nabla_\theta J = \mathbb{E}_{s, a}\!\left[\nabla_\theta \log \pi_\theta(a \mid s)\, Q^{\pi_\theta}(s, a)\right]$$

当 $\sigma \to 0$ 时，分布坍缩到 $\mu_\theta(s)$，上式收敛到：

$$\nabla_\theta J = \mathbb{E}_{s}\!\left[\nabla_\theta \mu_\theta(s) \cdot \nabla_a Q^{\mu_\theta}(s, a)\Big|_{a = \mu_\theta(s)}\right]$$

这正是确定性策略梯度定理（公式(5.3-2)）。因此，确定性策略梯度可以看作随机策略梯度在零方差极限下的特例。

注意确定性策略梯度的期望只对 $s$ 取，不需要对 $a$ 取期望。这是因为确定性策略下动作 $a = \mu_\theta(s)$ 是状态的确定函数，不存在需要边缘化的随机性。这意味着确定性策略梯度比随机策略梯度需要更少的样本来估计。

公式(5.3-2)中有一个关键要求：$Q^{\mu_\theta}(s, a)$ 必须关于 $a$ 可微。这意味着我们需要一个可微分的 $Q$ 函数近似器，而神经网络正好满足这个条件。这自然地引出了 Actor-Critic 架构：

  • Critic：一个神经网络 $Q_\phi(s, a)$，输入状态和动作，输出 $Q$ 值的估计。
  • Actor：一个神经网络 $\mu_\theta(s)$，输入状态，输出确定性动作。

Actor 的更新方向由 Critic 提供的梯度信号决定：将 $\mu_\theta(s)$ 代入 $Q_\phi(s, a)$，对整个复合函数 $Q_\phi\bigl(s, \mu_\theta(s)\bigr)$ 关于 $\theta$ 求梯度，然后沿梯度方向更新。

DDPG算法

有了确定性策略梯度定理，如何将其变成一个实用的深度强化学习算法？直接套用面临两个障碍：

  1. $Q^{\mu_\theta}$ 是未知的，需要用 Critic 网络 $Q_\phi(s,a)$ 来近似；但 Critic 本身也需要从数据中学习，训练初期估计不准。
  2. 确定性策略下，智能体不探索，采集的数据缺乏多样性，Critic 难以学到准确的 $Q$ 值。

Lillicrap et al.（2016）提出的 DDPG（Deep Deterministic Policy Gradient）算法借鉴了5.1节 DQN 的两个关键技术来解决这些问题。

经验回放

与 DQN 一样，DDPG 将每一步的转移 $(s, a, r, s')$ 存储到经验回放缓冲区 $\mathcal{D}$ 中，训练时随机抽取小批量数据。这带来两个好处：打破数据之间的时间相关性，以及实现 off-policy 学习，让过去的数据可以被反复利用。

注意，确定性策略梯度（公式(5.3-2)）中的期望是对状态分布 $d^{\mu_\theta}(s)$ 取的，不涉及对动作分布的积分。因此，即使数据来自过去的策略（动作不同），只要状态分布差异不大，梯度估计仍然近似正确。这使得确定性策略特别适合与经验回放结合使用。

目标网络

DDPG 同时维护 Actor 和 Critic 的目标网络 $\mu_{\theta'}$ 和 $Q_{\phi'}$，用于计算 TD 目标。与 DQN 不同的是，DDPG 使用软更新（soft update）而非定期的硬替换：

$$\theta' \leftarrow \tau \theta + (1 - \tau) \theta' \tag{5.3-3}$$ $$\phi' \leftarrow \tau \phi + (1 - \tau) \phi' \tag{5.3-4}$$

其中 $\tau$ 是一个很小的正数（通常 $\tau = 0.005$）。每次更新主网络后，目标网络的参数缓慢地向主网络靠拢。相比硬替换，软更新使目标值的变化更加平滑，有利于训练稳定性。

探索噪声

由于 $\mu_\theta(s)$ 是确定性的，训练时需要额外添加噪声来进行探索。最简单的方式是在策略输出上叠加高斯噪声：

$$a = \mu_\theta(s) + \epsilon, \quad \epsilon \sim \mathcal{N}(0, \sigma^2) \tag{5.3-5}$$

原始的 DDPG 论文使用了 Ornstein-Uhlenbeck（OU）过程产生时间相关的噪声，但后续研究发现简单的高斯噪声通常效果相当，且更容易实现和调参。

Critic 的更新

Critic 的学习目标是最小化 TD 误差。对于从经验回放中采样的一个小批量转移 $(s_i, a_i, r_i, s_i')$，构建 TD 目标：

$$y_i = r_i + \gamma\, Q_{\phi'}(s_i',\, \mu_{\theta'}(s_i')) \tag{5.3-6}$$

注意这里使用了目标 Actor $\mu_{\theta'}$ 来为下一个状态选择动作，使用目标 Critic $Q_{\phi'}$ 来评估该动作的价值。Critic 的损失函数为：

$$L(\phi) = \frac{1}{N} \sum_{i=1}^{N} \bigl(y_i - Q_\phi(s_i, a_i)\bigr)^2 \tag{5.3-7}$$

Actor 的更新

Actor 的目标是最大化 Critic 给出的 $Q$ 值。根据确定性策略梯度定理（公式(5.3-2)），Actor 的梯度为：

$$\nabla_\theta J \approx \frac{1}{N} \sum_{i=1}^{N} \nabla_a Q_\phi(s_i, a)\Big|_{a = \mu_\theta(s_i)} \cdot \nabla_\theta \mu_\theta(s_i) \tag{5.3-8}$$

在实现中，这等价于将 $\mu_\theta(s)$ 代入 $Q_\phi(s, a)$，对复合函数 $Q_\phi(s, \mu_\theta(s))$ 关于 $\theta$ 做梯度上升。PyTorch 的自动微分可以一步完成这个链式求导。

DDPG 算法
  1. 初始化 Critic 网络 $Q_\phi(s,a)$、Actor 网络 $\mu_\theta(s)$
  2. 初始化目标网络：$\phi' \leftarrow \phi$，$\theta' \leftarrow \theta$
  3. 初始化经验回放缓冲区 $\mathcal{D}$
  4. 对每个 episode：
    1. 观测初始状态 $s_0$
    2. 对每步 $t = 0, 1, 2, \ldots$：
      1. 选择动作 $a_t = \mu_\theta(s_t) + \epsilon_t$，$\epsilon_t \sim \mathcal{N}(0, \sigma^2)$
      2. 执行 $a_t$，观测 $r_t, s_{t+1}$
      3. 将 $(s_t, a_t, r_t, s_{t+1})$ 存入 $\mathcal{D}$
      4. 从 $\mathcal{D}$ 中随机采样小批量 $\{(s_i, a_i, r_i, s_i')\}$
      5. 计算 TD 目标：$y_i = r_i + \gamma\, Q_{\phi'}(s_i', \mu_{\theta'}(s_i'))$
      6. 更新 Critic：最小化 $\frac{1}{N}\sum_i (y_i - Q_\phi(s_i, a_i))^2$
      7. 更新 Actor：沿 $\nabla_\theta \frac{1}{N}\sum_i Q_\phi(s_i, \mu_\theta(s_i))$ 做梯度上升
      8. 软更新目标网络：$\phi' \leftarrow \tau\phi + (1-\tau)\phi'$，$\theta' \leftarrow \tau\theta + (1-\tau)\theta'$
DDPG = DPG + 经验回放 + 目标网络（软更新）+ 探索噪声。它将 DQN 的成功经验移植到了连续动作空间。
图 5.3-1：DDPG 的架构。Actor 输出确定性动作，Critic 评估动作价值。目标网络通过软更新（参数 $\tau$）缓慢跟踪主网络。经验回放缓冲区存储历史转移，供训练时随机采样。

DDPG的问题：$Q$ 值过估计

DDPG在许多连续控制任务中取得了不错的效果，但在实践中也暴露出一个严重的问题：$Q$ 值的过估计（overestimation）。

回顾3.2.3节对 Q-learning 过估计问题的讨论。当 $Q$ 函数近似器存在误差时，$\max_a Q(s,a)$ 倾向于选中那些因误差而被高估的动作，导致 $Q$ 值的系统性偏高。在 DQN 中，Double Q-learning 的思想成功缓解了这个问题。

在 DDPG 中，过估计问题实际上更加严重。原因在于确定性策略的 Actor 更新方式：

$$\theta \leftarrow \theta + \alpha \nabla_\theta Q_\phi\bigl(s, \mu_\theta(s)\bigr)$$

Actor 沿着 Critic 的梯度方向调整策略，目标是找到使 $Q_\phi(s, a)$ 最大的动作 $a$。如果 Critic 在某些区域存在过高的估计误差，Actor 就会学到专门利用这些误差的策略，将动作调整到 $Q_\phi$ 的虚假峰值处。

与 DQN 中的离散 $\max$ 操作相比，连续空间中的梯度上升更具破坏性。离散空间中 $\max$ 只能在有限个候选动作中选择；而在连续空间中，Actor 可以精确地爬到 $Q_\phi$ 的任意窄峰上，找到那些恰好被高估的"漏洞"。这就像是让一个学生来设计考试题目并自己参加考试一样，Actor（学生）会越来越擅长找到 Critic（评分标准）的弱点。

这种过估计形成恶性循环：

  1. Critic 对某些 $(s, a)$ 对的 $Q$ 值估计偏高；
  2. Actor 被更新到这些高估区域，输出的动作偏向这些虚假峰值；
  3. 这些偏差的动作又被用于计算 TD 目标 $y = r + \gamma Q_{\phi'}(s', \mu_{\theta'}(s'))$，进一步传播过估计；
  4. Critic 的过估计进一步加剧，形成正反馈循环。

在实践中，这会导致 DDPG 的训练不稳定，性能波动大，对超参数敏感。

TD3：三项改进

Fujimoto et al.（2018）对上述问题进行了系统分析，提出了 TD3（Twin Delayed DDPG）算法。TD3 在 DDPG 的基础上引入三项针对性改进，显著提升了训练的稳定性和最终性能。

改进一：双 Critic 网络（Twin Critics）

TD3 同时维护两个独立的 Critic 网络 $Q_{\phi_1}(s,a)$ 和 $Q_{\phi_2}(s,a)$，各自有独立的参数和目标网络。在计算 TD 目标时，取两者的较小值：

$$y = r + \gamma \min\bigl(Q_{\phi_1'}(s', \tilde{a}'),\; Q_{\phi_2'}(s', \tilde{a}')\bigr) \tag{5.3-9}$$

这一做法的思路与3.2.3节的 Double Q-learning 类似，但不完全相同。Double Q-learning 用一个网络选动作、另一个网络评估价值，解耦了选择与评估；而 TD3 的 Twin Critics 更加简单直接，直接取两个估计的最小值作为目标。这是一种保守估计策略：即使某个 Critic 过高估计了某个 $(s', a')$，只要另一个 Critic 没有犯同样的错误，取最小值就能抑制过估计。

为什么取 $\min$ 有效？
两个独立训练的 Critic 虽然都可能存在估计误差，但它们不太可能在同一个 $(s, a)$ 处同时严重高估。取最小值相当于选择更保守的那个估计。虽然这可能引入轻微的低估偏差，但低估比高估安全得多：高估会被 Actor 积极利用并放大，而低估只是让 Actor 稍微保守一些。

改进二：延迟策略更新（Delayed Policy Update）

在 DDPG 中，Actor 和 Critic 每一步都同时更新。但 Critic 的估计在训练初期并不准确，基于不准确的 Critic 来更新 Actor 可能导致 Actor 走偏。

TD3 的解决方案很简单：让 Actor 的更新频率低于 Critic。具体来说，每更新 Critic $d$ 次（通常 $d = 2$），才更新一次 Actor 和目标网络。这样做的效果是让 Critic 有更多的时间来"消化"新数据、减小估计误差，然后再用更准确的 Critic 来指导 Actor 更新。

改进三：目标策略平滑（Target Policy Smoothing）

在计算 TD 目标时，DDPG 使用 $\mu_{\theta'}(s')$ 来选择下一步的动作。如果 $Q_\phi$ 在 $\mu_{\theta'}(s')$ 附近的某个狭窄区域恰好有一个虚假的高值，这个误差就会被直接传入 TD 目标。

TD3 的第三项改进是对目标动作添加噪声：

$$\tilde{a}' = \mu_{\theta'}(s') + \epsilon, \quad \epsilon \sim \text{clip}\bigl(\mathcal{N}(0, \tilde{\sigma}^2),\, -c,\, c\bigr) \tag{5.3-10}$$

其中 $\tilde{\sigma}$ 是目标噪声的标准差，$c$ 是裁剪范围（通常 $\tilde{\sigma} = 0.2$，$c = 0.5$）。噪声被裁剪以确保目标动作不会偏离太远。

添加噪声的效果是对 $Q$ 函数在目标动作附近做了一次局部平滑。如果 $Q$ 的真实值在那个区域是平坦的，加噪声后的平均值不会变化太多；但如果 $Q$ 在那里有一个因估计误差产生的尖锐峰值，加噪声后峰值会被抹平。这正是我们想要的效果。

TD3 算法（Twin Delayed DDPG）
  1. 初始化两个 Critic 网络 $Q_{\phi_1}$、$Q_{\phi_2}$ 和 Actor 网络 $\mu_\theta$
  2. 初始化目标网络：$\phi_1' \leftarrow \phi_1$，$\phi_2' \leftarrow \phi_2$，$\theta' \leftarrow \theta$
  3. 初始化经验回放缓冲区 $\mathcal{D}$
  4. 对每个 episode：
    1. 观测初始状态 $s_0$
    2. 对每步 $t = 0, 1, 2, \ldots$：
      1. 选择动作 $a_t = \mu_\theta(s_t) + \epsilon$，$\epsilon \sim \mathcal{N}(0, \sigma^2)$（探索噪声）
      2. 执行 $a_t$，观测 $r_t, s_{t+1}$
      3. 将 $(s_t, a_t, r_t, s_{t+1})$ 存入 $\mathcal{D}$
      4. 从 $\mathcal{D}$ 中随机采样小批量 $\{(s_i, a_i, r_i, s_i')\}_{i=1}^N$
      5. 计算目标动作（含平滑噪声）：$\tilde{a}_i' = \mu_{\theta'}(s_i') + \text{clip}(\epsilon, -c, c)$，$\epsilon \sim \mathcal{N}(0, \tilde{\sigma}^2)$
      6. 计算 TD 目标：$y_i = r_i + \gamma \min\bigl(Q_{\phi_1'}(s_i', \tilde{a}_i'),\, Q_{\phi_2'}(s_i', \tilde{a}_i')\bigr)$
      7. 更新两个 Critic：分别最小化 $\frac{1}{N}\sum_i (y_i - Q_{\phi_j}(s_i, a_i))^2$，$j=1,2$
      8. 每 $d$ 步执行以下更新（延迟更新）：
        • 更新 Actor：$\theta \leftarrow \theta + \alpha_\theta \frac{1}{N}\sum_i \nabla_\theta Q_{\phi_1}(s_i, \mu_\theta(s_i))$
        • 软更新所有目标网络：$\phi_j' \leftarrow \tau\phi_j + (1-\tau)\phi_j'$，$\theta' \leftarrow \tau\theta + (1-\tau)\theta'$
TD3 相比 DDPG 的三项改进：(1) 双 Critic 取最小值，抑制过估计；(2) 延迟更新 Actor，让 Critic 先稳定；(3) 目标策略平滑，防止利用 $Q$ 函数窄峰。注意 Actor 更新时只使用 $Q_{\phi_1}$。

例子：广告实时竞价

为了展示 TD3 的完整实现，我们考虑一个简化的广告实时竞价（Real-Time Bidding, RTB）问题。在这个场景中，广告主需要在预算约束下，对每次广告展示机会决定出价金额，以最大化获得的点击总价值。

问题建模

我们将 RTB 问题建模为一个 episodic MDP。一个 episode 代表一天的竞价周期，共 $T = 100$ 个时间步，每步对应一次竞价机会。

广告实时竞价 MDP

状态 $s_t \in \mathbb{R}^4$：

  • $s_t^{(1)}$：剩余预算比例 $\in [0, 1]$
  • $s_t^{(2)}$：剩余时间比例 $\in [0, 1]$
  • $s_t^{(3)}$：当前广告位的预估点击率 $\in [0, 1]$
  • $s_t^{(4)}$：竞争强度（市场基础出价）$\in [0, 1]$

动作 $a_t \in [0, 1]$：出价金额，表示为最大出价的比例。

奖励：

$$r_t = \begin{cases} v \cdot \text{click}_t - \text{cost}_t & \text{if } a_t \geq \text{threshold}_t \text{ and budget} > 0 \\ 0 & \text{otherwise} \end{cases} \tag{5.3-11}$$

其中 $v$ 是每次点击的价值，$\text{click}_t \sim \text{Bernoulli}(\text{ctr}_t)$ 表示是否被点击，$\text{cost}_t$ 是赢得竞价的费用（第二价格：等于市场出价），$\text{threshold}_t$ 是竞争对手的最高出价。

这个问题的核心挑战在于：出价太高会快速耗尽预算，后期无法参与竞价；出价太低则赢不到好的广告位。智能体需要根据剩余预算、时间和当前广告位质量来动态调整出价策略。

TD3 实现

下面给出完整的 PyTorch 实现代码，包含环境模拟器和 TD3 算法。

import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np
from collections import deque
import random

# ========== 简易 RTB 环境 ==========
class RTBEnv:
    """简化的实时竞价环境"""
    def __init__(self, T=100, budget=50.0, click_value=5.0):
        self.T = T
        self.init_budget = budget
        self.click_value = click_value

    def reset(self):
        self.t = 0
        self.budget = self.init_budget
        self.done = False
        return self._obs()

    def _obs(self):
        ctr = np.clip(np.random.beta(2, 5), 0.05, 0.95)  # 点击率
        competition = np.clip(np.random.beta(3, 3), 0.1, 0.9)  # 竞争强度
        return np.array([
            self.budget / self.init_budget,    # 剩余预算比例
            1.0 - self.t / self.T,             # 剩余时间比例
            ctr,                               # 预估点击率
            competition                        # 竞争强度
        ], dtype=np.float32)

    def step(self, action):
        bid = float(np.clip(action, 0, 1))  # 出价 ∈ [0,1]
        obs = self._obs()
        ctr, competition = obs[2], obs[3]
        # 市场出价 = 竞争强度 + 噪声
        market_price = competition * 0.6 + np.random.uniform(0, 0.3)

        reward = 0.0
        if bid >= market_price and self.budget > 0:
            cost = market_price  # 第二价格
            self.budget -= cost
            clicked = np.random.random() < ctr
            reward = self.click_value * clicked - cost

        self.t += 1
        self.done = (self.t >= self.T) or (self.budget <= 0)
        next_obs = self._obs() if not self.done else np.zeros(4, dtype=np.float32)
        return next_obs, reward, self.done

# ========== 网络定义 ==========
class Actor(nn.Module):
    def __init__(self, state_dim, action_dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, action_dim), nn.Sigmoid()  # 输出 ∈ [0,1]
        )
    def forward(self, s):
        return self.net(s)

class Critic(nn.Module):
    def __init__(self, state_dim, action_dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, 1)
        )
    def forward(self, s, a):
        return self.net(torch.cat([s, a], dim=-1))

# ========== TD3 算法 ==========
class TD3:
    def __init__(self, state_dim=4, action_dim=1, gamma=0.99, tau=0.005,
                 policy_noise=0.2, noise_clip=0.5, policy_delay=2,
                 lr=3e-4, buffer_size=100000, batch_size=128):
        self.gamma = gamma
        self.tau = tau
        self.policy_noise = policy_noise
        self.noise_clip = noise_clip
        self.policy_delay = policy_delay
        self.batch_size = batch_size

        # 主网络
        self.actor = Actor(state_dim, action_dim)
        self.critic1 = Critic(state_dim, action_dim)
        self.critic2 = Critic(state_dim, action_dim)
        # 目标网络
        self.actor_target = Actor(state_dim, action_dim)
        self.critic1_target = Critic(state_dim, action_dim)
        self.critic2_target = Critic(state_dim, action_dim)
        # 初始化目标网络 = 主网络
        self.actor_target.load_state_dict(self.actor.state_dict())
        self.critic1_target.load_state_dict(self.critic1.state_dict())
        self.critic2_target.load_state_dict(self.critic2.state_dict())
        # 优化器
        self.actor_opt = torch.optim.Adam(self.actor.parameters(), lr=lr)
        self.critic1_opt = torch.optim.Adam(self.critic1.parameters(), lr=lr)
        self.critic2_opt = torch.optim.Adam(self.critic2.parameters(), lr=lr)
        # 经验回放
        self.buffer = deque(maxlen=buffer_size)
        self.total_updates = 0

    def select_action(self, state, explore_noise=0.1):
        s = torch.FloatTensor(state).unsqueeze(0)
        with torch.no_grad():
            a = self.actor(s).squeeze(0).numpy()
        a += np.random.normal(0, explore_noise, size=a.shape)
        return np.clip(a, 0, 1)

    def store(self, s, a, r, s_next, done):
        self.buffer.append((s, a, r, s_next, float(done)))

    def update(self):
        if len(self.buffer) < self.batch_size:
            return
        batch = random.sample(self.buffer, self.batch_size)
        s, a, r, s2, d = zip(*batch)
        s  = torch.FloatTensor(np.array(s))
        a  = torch.FloatTensor(np.array(a))
        r  = torch.FloatTensor(r).unsqueeze(1)
        s2 = torch.FloatTensor(np.array(s2))
        d  = torch.FloatTensor(d).unsqueeze(1)

        with torch.no_grad():
            # 目标策略平滑
            noise = (torch.randn_like(a) * self.policy_noise
                     ).clamp(-self.noise_clip, self.noise_clip)
            a2 = (self.actor_target(s2) + noise).clamp(0, 1)
            # Twin Critics 取最小值
            target_q = torch.min(
                self.critic1_target(s2, a2),
                self.critic2_target(s2, a2)
            )
            y = r + (1 - d) * self.gamma * target_q

        # 更新两个 Critic
        loss1 = F.mse_loss(self.critic1(s, a), y)
        self.critic1_opt.zero_grad(); loss1.backward(); self.critic1_opt.step()
        loss2 = F.mse_loss(self.critic2(s, a), y)
        self.critic2_opt.zero_grad(); loss2.backward(); self.critic2_opt.step()

        self.total_updates += 1
        # 延迟更新 Actor 和目标网络
        if self.total_updates % self.policy_delay == 0:
            actor_loss = -self.critic1(s, self.actor(s)).mean()
            self.actor_opt.zero_grad(); actor_loss.backward(); self.actor_opt.step()
            # 软更新目标网络
            for p, pt in zip(self.actor.parameters(), self.actor_target.parameters()):
                pt.data.copy_(self.tau * p.data + (1 - self.tau) * pt.data)
            for p, pt in zip(self.critic1.parameters(), self.critic1_target.parameters()):
                pt.data.copy_(self.tau * p.data + (1 - self.tau) * pt.data)
            for p, pt in zip(self.critic2.parameters(), self.critic2_target.parameters()):
                pt.data.copy_(self.tau * p.data + (1 - self.tau) * pt.data)

# ========== 训练 ==========
env = RTBEnv()
agent = TD3()
episode_rewards = []

for ep in range(300):
    obs = env.reset()
    total_reward = 0
    while not env.done:
        action = agent.select_action(obs, explore_noise=0.1)
        next_obs, reward, done = env.step(action[0])
        agent.store(obs, action, reward, next_obs, done)
        agent.update()
        obs = next_obs
        total_reward += reward
    episode_rewards.append(total_reward)
    if (ep + 1) % 50 == 0:
        avg = np.mean(episode_rewards[-50:])
        print(f"Episode {ep+1}, 近50轮平均回报: {avg:.2f}")

代码中标注了 TD3 的三项核心改进所在：update 方法中的 Twin Critics（两个 Critic 分别更新）、目标策略平滑（noise 和 a2 的计算）以及延迟更新（policy_delay 条件判断）。

值得注意实现中的几个细节：

  • 动作范围。Actor 的最后一层使用 $\text{Sigmoid}$ 激活函数，确保输出在 $[0, 1]$ 范围内。更一般的情况下（如动作范围为 $[a_{\text{low}}, a_{\text{high}}]$），可以用 $\text{Tanh}$ 激活后做线性变换。
  • Actor 损失。actor_loss = -self.critic1(s, self.actor(s)).mean() 对应公式(5.3-8)的负值（因为 PyTorch 做梯度下降，取负号变成梯度上升）。注意只使用 $Q_{\phi_1}$，不使用 $Q_{\phi_2}$。
  • 探索噪声与目标噪声。探索噪声（explore_noise=0.1）用于数据采集时的探索，目标噪声（policy_noise=0.2）用于计算 TD 目标时的平滑。两者的作用不同。

小结

本节从确定性策略出发，介绍了 DDPG 和 TD3 两种面向连续动作空间的 off-policy 深度强化学习算法。

DDPG 将确定性策略梯度定理与 DQN 的经验回放和目标网络相结合，首次实现了端到端的连续控制深度强化学习。但 DDPG 受 $Q$ 值过估计问题困扰，训练稳定性不足。TD3 通过三项简洁的改进（双 Critic、延迟更新、目标平滑）系统性地解决了这个问题。

TD3 仍然需要手动设置探索噪声的大小，这个超参数对性能有显著影响：噪声太小则探索不足，噪声太大则数据质量差。下一节将要介绍的 Soft Actor-Critic（SAC）从一个不同的角度解决了这个问题：它将策略的随机性（熵）纳入优化目标，让智能体自动学习合适的探索程度。

核心要点：DDPG 用确定性策略 + DQN 技术实现了连续动作的 off-policy 学习；TD3 通过双 Critic（抑制过估计）、延迟更新（稳定 Actor 学习）、目标平滑（避免利用 $Q$ 的窄峰）三项改进，显著提升了训练的稳定性和性能。在连续控制的 off-policy 场景中，TD3 是一个简洁、可靠的基线选择。
