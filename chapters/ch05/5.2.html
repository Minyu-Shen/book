5.2 Policy Gradient与PPO

为什么直接优化策略

5.1节介绍的DQN沿用了Q-learning的思路：先学习一个动作值函数 $Q_\theta(s,a)$，再从中推导出策略 $\pi(s) = \arg\max_a Q_\theta(s,a)$。这种基于值函数的方法用神经网络替代了Q-table，但核心逻辑没有改变，策略始终是从值函数间接得到的。

这种间接方式在很多场景下效果很好，但也存在局限性：

  • 连续动作空间。当动作是连续值时（例如设定补货数量、调整信号灯配时的秒数），$\arg\max_a Q(s,a)$ 需要对一个连续空间做优化，这本身就是一个困难的子问题。DQN只能处理离散且有限的动作集合。
  • 随机策略。有时最优策略本身就是随机的。例如在博弈论场景中，混合策略优于任何确定性策略。$\arg\max$ 只能给出确定性策略。
  • 策略更平滑。值函数的微小变化可能导致 $\arg\max$ 对应的动作发生突变。直接参数化策略则可以实现更平滑、更稳定的更新。

回忆2.2.1节的策略迭代算法：它始终显式地维护一个策略 $\pi$，通过交替进行策略评估（计算 $V^\pi$）和策略改进（贪心更新 $\pi$）来逼近最优策略。策略梯度方法继承了这个思想，但做了两个关键改变：

  1. 用参数 $\theta$ 表示策略 $\pi_\theta$，使策略空间变为连续的；
  2. 用梯度上升替代贪心改进，沿着目标函数增长最快的方向调整参数。
与策略迭代的对照
策略迭代（2.2.1节）：评估 $\pi$ $\to$ 贪心改进 $\to$ 重复
策略梯度：估计 $\nabla_\theta J(\theta)$ $\to$ 梯度上升 $\theta \leftarrow \theta + \alpha \nabla_\theta J(\theta)$ $\to$ 重复
两者都在"理解当前策略有多好"与"让策略变得更好"之间交替迭代。区别在于：策略梯度使用基于梯度的连续优化，而非离散的贪心切换。

策略参数化

为了使用梯度方法优化策略，首先需要将策略表示为参数 $\theta$ 的可微函数 $\pi_\theta(a \mid s)$。根据动作空间的类型，有两种常见的参数化方式。

离散动作：Softmax策略

当动作空间 $\mathcal{A}$ 是有限离散集合时，策略网络输出每个动作的偏好值 $h_\theta(s,a)$，再通过softmax函数转化为概率分布：

$$\pi_\theta(a \mid s) = \frac{\exp\bigl(h_\theta(s, a)\bigr)}{\displaystyle\sum_{a' \in \mathcal{A}} \exp\bigl(h_\theta(s, a')\bigr)} \tag{5.2-1}$$

其中 $h_\theta(s,a)$ 可以由神经网络计算。例如，一个简单的线性策略令 $h_\theta(s,a) = \theta^\top \phi(s,a)$，其中 $\phi(s,a)$ 是状态-动作的特征向量。更一般地，$h_\theta$ 可以是一个深度神经网络，输入状态 $s$，输出 $|\mathcal{A}|$ 维向量，每一维对应一个动作的偏好值。

连续动作：高斯策略

当动作 $a \in \mathbb{R}^d$ 是连续值时，常见做法是让策略输出一个高斯分布的均值和标准差：

$$\pi_\theta(a \mid s) = \mathcal{N}\bigl(a \;\big|\; \mu_\theta(s),\, \sigma_\theta(s)^2\bigr) \tag{5.2-2}$$

其中 $\mu_\theta(s)$ 和 $\sigma_\theta(s)$ 都由神经网络输出。在执行时，从这个高斯分布中采样得到具体的动作值。这种参数化天然地支持连续动作空间，而且采样和求导都很方便。

在管理科学的应用中，离散动作（如选择哪条路线、是否补货）和连续动作（如补货数量、定价水平）都很常见。策略梯度方法能够统一处理这两类问题，这是它相比DQN的重要优势。

策略梯度定理

有了参数化的策略 $\pi_\theta$，接下来需要定义优化目标。我们希望找到参数 $\theta$，使得策略的期望累计回报最大：

$$J(\theta) = \mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{T} \gamma^t R_t\right] \tag{5.2-3}$$

其中期望是对策略 $\pi_\theta$ 产生的轨迹 $(S_0, A_0, R_0, S_1, A_1, R_1, \ldots)$ 取的。为了用梯度上升优化 $J(\theta)$，我们需要计算 $\nabla_\theta J(\theta)$。关键困难在于：改变 $\theta$ 不仅改变了每步选择动作的概率，还改变了后续访问各状态的概率分布。这使得直接求导非常复杂。

策略梯度定理（Sutton et al., 2000）给出了一个优雅的结果，它将梯度表达为一个可以用采样估计的期望：

$$\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{T} \nabla_\theta \log \pi_\theta(A_t \mid S_t) \cdot G_t\right] \tag{5.2-4}$$

其中 $G_t = \sum_{k=0}^{T-t} \gamma^k R_{t+k}$ 是从时间步 $t$ 开始的折扣回报。

直觉理解

公式(5.2-4)的含义非常直观。对于轨迹中的每一步 $(S_t, A_t)$：

  • $\nabla_\theta \log \pi_\theta(A_t \mid S_t)$ 是使动作 $A_t$ 在状态 $S_t$ 下的概率增大的参数调整方向；
  • $G_t$ 是执行 $A_t$ 之后获得的实际回报，衡量这个动作有多好。

两者的乘积告诉我们：如果 $G_t$ 较大（动作带来了高回报），就增大该动作的概率；如果 $G_t$ 较小（动作带来了低回报），就减小该动作的概率。这正是试错学习的数学表达：好的行为被强化，差的行为被抑制。

选读：策略梯度定理的推导

考虑一条完整轨迹 $\tau = (S_0, A_0, R_0, \ldots, S_T, A_T, R_T)$，其在策略 $\pi_\theta$ 下的概率为：

$$P(\tau \mid \theta) = p(S_0) \prod_{t=0}^{T} \pi_\theta(A_t \mid S_t) \cdot P(S_{t+1} \mid S_t, A_t)$$

目标函数可写为 $J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau)]$，其中 $R(\tau) = \sum_{t=0}^T \gamma^t R_t$。对 $\theta$ 求导：

$$\nabla_\theta J(\theta) = \nabla_\theta \sum_\tau P(\tau \mid \theta) R(\tau) = \sum_\tau P(\tau \mid \theta) \frac{\nabla_\theta P(\tau \mid \theta)}{P(\tau \mid \theta)} R(\tau) = \mathbb{E}_{\tau \sim \pi_\theta}\!\left[\nabla_\theta \log P(\tau \mid \theta) \cdot R(\tau)\right]$$

这里用了 $\nabla \log f = \nabla f / f$ 这一恒等式（称为对数导数技巧，log-derivative trick）。由于转移概率 $P(S_{t+1} \mid S_t, A_t)$ 和初始分布 $p(S_0)$ 都与 $\theta$ 无关，取对数后求导只剩下策略的部分：

$$\nabla_\theta \log P(\tau \mid \theta) = \sum_{t=0}^{T} \nabla_\theta \log \pi_\theta(A_t \mid S_t)$$

进一步利用因果性（$t$ 时刻的动作不影响之前的奖励），可以将 $R(\tau)$ 替换为 $G_t$，得到公式(5.2-4)。

REINFORCE算法

有了策略梯度定理，最直接的算法就是用蒙特卡洛采样来估计梯度。这就是Williams（1992）提出的REINFORCE算法：运行策略 $\pi_\theta$ 生成一条完整轨迹，用实际的回报 $G_t$ 代入公式(5.2-4)进行梯度更新。

REINFORCE 算法
  1. 初始化策略参数 $\theta$
  2. 对每个回合（episode）：
    1. 使用策略 $\pi_\theta$ 生成一条完整轨迹：$S_0, A_0, R_0, S_1, A_1, R_1, \ldots, S_T, A_T, R_T$
    2. 对每个时间步 $t = 0, 1, \ldots, T$：
      1. 计算回报 $G_t = \sum_{k=0}^{T-t} \gamma^k R_{t+k}$
      2. 更新参数 $\theta \leftarrow \theta + \alpha \gamma^t \nabla_\theta \log \pi_\theta(A_t \mid S_t) \cdot G_t$
REINFORCE是蒙特卡洛方法：必须等一个回合结束后才能计算 $G_t$ 并更新参数。

REINFORCE算法简洁且理论上能保证收敛到局部最优，但它有一个突出的问题：方差大。由于 $G_t$ 是单条轨迹的实际回报，它受到环境随机性和策略随机性的双重影响，不同轨迹之间的 $G_t$ 可能差异巨大。高方差意味着梯度估计的噪声很大，需要大量的样本才能让参数朝着正确的方向更新，训练效率低下。

基线与方差缩减

缓解REINFORCE高方差问题的核心技巧是引入一个基线函数 $b(s)$。将策略梯度公式中的 $G_t$ 替换为 $G_t - b(S_t)$：

$$\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{T} \nabla_\theta \log \pi_\theta(A_t \mid S_t) \cdot \bigl(G_t - b(S_t)\bigr)\right] \tag{5.2-5}$$

为什么减去基线不会引入偏差？因为对任何只依赖于状态的函数 $b(s)$，有：

$$\mathbb{E}_{A_t \sim \pi_\theta(\cdot \mid S_t)}\!\left[\nabla_\theta \log \pi_\theta(A_t \mid S_t) \cdot b(S_t)\right] = b(S_t) \sum_{a} \nabla_\theta \pi_\theta(a \mid S_t) = b(S_t) \cdot \nabla_\theta 1 = 0 \tag{5.2-6}$$

这里利用了 $\sum_a \pi_\theta(a \mid s) = 1$ 对 $\theta$ 求导等于零的事实。因此，减去 $b(S_t)$ 不改变梯度的期望值，但可以显著减小方差。

最常用的基线是状态值函数 $V(S_t)$，即从状态 $S_t$ 出发的平均回报。这时 $G_t - V(S_t)$ 衡量的是在状态 $S_t$ 下执行动作 $A_t$ 所获得的回报相对于平均水平的差异，这就是优势函数（advantage function）：

$$A_t = G_t - V(S_t) \tag{5.2-7}$$
优势函数的直觉

$A_t > 0$ 表示动作 $A_t$ 比平均水平好，应该增大其概率；$A_t < 0$ 表示动作 $A_t$ 比平均水平差，应该减小其概率。与原始的 $G_t$ 相比，优势函数去除了"状态本身好坏"的干扰——一个好状态下的普通动作不会因为 $G_t$ 碰巧较大而被不当强化，一个差状态下的好动作也不会因为 $G_t$ 整体较小而被忽视。

Actor-Critic

使用基线 $V(S_t)$ 需要知道状态值函数，但我们一般不知道真实的 $V^\pi(s)$。解决办法是：像策略一样，也用一个参数化的函数来近似它。这就引出了 Actor-Critic 框架，它同时维护两个组件：

  • Actor（演员）：策略网络 $\pi_\theta(a \mid s)$，负责选择动作。
  • Critic（评论家）：值函数网络 $V_\phi(s)$，负责评估当前状态的好坏。

Actor根据Critic提供的评价来改进策略，Critic则根据Actor产生的经验来更新自己对值函数的估计。

再看策略迭代的对应
回忆2.2.1节的策略迭代：策略评估阶段计算 $V^\pi$，策略改进阶段用 $V^\pi$ 贪心地更新 $\pi$。Actor-Critic正是这个思想的深度学习版本：Critic做策略评估（学习 $V_\phi \approx V^{\pi_\theta}$），Actor做策略改进（利用 $V_\phi$ 计算优势，梯度上升更新 $\pi_\theta$）。不同之处在于两者同时学习、同时更新，而非严格地交替到收敛。

Critic不仅提供基线，还让我们不必等到回合结束才更新。利用TD思想，可以用TD误差来近似优势函数：

$$\delta_t = R_t + \gamma V_\phi(S_{t+1}) - V_\phi(S_t) \tag{5.2-8}$$

$\delta_t$ 是优势函数 $A_t$ 的无偏估计（在Critic准确时）。它只需要一步转移 $(S_t, A_t, R_t, S_{t+1})$，无需等到回合结束。

Actor-Critic 算法（单步版本）
  1. 初始化策略参数 $\theta$（Actor）和值函数参数 $\phi$（Critic）
  2. 对每个回合：
    1. 初始化状态 $S_0$
    2. 对每个时间步 $t = 0, 1, 2, \ldots$：
      1. 根据 $\pi_\theta(\cdot \mid S_t)$ 采样动作 $A_t$
      2. 执行 $A_t$，观察 $R_t$ 和 $S_{t+1}$
      3. 计算TD误差 $\delta_t = R_t + \gamma V_\phi(S_{t+1}) - V_\phi(S_t)$
      4. 更新Critic：$\phi \leftarrow \phi - \alpha_\phi \nabla_\phi \bigl(V_\phi(S_t) - (R_t + \gamma V_\phi(S_{t+1}))\bigr)^2$
      5. 更新Actor：$\theta \leftarrow \theta + \alpha_\theta \nabla_\theta \log \pi_\theta(A_t \mid S_t) \cdot \delta_t$
Actor和Critic可以共享神经网络的底层特征提取部分，只在输出层分叉：一个输出动作概率，一个输出状态值。

Actor-Critic融合了策略梯度（直接优化策略）和值函数估计（减小方差）的优点，成为现代深度强化学习算法的基础框架。接下来介绍的PPO就是Actor-Critic的一个重要改进。

PPO：近端策略优化

Actor-Critic虽然比REINFORCE更高效，但在实际训练中仍然面临一个棘手的问题：步长的选择。学习率太大，策略可能在一次更新后急剧变差，导致后续采集的数据质量下降，形成恶性循环；学习率太小，训练速度又过慢。

从参数空间到策略空间：步长问题的根源

标准梯度上升在参数空间中以欧氏距离度量步长：$\|\theta_{\text{new}} - \theta_{\text{old}}\|_2$。但参数空间中的距离与策略分布之间的差异并不对应。考虑一个 softmax 策略：

  • 如果某个状态下两个动作的偏好值 $h_\theta(s,a_0) = 100$ 和 $h_\theta(s,a_1) = 100$，那么 $\pi_\theta(a_0|s) = 0.5$。将 $h_\theta(s,a_0)$ 增大 $0.01$，策略几乎不变。
  • 但如果偏好值为 $h_\theta(s,a_0) = 0$ 和 $h_\theta(s,a_1) = 0$，同样增大 $0.01$，策略变化也很小；而将 $h_\theta(s,a_0)$ 增大 $5$，策略从均匀分布变为 $\pi_\theta(a_0|s) \approx 0.99$，发生了剧烈变化。

同样大小的参数更新，在不同的参数区域可能导致截然不同的策略变化幅度。这意味着固定学习率无法可靠地控制策略变化的大小，这正是训练不稳定的根源。

一个自然的想法是：不要在参数空间里度量步长，而是直接在策略分布空间里度量步长。也就是说，控制新旧策略之间的分布差异，而不是参数的欧氏距离。衡量两个概率分布差异的标准工具是 $\text{KL}$ 散度：

$$\text{KL}\big(\pi_{\theta_{\text{old}}}(\cdot|s) \| \pi_\theta(\cdot|s)\big) = \sum_a \pi_{\theta_{\text{old}}}(a|s) \log \frac{\pi_{\theta_{\text{old}}}(a|s)}{\pi_\theta(a|s)}$$

沿这条思路，研究者们发展出了三代算法，逐步将"信任区域"的思想从理论变为实用。

自然策略梯度

Kakade（2001）首先指出，在策略优化中应当使用自然梯度替代普通梯度。自然梯度用 $\text{Fisher}$ 信息矩阵 $F(\theta)$ 对梯度方向进行修正：

$$\theta \leftarrow \theta + \alpha\, F(\theta)^{-1} \nabla_\theta J(\theta) \tag{5.2-8a}$$

其中 $\text{Fisher}$ 信息矩阵定义为：

$$F(\theta) = \mathbb{E}_{s \sim d^{\pi_\theta},\, a \sim \pi_\theta}\!\left[\nabla_\theta \log \pi_\theta(a|s)\, \nabla_\theta \log \pi_\theta(a|s)^\top\right]$$

$F(\theta)$ 描述了参数 $\theta$ 的微小变化对策略分布的影响程度。用 $F(\theta)^{-1}$ 左乘梯度后，每一步更新在 $\text{KL}$ 散度意义下的步长近似恒定，不再依赖参数所处的位置。直觉上，在策略对参数敏感的方向走得更谨慎，在策略对参数不敏感的方向走得更大胆。

TRPO：信任区域策略优化

自然策略梯度的步长 $\alpha$ 仍然是一个固定的超参数。Schulman et al.（2015）提出的 TRPO（Trust Region Policy Optimization）将步长控制从固定值改为硬约束：直接要求新旧策略的 $\text{KL}$ 散度不超过一个阈值 $\delta$。

$$\max_\theta \;\; \mathbb{E}_{s,a \sim \pi_{\theta_{\text{old}}}}\!\left[\frac{\pi_\theta(a|s)}{\pi_{\theta_{\text{old}}}(a|s)}\, A^{\pi_{\theta_{\text{old}}}}(s,a)\right] \quad \text{s.t.} \quad \mathbb{E}_{s \sim \pi_{\theta_{\text{old}}}}\!\left[\text{KL}\big(\pi_{\theta_{\text{old}}}(\cdot|s) \| \pi_\theta(\cdot|s)\big)\right] \leq \delta \tag{5.2-8b}$$

与自然策略梯度相比，TRPO的硬约束更加鲁棒：无论目标函数的曲率如何变化，更新幅度都不会超出信任区域。在实现上，TRPO用 $\text{Fisher}$ 信息矩阵的二次近似来表示 $\text{KL}$ 约束，然后通过共轭梯度法求解方向、线搜索确定步长。这套二阶优化流程保证了理论上的单调改进，但实现较为复杂，计算成本也较高。

PPO：用裁剪替代约束

PPO（Proximal Policy Optimization，Schulman et al., 2017）的核心观察是：TRPO的 $\text{KL}$ 约束虽然有效，但实现代价太大。能否用一种更简单的方式达到类似效果？PPO给出的答案是：通过裁剪目标函数来限制策略更新的幅度，无需显式的约束或二阶计算。

从自然策略梯度到PPO的演进

自然策略梯度（Kakade, 2001）：用 $\text{Fisher}$ 信息矩阵修正梯度方向，在 $\text{KL}$ 散度空间中走等大的步。但步长 $\alpha$ 仍是固定超参数。

TRPO（Schulman et al., 2015）：将步长控制升级为硬约束 $\text{KL} \leq \delta$，保证每次更新都在信任区域内。但需要共轭梯度和线搜索，实现复杂。

PPO（Schulman et al., 2017）：用裁剪目标函数替代 $\text{KL}$ 约束，几行代码即可实现，效果与TRPO相当。成为目前应用最广泛的策略优化算法。

重要性采样比率

PPO的关键思想是重用旧策略 $\pi_{\theta_{\text{old}}}$ 收集的数据来更新新策略 $\pi_\theta$。为此引入重要性采样比率：

$$r_t(\theta) = \frac{\pi_\theta(A_t \mid S_t)}{\pi_{\theta_{\text{old}}}(A_t \mid S_t)} \tag{5.2-9}$$

当 $\theta = \theta_{\text{old}}$ 时，$r_t(\theta) = 1$。如果新策略增大了动作 $A_t$ 的概率，则 $r_t > 1$；反之 $r_t < 1$。

用 $r_t(\theta)$ 可以将策略梯度目标改写为：

$$L^{IS}(\theta) = \mathbb{E}\left[r_t(\theta) \cdot A_t\right] \tag{5.2-10}$$

这个目标允许我们用旧数据来估计新策略的性能。但如果不加限制地最大化它，$r_t(\theta)$ 可能变得很大，导致更新过于激进。

PPO-Clip目标函数

PPO通过裁剪 $r_t(\theta)$ 来解决这个问题：

$$L^{\text{CLIP}}(\theta) = \mathbb{E}\left[\min\!\Big(r_t(\theta)\, A_t,\;\; \text{clip}\big(r_t(\theta),\; 1-\epsilon,\; 1+\epsilon\big)\, A_t\Big)\right] \tag{5.2-11}$$

其中 $\epsilon$ 是裁剪参数，通常取 $0.1$ 到 $0.2$。$\text{clip}(r, 1-\epsilon, 1+\epsilon)$ 将比率限制在 $[1-\epsilon, 1+\epsilon]$ 范围内。

取 $\min$ 的效果是：

  • 当 $A_t > 0$（动作比平均好）：目标函数鼓励增大 $r_t$（即增大该动作的概率），但一旦 $r_t > 1+\epsilon$，裁剪使得目标不再增长，阻止过度增大。
  • 当 $A_t < 0$（动作比平均差）：目标函数鼓励减小 $r_t$（即减小该动作的概率），但一旦 $r_t < 1-\epsilon$，裁剪使得目标不再增长，阻止过度减小。

这样，PPO-Clip在不使用二阶优化的前提下，实现了类似信任区域的效果：每次更新的幅度被自动限制在 $\epsilon$ 的范围内。

图 5.2-1：PPO-Clip目标函数的示意图。蓝色实线是 $L^{\text{CLIP}}$，灰色虚线是未裁剪的 $rA$。裁剪区域内目标函数不再随 $r$ 变化，阻止策略更新过大。
PPO-Clip 算法
  1. 初始化策略参数 $\theta$（Actor）和值函数参数 $\phi$（Critic）
  2. 对每次迭代：
    1. 用当前策略 $\pi_{\theta_{\text{old}}}$ 收集一批轨迹数据
    2. 对每条轨迹中的每步，计算优势估计 $\hat{A}_t$（通常使用GAE，见下文）
    3. 对收集到的数据进行 $K$ 轮（epoch）小批量梯度更新：
      1. 计算 $r_t(\theta) = \pi_\theta(A_t \mid S_t) / \pi_{\theta_{\text{old}}}(A_t \mid S_t)$
      2. 计算裁剪目标 $L^{\text{CLIP}}(\theta)$（公式(5.2-11)）
      3. 更新Actor：$\theta \leftarrow \theta + \alpha_\theta \nabla_\theta L^{\text{CLIP}}(\theta)$
      4. 更新Critic：最小化 $\bigl(V_\phi(S_t) - G_t\bigr)^2$
    4. 令 $\theta_{\text{old}} \leftarrow \theta$
PPO对同一批数据进行多轮更新（通常 $K=3 \sim 10$），大幅提高了数据利用率。裁剪机制保证即使多轮更新，策略也不会偏离太远。

广义优势估计（GAE）

PPO实践中通常使用广义优势估计（Generalized Advantage Estimation, GAE, Schulman et al., 2016）来计算 $\hat{A}_t$。GAE的公式与3.1.4节推导的 $\lambda$-回报 TD 误差表示（式(3.1-14)）形式完全相同——将 $\lambda$-回报的思想应用于优势函数估计：

$$\hat{A}_t^{\text{GAE}} = \sum_{l=0}^{T-t} (\gamma\lambda)^l \delta_{t+l} \tag{5.2-12}$$

其中 $\delta_t = R_t + \gamma V_\phi(S_{t+1}) - V_\phi(S_t)$。参数 $\lambda \in [0, 1]$ 控制偏差-方差的平衡：$\lambda = 0$ 时 GAE 退化为单步 TD 误差 $\delta_t$（低方差、高偏差）；$\lambda = 1$ 时 GAE 等价于蒙特卡洛回报减去基线（高方差、低偏差）。实践中 $\lambda = 0.95$ 是常见的选择。实现上，GAE 在收集的轨迹上从后往前递推计算：$\hat{A}_t^{\text{GAE}} = \delta_t + \gamma\lambda\,\hat{A}_{t+1}^{\text{GAE}}$，只需一次反向遍历即可完成。

为什么PPO是默认选择

PPO自2017年提出以来，迅速成为学术研究和工程实践中最常用的策略优化算法。它的核心优势在于：

  • 实现简洁。相比TRPO需要共轭梯度法和线搜索，PPO只需在标准的随机梯度上升中加入一个裁剪操作，几行代码即可实现。
  • 训练稳定。裁剪机制提供了隐式的信任区域，避免了策略的突然崩溃。超参数（$\epsilon$, $K$, $\lambda$）在大部分任务中使用默认值就能取得不错的效果。
  • 数据效率。通过重要性采样和多轮更新，PPO可以对一批数据进行多次利用，比REINFORCE等纯在策略方法高效得多。
  • 适用范围广。PPO对离散和连续动作空间都适用，在Atari游戏、机器人控制、大语言模型对齐（RLHF）以及各种管理科学场景中都有成功应用。

例子：机票动态定价的PPO实现

用一个简化的机票定价问题演示 PPO 的完整实现。航空公司在出发前 $T=30$ 天内每天设定票价，面对价格敏感的随机需求，目标是最大化总收入。

  • 状态：$s = (\text{剩余座位} / 50,\; \text{剩余天数} / 30)$，归一化到 $[0,1]^2$
  • 动作：从4个价格档位中选择——$a \in \{200, 400, 600, 800\}$ 元
  • 需求：每天的购票人数 $D \sim \text{Poisson}(\mu)$，其中 $\mu = 5 \cdot e^{-\text{price}/400}$（价格越高，需求越低）
  • 奖励：$R = \text{price} \times \min(D, \text{剩余座位})$
import torch
import torch.nn as nn
import numpy as np

# --- 环境 ---
class TicketPricingEnv:
    def __init__(self, capacity=50, horizon=30):
        self.capacity = capacity
        self.horizon = horizon

    def reset(self):
        self.seats = self.capacity
        self.day = 0
        return self._obs()

    def _obs(self):
        return [self.seats / self.capacity, (self.horizon - self.day) / self.horizon]

    def step(self, action_idx):
        price = [200, 400, 600, 800][action_idx]
        lam = 5 * np.exp(-price / 400)          # 需求强度
        demand = np.random.poisson(lam)
        sold = min(demand, self.seats)
        reward = price * sold
        self.seats -= sold
        self.day += 1
        done = (self.day >= self.horizon) or (self.seats <= 0)
        return self._obs(), reward, done

# --- Actor-Critic 网络 ---
class ActorCritic(nn.Module):
    def __init__(self):
        super().__init__()
        self.shared = nn.Sequential(nn.Linear(2, 64), nn.ReLU())
        self.actor = nn.Linear(64, 4)       # 4个价格档位
        self.critic = nn.Linear(64, 1)      # V(s)

    def forward(self, x):
        h = self.shared(x)
        return torch.softmax(self.actor(h), dim=-1), self.critic(h)

# --- PPO 训练 ---
env = TicketPricingEnv()
model = ActorCritic()
optimizer = torch.optim.Adam(model.parameters(), lr=3e-4)
gamma, lam, eps_clip = 0.99, 0.95, 0.2

for epoch in range(300):
    # 收集一批轨迹
    states, actions, rewards, log_probs_old, values = [], [], [], [], []
    for _ in range(8):  # 8条轨迹
        obs = env.reset()
        done = False
        while not done:
            s_t = torch.tensor([obs], dtype=torch.float32)
            with torch.no_grad():
                probs, val = model(s_t)
            dist = torch.distributions.Categorical(probs)
            a = dist.sample()
            obs2, r, done = env.step(a.item())
            states.append(obs); actions.append(a.item())
            rewards.append(r); log_probs_old.append(dist.log_prob(a).item())
            values.append(val.item()); obs = obs2
        values.append(0)  # terminal value

    # GAE计算（式3.1-14的递推形式）
    advantages, returns = [], []
    gae = 0
    for t in reversed(range(len(rewards))):
        delta = rewards[t] + gamma * values[t+1] - values[t]
        gae = delta + gamma * lam * gae
        advantages.insert(0, gae)
        returns.insert(0, gae + values[t])

    # PPO更新（3轮）
    s_batch = torch.tensor(states, dtype=torch.float32)
    a_batch = torch.tensor(actions, dtype=torch.long)
    adv = torch.tensor(advantages, dtype=torch.float32)
    adv = (adv - adv.mean()) / (adv.std() + 1e-8)
    ret = torch.tensor(returns, dtype=torch.float32)
    old_lp = torch.tensor(log_probs_old, dtype=torch.float32)

    for _ in range(3):
        probs, vals = model(s_batch)
        dist = torch.distributions.Categorical(probs)
        new_lp = dist.log_prob(a_batch)
        ratio = (new_lp - old_lp).exp()
        # PPO-Clip损失
        loss_actor = -torch.min(ratio * adv,
                                ratio.clamp(1-eps_clip, 1+eps_clip) * adv).mean()
        loss_critic = nn.functional.mse_loss(vals.squeeze(), ret)
        loss = loss_actor + 0.5 * loss_critic
        optimizer.zero_grad(); loss.backward(); optimizer.step()

    if epoch % 50 == 0:
        # 评估
        obs = env.reset(); total = 0; done = False
        while not done:
            with torch.no_grad():
                probs, _ = model(torch.tensor([obs], dtype=torch.float32))
            obs, r, done = env.step(probs.argmax().item())
            total += r
        print(f"Epoch {epoch}, 总收入={total:.0f}")

# 查看学到的定价策略
print("\n学到的定价策略：")
for seats in [50, 30, 10, 5]:
    for day_left in [25, 15, 5]:
        obs = [seats/50, day_left/30]
        with torch.no_grad():
            probs, v = model(torch.tensor([obs], dtype=torch.float32))
        price = [200,400,600,800][probs.argmax().item()]
        print(f"  剩余{seats:2d}座, 还有{day_left:2d}天 → 定价{price}元 (V={v.item():.0f})")

可以观察到学到的策略符合收益管理的直觉：座位充裕时低价吸引客流，座位紧张时提高价格。代码中的 GAE 计算直接使用了3.1.4节式(3.1-14)的递推形式。
