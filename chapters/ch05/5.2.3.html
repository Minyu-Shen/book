5.2.3 例子：机票动态定价

问题背景

航空公司的收益管理是一个经典的序贯决策问题。一架航班有固定数量的座位，在出发前的若干天内，航空公司每天需要决定明天以什么价格售票。定价过高会导致座位卖不出去，出发时空座浪费；定价过低则白白损失了潜在收入。最优定价策略需要综合考虑当前剩余座位数、距出发的天数以及不确定的市场需求。

传统的收益管理方法通常先估计一个参数化的需求模型（如用历史数据拟合需求曲线），再基于这个模型求解动态规划。但在实际中，需求模式可能随季节、竞争对手、宏观经济等因素变化，需求模型的准确性难以保证。强化学习提供了另一种思路：不假设需求分布已知，而是让算法通过反复试错，直接从与环境的交互中学习定价策略。下面用 PPO 来求解这个问题。

MDP 建模

将这个问题建模为以下 MDP。每个时间步对应一天，航空公司在每天结束时观察状态，为明天设定票价。

机票动态定价 MDP
  • 状态：$s_t = \bigl(\text{剩余座位数} / 50,\;\; \text{剩余天数} / 30\bigr) \in [0,1]^2$
  • 动作：从4个价格档位中选择，$a_t \in \{200, 400, 600, 800\}$ 元
  • 需求：每天购票人数 $D_t \sim \text{Poisson}(\lambda_t)$，需求强度随价格递减：
$$\lambda_t = 8 \cdot e^{-\text{price}_t / 400} \tag{5.2-14}$$
  • 转移：$\text{售出}_t = \min(D_t,\, \text{剩余座位}_t)$，剩余座位减少相应数量
  • 奖励：当天售票收入 $R_t = \text{price}_t \times \text{售出}_t$
  • 回合终止：30天用完或座位售罄

折扣因子 $\gamma = 0.99$。RL 智能体不知道式(5.2-14)的需求模型，只能通过观察每天的销售结果来学习。

需求模型（式(5.2-14)）描述了价格弹性：价格越高，需求强度 $\lambda_t$ 越低。例如，$\text{price} = 200$ 时 $\lambda \approx 4.9$（平均每天近5人购票），$\text{price} = 800$ 时 $\lambda \approx 1.1$（平均约1人）。各价格档的期望日收入为：

从单日期望收入看，400元档最优（1,177元/天）。但注意：30天的总需求约为 $2.67 \times 30 \approx 80$ 人次，远超50个座位。这意味着座位是稀缺资源——如果一味选择低价，虽然日收入高，但大量座位被廉价售出，总收入反而不如用高价慢慢卖。例如始终选600元档，日需求1.79、30天总需求约54，刚好卖完50座，总收入 $600 \times 50 = 30{,}000$ 元——远高于始终选400元的 $400 \times 50 = 20{,}000$ 元。动态规划（2.2节）在需求模型已知时可以精确求解，最优策略的期望总收入约为 $30{,}200$ 元。但我们假装不知道需求模型，用 PPO 来学习。

为什么选择离散价格？本节的动作空间是4个离散档位，PPO 用 softmax 参数化输出每个价格的选择概率。这与5.1.3节 DQN 的库存管理例子设置类似，方便直接对比两种算法。连续定价（将价格建模为连续动作 $a \in [200, 800]$）将在5.3节和5.4节中展示。

Step 1：基础 PPO 训练

先用标准配置的 PPO（裁剪参数 $\epsilon = 0.2$，每批数据更新 $K = 3$ 轮，GAE $\lambda = 0.95$）来求解这个定价问题。策略网络的输入是归一化的状态向量 $(s_1, s_2) = (\text{剩余座位}/50,\; \text{剩余天数}/30)$，经过一个64维隐藏层，输出4个价格的 softmax 概率。

Step 2：数据复用轮数 $K$ 的影响

PPO 相比 REINFORCE 的一个核心优势是数据复用：收集一批轨迹后，可以用同一批数据更新策略多轮（$K$ 轮），而不是用完即丢。裁剪机制保证了即使多次更新，新策略也不会偏离旧策略太远。

下面对比 $K = 1$（每批数据只更新一次，接近 REINFORCE）与 $K = 3$（标准设置）。其余超参数相同（$\epsilon = 0.2$，$\lambda = 0.95$）。

$K=1$ 时每批数据只用一次就丢弃，样本效率低，训练进展慢。$K=3$ 将同一批数据复用三轮，利用裁剪机制防止策略偏移过远，在样本效率和稳定性之间取得了更好的平衡。从阴影带可以看到，$K=3$ 不仅均值更高，方差也更小。PPO 论文推荐 $K$ 取3到10之间；在更复杂的问题上，$K$ 过大会导致重要性采样比率 $r_t(\theta)$ 偏离1太远，即使有裁剪也会损害学习质量，但在本例这样的低维问题上这一效应不明显。

Step 3：裁剪机制的效果

裁剪是 PPO 最核心的设计。为了直观展示它的作用，下面将裁剪关闭（使用原始的重要性采样策略梯度）与裁剪开启（$\epsilon = 0.2$）做对比。两者其余配置完全相同（$K=3$，GAE $\lambda = 0.95$）。

在本例中，两者的均值接近——这是因为 GAE 已经使优势估计较为平滑，降低了裁剪的必要性。裁剪的主要价值体现在稳定性上：消融实验（图 5.2-11）中对比 +多轮（无裁剪、$K=3$、无 GAE）与 +裁剪+多轮，后者的方差带远窄于前者。换言之，当优势估计噪声较大（如关闭 GAE）且多轮更新放大策略漂移时，裁剪的约束作用最为关键。$\epsilon = 0.2$ 是 PPO 原始论文（Schulman et al., 2017）推荐的默认值，在大多数问题上无需调整。

消融实验：每项设计贡献了多少？

5.2.1节介绍的 PPO 包含三项关键设计：多轮更新（$K > 1$）、裁剪机制（$\epsilon$-clip）和 GAE（$\lambda > 0$）。它们分别解决不同的问题：多轮更新提高样本效率，裁剪防止策略更新过大，GAE 平衡优势估计的偏差和方差。下面通过消融实验看清每项设计的贡献。

三项设计的开关为：多轮更新（开：$K=3$，关：$K=1$）、裁剪（开：$\epsilon=0.2$，关：不裁剪）、GAE（开：$\lambda=0.95$，关：$\lambda=0$，即纯单步 TD）。$2^3 = 8$ 种组合的训练曲线如下，勾选复选框来显示或隐藏各配置。

图 5.2-11   PPO 三项设计的消融实验。灰色为基线（无裁剪、$K=1$、纯 TD），绿色为完整 PPO（裁剪 + $K=3$ + GAE）。勾选复选框可查看中间配置。

从消融实验中可以读出以下观察（注意每条曲线的阴影宽度，它反映了不同随机种子下的训练方差）：

  • GAE 的贡献体现在均值：对比基线（灰色，无 GAE、$K=1$、无裁剪）与 +GAE（红色），加入 GAE 后平均收入从约 $27{,}000$ 跃升到 $29{,}000$，提升最为显著。GAE 用多步信息估计优势函数，大幅降低了纯单步 TD 的偏差。
  • 裁剪的贡献体现在方差：对比 +多轮（黄色）与 +裁剪+多轮（青色），两者均值接近，但 +多轮 的阴影带远宽于 +裁剪+多轮，说明没有裁剪保护的多轮更新非常不稳定——某些种子表现很好，另一些则出现策略崩溃。裁剪并不提升均值上限，而是消除了最坏情况。
  • 多轮更新提高样本效率：对比基线（$K=1$）与 +多轮（$K=3$），后者收敛更快，但不稳定性也随之增加。这正是裁剪要解决的问题。
  • 三者组合最优：完整 PPO（绿色）兼具高均值和窄方差带，说明三项设计在不同层面互为补充。
核心观察：PPO 的三项设计各有侧重：GAE 提升平均性能（改善优势估计的质量），裁剪提升稳定性（约束多轮更新的偏移），多轮更新提升样本效率。仅看单次训练的曲线可能忽略裁剪的价值；多次训练取平均并观察方差带，才能看到裁剪将不稳定的多轮更新变为可靠的收敛过程。

结果分析

训练收敛后，PPO 学到的定价策略呈现出明显的座位稀缺性驱动特征：

  • 座位效应（主效应）：剩余座位的稀缺程度是定价的主要驱动因素。在本例中需求旺盛（总需求约80 > 50座），即使座位相对充裕（如40座/25天），策略也选择600元而非更低的400元——因为以600元的需求速率（1.79人/天），刚好能在剩余天数内卖完。座位更加稀缺时（如5座/25天），策略进一步提升到800元，因为有充裕的时间逐张高价售出。
  • 时间与座位的交互：时间效应不能脱离座位数量来讨论。同样剩余5个座位，距出发25天时策略坚定选择800元（有时间等高价成交）；只剩3天时策略转向600—800元之间的折中（时间紧迫，需要平衡单价和成交概率）。读者可以在交互演示1中切换不同状态来验证这些规律。

图 5.2-12 将收敛后的策略在整个状态空间上可视化为热力图，直观展示了上述规律。

图 5.2-12   PPO 收敛后的定价策略热力图。横轴为剩余天数，纵轴为剩余座位数，颜色反映该状态下的期望定价。黄色区域（座位相对充裕）以600元为主；红色区域（座位稀缺）以800元为主。由于需求旺盛，策略几乎不选择400元或200元。

热力图显示策略几乎只在600元和800元之间切换，决定因素是座位与剩余时间的比值：座位少、时间多时选800元（慢慢高价卖出），座位多或时间紧时回落到600元（加快周转）。这与传统收益管理文献中的最优定价结论一致，说明 PPO 能够在不知道需求分布的情况下发现合理的状态依赖定价策略。

与5.1.3节库存管理例子的对比。5.1.3节的库存管理用 DQN 求解，动作是离散的补货量。本节的机票定价也使用离散动作（4个价格档位），但用 PPO 求解。两个例子的核心区别在于算法思路：DQN 学习 $Q(s,a)$ 后用 $\arg\max$ 选动作；PPO 直接学习策略概率 $\pi_\theta(a|s)$，不需要先估计值函数再推导策略。5.1.3节的消融实验测试了 DQN 的三项改进（Double、Dueling、PER），本节则测试了 PPO 的三项设计（裁剪、多轮更新、GAE），形式上构成对照。

训练代码

以下代码可完整复现本节所有训练曲线和热力图。依赖：PyTorch、NumPy。

import torch, torch.nn as nn, numpy as np, math

# ==================== 环境 ====================
class TicketPricingEnv:
    PRICES  = [200, 400, 600, 800]
    LAMBDAS = [8.0 * math.exp(-p / 400.0) for p in PRICES]

    def __init__(self, total_seats=50, total_days=30):
        self.total_seats = total_seats
        self.total_days  = total_days

    def reset(self):
        self.seats     = self.total_seats
        self.days_left = self.total_days
        return (self.seats / self.total_seats,
                self.days_left / self.total_days)

    def step(self, a):
        price  = self.PRICES[a]
        demand = np.random.poisson(self.LAMBDAS[a])
        sold   = min(demand, self.seats)
        reward = price * sold
        self.seats     -= sold
        self.days_left -= 1
        done = self.days_left <= 0 or self.seats <= 0
        return (self.seats / self.total_seats,
                self.days_left / self.total_days), reward, done

# ==================== Actor-Critic 网络 ====================
class ActorCritic(nn.Module):
    def __init__(self, hidden=64):
        super().__init__()
        self.pi = nn.Sequential(
            nn.Linear(2, hidden), nn.Tanh(),
            nn.Linear(hidden, hidden), nn.Tanh(),
            nn.Linear(hidden, 4))
        self.v = nn.Sequential(
            nn.Linear(2, hidden), nn.Tanh(),
            nn.Linear(hidden, hidden), nn.Tanh(),
            nn.Linear(hidden, 1))

    def forward(self, x):
        return self.pi(x), self.v(x).squeeze(-1)

# ==================== GAE ====================
def compute_gae(rewards, values, dones, gamma, lam):
    n = len(rewards)
    adv = [0.0] * n
    g = 0.0
    for t in range(n - 1, -1, -1):
        nv = 0.0 if (t == n-1 or dones[t]) else values[t+1]
        delta = rewards[t] + gamma * nv - values[t]
        g = delta if dones[t] else delta + gamma * lam * g
        adv[t] = g
    return adv

# ==================== PPO 训练 ====================
REWARD_SCALE = 1000.0

def train_ppo(clip_eps=0.2, K=3, gae_lambda=0.95,
              n_epochs=200, batch_size=32, gamma=0.99,
              lr=1e-3, seed=42):
    torch.manual_seed(seed)
    np.random.seed(seed)

    env   = TicketPricingEnv()
    model = ActorCritic()
    opt   = torch.optim.Adam(model.parameters(), lr=lr)
    use_clip = clip_eps is not None
    obs_buf  = torch.zeros(1, 2)
    rew_hist = []

    for epoch in range(n_epochs):
        S, A, R, D, LP, V = [], [], [], [], [], []
        total_reward = 0.0

        for _ in range(batch_size):          # 采集 batch_size 个完整回合
            o = env.reset(); ep_r = 0.0
            while True:
                obs_buf[0, 0], obs_buf[0, 1] = o
                with torch.no_grad():
                    logits, v = model(obs_buf)
                dist = torch.distributions.Categorical(logits=logits[0])
                a = dist.sample()
                S.append(o); A.append(a.item())
                LP.append(dist.log_prob(a).item())
                V.append(v.item())
                o, r, d = env.step(a.item())
                R.append(r / REWARD_SCALE); D.append(d)
                ep_r += r
                if d: break
            total_reward += ep_r
        rew_hist.append(total_reward / batch_size)

        # GAE 优势估计
        advs  = compute_gae(R, V, D, gamma, gae_lambda)
        s_t   = torch.tensor(S)
        a_t   = torch.tensor(A, dtype=torch.long)
        olp_t = torch.tensor(LP)
        adv_t = torch.tensor(advs, dtype=torch.float32)
        ret_t = adv_t + torch.tensor(V, dtype=torch.float32)
        if adv_t.std() > 1e-8:
            adv_t = (adv_t - adv_t.mean()) / (adv_t.std() + 1e-8)

        # K 轮策略更新
        for _ in range(K):
            logits, v = model(s_t)
            dist  = torch.distributions.Categorical(logits=logits)
            nlp   = dist.log_prob(a_t)
            ratio = torch.exp(nlp - olp_t)
            if use_clip:
                s1 = ratio * adv_t
                s2 = torch.clamp(ratio, 1-clip_eps, 1+clip_eps) * adv_t
                pi_loss = -torch.min(s1, s2).mean()
            else:
                pi_loss = -(ratio.clamp(0.01, 100) * adv_t).mean()
            v_loss = (ret_t - v).pow(2).mean()
            loss = pi_loss + 0.5 * v_loss - 0.01 * dist.entropy().mean()
            opt.zero_grad(); loss.backward()
            nn.utils.clip_grad_norm_(model.parameters(), 0.5)
            opt.step()

        if (epoch+1) % 50 == 0:
            print(f"epoch {epoch+1:3d}  reward = {rew_hist[-1]:,.0f}")

    return rew_hist, model

# ==================== 运行 ====================
rewards, model = train_ppo()          # 标准 PPO
rewards_K1, _  = train_ppo(K=1)       # K=1 对比
rewards_nc, _  = train_ppo(clip_eps=None)  # 无裁剪对比
