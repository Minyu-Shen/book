5.1.3 例子：库存管理

前两节分别介绍了 DQN 的基本算法（5.1.1节）和三项关键改进：Double DQN、Dueling 架构、优先经验回放（5.1.2节）。理论讲完了，但一个核心问题仍然悬而未决：这些改进究竟能带来多大的实际收益？它们各自解决了什么具体问题？本节通过一个库存管理的例子，让读者在交互演示中逐一观察每项改进的效果，最终通过消融实验（ablation study）看清它们的互补关系。

问题建模

考虑一个零售商每天需要决定向供应商订多少货。需求是随机的，库存太多会产生持有成本，库存不足则面临缺货损失。这是运筹学中经典的报童问题（newsvendor problem）的多期推广，也是管理科学中最自然的序贯决策场景之一。

库存管理 MDP
  • 状态：当前库存水平 $s \in \{0, 1, \ldots, 30\}$，共31个离散状态
  • 动作：订货量 $a \in \{0, 5, 10, 15, 20\}$，共5个离散档位
  • 需求：$D \sim \text{Poisson}(8)$，每天独立抽取
  • 状态转移：$s' = \min\!\big(\max(s + a - D,\; 0),\; 30\big)$
  • 奖励：
$$R = \underbrace{10 \cdot \min(s, D)}_{\text{销售收入}} - \underbrace{1 \cdot (s + a - D)^+}_{\text{持有成本}} - \underbrace{5 \cdot (D - s)^+}_{\text{缺货惩罚}} - \underbrace{2 \cdot a}_{\text{订货成本}} \tag{5.1-13}$$

其中 $(x)^+ = \max(x, 0)$。库存上限为30，下限为0（不允许欠货）。折扣因子 $\gamma = 0.99$。

奖励函数（式(5.1-13)）的四项分别对应零售商面临的四种经济因素。销售收入取决于实际能卖出的量 $\min(s, D)$，即库存和需求中较小者。持有成本按期末剩余库存 $(s + a - D)^+$ 计算。缺货惩罚按未满足的需求 $(D - s)^+$ 计算，单位惩罚（5元）高于持有成本（1元），反映了"缺货比过剩代价更高"的常见商业逻辑。订货成本与订货量 $a$ 成正比。

这个问题虽然只有31个状态和5个动作，用动态规划可以精确求解（最优策略的长期平均回合奖励约为 $50.2$），但它包含了管理科学问题中的典型特征：随机需求、库存约束、多种成本的权衡。更重要的是，它足够小，使我们能在浏览器中实时运行训练并可视化每项 DQN 改进的效果。

Step 1：基础 DQN

先用最基本的 DQN（经验回放 + 目标网络，不含任何改进）来解决这个库存管理问题。Q网络的输入是归一化的库存水平 $s/30$，经过两个64维隐藏层，输出5个动作的 $Q$ 值。

可以观察到几个现象。首先，训练初期奖励波动剧烈，直到大约第120回合左右才趋于稳定。其次，热力图中的 $Q$ 值分布逐渐呈现出合理的模式：库存低时（$s \leq 5$），$Q$ 值最高的动作是大量订货（$a = 15$ 或 $a = 20$）；库存高时（$s \geq 20$），最优动作是不订货（$a = 0$）。这符合直觉。但如果仔细观察 $Q$ 值的绝对大小，会发现它们普遍偏高，这正是 $\max$ 运算导致的过高估计问题。

Step 2：加入 Double DQN

在库存管理中，过高估计有一个具体的危害：当库存较低且需求不确定时，基础 DQN 容易高估大订货量的 $Q$ 值。例如在 $s=2$ 时，$a=20$ 的实际价值可能并不比 $a=15$ 高多少（因为大量订货会带来额外的持有成本），但由于估计噪声，$\max$ 运算倾向于选出被高估的那个。这种偏差会导致智能体系统性地过度订货。

Double DQN 的修复方式在5.1.2节已经介绍：用在线网络选动作，用目标网络估值。下面的可视化直接对比两种方法的训练效果。

Step 3：加入 Dueling 架构

Dueling 架构将 $Q(s,a)$ 分解为状态价值 $V(s)$ 和优势函数 $A(s,a)$ 两个分支。在库存管理中，$V$ 分支负责回答"当前库存水平本身好不好"，$A$ 分支回答"在当前库存下，每个订货量相对于平均水平好多少"。

$V$ 分支能快速学到一个关键模式：中等库存水平（$s \approx 8 \sim 12$）最好，因为既不会因库存过多而承受高持有成本，也不会因库存过少而遭受缺货惩罚。$A$ 分支则捕捉了更细粒度的信息：在库存为0时，$A(0, 20)$ 远大于 $A(0, 0)$；在库存为25时，$A(25, 0)$ 远大于 $A(25, 20)$。

Dueling 架构的优势在于：$V(s)$ 的学习可以利用所有动作的经验，不需要为每个动作单独积累样本。在训练初期，即使某些 $(s,a)$ 对还没有被充分探索，$V(s)$ 也能从其他动作的转移中获得信息。这使得 Dueling 架构在训练早期就能给出较好的 $Q$ 值估计，尤其在动作空间较大时效果更为显著。

Step 4：加入优先经验回放

在库存管理中，缺货事件是一个典型的"稀有但重要"的经验。当库存为0且需求很高时（如 $D = 15$），智能体会遭受巨大的缺货惩罚 $5 \times 15 = 75$ 元，此时的 TD 误差也会很大。然而在均匀采样的经验回放中，这类转移被抽到的概率与普通转移相同，导致网络需要很多回合才能充分学习这类"极端情况"下的正确 $Q$ 值。

优先经验回放根据 TD 误差的大小分配采样概率：TD 误差越大的转移，被回放的概率越高。这意味着那些缺货事件（高 TD 误差）会被反复回放，加速了网络对这类关键经验的学习。

消融实验：每项改进贡献了多少？

前面逐一展示了三项改进各自的效果。最后一个关键问题是：将它们组合起来会怎样？是否存在重复收益（两项改进解决了同一个问题），还是互补收益（各自解决不同问题，收益叠加）？

消融实验（ablation study）是回答这类问题的标准方法：从完整配置开始，逐一关闭各项改进，观察性能变化。下面的交互演示预计算了所有 $2^3 = 8$ 种组合的训练曲线，读者可以通过勾选/取消复选框来对比任意组合。

从消融实验中可以得出以下结论：

  • PER 对收敛速度的提升最为显著。在库存管理中，缺货和过度订货是两类关键的"信息量大"的经验，PER 确保这些经验被充分学习
  • Double DQN 主要改善了稳定性。训练曲线的波动更小，最终收敛值更接近最优
  • Dueling 架构加速了早期学习。$V$ 分支快速学到"中等库存最好"的模式，即使 $A$ 分支尚未收敛，整体 $Q$ 值也已比较合理
  • 三项改进是互补的。全部启用时（绿色曲线），收敛速度最快、最终性能最高、波动最小。这与 Rainbow（Hessel et al., 2018）的实验结论一致
三项 DQN 改进各自解决不同的问题，互补性强：
Double DQN 纠正过高估计，Dueling 加速状态价值学习，PER 聚焦关键经验。
在管理科学的离散决策问题中，推荐同时启用三者。

完整 Python 实现

通过上面的交互演示，读者已经直观地理解了每项改进的作用。下面给出将三项改进（Double DQN + Dueling 架构 + 优先经验回放）整合在一起的完整 PyTorch 代码，供实际使用参考。

import torch
import torch.nn as nn
import numpy as np
from collections import namedtuple

# ==================== 环境 ====================
class InventoryEnv:
    """库存管理环境：状态为库存水平，动作为订货量"""
    def __init__(self, max_inv=30, mean_demand=8):
        self.max_inv = max_inv
        self.mean_demand = mean_demand
        self.actions = [0, 5, 10, 15, 20]

    def reset(self):
        self.state = np.random.randint(0, self.max_inv + 1)
        return self.state

    def step(self, action_idx):
        order = self.actions[action_idx]
        demand = np.random.poisson(self.mean_demand)
        sold = min(self.state, demand)
        reward = (10 * sold
                  - 1 * max(self.state + order - demand, 0)
                  - 5 * max(demand - self.state, 0)
                  - 2 * order)
        self.state = min(max(self.state + order - demand, 0), self.max_inv)
        return self.state, reward, False  # 无限期，不终止

# ==================== Dueling Q 网络 ====================
class DuelingQNet(nn.Module):
    def __init__(self, n_actions=5):
        super().__init__()
        self.shared = nn.Sequential(
            nn.Linear(1, 64), nn.ReLU(),
            nn.Linear(64, 64), nn.ReLU()
        )
        self.val_head = nn.Linear(64, 1)          # V(s) 分支
        self.adv_head = nn.Linear(64, n_actions)   # A(s,a) 分支

    def forward(self, x):
        h = self.shared(x)
        v = self.val_head(h)                       # (batch, 1)
        a = self.adv_head(h)                       # (batch, n_actions)
        # Q = V + (A - mean(A))
        return v + a - a.mean(dim=1, keepdim=True)

# ==================== 优先经验回放 ====================
class SumTree:
    """用于 O(log N) 优先采样的线段树"""
    def __init__(self, capacity):
        self.capacity = capacity
        self.tree = np.zeros(2 * capacity - 1)
        self.data = [None] * capacity
        self.ptr = 0
        self.size = 0

    def _propagate(self, idx, change):
        parent = (idx - 1) // 2
        self.tree[parent] += change
        if parent != 0:
            self._propagate(parent, change)

    def update(self, idx, priority):
        change = priority - self.tree[idx]
        self.tree[idx] = priority
        self._propagate(idx, change)

    def add(self, priority, data):
        idx = self.ptr + self.capacity - 1
        self.data[self.ptr] = data
        self.update(idx, priority)
        self.ptr = (self.ptr + 1) % self.capacity
        self.size = min(self.size + 1, self.capacity)

    def get(self, s):
        idx = 0
        while idx < self.capacity - 1:
            left = 2 * idx + 1
            if s <= self.tree[left]:
                idx = left
            else:
                s -= self.tree[left]
                idx = left + 1
        data_idx = idx - self.capacity + 1
        return idx, self.tree[idx], self.data[data_idx]

    @property
    def total(self):
        return self.tree[0]

Transition = namedtuple('Transition', ['s', 'a', 'r', 's_next'])

class PrioritizedReplayBuffer:
    def __init__(self, capacity, alpha=0.6):
        self.tree = SumTree(capacity)
        self.alpha = alpha
        self.max_priority = 1.0
        self.epsilon = 1e-5

    def push(self, *args):
        self.tree.add(self.max_priority ** self.alpha, Transition(*args))

    def sample(self, batch_size, beta=0.4):
        batch, indices, weights = [], [], []
        segment = self.tree.total / batch_size
        for i in range(batch_size):
            s = np.random.uniform(segment * i, segment * (i + 1))
            idx, priority, data = self.tree.get(s)
            batch.append(data)
            indices.append(idx)
            prob = priority / self.tree.total
            weights.append((self.tree.size * prob) ** (-beta))
        max_w = max(weights)
        weights = [w / max_w for w in weights]
        return batch, indices, torch.tensor(weights, dtype=torch.float32)

    def update_priorities(self, indices, td_errors):
        for idx, td in zip(indices, td_errors):
            priority = (abs(td) + self.epsilon) ** self.alpha
            self.max_priority = max(self.max_priority, priority)
            self.tree.update(idx, priority)

    def __len__(self):
        return self.tree.size

# ==================== 训练 ====================
env = InventoryEnv()
q_net = DuelingQNet()
target_net = DuelingQNet()
target_net.load_state_dict(q_net.state_dict())
optimizer = torch.optim.Adam(q_net.parameters(), lr=1e-3)
buffer = PrioritizedReplayBuffer(capacity=10000, alpha=0.6)

gamma = 0.99
eps = 1.0
batch_size = 64
beta_start, beta_end = 0.4, 1.0
n_episodes = 500
steps_per_episode = 200

for episode in range(n_episodes):
    s = env.reset()
    ep_reward = 0
    beta = beta_start + (beta_end - beta_start) * episode / n_episodes

    for t in range(steps_per_episode):
        # epsilon-greedy 探索
        if np.random.random() < eps:
            a = np.random.randint(5)
        else:
            with torch.no_grad():
                q = q_net(torch.tensor([[s / 30.0]], dtype=torch.float32))
            a = q.argmax().item()

        s_next, reward, _ = env.step(a)
        buffer.push(s, a, reward, s_next)
        ep_reward += reward
        s = s_next

        # 从优先回放缓冲区采样并训练
        if len(buffer) >= batch_size:
            batch, indices, is_weights = buffer.sample(batch_size, beta)
            ss = torch.tensor([[b.s / 30.0] for b in batch], dtype=torch.float32)
            aa = torch.tensor([b.a for b in batch], dtype=torch.long)
            rr = torch.tensor([b.r for b in batch], dtype=torch.float32)
            ss2 = torch.tensor([[b.s_next / 30.0] for b in batch], dtype=torch.float32)

            # Double DQN 目标：在线网络选动作，目标网络估值
            with torch.no_grad():
                best_a = q_net(ss2).argmax(dim=1)
                q_target = target_net(ss2).gather(1, best_a.unsqueeze(1)).squeeze()
            target = rr + gamma * q_target
            q_pred = q_net(ss).gather(1, aa.unsqueeze(1)).squeeze()

            # 加权 MSE 损失（重要性采样修正）
            td_errors = (target - q_pred).detach().numpy()
            loss = (is_weights * (q_pred - target) ** 2).mean()

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            # 更新优先级
            buffer.update_priorities(indices, td_errors)

    eps = max(0.05, eps * 0.99)
    if episode % 10 == 0:
        target_net.load_state_dict(q_net.state_dict())
    if episode % 50 == 0:
        print(f"Episode {episode}, reward={ep_reward:.0f}, eps={eps:.2f}")

# 查看学到的策略
print("\n学到的补货策略：")
for s in range(0, 31, 5):
    with torch.no_grad():
        q = q_net(torch.tensor([[s / 30.0]], dtype=torch.float32))
    action = [0, 5, 10, 15, 20][q.argmax().item()]
    print(f"  库存={s:2d} -> 订货 {action}")

代码结构清晰地对应了三项改进：DuelingQNet 类实现了 Dueling 架构（$V$ 和 $A$ 两个分支），PrioritizedReplayBuffer 类实现了基于 SumTree 的优先经验回放，而训练循环中 Double DQN 的目标计算只需两行代码（best_a = q_net(ss2).argmax(dim=1) 选动作，q_target = target_net(ss2).gather(...) 估值）。三项改进的代码完全解耦，读者可以自行尝试开启或关闭任意一项，观察与前面交互演示一致的效果。
