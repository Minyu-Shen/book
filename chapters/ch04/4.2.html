4.2 时域建模与回合定义

上一节介绍了将管理科学问题抽象为 MDP 的整体框架。在进入状态、动作、奖励等具体设计之前，有一个更基本的建模决策需要先回答：管理者的规划时域是有限的还是无限的？这个选择会如何影响最优策略？

这些问题在初次将 RL 应用于管理科学问题时容易被忽视。我们通过一个精确可解的小例子来把这件事说清楚。

管理者的目标决定时域建模

建模的第一步不是写状态空间和动作空间，而是搞清楚一个问题：管理者真正关心的是什么，约束是什么。同一套设备、同样的状态转移，管理者面对不同的约束条件，最优策略可以截然不同。

下面用一个设备维护的例子来展示这一点。

例子：设备预防性维护

一台生产设备有两个状态。

  • 正常（$N$）：每期产出收益 $+10$。但设备会自然退化，每期有 $50\%$ 的概率变为磨损。
  • 磨损（$W$）：产出效率下降，每期收益仅 $+3$。设备不会自行恢复。

在磨损状态下，管理者面临选择：

  • 继续生产：当期收益 $+3$，设备保持磨损。
  • 停机维护：当期净收益 $-7$（停产损失加维修费用），但设备恢复正常。

折扣因子 $\gamma = 0.9$。用 MDP 的语言，这个问题的转移结构如下图所示：

图 4.2-1　设备维护 MDP 的状态转移图。正常状态每期以 50% 概率退化为磨损；磨损状态下，管理者选择继续生产（低收益但零成本）或停机维护（付出成本但恢复正常）。

维护本质上是一项先投资、后收益的决策：当期付出 $7$（收益从 $+3$ 降为 $-7$，落差为 $10$），换取后续若干期产出从 $3$ 提升到 $10$（每期多赚 $7$）。直觉上大约需要 $10 \div 7 \approx 1.4$ 期才能回本。那么，管理者应该维护吗？

答案取决于管理者面对的时间约束。

场景 A：设备租赁 20 期后归还

问题设定

管理者签订了 20 期的设备租赁合同。第 20 期结束后，设备归还出租方，设备此后的状态与自己无关。管理者的目标是最大化这 20 期内的折扣总收益。

这是一个有限时域 MDP，$T = 20$。有限时域的标准求解方法是反向归纳（backward induction）。它的思路与 1.4.2 节的值迭代一脉相承，都是反复应用贝尔曼最优方程来求解最优值函数。区别在于：无限时域值迭代从一个任意初始猜测出发，反复迭代直到收敛；反向归纳则从一个已知的边界条件（终止时刻的值为零）出发，逐期向前推导，一次遍历即可得到每一期的精确解。读者如果学过运筹学中的动态规划（如胡运权《运筹学》第 9 章），会发现反向归纳就是那里的逆序求解法。

设 $V_k(s)$ 表示还剩 $k$ 期时处于状态 $s$ 的最优值。边界条件 $V_0(N) = V_0(W) = 0$（租约到期，无剩余价值）。

递推关系为：

$$V_k(N) = 10 + \gamma\big[0.5\, V_{k-1}(N) + 0.5\, V_{k-1}(W)\big] \tag{4.2-1}$$ $$V_k(W) = \max\Big\{\underbrace{3 + \gamma\, V_{k-1}(W)}_{\text{继续生产}},\;\; \underbrace{-7 + \gamma\, V_{k-1}(N)}_{\text{停机维护}}\Big\} \tag{4.2-2}$$

手动计算前几步（验证直觉）：

$k=1$（最后 1 期）：$Q(W, \text{继续}) = 3$，$Q(W, \text{维护}) = -7$。选择继续生产。还剩 1 期根本来不及回本。

$k=2$：$Q(W, \text{继续}) = 3 + 0.9 \times 3 = 5.70$，$Q(W, \text{维护}) = -7 + 0.9 \times 10 = 2.00$。选择继续生产。

$k=3$：$Q(W, \text{继续}) = 3 + 0.9 \times 5.70 = 8.13$，$Q(W, \text{维护}) = -7 + 0.9 \times 15.85 = 7.27$。继续生产仍然更好，但差距已经收窄到 $0.86$。

$k=4$：$Q(W, \text{继续}) = 10.32$，$Q(W, \text{维护}) = 10.73$。维护首次胜出！

用代码完成全部 20 期的反向归纳：

import numpy as np

gamma = 0.9
T = 20

# V[k] = 还剩 k 期时的值函数，V[0] = 终止值 = 0
V = [{'N': 0.0, 'W': 0.0} for _ in range(T + 1)]

for k in range(1, T + 1):
    V[k]['N'] = 10 + gamma * (0.5 * V[k-1]['N'] + 0.5 * V[k-1]['W'])
    q_operate  = 3  + gamma * V[k-1]['W']
    q_maintain = -7 + gamma * V[k-1]['N']
    V[k]['W'] = max(q_operate, q_maintain)
    action = '维护' if q_maintain >= q_operate else '生产'
    print(f'k={k:2d}  Q(继续)={q_operate:6.2f}  '
          f'Q(维护)={q_maintain:6.2f}  → {action}')

有限时域的最优策略是非平稳的：剩余期数 $k \geq 4$ 时维护，$k \leq 3$ 时继续生产。最后 3 期不维护是完全合理的决策。何必花 7 万给即将归还的设备做保养？

场景 B：自有设备持续运营

问题设定

管理者拥有这台设备，计划长期使用，没有确定的退出时间。管理者的目标是最大化长期折扣总收益。

这是一个无限时域 MDP，没有终止时刻，因此无法从"最后一期"反向推导。用 1.4.2 节介绍的值迭代求解：从任意初始值出发，反复应用贝尔曼最优方程直到收敛。

$$V^*(N) = 10 + \gamma\big[0.5\, V^*(N) + 0.5\, V^*(W)\big] \tag{4.2-3}$$ $$V^*(W) = \max\big\{3 + \gamma\, V^*(W),\;\; {-7} + \gamma\, V^*(N)\big\} \tag{4.2-4}$$

假设维护是最优的，则 $V^*(W) = -7 + \gamma\, V^*(N)$，代入式(4.2-3)：

$$V^*(N) = 10 + \gamma\big[0.5\, V^*(N) + 0.5\,({-7} + \gamma\, V^*(N))\big] = 6.85 + 0.855\, V^*(N)$$

解得 $V^*(N) = 6.85 \,/\, 0.145 = 47.24$，进而 $V^*(W) = -7 + 0.9 \times 47.24 = 35.52$。验证：

$$Q^*(W, \text{继续}) = 3 + 0.9 \times 35.52 = 34.97,\quad Q^*(W, \text{维护}) = 35.52$$

维护确实是最优的（$35.52 > 34.97$），假设成立。

# 无限时域值迭代
V = {'N': 0.0, 'W': 0.0}
for _ in range(1000):
    V = {
        'N': 10 + gamma * (0.5 * V['N'] + 0.5 * V['W']),
        'W': max(3 + gamma * V['W'],       # 继续生产
                 -7 + gamma * V['N'])       # 停机维护
    }
print(f"V*(N) = {V['N']:.2f}, V*(W) = {V['W']:.2f}")
# 输出: V*(N) = 47.24, V*(W) = 35.52

无限时域的最优策略是平稳的：不管当前是第几期，最优动作只取决于设备状态。

设备状态最优动作$V^*(s)$
$N$（正常）继续生产（唯一动作）$47.24$
$W$（磨损）停机维护$35.52$

与场景 A 的对比鲜明：场景 A 中，磨损状态的最优动作随剩余期数变化（$k \geq 4$ 维护，$k \leq 3$ 继续生产）；场景 B 中，磨损时始终维护，因为管理者永远有足够的未来来回收维护投资。

对比：时域假设如何改变最优策略

将两个场景的结果并列对比：

场景 A：租赁 20 期 场景 B：自有设备
时域有限（$T = 20$）无限
求解方法值迭代（从终止条件反向推导）值迭代（从初始猜测迭代至收敛）

核心洞察

两个场景的状态空间、动作空间和转移概率完全相同，区别仅在于管理者面对的时间约束。同一套系统动力学，不同的时域假设，给出不同的最优策略。这不是算法问题，而是建模问题。

场景 A 的策略是非平稳的：最优动作取决于状态 $s$ 和剩余期数 $k$，即 $\pi^*_k(s)$。而本书从第一章起讨论的都是平稳策略 $\pi(s)$。有没有办法将场景 A 也纳入平稳策略的框架？下面从两个角度来解决这个问题。

视角 C：将剩余时间纳入状态

场景 A 的策略之所以"非平稳"，是因为最优动作依赖于一个没有被纳入状态的信息：剩余期数 $k$。一个自然的想法是：既然最优动作依赖于 $k$，就把 $k$ 也作为状态的一部分。

将剩余期数 $k$ 编码进状态，得到增广状态空间 $\{(N, k),\, (W, k) \mid k = 1, \ldots, 20\}$，共 $40$ 个状态。增广后的状态转移有两个维度：设备状态 $s$ 的转移与场景 A 相同（随机的），而剩余期数 $k$ 每一步确定性地减一，$k$ 到达 $0$ 时回合结束。贝尔曼最优方程为：

$$V^*\big((N, k)\big) = 10 + \gamma\big[0.5\, V^*\big((N, k{-}1)\big) + 0.5\, V^*\big((W, k{-}1)\big)\big] \tag{4.2-5}$$ $$V^*\big((W, k)\big) = \max\big\{3 + \gamma\, V^*\big((W, k{-}1)\big),\;\; {-7} + \gamma\, V^*\big((N, k{-}1)\big)\big\} \tag{4.2-6}$$

边界条件 $V^*\big((N, 0)\big) = V^*\big((W, 0)\big) = 0$。由于状态 $(s, k)$ 只会转移到 $(s', k{-}1)$，从 $k = 1$ 逐层推到 $k = 20$，一次遍历即可求出所有值：

# 视角 C: 增广状态 (s, k), 从 k=1 逐层推到 k=20
gamma, T = 0.9, 20
V = {(s, 0): 0.0 for s in ['N', 'W']}

for k in range(1, T + 1):
    V[('N', k)] = 10 + gamma * (0.5*V[('N',k-1)] + 0.5*V[('W',k-1)])
    q_op = 3  + gamma * V[('W', k-1)]
    q_mt = -7 + gamma * V[('N', k-1)]
    V[('W', k)] = max(q_op, q_mt)
    print(f"k={k:2d}  V(W,k)={V[('W',k)]:6.2f}  "
          f"{'维护' if q_mt >= q_op else '生产'}")

运行结果与场景 A 逐值相等：$k \leq 3$ 继续生产，$k \geq 4$ 停机维护。这并不意外，因为公式 (4.2-5)(4.2-6) 与场景 A 的公式 (4.2-1)(4.2-2) 形式完全一致。区别在于理解方式：在场景 A 中，$\pi^*_k(s)$ 是一个随 $k$ 变化的非平稳策略；而在视角 C 中，$\pi^*\big((s, k)\big)$ 是增广状态空间上的平稳策略。$(W, 3)$ 和 $(W, 16)$ 是两个不同的状态，对应不同的最优动作，就像场景 B 中 $N$ 和 $W$ 对应不同动作一样。

视角 C 回答了一个问题：有限时域问题的最优策略也可以是平稳的，只要把剩余时间视为状态的一部分。

视角 D：在场景 B 的 MDP 上引入吸收状态

视角 C 把非平稳变为平稳，但求解方式仍是逐层推导。一个更根本的问题是：能不能把场景 A 的有限时域问题本身转化为无限时域问题？如果可以，就能直接复用场景 B 的值迭代算法，不需要为有限时域单独开发求解方法。

吸收状态构造正是实现这一转化的方法。在视角 C 的增广状态空间上增加一个吸收状态 $s_{\text{abs}}$，使整个问题成为一个无限时域 MDP：

吸收状态构造

增广状态空间 $\mathcal{S}' = \big\{(N, 20), \ldots, (N, 1),\; (W, 20), \ldots, (W, 1),\; s_{\text{abs}}\big\}$，共 $41$ 个状态。$s_{\text{abs}}$ 满足 $R(s_{\text{abs}}, a) = 0$，$P(s_{\text{abs}} \mid s_{\text{abs}}, a) = 1$。$k = 1$ 的下一步转移到 $s_{\text{abs}}$，之后永远留在那里。

进入吸收状态后，无限求和自动退化为有限求和：

$$G_t = \underbrace{\sum_{k=0}^{T-1-t} \gamma^k R_{t+k}}_{\text{有限回报}} + \underbrace{\sum_{k=T-t}^{\infty} \gamma^k \cdot 0}_{\text{吸收状态后全部为零}} = \sum_{k=0}^{T-1-t} \gamma^k R_{t+k} \tag{4.2-7}$$

现在用场景 B 完全相同的无限时域值迭代求解：对所有 $41$ 个状态反复迭代贝尔曼最优方程直到收敛。

# 视角 D: 无限时域值迭代 (与场景 B 完全相同的算法)
gamma, T = 0.9, 20
V = {(s, k): 0.0 for s in ['N', 'W'] for k in range(T + 1)}

for iteration in range(1000):    # 反复迭代直到收敛
    V_new = {(s, 0): 0.0 for s in ['N', 'W']}  # 吸收状态: V=0
    for k in range(1, T + 1):
        V_new[('N', k)] = 10 + gamma * (
            0.5 * V[('N', k-1)] + 0.5 * V[('W', k-1)])
        q_op = 3  + gamma * V[('W', k-1)]
        q_mt = -7 + gamma * V[('N', k-1)]
        V_new[('W', k)] = max(q_op, q_mt)
    if max(abs(V_new[key] - V[key]) for key in V) < 1e-12:
        print(f'收敛于第 {iteration+1} 轮')
        break
    V = V_new

# 验证: 与场景 A、C 的结果逐值相等
for k in [1, 2, 3, 4, 5, 20]:
    q_op = 3  + gamma * V[('W', k-1)]
    q_mt = -7 + gamma * V[('N', k-1)]
    action = '维护' if q_mt >= q_op else '生产'
    print(f'k={k:2d}: {action}  Q(继续)={q_op:.2f} Q(维护)={q_mt:.2f}')

运行结果与场景 A 的反向归纳、视角 C 的逐层推导逐值相等：$k \leq 3$ 继续生产，$k \geq 4$ 停机维护。三条求解路径殊途同归。

视角 D 的意义在于：场景 A 的有限时域问题，可以被嵌入场景 B 的无限时域框架中，用同一套值迭代算法求解。这是本书选择无限时域作为统一数学语言的主要原因：贝尔曼方程、策略梯度定理等核心结论只需推导一次，有限时域问题通过吸收状态构造自动纳入。

什么时候应该把时间放进状态

视角 C 和 D 的核心操作是将时间 $k$ 纳入状态。但并非所有问题都需要这样做。判断标准：最优动作是否应该随时间变化？

  • 有限期问题（租赁设备、航班起飞前的收益管理、季节性商品清仓）：将剩余时间 $k$ 放进状态。策略可以表达"最后 3 期不维护"这样的时间依赖行为。
  • 持续运营问题（自有设备维护、常规库存补货、交通信号控制）：不放。最优策略只取决于系统状态，与"这是第几期"无关。
  • 周期性问题（网约车早晚高峰、季节性需求波动）：放时段特征（如一天中的小时数），但不放"训练回合的剩余步数"。前者反映系统的真实周期性，后者是训练的人为产物。
一条判断规则   如果改变截止时刻 $T$（比如从 20 期改为 30 期）会改变中间某一期的最优动作，那么时间应该进状态。如果不会（即最优动作只取决于当前系统状态），那么时间不应该进状态。在设备维护的例子中：场景 A 改变 $T$ 会影响策略（$T = 10$ 时最后 3 期不维护，$T = 30$ 时也是最后 3 期不维护，但"最后 3 期"指向不同的日历时间）；场景 B 没有 $T$，策略完全由 $(N, W)$ 决定。

回合是一个建模选择

在管理科学中，回合的边界往往不是问题天然给定的，而是建模者的选择。选择回合边界的一般原则：找到系统中自然的重置点。如果系统状态在某些时刻会回到（或接近）某个初始条件，那里就是回合的自然分界线。回合过短则智能体看不到长期效应，过长则蒙特卡洛回报 $G_t$ 方差过大。

RL 的训练范式是有限回合 + 重复采样：用仿真器模拟一个回合，计算回报并更新参数，然后重复成百上千次。我们优化的不是某一个特定回合的表现，而是跨回合的平均表现。

折扣因子 $\gamma$ 的双重角色

在无限时域设定中，$\gamma < 1$ 是保证 $\sum_{k=0}^{\infty} \gamma^k R_{t+k}$ 收敛的数学必要条件。在有限时域中，$\gamma = 1$ 也完全合法。实践中通常仍然使用 $\gamma < 1$，但原因从数学必要性变为建模选择：

  • 时间偏好：$\gamma$ 直接对应金融中的贴现率。$\gamma = 0.99$ 意味着一期后的一元钱在当前只值 $0.99$ 元，与净现值分析中的折现完全一致。
  • 降低方差：$\gamma < 1$ 让远期奖励的权重指数衰减，使回报 $G_t$ 方差更小，梯度估计更稳定。
  • 隐含的有效视野：$\gamma$ 隐式定义了规划范围。$\gamma^k$ 在 $k \approx 1/(1-\gamma)$ 步后衰减到可忽略的水平。$\gamma = 0.99$ 对应约 $100$ 步的有效视野。

TD 学习为什么让这个区别变得不那么显眼

读到这里，有一个自然的疑问：既然实践中几乎都用 TD 学习（如 Q-learning、PPO），而 TD 更新只需要一步转移 $(s_t, a_t, r_{t+1}, s_{t+1})$，不像蒙特卡洛方法那样需要等到回合结束计算完整回报 $G_t$，那么有限与无限的区别在哪里？

这个疑问揭示了 TD 方法的一个重要性质：在算法运行的层面，TD 确实不受回合长度的限制。蒙特卡洛方法需要完整轨迹来计算 $G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$，无限时域意味着这个级数永远算不完，MC 因此无法用于真正的无限时域问题。TD 通过自举（用 $V(s_{t+1})$ 替代未来回报的精确值）绕过了这个限制。不管回合有 20 步还是无限步，TD 的更新公式都一样：

$$V(s_t) \leftarrow V(s_t) + \alpha \big[r_{t+1} + \gamma\, V(s_{t+1}) - V(s_t)\big] \tag{4.2-8}$$

但有限与无限的区别并没有消失，它只是从算法层面转移到了建模层面。上面的设备维护例子已经清楚地表明：即使使用完全相同的 TD 更新公式，如果将问题建模为有限时域（场景 A），学到的最优策略是非平稳的（最后 3 期不维护）；如果建模为无限时域（场景 B），学到的最优策略是平稳的（始终维护）。算法没有变，变的是 MDP 模型的结构，而最优策略因此完全不同。

这意味着，建模者必须先回答"管理者面对的是什么样的时间约束"，再选择对应的 MDP 类型。这个建模决策先于算法选择，也比算法选择更重要。

截断陷阱：当模型与现实不匹配时

理解了上述区别后，一种常见的实践错误就很容易理解了。

假设管理者拥有设备并计划长期使用（场景 B），但研究者在训练时为了方便，把仿真切成 20 步一段，并在回合末尾设置 $V(s_T) = 0$。这等于告诉智能体"第 20 期后设备消失了"。智能体会"理性地"在最后 3 期放弃维护，学到的策略如下表所示：

时间步 $t$ 0—16 17 18 19
磨损时动作 停机维护 ✓ 继续生产 ✗ 继续生产 ✗ 继续生产 ✗

这个策略对于"20 期后设备消失"的假想世界是正确的。但管理者的设备并不会在第 20 期消失。在真实的持续运营中，第 17—19 期放弃维护意味着设备带着磨损进入下一段运营，性能白白损失。

截断陷阱的根源不是 TD 或 MC 的算法选择，而是MDP 模型与现实之间的不匹配。研究者为了训练方便引入的回合边界，被智能体当成了真实的终止条件。问题不在于截断本身，而在于截断时把 $V(s_T) = 0$。

没有自然终点时怎么办

有些管理科学问题确实没有明显的回合边界：在线定价系统持续运行，生产排程周而复始，仓库不间断地补货出货。这类持续运营问题需要人为截断为回合来训练，但必须正确处理边界。常用的方法有两种。

方法一：在截断处自举

回合结束时不置 $V(s_T) = 0$，而是用当前值函数的估计 $\hat{V}(s_T)$ 作为最后一步的自举目标。这告诉智能体：回合结束后世界并没有消失，$s_T$ 之后仍有价值。在设备维护的例子中，我们已经验证过：用 $V^*$ 自举后，反向归纳在所有 20 个时间步都给出与无限时域一致的"始终维护"策略。

方法二：冗余缓冲段

实际模拟 $H_{\text{total}}$ 步，但只对前 $H_{\text{eval}}$ 步计算回报和更新参数。后面的 $H_{\text{total}} - H_{\text{eval}}$ 步仅用于提供未来奖励的估计，不参与梯度计算。在 $H_{\text{eval}}$ 时刻，智能体并不知道自己处于回合的哪个位置，边界效应就被消除了。

判断标准   面对一个管理科学问题，先问自己：回合结束时，系统状态对管理者还有价值吗？如果有（如自有设备仍在运行），就必须自举或使用缓冲段；如果没有（如租约到期设备归还），终止处理 $V = 0$ 就是正确的。

小结

  1. 有限时域与无限时域的区别不在于算法（TD 两者都能用），而在于管理者面对的时间约束。同一个 MDP，不同的时域假设给出不同的最优策略。
  2. 视角 C 将剩余时间纳入状态，把非平稳策略转化为增广状态空间上的平稳策略。是否把时间放进状态取决于最优动作是否依赖于时间。
  3. 视角 D 在场景 B 的无限时域 MDP 上引入吸收状态，用同一套值迭代算法求解，结果与场景 A 的反向归纳完全一致。这验证了无限时域框架的一般性。
  4. 截断陷阱的根源是模型与现实的不匹配。对持续运营问题，必须在截断处自举或使用缓冲段。
