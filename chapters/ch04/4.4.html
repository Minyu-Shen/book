4.4 回合定义与时域建模

上一节讨论了一个回合内部的时间结构。本节讨论一个更基本的问题：一个回合有多长？管理者的规划时域是有限的还是无限的？这个建模选择会如何影响最优策略？

这些问题看似基础，却几乎困扰过每一个初次将 RL 应用于管理科学问题的研究者。我们通过一个精确可解的小例子来把这件事彻底说清楚。

管理者的目标决定时域建模

建模的第一步不是写状态空间和动作空间，而是搞清楚一个问题：管理者真正关心的是什么，约束是什么。同一套设备、同样的状态转移，管理者面对不同的约束条件，最优策略可以截然不同。

下面用一个设备维护的例子来展示这一点。

例子：设备预防性维护

一台生产设备有两个状态。

  • 正常（$N$）：每期产出收益 $+10$。但设备会自然退化，每期有 $50\%$ 的概率变为磨损。
  • 磨损（$W$）：产出效率下降，每期收益仅 $+3$。设备不会自行恢复。

在磨损状态下，管理者面临选择：

  • 继续生产：当期收益 $+3$，设备保持磨损。
  • 停机维护：当期净收益 $-7$（停产损失加维修费用），但设备恢复正常。

折扣因子 $\gamma = 0.9$。用 MDP 的语言，这个问题的转移结构如下图所示：

图 4.4-1　设备维护 MDP 的状态转移图。正常状态每期以 50% 概率退化为磨损；磨损状态下，管理者选择继续生产（低收益但零成本）或停机维护（付出成本但恢复正常）。

维护本质上是一项先投资、后收益的决策：当期付出 $7$（收益从 $+3$ 降为 $-7$，落差为 $10$），换取后续若干期产出从 $3$ 提升到 $10$（每期多赚 $7$）。直觉上大约需要 $10 \div 7 \approx 1.4$ 期才能回本。那么，管理者应该维护吗？

答案取决于管理者面对的时间约束。

场景 A：设备租赁 20 期后归还

管理者的处境

管理者签订了 20 期的设备租赁合同。第 20 期结束后，设备归还出租方，设备此后的状态与自己无关。管理者的目标是最大化这 20 期内的折扣总收益。

这是一个有限时域 MDP，$T = 20$。用反向归纳（backward induction，即有限时域的值迭代）求解。从最后一期倒推：设 $V_k(s)$ 表示还剩 $k$ 期时处于状态 $s$ 的最优值。边界条件 $V_0(N) = V_0(W) = 0$（租约到期，无剩余价值）。

递推关系为：

$$V_k(N) = 10 + \gamma\big[0.5\, V_{k-1}(N) + 0.5\, V_{k-1}(W)\big] \tag{4.4-1}$$ $$V_k(W) = \max\Big\{\underbrace{3 + \gamma\, V_{k-1}(W)}_{\text{继续生产}},\;\; \underbrace{-7 + \gamma\, V_{k-1}(N)}_{\text{停机维护}}\Big\} \tag{4.4-2}$$

手动计算前几步（验证直觉）：

$k=1$（最后 1 期）：$Q(W, \text{继续}) = 3$，$Q(W, \text{维护}) = -7$。选择继续生产。还剩 1 期根本来不及回本。

$k=2$：$Q(W, \text{继续}) = 3 + 0.9 \times 3 = 5.70$，$Q(W, \text{维护}) = -7 + 0.9 \times 10 = 2.00$。选择继续生产。

$k=3$：$Q(W, \text{继续}) = 3 + 0.9 \times 5.70 = 8.13$，$Q(W, \text{维护}) = -7 + 0.9 \times 15.85 = 7.27$。继续生产仍然更好，但差距已经收窄到 $0.86$。

$k=4$：$Q(W, \text{继续}) = 10.32$，$Q(W, \text{维护}) = 10.73$。维护首次胜出！

用代码完成全部 20 期的反向归纳：

import numpy as np

gamma = 0.9
T = 20

# V[k] = 还剩 k 期时的值函数，V[0] = 终止值 = 0
V = [{'N': 0.0, 'W': 0.0} for _ in range(T + 1)]

for k in range(1, T + 1):
    V[k]['N'] = 10 + gamma * (0.5 * V[k-1]['N'] + 0.5 * V[k-1]['W'])
    q_operate  = 3  + gamma * V[k-1]['W']
    q_maintain = -7 + gamma * V[k-1]['N']
    V[k]['W'] = max(q_operate, q_maintain)
    action = '维护' if q_maintain >= q_operate else '生产'
    print(f'k={k:2d}  Q(继续)={q_operate:6.2f}  '
          f'Q(维护)={q_maintain:6.2f}  → {action}')

有限时域的最优策略是非平稳的：剩余期数 $k \geq 4$ 时维护，$k \leq 3$ 时继续生产。最后 3 期不维护是完全合理的决策。何必花 7 万给即将归还的设备做保养？

场景 B：自有设备持续运营

管理者的处境

管理者拥有这台设备，计划长期使用，没有确定的退出时间。管理者的目标是最大化长期折扣总收益。

这是一个无限时域 MDP。用值迭代（1.4.2节）求解。贝尔曼最优方程为：

$$V^*(N) = 10 + \gamma\big[0.5\, V^*(N) + 0.5\, V^*(W)\big] \tag{4.4-3}$$ $$V^*(W) = \max\big\{3 + \gamma\, V^*(W),\;\; {-7} + \gamma\, V^*(N)\big\} \tag{4.4-4}$$

假设维护是最优的，则 $V^*(W) = -7 + \gamma\, V^*(N)$，代入式(4.4-3)：

$$V^*(N) = 10 + \gamma\big[0.5\, V^*(N) + 0.5\,({-7} + \gamma\, V^*(N))\big] = 6.85 + 0.855\, V^*(N)$$

解得 $V^*(N) = 6.85 \,/\, 0.145 = 47.24$，进而 $V^*(W) = -7 + 0.9 \times 47.24 = 35.52$。验证：

$$Q^*(W, \text{继续}) = 3 + 0.9 \times 35.52 = 34.97,\quad Q^*(W, \text{维护}) = 35.52$$

维护确实是最优的（$35.52 > 34.97$），假设成立。

# 无限时域值迭代
V = {'N': 0.0, 'W': 0.0}
for _ in range(1000):
    V = {
        'N': 10 + gamma * (0.5 * V['N'] + 0.5 * V['W']),
        'W': max(3 + gamma * V['W'],       # 继续生产
                 -7 + gamma * V['N'])       # 停机维护
    }
print(f"V*(N) = {V['N']:.2f}, V*(W) = {V['W']:.2f}")
# 输出: V*(N) = 47.24, V*(W) = 35.52

无限时域的最优策略是平稳的：无论处于第 1 期还是第 1000 期，只要设备磨损就应该维护。管理者永远有足够的未来来回收维护投资。

核心洞察

两个场景的状态空间、动作空间和转移概率完全相同，唯一的区别是管理者面对的时间约束。同一个 MDP，不同的时域假设，给出截然不同的最优策略。这不是算法问题，而是建模问题：你的模型有没有忠实反映管理者的真实处境。

回合是一个建模选择

在 Atari 游戏或棋类博弈中，一个回合有天然的边界：游戏结束就是回合终止。但在管理科学中，回合的边界往往不是问题天然给定的，而是建模者的选择。

例：同一个系统的不同回合定义

考虑一个城市的网约车派单系统。以下回合定义都是合理的：

  • 方案 A：一个回合 = 早高峰 8:00—10:00（2小时，约120个决策步）
  • 方案 B：一个回合 = 一整天 0:00—24:00（24小时，约1440个决策步）
  • 方案 C：一个回合 = 一周的连续运营（168小时）

选择回合边界的一般原则：找到系统中自然的重置点。如果系统状态在某些时刻会回到（或接近）某个初始条件，那里就是回合的自然分界线。深夜时段车辆和需求都回到低水平，是一天回合的自然终点；回合过短则智能体看不到长期效应，过长则蒙特卡洛回报 $G_t$ 方差过大。

RL 的训练范式是有限回合 + 重复采样：用仿真器模拟一个回合，计算回报并更新参数，然后重复成百上千次。我们优化的不是某一个特定回合的表现，而是跨回合的平均表现。

有限时域如何纳入无限时域框架

本书从1.3节起统一采用无限时域折扣 MDP 作为数学框架。有限时域问题通过一个简单的构造被纳入。

吸收状态构造（absorbing state）

给定一个 $T$ 步的有限时域 MDP，在状态空间中增加一个吸收状态 $s_{\text{abs}}$。在原 MDP 的最后一步 $t = T-1$，下一状态转移到 $s_{\text{abs}}$。吸收状态满足 $R(s_{\text{abs}}, a) = 0$ 且 $P(s_{\text{abs}} \mid s_{\text{abs}}, a) = 1$。

加入吸收状态后，无限时域的回报自动退化为有限求和：

$$G_t = \underbrace{\sum_{k=0}^{T-1-t} \gamma^k R_{t+k}}_{\text{有限回报}} + \underbrace{\sum_{k=T-t}^{\infty} \gamma^k \cdot 0}_{\text{全部为零}} = \sum_{k=0}^{T-1-t} \gamma^k R_{t+k} \tag{4.4-5}$$

这就是本书选择无限时域作为统一框架的原因：不是假设现实问题是无限期的，而是因为这个框架足够一般，有限时域作为特例被自然包含。贝尔曼方程、策略梯度定理、所有算法都无需修改即可直接应用。

折扣因子 $\gamma$ 的双重角色

在无限时域设定中，$\gamma < 1$ 是保证 $\sum_{k=0}^{\infty} \gamma^k R_{t+k}$ 收敛的数学必要条件。在有限时域中，$\gamma = 1$ 也完全合法。实践中通常仍然使用 $\gamma < 1$，但原因从数学必要性变为建模选择：

  • 时间偏好：$\gamma$ 直接对应金融中的贴现率。$\gamma = 0.99$ 意味着一期后的一元钱在当前只值 $0.99$ 元，与净现值分析中的折现完全一致。
  • 降低方差：$\gamma < 1$ 让远期奖励的权重指数衰减，使回报 $G_t$ 方差更小，梯度估计更稳定。
  • 隐含的有效视野：$\gamma$ 隐式定义了规划范围。$\gamma^k$ 在 $k \approx 1/(1-\gamma)$ 步后衰减到可忽略的水平。$\gamma = 0.99$ 对应约 $100$ 步的有效视野。

TD 学习为什么让这个区别变得不那么显眼

读到这里，有一个自然的疑问：既然实践中几乎都用 TD 学习（如 Q-learning、PPO），而 TD 更新只需要一步转移 $(s_t, a_t, r_{t+1}, s_{t+1})$，不像蒙特卡洛方法那样需要等到回合结束计算完整回报 $G_t$，那么有限与无限的区别在哪里？

这个疑问揭示了 TD 方法的一个重要性质：在算法运行的层面，TD 确实不受回合长度的限制。蒙特卡洛方法需要完整轨迹来计算 $G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$，无限时域意味着这个级数永远算不完，MC 因此无法用于真正的无限时域问题。TD 通过自举（用 $V(s_{t+1})$ 替代未来回报的精确值）绕过了这个限制。不管回合有 20 步还是无限步，TD 的更新公式都一样：

$$V(s_t) \leftarrow V(s_t) + \alpha \big[r_{t+1} + \gamma\, V(s_{t+1}) - V(s_t)\big] \tag{4.4-6}$$

但有限与无限的区别并没有消失，它只是从算法层面转移到了建模层面。上面的设备维护例子已经清楚地表明：

  • 有限时域的最优策略随时间变化（非平稳）：第 1 期和第 18 期的最优维护决策不同。
  • 无限时域的最优策略不随时间变化（平稳）：任何时候磨损了都应该维护。

用 TD 还是 MC 都不会改变这个建模层面的区别。建模者必须先回答"管理者面对的是什么样的时间约束"，再选择对应的 MDP 类型。选对了模型，TD 和 MC 都会给出正确答案；选错了模型，用什么算法都会学到有偏的策略。

截断陷阱：当模型与现实不匹配时

理解了上述区别后，一种常见的实践错误就很容易理解了。

假设管理者拥有设备并计划长期使用（场景 B），但研究者在训练时为了方便，把仿真切成 20 步一段，并在回合末尾设置 $V(s_T) = 0$。这等于告诉智能体"第 20 期后设备消失了"。智能体会"理性地"在最后 3 期放弃维护，学到的策略如下表所示：

时间步 $t$ 0—16 17 18 19
磨损时动作 停机维护 ✓ 继续生产 ✗ 继续生产 ✗ 继续生产 ✗

这个策略对于"20 期后设备消失"的假想世界是正确的。但管理者的设备并不会在第 20 期消失。在真实的持续运营中，第 17—19 期放弃维护意味着设备带着磨损进入下一段运营，性能白白损失。

截断陷阱的根源不是 TD 或 MC 的算法选择，而是MDP 模型与现实之间的不匹配。研究者为了训练方便引入的回合边界，被智能体当成了真实的终止条件。问题不在于截断本身，而在于截断时把 $V(s_T) = 0$。

没有自然终点时怎么办

有些管理科学问题确实没有明显的回合边界：在线定价系统持续运行，生产排程周而复始，仓库不间断地补货出货。这类持续运营问题需要人为截断为回合来训练，但必须正确处理边界。常用的方法有两种。

方法一：在截断处自举

回合结束时不置 $V(s_T) = 0$，而是用当前值函数的估计 $\hat{V}(s_T)$ 作为最后一步的自举目标。这告诉智能体：回合结束后世界并没有消失，$s_T$ 之后仍有价值。在设备维护的例子中，我们已经验证过：用 $V^*$ 自举后，反向归纳在所有 20 个时间步都给出与无限时域一致的"始终维护"策略。

方法二：冗余缓冲段

实际模拟 $H_{\text{total}}$ 步，但只对前 $H_{\text{eval}}$ 步计算回报和更新参数。后面的 $H_{\text{total}} - H_{\text{eval}}$ 步仅用于提供未来奖励的估计，不参与梯度计算。在 $H_{\text{eval}}$ 时刻，智能体并不知道自己处于回合的哪个位置，边界效应就被消除了。

判断标准   面对一个管理科学问题，先问自己：回合结束时，系统状态对管理者还有价值吗？如果有（如自有设备仍在运行），就必须自举或使用缓冲段；如果没有（如租约到期设备归还），终止处理 $V = 0$ 就是正确的。

小结

  1. 有限时域与无限时域的区别不在于算法（TD 两者都能用），而在于管理者面对的时间约束。同一个 MDP，不同的时域假设给出不同的最优策略。
  2. 有限时域的最优策略是非平稳的（随剩余时间变化），无限时域的最优策略是平稳的。
  3. 回合的定义是建模者的选择。训练时用有限回合 + 重复采样。
  4. 有限时域通过吸收状态纳入无限时域框架，所有理论结论自动适用。
  5. 截断陷阱的根源是模型与现实的不匹配。对持续运营问题，必须在截断处自举或使用缓冲段。
