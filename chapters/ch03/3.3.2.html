3.3.2 REINFORCE算法

回顾策略梯度定理

上一节的策略梯度定理(3.3-3)给出了目标函数 $J(\theta)$ 关于参数 $\theta$ 的梯度：

$$\nabla_\theta J(\theta) = \frac{1}{1-\gamma}\,\mathbb{E}_{S \sim d^{\pi_\theta}}\!\bigg[\mathbb{E}_{A \sim \pi_\theta(\cdot|S)}\!\Big[\nabla_\theta \ln \pi_\theta(A \mid S) \cdot Q^{\pi_\theta}(S, A)\Big]\bigg]$$

这个公式虽然精确，但无法直接用于计算：动作值函数 $Q^{\pi_\theta}(s,a)$ 是未知的，稳态分布 $d^{\pi_\theta}$ 下的期望也无法精确求解。REINFORCE算法的贡献在于给出了一种纯粹基于采样的方法来估计这个梯度，从而将定理转化为可执行的算法。

从定理到算法

将策略梯度定理转化为实际算法，需要解决两个问题：如何处理未知的 $Q^{\pi_\theta}$，以及如何用采样代替期望。

第一步：用蒙特卡洛回报替代 $Q^{\pi_\theta}$

回忆动作值函数的定义：$Q^{\pi_\theta}(s,a) = \mathbb{E}_{\pi_\theta}[G_t \mid S_t = s, A_t = a]$，即从状态 $s$ 执行动作 $a$ 后，按策略 $\pi_\theta$ 继续行动所获得的期望回报。虽然我们不知道 $Q^{\pi_\theta}$ 的精确值，但如果实际走一条轨迹，沿途计算的回报 $G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$ 就是 $Q^{\pi_\theta}(S_t, A_t)$ 的一个无偏样本。于是，策略梯度中的关键乘积可以用采样近似：

$$\nabla_\theta \ln \pi_\theta(A_t \mid S_t) \cdot Q^{\pi_\theta}(S_t, A_t) \quad\longrightarrow\quad \nabla_\theta \ln \pi_\theta(A_t \mid S_t) \cdot G_t$$

这正是蒙特卡洛思想在策略梯度中的应用：用实际采样的回报代替期望值。

第二步：用轨迹采样代替期望

策略梯度定理中的双重期望 $\mathbb{E}_{S \sim d^{\pi_\theta}}\mathbb{E}_{A \sim \pi_\theta}[\cdot]$ 是对稳态分布下所有状态和动作的平均。让智能体按策略 $\pi_\theta$ 与环境交互，生成一条轨迹 $S_0, A_0, R_1, S_1, A_1, R_2, \ldots$。2.1.3节指出，在策略 $\pi_\theta$ 下马尔可夫链的状态分布收敛到 $d^{\pi_\theta}$，因此轨迹上每个状态 $S_t$ 近似是 $d^{\pi_\theta}$ 的一个样本；而每个动作 $A_t$ 是按 $\pi_\theta(\cdot \mid S_t)$ 采样的。这样，轨迹上的每一对 $(S_t, A_t)$ 都为双重期望贡献一个样本。将所有时刻的贡献求和，就得到单条轨迹的梯度估计：

$$\nabla_\theta J(\theta) \approx \sum_{t=0}^{T-1} \gamma^t \, \nabla_\theta \ln \pi_\theta(A_t \mid S_t) \cdot G_t$$

注意求和遍历了轨迹上的每个时刻：不只是起始状态 $S_0$，而是 $S_0, S_1, \ldots, S_{T-1}$ 中的每一个都参与更新，各自以自己时刻的 $G_t$ 和折扣权重 $\gamma^t$ 加权。$\gamma^t$ 确保早期时刻的贡献更大，因为早期决策对总回报的影响更显著。前面的常数 $\frac{1}{1-\gamma}$ 只影响梯度大小不影响方向，被吸收进学习率 $\alpha$。

得到更新规则

将上述梯度估计代入梯度上升 $\theta \leftarrow \theta + \alpha \nabla_\theta J$，对轨迹中每个时刻 $t$ 分别更新：

$$\theta \leftarrow \theta + \alpha \cdot \gamma^t \cdot G_t \cdot \nabla_\theta \ln \pi_\theta(A_t \mid S_t) \tag{3.3-4}$$

这个更新规则的直觉很清晰。$\nabla_\theta \ln \pi_\theta(A_t \mid S_t)$ 指向增大动作 $A_t$ 概率的方向，$G_t$ 衡量这个动作之后的实际回报。两者相乘意味着：如果 $G_t$ 较大（这次经历回报高），就沿梯度方向迈一大步，显著增大选择该动作的概率；如果 $G_t$ 较小，更新幅度也相应减小。

采样动作与计算对数策略梯度

更新公式(3.3-4)中，$G_t$ 的计算已经在3.1.1节介绍过（从后向前递推 $G_t = R_{t+1} + \gamma\,G_{t+1}$）。剩下的关键问题是如何按策略概率采样动作，以及如何计算 $\nabla_\theta \ln \pi_\theta(A_t \mid S_t)$。两者都取决于策略的具体参数化形式。

回忆3.3.1节的softmax参数化：对每个动作计算分数 $f_\theta(s,a)$，通过softmax映射为概率：

$$\pi_\theta(a \mid s) = \frac{\exp\!\big(f_\theta(s,a)\big)}{\sum_{a'} \exp\!\big(f_\theta(s,a')\big)}$$

采样动作很直接：给定状态 $s$，计算所有动作的 $\pi_\theta(a \mid s)$，然后按此概率分布随机抽取一个动作执行。

计算 $\nabla_\theta \ln \pi_\theta$ 的具体形式取决于 $f_\theta$ 的选择。当 $f_\theta$ 是神经网络时，$\nabla_\theta \ln \pi_\theta$ 没有解析表达式，但可以通过反向传播自动计算，第五章将介绍。下面以本章使用的线性形式 $f_\theta(s,a) = \theta^\top \phi(s,a)$ 为例，手动推导。

代入后：

$$\pi_\theta(a \mid s) = \frac{\exp\!\big(\theta^\top \phi(s,a)\big)}{\sum_{a'} \exp\!\big(\theta^\top \phi(s,a')\big)}$$

取对数，分式拆为分子减分母：

$$\ln \pi_\theta(a \mid s) = \theta^\top \phi(s,a) - \ln \sum_{a'} \exp\!\big(\theta^\top \phi(s,a')\big)$$
推导 $\nabla_\theta \ln \pi_\theta$（线性 softmax）

对 $\theta$ 求梯度。第一项 $\theta^\top \phi(s,a)$ 是 $\theta$ 的线性函数，梯度就是系数向量本身：$\nabla_\theta\!\big(\theta^\top \phi(s,a)\big) = \phi(s,a)$。第二项的形式是 $\ln g(\theta)$，其中 $g(\theta) = \sum_{a'} \exp\!\big(\theta^\top \phi(s,a')\big)$。对数的链式法则给出 $\nabla_\theta \ln g = \frac{\nabla_\theta g}{g}$，代入计算：

$$\nabla_\theta \ln \sum_{a'} \exp\!\big(\theta^\top \phi(s,a')\big) = \frac{\sum_{a'} \exp\!\big(\theta^\top \phi(s,a')\big) \cdot \phi(s,a')}{\sum_{a'} \exp\!\big(\theta^\top \phi(s,a')\big)} = \sum_{a'} \pi_\theta(a' \mid s)\,\phi(s,a')$$

最后一个等号是因为每个 $\frac{\exp(\theta^\top \phi(s,a'))}{\sum_{a''} \exp(\theta^\top \phi(s,a''))}$ 恰好就是 $\pi_\theta(a' \mid s)$ 的定义。两项相减，得到：

$$\nabla_\theta \ln \pi_\theta(a \mid s) = \phi(s,a) - \sum_{a'} \pi_\theta(a' \mid s) \, \phi(s,a')$$

即被选中动作的特征 $\phi(s,a)$，减去策略下的期望特征 $\mathbb{E}_{A \sim \pi_\theta}[\phi(s,A)]$。当 $\phi$ 是 one-hot 特征时（如网约车例子），这退化为分量形式：

$$\frac{\partial \ln \pi_\theta(a \mid s)}{\partial \theta_{s,a'}} = \begin{cases} 1 - \pi_\theta(a \mid s), & \text{if } a' = a \\ -\pi_\theta(a' \mid s), & \text{if } a' \neq a \end{cases}$$

即被选中的动作参数增大，增幅为 $1 - \pi_\theta(a \mid s)$（概率越小增幅越大）；未被选中动作的参数减小，减幅为 $\pi_\theta(a' \mid s)$。所有变化量由 $\alpha \gamma^t G_t$ 统一缩放。

REINFORCE 算法

输入：特征函数 $\phi(s,a)$，学习率 $\alpha$，折扣因子 $\gamma$

  1. 初始化参数 $\theta$（如全零向量），此时 $\pi_\theta$ 为均匀随机策略
  2. 对每个回合：
    1. 生成轨迹：从 $S_0$ 出发，在每个状态 $S_t$ 计算 $\pi_\theta(a \mid S_t) = \text{softmax}\!\big(\theta^\top \phi(S_t, \cdot)\big)$，按此概率采样动作 $A_t$，观察 $R_{t+1}$ 和 $S_{t+1}$，直到回合结束
    2. 计算回报：从后向前递推 $G_t = R_{t+1} + \gamma\,G_{t+1}$
    3. 更新参数：对每个时刻 $t$，计算梯度 $\nabla_\theta \ln \pi_\theta(A_t \mid S_t) = \phi(S_t, A_t) - \sum_{a'} \pi_\theta(a' \mid S_t)\,\phi(S_t, a')$，更新 $\theta \leftarrow \theta + \alpha \, \gamma^t \, G_t \cdot \nabla_\theta \ln \pi_\theta(A_t \mid S_t)$
REINFORCE需要等到回合结束才能更新，因为 $G_t$ 的计算依赖整条轨迹的奖励。这与3.1.1节的蒙特卡洛方法一致：两者都使用完整回报 $G_t$，区别在于MC更新的是值函数，REINFORCE更新的是策略参数。

例子：用REINFORCE学习网约车最优策略

下面用REINFORCE学习网约车MDP的最优策略（$\gamma = 0.9$，$\alpha = 0.01$）。由于是无限期MDP，每条轨迹模拟 $H_{\text{total}} = 200$ 步，仅对前 $H_{\text{eval}} = 100$ 步更新参数。代码实现直接对应上面算法的三个步骤：

import numpy as np
gamma, alpha = 0.9, 0.01
H_total, H_eval = 200, 100

# MDP 转移模型
T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}
states = ['c', 'b', 'a']
actions = {s: list(T[s].keys()) for s in states}

def step(s, action):
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum: return s_next, reward
    return s_next, reward

# ── 策略参数化：θ 向量 + softmax ──
theta = {s: {a: 0.0 for a in actions[s]} for s in states}

def policy_probs(s):                     # 步骤 2a: 计算 π_θ(a|s)
    acts = actions[s]
    logits = [theta[s][a] for a in acts]  # f_θ(s,a) = θ_{s,a}
    max_l = max(logits)
    exps = [np.exp(l - max_l) for l in logits]
    total = sum(exps)
    return {a: e / total for a, e in zip(acts, exps)}

def sample_action(s):                    # 步骤 2a: 按概率采样动作
    probs = policy_probs(s)
    acts = list(probs.keys())
    ps = [probs[a] for a in acts]
    return acts[np.random.choice(len(acts), p=ps)]

# ── REINFORCE 主循环 ──
for ep in range(5000):
    # 步骤 2a: 生成完整轨迹
    s = np.random.choice(states)
    trajectory = []
    for _ in range(H_total):
        a = sample_action(s)
        s_next, r = step(s, a)
        trajectory.append((s, a, r))
        s = s_next

    # 步骤 2b: 从后向前计算回报
    returns = [0.0] * len(trajectory)
    G = 0.0
    for t in range(len(trajectory) - 1, -1, -1):
        G = trajectory[t][2] + gamma * G
        returns[t] = G

    # 步骤 2c: 计算梯度并更新参数
    for t in range(H_eval):
        s_t, a_t, _ = trajectory[t]
        G_t = returns[t]
        probs = policy_probs(s_t)
        # ∂ln π / ∂θ_{s,a'} = 1{a'=a} - π(a'|s)
        for a in actions[s_t]:
            if a == a_t:
                theta[s_t][a] += alpha * (gamma ** t) * G_t * (1 - probs[a])
            else:
                theta[s_t][a] -= alpha * (gamma ** t) * G_t * probs[a]

# 输出学到的策略
for s in states:
    probs = policy_probs(s)
    best = max(probs, key=probs.get)
    print(f"  {s}: {best} (π = {probs[best]:.2f})")

运行后，REINFORCE会学到与SARSA、Q-learning相同的最优策略：市中心接单，郊区空驶去市中心，机场接单。不过由于策略梯度的方差较大，收敛速度通常慢于基于值函数的方法。

高方差问题

REINFORCE使用完整回报 $G_t$ 作为梯度估计中的信号，这与3.1.1节的蒙特卡洛方法面临同样的问题：$G_t$ 的方差很大。一条轨迹中，$G_t$ 取决于从时刻 $t$ 到回合结束的所有奖励和状态转移，其中每一步都引入随机性。这些随机性逐步累积，使得不同轨迹的 $G_t$ 差异悬殊。在网约车例子中，同样从郊区 $b$ 出发、选择相同的动作，一条轨迹可能因为连续接到高价订单得到 $G_t = 70$，另一条可能因为反复在郊区循环而只有 $G_t = 15$。

高方差意味着每次梯度更新的方向波动剧烈，需要大量轨迹才能使平均梯度收敛到正确方向。在上面的交互演示中，你可能已经注意到策略概率在训练过程中并非平稳上升，而是有明显的波动。在更复杂的问题中，这种方差会成为严重的瓶颈。

一个自然的想法是借鉴3.1.2节的时序差分思想：不等到回合结束再用 $G_t$，而是用TD目标 $R_{t+1} + \gamma \, V(S_{t+1})$ 来替代 $G_t$，用一步的实际奖励加上后续状态的值函数估计来降低方差。这正是下一节将要介绍的 Actor-Critic 方法的核心思想。
