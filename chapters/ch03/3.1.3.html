    • 从预测到控制：为什么需要 Q 值

    上一节介绍的MC和TD方法解决的是预测问题——给定一个策略 $\pi$，估计它的状态值函数 $V^\pi(s)$。然而，我们最终的目标是控制问题：找到最优策略 $\pi^*$。如果我们只有 $V(s)$，要根据它改善策略，就必须知道转移概率 $P(s' \mid s, a)$，因为需要计算每个动作的期望回报来比较哪个好：$\sum_{s'} P(s' \mid s, a)\,[R + \gamma\,V(s')]$。这又回到了"需要模型"的老路上。

    解决方案是直接学习动作值函数（Q值）$Q(s, a)$。有了Q值，智能体不需要知道任何转移概率，只需在每个状态选择 $\arg\max_a Q(s, a)$ 即可得到策略。这正是SARSA和Q-learning的核心思想。

    但直接贪心地选择当前最优动作存在一个问题：如果某个动作从未被尝试过，它的Q值可能被低估，从而永远不会被选中。这就是经典的探索-利用困境（exploration-exploitation dilemma）。我们需要一种策略，既能利用已有知识选择好的动作，又能偶尔探索未知的动作。最常用的方案是 $\varepsilon$-greedy策略：

    $\varepsilon$-greedy 策略

    以 $1-\varepsilon$ 的概率选择当前Q值最大的动作（利用），以 $\varepsilon$ 的概率随机选择一个动作（探索）：

    $$\pi(a \mid s) = \begin{cases} 1 - \varepsilon + \dfrac{\varepsilon}{|\mathcal{A}(s)|}, & \text{if } a = \arg\max_{a'} Q(s, a') \\ \dfrac{\varepsilon}{|\mathcal{A}(s)|}, & \text{otherwise} \end{cases} \tag{3.1-7}$$

    当 $\varepsilon = 0.1$ 且有2个动作时，最优动作被选中的概率是 $0.95$，非最优动作被选中的概率是 $0.05$。

    SARSA：在策略上的TD控制

    上一节的TD(0)方法将V值每步更新为 $V(S_t) \leftarrow V(S_t) + \alpha[R_t + \gamma\,V(S_{t+1}) - V(S_t)]$。现在我们将同样的思想从V值扩展到Q值：用当前的Q估计来更新Q估计。SARSA的名字来源于更新所需的五元组 $(S_t, A_t, R_t, S_{t+1}, A_{t+1})$——在当前状态 $S_t$ 执行动作 $A_t$，获得奖励 $R_t$，进入下一个状态 $S_{t+1}$，并在 $S_{t+1}$ 下实际选择了动作 $A_{t+1}$。

    SARSA 更新规则
    $$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[\, R_t + \gamma \, Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t) \,\right] \tag{3.1-8}$$

    其中 $A_{t+1}$ 是在 $S_{t+1}$ 下由当前策略（如 $\varepsilon$-greedy）实际选出的动作。TD目标是 $R_t + \gamma\,Q(S_{t+1}, A_{t+1})$。

    SARSA 算法
    1. 初始化 $Q(s, a)$ 为任意值（如全0），对所有 $s \in \mathcal{S}$，$a \in \mathcal{A}(s)$
    2. 对每个回合（episode）：
      1. 初始化状态 $S$
      2. 根据 $\varepsilon$-greedy 从 $Q$ 中选择动作 $A$
      3. 对回合中的每一步：
        1. 执行动作 $A$，观察奖励 $R$ 和下一状态 $S'$
        2. 根据 $\varepsilon$-greedy 从 $Q$ 中选择 $A'$
        3. $Q(S, A) \leftarrow Q(S, A) + \alpha\,[R + \gamma\,Q(S', A') - Q(S, A)]$
        4. $S \leftarrow S'$，$A \leftarrow A'$
    SARSA是在策略（on-policy）算法：更新时用的 $A'$ 与行为策略选出的动作相同。它学到的是 $\varepsilon$-greedy 策略对应的Q值，而非纯贪心策略的Q值。

    例子：网约车司机的SARSA学习

    继续使用网约车MDP（3个状态、6个状态-动作对），用SARSA算法从零开始学习Q值。设 $\gamma = 0.9$，$\alpha = 0.1$，$\varepsilon = 0.1$。

    import numpy as np
gamma, alpha, epsilon = 0.9, 0.1, 0.1
T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}
def step(s, action):
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum: return s_next, reward
    return s_next, reward

def eps_greedy(s, Q):
    acts = list(T[s].keys())
    if np.random.random() < epsilon:
        return np.random.choice(acts)
    return max(acts, key=lambda a: Q[s][a])

Q = {s: {a: 0.0 for a in T[s]} for s in T}
for ep in range(10000):
    s = ['c', 'b', 'a'][ep % 3]
    a = eps_greedy(s, Q)
    for _ in range(200):
        s2, R = step(s, a)
        a2 = eps_greedy(s2, Q)
        Q[s][a] += alpha * (R + gamma * Q[s2][a2] - Q[s][a])   # SARSA
        s, a = s2, a2

for s in T:
    for a in T[s]:
        print(f'Q({s}, {a}) = {Q[s][a]:.1f}')

    注意SARSA收敛后的Q值会略低于理论最优 $Q^*$。这是因为SARSA是在策略算法——它学到的是 $\varepsilon$-greedy 策略的Q值，而该策略以 $\varepsilon$ 的概率执行随机动作，因此其期望回报略低于纯贪心策略。

    Q-learning：离策略的TD控制

    Q-learning（Watkins, 1989）是强化学习中最重要的算法之一。与SARSA不同，Q-learning在更新时不使用实际选择的下一个动作 $A_{t+1}$，而是直接取下一状态所有动作中Q值最大的那个：

    Q-learning 更新规则
    $$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[\, R_t + \gamma \, \max_{a} Q(S_{t+1}, a) - Q(S_t, A_t) \,\right] \tag{3.1-9}$$

    TD目标是 $R_t + \gamma\,\max_a Q(S_{t+1}, a)$，与实际选择的 $A_{t+1}$ 无关。

    对比2.2节中的贝尔曼最优方程，Q-learning的更新目标正是其采样版本：

    $$\underbrace{R_t + \gamma \max_{a} Q(S_{t+1}, a)}_{\text{Q-learning TD目标（单次采样）}} \quad \approx \quad \underbrace{\sum_{s'} P(s' \mid s, a)\left[R(s,a) + \gamma \max_{a'} Q^*(s', a')\right]}_{\text{贝尔曼最优方程右端（需要模型）}}$$

    Q-learning用一次真实转移的样本 $(S_t, A_t, R_t, S_{t+1})$ 来近似贝尔曼最优方程中对所有可能后继状态的求和。经过足够多次的采样和更新，Q值会收敛到 $Q^*$——这已被严格证明（只要每个状态-动作对被访问无穷多次，且学习率满足一定条件）。

    Q-learning是离策略（off-policy）算法：行为策略（用来选择动作探索环境的策略）可以是 $\varepsilon$-greedy，但学到的Q值对应的是纯贪心的最优策略。换言之，智能体一边用 $\varepsilon$-greedy 探索，一边学习的却是最优策略的Q值——两个策略可以不同。

    Q-learning 算法
    1. 初始化 $Q(s, a)$ 为任意值（如全0），对所有 $s \in \mathcal{S}$，$a \in \mathcal{A}(s)$
    2. 对每个回合（episode）：
      1. 初始化状态 $S$
      2. 对回合中的每一步：
        1. 根据 $\varepsilon$-greedy 从 $Q$ 中选择动作 $A$
        2. 执行动作 $A$，观察奖励 $R$ 和下一状态 $S'$
        3. $Q(S, A) \leftarrow Q(S, A) + \alpha\,[R + \gamma\,\max_a Q(S', a) - Q(S, A)]$
        4. $S \leftarrow S'$
    Q-learning是离策略（off-policy）算法：更新使用 $\max_a Q(S', a)$，与行为策略选择的实际动作无关。

    例子：网约车司机的Q-learning

    继续使用网约车MDP，用Q-learning从零开始学习。同样设 $\gamma = 0.9$，$\alpha = 0.1$，$\varepsilon = 0.1$。初始时所有6个Q值均为0。让我们手算前几步的更新过程。

    第1步：在市中心(c)，选择"接单"，乘客去郊区(b)，车费 $R=8$
    $Q(c, \text{接单}) \leftarrow 0 + 0.1\times[8 + 0.9\times\max(0, 0) - 0] = 0.1 \times 8 = 0.8$
    第2步：在郊区(b)，选择"接单"，乘客留在郊区(b)，车费 $R=1$
    $Q(b, \text{接单}) \leftarrow 0 + 0.1\times[1 + 0.9\times\max(0, 0) - 0] = 0.1 \times 1 = 0.1$
    第3步：在郊区(b)，选择"空驶去市中心"，油费 $R=-3$，转移到市中心(c)
    $Q(b, \text{空驶}) \leftarrow 0 + 0.1\times[-3 + 0.9\times\max(0.8, 0) - 0] = 0.1 \times (-3 + 0.72) = -0.228$
    第4步：在市中心(c)，选择"空驶去机场"，油费 $R=-4$，转移到机场(a)
    $Q(c, \text{空驶}) \leftarrow 0 + 0.1\times[-4 + 0.9\times\max(0, 0) - 0] = 0.1 \times (-4) = -0.4$

    在学习初期，"空驶"动作的Q值为负（因为空驶本身有成本），看起来不如"接单"。但随着学习的深入，智能体会逐渐发现：从郊区空驶到市中心虽然短期亏损，但到了市中心后能获得更高的后续回报。经过数万步的学习，Q值会逐步收敛到理论最优值 $Q^*$：

    状态$Q^*$(接单)$Q^*$(空驶)最优动作
    市中心 (c)56.350.0接单
    郊区 (b)46.947.7空驶去市中心
    机场 (a)60.045.7接单
    Q-learning得到的最优策略：市中心→接单，郊区→空驶去市中心，机场→接单
    与2.2节动态规划得到的结果完全一致！

    关键区别在于：动态规划需要事先知道所有的转移概率（如市中心接单后去郊区的概率是30%），而Q-learning完全不需要这些信息。它仅通过反复在三个区域间运营，逐步从经验中学会了同样的最优策略。

    import numpy as np
gamma, alpha, epsilon = 0.9, 0.1, 0.1
T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}
def step(s, action):
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum: return s_next, reward
    return s_next, reward

Q = {s: {a: 0.0 for a in T[s]} for s in T}
for ep in range(10000):
    s = ['c', 'b', 'a'][ep % 3]
    for _ in range(200):
        acts = list(T[s].keys())
        a = np.random.choice(acts) if np.random.random() < epsilon \
            else max(acts, key=lambda x: Q[s][x])
        s2, R = step(s, a)
        Q[s][a] += alpha * (R + gamma * max(Q[s2].values()) - Q[s][a])  # Q-learning
        s = s2

for s in T:
    for a in T[s]:
        print(f'Q({s}, {a}) = {Q[s][a]:.1f}')

    SARSA 与 Q-learning 的对比

    比较维度SARSAQ-learning
    更新目标$R + \gamma\,Q(S', A')$
    （$A'$由策略选出）    		$R + \gamma\,\max_a Q(S', a)$
    （取最大值）
    学到的Q值当前 $\varepsilon$-greedy 策略的Q值最优策略的Q值 $Q^*$
    策略类型在策略（on-policy）离策略（off-policy）
    探索的影响探索会降低学到的Q值
    （因为考虑了探索动作的后果）    		探索不影响Q值的收敛目标
    （始终朝 $Q^*$ 收敛）
    安全性更保守：会避开"探索可能带来灾难"的区域更激进：假设未来总是最优行动

    从表格到函数近似

    在网约车例子中，状态空间只有3个状态、6个Q值，用表格存储绰绰有余。但在实际的管理科学问题中——比如有成百上千个区域的网约车调度、连续变化的库存水平、高维的交通状态——状态空间可能是巨大的甚至连续的，表格方法不再可行。解决方案是用函数近似（function approximation）来表示Q值：用一个参数化的函数 $Q_\theta(s, a)$（如神经网络）来代替查找表，输入状态-动作对，输出Q值估计。这就是Deep Q-Network（DQN）等深度强化学习算法的起点，我们将在第五章详细展开。

    小结

    算法更新目标是否需要等回合结束问题类型
    MC（蒙特卡洛）$G_t$（实际回报）是预测 / 控制
    SARSA$R + \gamma\,Q(S', A')$否（每步更新）控制（on-policy）
    Q-learning$R + \gamma\,\max_a Q(S', a)$否（每步更新）控制（off-policy）

    本节对比了 MC 和 TD(0) 的核心差异：MC 使用完整回报 $G_t$（无偏但高方差），TD(0) 使用一步自举目标 $R_{t+1} + \gamma V(S_{t+1})$（低方差但有偏）。两者各有优劣，那么一个自然的问题是：能否在它们之间找到一个连续的折中？下一节将介绍的 TD($\lambda$) 正是解决这个问题的方法，它通过一个参数 $\lambda$ 将 MC 和 TD(0) 统一到同一个框架中。
