3.1.2 时序差分学习

上一节的蒙特卡洛方法有一个明显的局限：必须等到整条轨迹结束，才能计算累计回报 $G_t$，然后才能更新值函数。对于持续运行的任务（如网约车24小时不停运营）或轨迹很长的问题，这带来了很大的延迟。我们能否在每走一步后就立即更新值函数？时序差分（Temporal Difference, TD）学习给出了肯定的答案。TD学习将蒙特卡洛方法的无模型学习能力与动态规划的递推更新思想结合在一起，是强化学习中最核心的算法思想之一。

TD(0)：一步自举更新

TD方法的思路来自贝尔曼方程。回忆2.1.3节的贝尔曼期望方程：

$$V^\pi(s) = \mathbb{E}_\pi\!\left[\,R_t + \gamma\, V^\pi(S_{t+1}) \mid S_t = s\,\right]$$

这个方程告诉我们：如果值函数是准确的，那么 $V^\pi(S_t)$ 应该等于 $R_t + \gamma\, V^\pi(S_{t+1})$ 的期望。蒙特卡洛方法用实际的完整回报 $G_t = R_t + \gamma R_{t+1} + \gamma^2 R_{t+2} + \cdots$ 来更新 $V(S_t)$，而TD方法只看一步：用当前的估计值 $V(S_{t+1})$ 来替代未来所有的回报，构造一个更新目标 $R_t + \gamma\,V(S_{t+1})$。

这个思路其实和2.2节的动态规划一脉相承：策略迭代和值迭代也是用下一状态的值函数估计 $V(s')$ 来更新当前状态的 $V(s)$，只不过DP需要对所有可能的 $s'$ 求加权平均（因为知道转移概率），而TD只使用实际采样到的那一个 $s'$。这种用已有估计值来更新估计值的做法，在强化学习中称为自举（bootstrapping），"自举"的字面意思是"拽着自己的鞋带把自己提起来"，形象地描述了"用估计更新估计"的递推过程。

TD(0) 更新规则

在时刻 $t$，智能体执行动作后观察到奖励 $R_t$ 和下一状态 $S_{t+1}$，立即更新：

$$V(S_t) \leftarrow V(S_t) + \alpha \left[\, \underbrace{R_t + \gamma \, V(S_{t+1})}_{\text{TD目标}} - V(S_t) \,\right] \tag{3.1-5}$$

其中 $\alpha$ 是学习率。与MC更新（式1.5-2）的关键区别在于：MC用实际回报 $G_t$ 作为更新目标，TD用一步估计 $R_t + \gamma\,V(S_{t+1})$ 作为更新目标。TD只需要走一步就能更新，无需等到轨迹结束。

直觉：用网约车例子理解TD

假设司机目前在市中心，算法对市中心的价值估计为 $V(c) = 40$（还不准确，真实值约为52.3）。司机按策略 $\pi_1$ 接了一单，乘客去郊区，车费8元。到达郊区后，算法对郊区的价值估计为 $V(b) = 35$。

图 3.1-3   TD更新的直觉。蓝色实线段是已经观测到的真实车费，黄色虚线段是算法对未来收入的估计。TD目标结合了真实观测和已有估计。

两种更新方式的区别如下：

  • MC做法：必须让司机继续运营完整个截断窗口（200步），记录从市中心出发的实际累计折扣收入 $G_t$（比如这次跑出来是53.2元），才能用 $G_t$ 更新 $V(c)$
  • TD做法：到达郊区的那一刻就可以更新。TD目标 $= 8 + 0.9 \times 35 = 39.5$，立即更新 $V(c) \leftarrow 40 + \alpha\,(39.5 - 40)$

TD目标39.5中仍包含对 $V(b)$ 的估计（可能不准确），但它比纯粹的旧估计40更可靠，因为其中的8元是司机真实赚到的车费。这正是自举的精髓：用一部分真实观测加一部分已有估计，构造一个比纯估计更好的更新目标。

TD误差

TD更新的核心量是 TD误差（TD Error），它衡量了一步转移之后的发现与之前的预期之间的差异：

TD误差的定义
$$\delta_t = R_t + \gamma \, V(S_{t+1}) - V(S_t) \tag{3.1-6}$$

TD更新可以简写为 $V(S_t) \leftarrow V(S_t) + \alpha\,\delta_t$。

TD误差的直觉含义是一种惊喜信号。如果 $\delta_t > 0$，说明这一步获得的奖励加上对未来的估计比之前预期的更好，应该上调 $V(S_t)$；如果 $\delta_t < 0$，则这一步的结果不如预期，应该下调 $V(S_t)$。这种"预测误差驱动学习"的机制与行为科学和神经经济学中的发现高度一致：多巴胺神经元的放电模式与TD误差的正负号表现出惊人的相似性（Schultz et al., 1997）。

MC vs TD：偏差与方差的权衡

MC和TD都能从交互经验中估计 $V^\pi$，但它们使用不同的更新目标，因此在统计性质上存在根本性的差异。

比较维度蒙特卡洛（MC）时序差分（TD）
更新目标实际回报 $G_t$一步估计 $R_t + \gamma\,V(S_{t+1})$
何时更新必须等轨迹结束每走一步立即更新
偏差无偏：$G_t$ 是 $V^\pi(S_t)$ 的无偏估计有偏：依赖当前 $V(S_{t+1})$ 的准确性
方差高方差：$G_t$ 受整条轨迹所有随机性影响低方差：只涉及一步转移的随机性
适用场景轨迹较短、有明确终止；无限期任务可通过截断+冗余段处理（见3.1.1节），但仍需等截断完成轨迹长或持续任务、需要在线学习

MC的更新目标 $G_t$ 是 $V^\pi(S_t)$ 的无偏估计，但它依赖于从时刻 $t$ 到轨迹结束的所有随机转移和奖励，因此方差很大。TD的更新目标 $R_t + \gamma\,V(S_{t+1})$ 只涉及一步转移的随机性，方差较低，但由于 $V(S_{t+1})$ 本身是一个估计值（可能不准确），TD目标存在偏差。这是统计学习中经典的偏差-方差权衡（bias-variance tradeoff）。

用网约车例子感受偏差与方差

让我们用具体数字来感受这个权衡。从状态 $c$（市中心）出发，按策略 $\pi_1$ 接单：

MC的更新目标 $G_t$：跑完200步截断窗口后的累计折扣收入。每次运行的结果都不同，因为200步中涉及大量随机转移。五次独立运行可能得到：

$$G_t = 67.1,\; 38.4,\; 52.8,\; 71.3,\; 29.6,\; \ldots$$

平均值收敛到 $V^{\pi_1}(c) = 52.3$（无偏），但单次结果的波动范围很大。

TD的更新目标 $R_t + \gamma\,V(S_{t+1})$：只涉及一步转移，从 $c$ 接单只有三种可能结果：

乘客目的地概率TD目标
$c$（短途）0.5$5 + 0.9 \times V(c)$
$b$（中途）0.3$8 + 0.9 \times V(b)$
$a$（长途）0.2$12 + 0.9 \times V(a)$

如果当前 $V$ 值接近真实值（$V(c) \approx 52.3$，$V(b) \approx 42.3$，$V(a) \approx 55.9$），三种TD目标分别为52.1、46.1、62.3，取值范围仅16元，远小于MC。但如果学习初期 $V(c) = V(b) = V(a) = 0$，三种TD目标分别为5、8、12，期望值仅7.3，远偏离真实值52.3。这就是偏差的代价。

切换到"学习初期"可以看到：TD箱型图虽然很窄（低方差），但整体偏移到了左侧，远离真实值（高偏差）。切换回"接近收敛"，两个箱型图都以真实值为中心，但MC的宽度远大于TD。随着学习的推进，$V$ 越来越准确，TD的偏差逐渐消失，而方差优势始终存在。这就是TD方法在实践中通常收敛更快的原因。

实际上，MC和TD可以统一为更一般的 $n$ 步回报框架：MC对应 $n = \infty$（使用完整回报），TD(0)对应 $n = 1$（使用一步估计），中间的 $n$ 步TD使用部分实际回报加自举，可以在偏差和方差之间取得更灵活的平衡。

DP、MC、TD三者的关系

现在让我们把动态规划（DP）、蒙特卡洛（MC）和时序差分（TD）放在一起比较。这三种方法可以从两个维度来理解：是否需要环境模型，以及是否使用自举。

动态规划（DP）

需要模型：是

更新方式：全宽度备份，对所有可能的下一状态求期望

自举：是

代表算法：策略迭代、值迭代

蒙特卡洛（MC）

需要模型：否

更新方式：基于采样，用完整轨迹的实际回报 $G_t$ 更新

自举：否

代表算法：首次访问MC、每次访问MC

时序差分（TD）

需要模型：否

更新方式：基于采样，用一步转移 $R_t + \gamma\,V(S_{t+1})$ 更新

自举：是

代表算法：TD(0)、SARSA、Q-learning

三者的区别可以用备份图（backup diagram）直观地展示。备份图描绘了一棵决策树，红色阴影标出了每种方法实际用来更新 $V(S_t)$ 的信息范围。

DP备份

$V(s) \leftarrow \mathbb{E}_\pi[R_t + \gamma\,V(S_{t+1})]$

需要模型，全宽度展开

MC备份

$V(S_t) \leftarrow V(S_t) + \alpha[G_t - V(S_t)]$

无需模型，沿一条轨迹采样到底

TD备份

$V(S_t) \leftarrow V(S_t) + \alpha[R_t + \gamma\,V(S_{t+1}) - V(S_t)]$

无需模型，只看一步就更新

状态节点   动作节点   可能的转移   用于更新的信息

图 3.1-5   DP、MC、TD 三种方法的备份图对比。红色阴影标出了每种方法用于更新 $V(S_t)$ 的信息范围：DP展开所有分支（需要模型），MC沿一条轨迹采样到底（无需模型，不自举），TD只采样一步（无需模型，自举）。

TD方法兼取了MC和DP各自的优势：它像MC一样不需要环境模型，直接从交互经验中学习；又像DP一样利用贝尔曼方程的递推结构进行自举更新，不需要等到轨迹结束。这种融合使TD成为现代强化学习中最广泛使用的算法基础。

例子：用TD(0)评估网约车策略

继续使用网约车MDP（3个状态：市中心 $c$、郊区 $b$、机场 $a$，折扣因子 $\gamma = 0.9$）。用TD(0)来评估策略 $\pi_1$（所有状态都选择"接单"）。与MC不同的是，TD(0)在每一步转移之后就立即更新值函数，无需等到轨迹结束。Python代码如下：

import numpy as np
gamma, alpha = 0.9, 0.01

# 网约车MDP转移模型
T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}

def step(s, action):
    """根据目的地概率采样下一状态，并返回对应车费"""
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum:
            return s_next, reward
    return s_next, reward

policy = {'c': '接单', 'b': '接单', 'a': '接单'}   # π₁
V = {'c': 0.0, 'b': 0.0, 'a': 0.0}

for ep in range(5000):
    s = np.random.choice(['c', 'b', 'a'])       # 随机选择起始状态
    for _ in range(200):                          # 每条轨迹最多200步
        s_next, R = step(s, policy[s])
        V[s] += alpha * (R + gamma * V[s_next] - V[s])   # TD(0)更新
        s = s_next

for s in ['c', 'b', 'a']:
    print(f'V({s}) = {V[s]:.1f}')
# 期望输出: V(c)≈52.3, V(b)≈42.3, V(a)≈55.9

注意代码中的关键区别：MC需要先运行完整轨迹、记录所有回报，然后回头逐状态更新；而TD(0)在每一步转移 $(s, a, R, s')$ 发生后就立即执行 V[s] += alpha * (R + gamma * V[s_next] - V[s])，无需等待。

从预测到控制

蒙特卡洛方法和时序差分学习都解决了策略评估（预测）问题：在不知道环境模型的情况下，估计给定策略 $\pi$ 的状态值函数 $V^\pi$。但我们的最终目标是找到最优策略（控制问题）。要实现控制，需要从学习 $V$ 值转向学习 $Q$ 值，因为 $Q(s,a)$ 直接给出了在状态 $s$ 选择动作 $a$ 的好坏，智能体无需知道转移概率就能据此选择动作。3.2节将介绍两种基于TD思想的控制算法：在策略的SARSA和离策略的Q-learning。
