3.3 Soft Actor-Critic

为什么需要SAC

回顾前两节：3.1节的DQN将Q-learning与深度神经网络结合，实现了离策略学习，能通过经验回放缓冲区高效利用历史数据，但它只能处理离散动作空间。3.2节的PPO采用策略梯度方法，天然支持连续动作空间，训练也比较稳定，但它是在策略算法，每批数据用完后就必须丢弃，样本效率较低。

在管理科学的实际应用中，许多决策变量天然是连续的：物流中的车辆调度速度、库存管理中的补货数量、金融中的投资组合权重、交通信号的绿灯时长等。对于这类问题，我们希望算法同时具备以下三个特性：

  • 支持连续动作空间，不需要将决策变量人为离散化；
  • 离策略学习，能够重复利用历史经验，提高样本效率；
  • 训练稳定，不容易因超参数设置不当而崩溃。

Soft Actor-Critic（SAC，Haarnoja et al., 2018）正是为了同时满足这三个需求而提出的。它的核心思想是在标准的强化学习目标中加入一个熵正则化项：不仅要最大化累积奖励，还要让策略尽可能保持随机性。这个看似简单的改动带来了深远的影响：它自然地鼓励探索，显著提升了训练的稳定性和鲁棒性。

SAC在管理科学中的优势：许多管理科学问题的仿真器运行成本高昂（例如，一次完整的交通仿真可能需要数分钟）。SAC的离策略特性使其能够充分利用每一条仿真数据，大幅减少所需的仿真次数。同时，它对超参数不敏感的特性，减少了研究人员的调参负担。

最大熵强化学习

标准强化学习的目标是最大化期望累积折扣奖励：

$$J_{\text{std}}(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t R_t\right]$$

SAC在此基础上增加了一个策略熵（policy entropy）奖励项，形成最大熵目标：

$$J(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t \Big(R_t + \alpha \,\mathcal{H}\big(\pi(\cdot \mid S_t)\big)\Big)\right] \tag{3.3-1}$$

其中 $\mathcal{H}\big(\pi(\cdot \mid s)\big)$ 是策略在状态 $s$ 下的熵。对连续动作空间下的策略分布 $\pi(\cdot \mid s)$，熵定义为：

$$\mathcal{H}\big(\pi(\cdot \mid s)\big) = -\int \pi(a \mid s) \log \pi(a \mid s) \, da = \mathbb{E}_{a \sim \pi(\cdot|s)}\big[-\log \pi(a \mid s)\big] \tag{3.3-2}$$

参数 $\alpha > 0$ 称为温度系数（temperature），它控制了熵项的权重，也就是探索与利用之间的平衡：

  • $\alpha$ 较大时，算法更倾向于保持策略的随机性（多探索）；
  • $\alpha$ 较小时，算法更倾向于追求高奖励（多利用）；
  • $\alpha \to 0$ 时，退化为标准的强化学习目标。
最大熵目标的直觉

想象一位基金经理需要在多个资产之间分配投资比例（连续动作）。最大熵目标要求他：在保证预期收益的前提下，尽可能分散投资。这样做的好处是：（1）避免过早押注于某个单一方案，在信息不充分时保持灵活性；（2）即使环境发生变化，分散的策略也更具鲁棒性；（3）学习过程中能持续探索不同的投资组合，不会因为偶然发现一个还不错的方案就停止尝试。

公式(3.3-1)可以等价地写成更紧凑的形式。将熵的定义代入，每一步的奖励变为 $R_t - \alpha \log \pi(A_t \mid S_t)$，即：

$$J(\pi) = \mathbb{E}_\pi\left[\sum_{t=0}^{\infty} \gamma^t \Big(R_t - \alpha \log \pi(A_t \mid S_t)\Big)\right] \tag{3.3-3}$$

这个形式更便于后续推导：可以把 $-\alpha \log \pi(a \mid s)$ 看作一个与动作概率相关的额外奖励。策略越随机（概率越分散），$-\log \pi$ 越大，获得的额外奖励越多。

软贝尔曼方程

回顾1.3.3节的标准贝尔曼期望方程，状态值函数与动作值函数之间满足：

$$V^\pi(s) = \mathbb{E}_{a \sim \pi}\big[Q^\pi(s, a)\big], \quad Q^\pi(s, a) = R(s,a) + \gamma \, \mathbb{E}_{s' \sim P}\big[V^\pi(s')\big]$$

在最大熵框架下，这些方程被修改为软贝尔曼方程（soft Bellman equation）。关键变化在于：状态值函数不再是Q值的简单期望，而是要减去策略的对数概率（即加上熵奖励）：

$$V(s) = \mathbb{E}_{a \sim \pi}\Big[Q(s, a) - \alpha \log \pi(a \mid s)\Big] \tag{3.3-4}$$
$$Q(s, a) = R(s, a) + \gamma \, \mathbb{E}_{s' \sim P}\Big[V(s')\Big] \tag{3.3-5}$$

将公式(3.3-4)代入公式(3.3-5)，可以消去 $V$ 得到：

$$Q(s, a) = R(s,a) + \gamma \, \mathbb{E}_{s'}\Big[\mathbb{E}_{a' \sim \pi}\big[Q(s', a') - \alpha \log \pi(a' \mid s')\big]\Big] \tag{3.3-6}$$

对比标准贝尔曼方程，软贝尔曼方程唯一的不同是多了 $-\alpha \log \pi(a \mid s)$ 项。这使得在固定点处，Q值不仅反映了累积奖励，还反映了未来策略的随机程度。

软贝尔曼方程的含义

在标准贝尔曼方程中，$V(s)$ 等于在状态 $s$ 下按照策略行动所获得的期望Q值。在软贝尔曼方程中，$V(s)$ 还包含了策略的熵奖励。这意味着：如果两个策略获得相同的Q值，但其中一个更随机（熵更高），那么更随机的策略对应更高的 $V(s)$。由此可以导出最大熵框架下的最优策略形式：

$$\pi^*(a \mid s) \propto \exp\!\Big(\frac{1}{\alpha} Q^*(s, a)\Big) \tag{3.3-7}$$

这是一个玻尔兹曼分布（Boltzmann distribution），温度系数 $\alpha$ 控制分布的尖锐程度。$\alpha$ 越小，分布越集中于Q值最大的动作；$\alpha$ 越大，分布越均匀。

SAC的网络架构

SAC使用三组神经网络协同工作：

1. 两个Q网络 $Q_{\theta_1}$ 和 $Q_{\theta_2}$

输入为状态-动作对 $(s, a)$，输出一个标量Q值。使用两个Q网络并取较小值是为了缓解Q值过高估计问题（与3.1节DQN中的Double DQN思想一致）。在计算目标值和更新策略时，总是取两者的最小值：

$$Q_{\min}(s, a) = \min\big(Q_{\theta_1}(s, a),\; Q_{\theta_2}(s, a)\big) \tag{3.3-8}$$

2. 策略网络 $\pi_\phi$

输入为状态 $s$，输出一个高斯分布的均值 $\mu_\phi(s)$ 和标准差 $\sigma_\phi(s)$。动作通过以下方式采样：

$$a = \tanh\!\big(\mu_\phi(s) + \sigma_\phi(s) \odot \epsilon\big), \quad \epsilon \sim \mathcal{N}(0, I) \tag{3.3-9}$$

这里的 $\tanh$ 将动作压缩到 $[-1, 1]$ 范围内，$\odot$ 表示逐元素相乘。这种采样方式称为重参数化技巧（reparameterization trick）：将随机性从策略网络中分离出来（体现在外部噪声 $\epsilon$ 中），使得梯度可以通过采样的动作反向传播到策略网络参数 $\phi$。

3. 目标Q网络 $Q_{\bar{\theta}_1}$ 和 $Q_{\bar{\theta}_2}$

这是两个Q网络的缓慢更新副本，用于计算TD目标值，以稳定训练过程（与3.1节DQN中的目标网络思想相同）：

$$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau) \bar{\theta}_i, \quad i = 1, 2 \tag{3.3-10}$$

其中 $\tau$ 通常取很小的值（如 $0.005$），使得目标网络缓慢跟踪主网络。

图 3.3-1　SAC 的网络架构。策略网络（绿色）输出高斯分布并采样动作；两个 Q 网络（黄色）评估动作价值；目标 Q 网络（红色虚线）通过软更新缓慢跟踪主网络，其最小值用于计算稳定的 TD 目标。

SAC算法

SAC的训练过程包含三个交替进行的更新步骤。每次从经验回放缓冲区中采样一个小批量 $(s, a, r, s')$，依次更新Q网络、策略网络和目标网络。

第一步：更新Q网络。首先计算TD目标值。用当前策略网络在下一状态 $s'$ 采样一个动作 $\tilde{a}'$，利用目标Q网络计算软Q值：

$$y = r + \gamma \Big(\min_{i=1,2} Q_{\bar{\theta}_i}(s', \tilde{a}') - \alpha \log \pi_\phi(\tilde{a}' \mid s')\Big), \quad \tilde{a}' \sim \pi_\phi(\cdot \mid s') \tag{3.3-11}$$

然后最小化每个Q网络的预测值与TD目标之间的均方误差：

$$L(\theta_i) = \mathbb{E}_{(s,a,r,s')}\left[\Big(Q_{\theta_i}(s, a) - y\Big)^2\right], \quad i = 1, 2 \tag{3.3-12}$$

第二步：更新策略网络。策略网络的目标是最大化期望的软Q值，即让策略选出的动作既有高Q值，又保持足够的随机性：

$$L(\phi) = \mathbb{E}_{s}\left[\mathbb{E}_{\tilde{a} \sim \pi_\phi}\Big[\alpha \log \pi_\phi(\tilde{a} \mid s) - Q_{\min}(s, \tilde{a})\Big]\right] \tag{3.3-13}$$

这里 $Q_{\min}(s, \tilde{a}) = \min(Q_{\theta_1}(s, \tilde{a}),\; Q_{\theta_2}(s, \tilde{a}))$。利用公式(3.3-9)的重参数化技巧，$\tilde{a}$ 关于 $\phi$ 可微，因此可以用标准的梯度下降来最小化 $L(\phi)$。

第三步：软更新目标网络。按照公式(3.3-10)缓慢更新目标网络参数。

Soft Actor-Critic (SAC) 算法
  1. 初始化策略网络参数 $\phi$，Q网络参数 $\theta_1, \theta_2$，令目标网络 $\bar{\theta}_i \leftarrow \theta_i$
  2. 初始化经验回放缓冲区 $\mathcal{D} \leftarrow \emptyset$
  3. 对每个时间步 $t = 0, 1, 2, \ldots$：
    1. 观察状态 $s_t$，从策略 $a_t \sim \pi_\phi(\cdot \mid s_t)$ 采样动作
    2. 执行 $a_t$，观察奖励 $r_t$ 和下一状态 $s_{t+1}$，将 $(s_t, a_t, r_t, s_{t+1})$ 存入 $\mathcal{D}$
    3. 从 $\mathcal{D}$ 中随机采样小批量 $\{(s, a, r, s')\}$
    4. 更新Q网络：对每个样本，用公式(3.3-11)计算TD目标 $y$，用公式(3.3-12)更新 $\theta_1, \theta_2$
    5. 更新策略网络：用公式(3.3-13)更新 $\phi$
    6. 软更新目标网络：$\bar{\theta}_i \leftarrow \tau \theta_i + (1 - \tau)\bar{\theta}_i$，$i = 1, 2$
SAC是离策略算法：步骤(c)中从缓冲区采样的数据可以来自任何历史策略。步骤(a)中的采样使用重参数化技巧（公式3.3-9）。

自动温度调节

温度系数 $\alpha$ 的选择对SAC的性能影响很大。手动调节 $\alpha$ 既费时又不可靠，因为最优的 $\alpha$ 值会随训练进程而变化（初期需要大量探索，后期应减少随机性）。SAC的改进版本（Haarnoja et al., 2018b）提出了一种自动温度调节机制：将 $\alpha$ 也作为一个可学习的参数，通过优化以下目标来自动调整：

$$L(\alpha) = \mathbb{E}_{a \sim \pi_\phi}\Big[-\alpha \big(\log \pi_\phi(a \mid s) + \bar{\mathcal{H}}\big)\Big] \tag{3.3-14}$$

其中 $\bar{\mathcal{H}}$ 是目标熵，通常设为 $-\dim(\mathcal{A})$（即负的动作空间维度）。这个约束的含义是：当策略的熵低于目标值时，自动增大 $\alpha$ 以鼓励更多探索；当熵高于目标值时，自动减小 $\alpha$ 以更多地利用已有知识。

在实践中，自动温度调节几乎是标配。它使得SAC成为一个超参数设置非常方便的算法：研究人员只需指定目标熵（通常用默认值即可），无需手动调节 $\alpha$。

例子：连续动作空间的库存补货决策

考虑一个单产品库存管理问题。某零售商每天需要决定向供应商订购多少单位的商品。

状态：当前库存水平 $s_t \in [0, 100]$。

动作：补货数量 $a_t \in [0, 50]$，是一个连续值。

状态转移：每天的顾客需求 $D_t$ 服从均值为20、标准差为8的正态分布（取非负部分），下一期库存为：

$$S_{t+1} = \max\big(0,\; S_t + a_t - D_t\big) \tag{3.3-15}$$

奖励：每天的利润等于销售收入减去库存持有成本和缺货惩罚：

$$R_t = \underbrace{p \cdot \min(S_t + a_t,\; D_t)}_{\text{销售收入}} - \underbrace{h \cdot \max(S_t + a_t - D_t,\; 0)}_{\text{持有成本}} - \underbrace{b \cdot \max(D_t - S_t - a_t,\; 0)}_{\text{缺货惩罚}} - \underbrace{c \cdot a_t}_{\text{订货成本}} \tag{3.3-16}$$

其中 $p = 10$ 为单位售价，$h = 1$ 为单位持有成本，$b = 5$ 为单位缺货惩罚，$c = 3$ 为单位订货成本。

为什么不离散化动作空间？

一种朴素的做法是将补货数量离散化为 $\{0, 1, 2, \ldots, 50\}$ 共51个离散动作，然后用DQN求解。但这有两个问题：（1）如果精度不够（例如只取 $\{0, 10, 20, 30, 40, 50\}$），最优补货量可能落在网格点之间，导致次优解；（2）如果精度太高，动作空间变得很大，Q网络需要对每个动作都估计一个Q值，计算量剧增。更严重的是，在多维连续动作场景（如同时决定多种商品的订货量），离散化导致的组合爆炸会使DQN完全不可行。SAC直接在连续空间上输出高斯分布，自然地避免了这些问题。

小结

SAC将离策略学习、Actor-Critic架构和最大熵正则化三者结合，在连续动作空间问题上表现出色。下面将本章介绍的三种算法从多个维度进行对比：

算法选择的经验法则：
动作空间离散且较小 → 优先考虑 DQN；
希望快速上手、追求训练稳定 → 优先考虑 PPO；
动作空间连续且仿真成本高 → 优先考虑 SAC。

实际研究中，建议先用PPO验证问题建模是否合理（因为PPO最容易调通），再根据需要切换到SAC以提高样本效率。下一节将讨论深度强化学习训练中的实践问题，包括超参数调优、奖励函数设计与训练稳定性。
