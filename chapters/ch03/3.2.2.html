3.2.2 Q-learning

  • 3.2.3 Double Q-learning

    • Q-learning：Off-policy 的 TD 控制

    Q-learning（Watkins, 1989）是强化学习中最重要的算法之一。它和 SARSA 的唯一区别在于 TD 目标的构造方式。回忆 SARSA 的更新规则(3.2-1)：TD 目标 $R_t + \gamma\, Q(S_{t+1}, A_{t+1})$ 中的 $A_{t+1}$ 是 $\varepsilon$-greedy 实际选出并将要执行的下一个动作。Q-learning 则不同：它不关心智能体下一步实际会选什么动作，而是直接取下一状态所有动作中 $Q$ 值最大的那个——好像在"想象"自己下一步一定会采取最优动作：

    Q-learning 更新规则
    $$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[\, R_t + \gamma \, \max_{a} Q(S_{t+1}, a) - Q(S_t, A_t) \,\right] \tag{3.2-3}$$

    TD 目标是 $R_t + \gamma\,\max_a Q(S_{t+1}, a)$，与实际选择的 $A_{t+1}$ 无关。

    对比2.2.2节中的贝尔曼最优方程(2.2-9)，Q-learning 的更新目标正是其采样版本：

    $$\underbrace{R_t + \gamma \max_{a} Q(S_{t+1}, a)}_{\text{Q-learning TD目标（单次采样）}} \quad \approx \quad \underbrace{R(s,a) + \gamma \sum_{s'} P(s' \mid s, a)\, \max_{a'} Q^*(s', a')}_{\text{贝尔曼最优方程(2.2-9)右端（需要模型）}}$$

    Q-learning 用一次真实转移的样本 $(S_t, A_t, R_t, S_{t+1})$ 来近似贝尔曼最优方程中对所有可能后继状态的求和。经过足够多次的采样和更新，$Q$ 值会收敛到 $Q^*$。这已被严格证明（Watkins & Dayan, 1992），证明思路与2.2.2节选读中的压缩映射论证一脉相承：Q-learning 的更新对应贝尔曼最优算子的随机逼近版本（只不过这里是对 $Q$ 值而非 $V$ 值），结合 Robbins-Monro 条件（每个状态-动作对被访问无穷多次，学习率递减且累积无穷大），可以保证 $Q_t \to Q^*$。

    为什么 Q-learning 是 off-policy 的

    对比两种算法的 TD 目标：

    $$\text{SARSA：}\; R_t + \gamma\,Q(S_{t+1},\, \underbrace{A_{t+1}}_{\text{实际选出}}) \qquad \text{Q-learning：}\; R_t + \gamma\,\underbrace{\max_{a} Q(S_{t+1}, a)}_{\text{想象的最优动作}} \tag{3.2-4}$$

    SARSA 用 $A_{t+1}$（$\varepsilon$-greedy 实际选出的动作）来估算未来回报。因此 SARSA 回答的问题是：如果我继续按 $\varepsilon$-greedy 行动（偶尔随机探索），从 $(S_t, A_t)$ 出发能获得多少期望回报？$Q$ 值收敛到的是 $\varepsilon$-greedy 策略本身的值 $Q^{\pi_\varepsilon}$。

    Q-learning 用 $\max_a Q(S_{t+1}, a)$ 来估算未来回报。这个 $\max$ 意味着什么？它假设在 $S_{t+1}$ 时，智能体一定会选 $Q$ 值最高的动作。因此 Q-learning 回答的问题是：如果我从下一步开始一直采取最优动作（不再探索），从 $(S_t, A_t)$ 出发能获得多少期望回报？$Q$ 值收敛到的是最优策略的值 $Q^*$。

    注意这里的关键矛盾：智能体实际上在用 $\varepsilon$-greedy 探索（否则无法学习新知识），但它学习的 $Q$ 值却对应一个纯贪心策略（每步都选最优动作）。用来生成数据的策略和 $Q$ 值所评估的策略不是同一个——这就是 off-policy 的含义。在 off-policy 的语境下，用来生成数据的 $\varepsilon$-greedy 称为行为策略（behavior policy），而 $Q$ 值所收敛到的纯贪心策略称为目标策略（target policy）。目标策略并没有被显式写成 $\pi(a|s) = \ldots$，而是隐含在 $\max$ 运算中——取 $\max$ 就等价于按纯贪心策略选动作。

    Q-learning 算法
    1. 初始化 $Q(s, a)$ 为任意值（如全0），对所有 $s \in \mathcal{S}$，$a \in \mathcal{A}(s)$
    2. 对每个回合（episode）：
      1. 初始化状态 $S$
      2. 对回合中的每一步：
        1. 根据 $\varepsilon$-greedy 从 $Q$ 中选择动作 $A$
        2. 执行动作 $A$，观察奖励 $R$ 和下一状态 $S'$
        3. $Q(S, A) \leftarrow Q(S, A) + \alpha\,[R + \gamma\,\max_a Q(S', a) - Q(S, A)]$
        4. $S \leftarrow S'$
    Q-learning 是 off-policy 算法：更新使用 $\max_a Q(S', a)$，与行为策略选择的实际动作无关。对比 SARSA 算法框，SARSA 需要先选出 $A_{t+1}$（既用于更新又用作下一步实际执行的动作），而 Q-learning 不需要。

    例子：网约车司机的 Q-learning

    继续使用网约车MDP，用 Q-learning 从零开始学习。同样设 $\gamma = 0.9$，$\alpha = 0.1$，$\varepsilon = 0.1$。初始时所有6个 $Q$ 值均为0。每步更新按公式(3.2-3)进行：

    $$Q(\underbrace{S_t}_{\text{当前状态}},\, \underbrace{A_t}_{\text{当前动作}}) \;\leftarrow\; Q(S_t, A_t) \;+\; \underbrace{\alpha}_{0.1} \Big[\, \underbrace{R_t}_{\text{即时奖励}} + \underbrace{\gamma}_{0.9}\, \max_{a} Q(\underbrace{S_{t+1}}_{\text{下一状态}}, a) - Q(S_t, A_t) \,\Big]$$

    与 SARSA 不同，更新所需的信息只有四元组 $(S_t, A_t, R_t, S_{t+1})$，不需要预先选出 $A_{t+1}$。点击下方按钮逐步观察每一步的更新。

    在学习初期，"空驶"动作的 $Q$ 值为负（因为空驶本身有成本），看起来不如"接单"。但随着学习的深入，智能体会逐渐发现：从郊区空驶到市中心虽然短期亏损，但到了市中心后能获得更高的后续回报。经过充分学习后，$Q$ 值会收敛到理论最优值 $Q^*$：

    关键区别在于：动态规划需要事先知道所有的转移概率（如市中心接单后去郊区的概率是30%），而 Q-learning 完全不需要这些信息。它仅通过反复在三个区域间运营，逐步从经验中学会了同样的最优策略。

    import numpy as np
gamma, epsilon = 0.9, 0.1
T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}
def step(s, action):
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum: return s_next, reward
    return s_next, reward

Q = {s: {a: 0.0 for a in T[s]} for s in T}
N = {s: {a: 0 for a in T[s]} for s in T}  # 访问计数
for ep in range(10000):
    s = ['c', 'b', 'a'][ep % 3]
    for _ in range(200):
        acts = list(T[s].keys())
        a = np.random.choice(acts) if np.random.random() < epsilon \
            else max(acts, key=lambda x: Q[s][x])
        s2, R = step(s, a)
        N[s][a] += 1
        alpha = 10 / (10 + N[s][a])          # 衰减学习率
        Q[s][a] += alpha * (R + gamma * max(Q[s2].values()) - Q[s][a])
        s = s2

for s in T:
    for a in T[s]:
        print(f'Q({s}, {a}) = {Q[s][a]:.1f}')

    从表格到函数近似

    在网约车例子中，状态空间只有3个状态、6个Q值，用表格存储绰绰有余。但在实际的管理科学问题中——比如有成百上千个区域的网约车调度、连续变化的库存水平、高维的交通状态——状态空间可能是巨大的甚至连续的，表格方法不再可行。解决方案是用函数近似（function approximation）来表示Q值：用一个参数化的函数 $Q_\theta(s, a)$（如神经网络）来代替查找表，输入状态-动作对，输出Q值估计。这就是Deep Q-Network（DQN）等深度强化学习算法的起点，我们将在第五章详细展开。

    至此，3.2节完成了从策略评估到最优控制的最后一步：SARSA（on-policy）和 Q-learning（off-policy）都能在不知道环境模型的情况下，通过与环境交互直接学习最优的动作选择。它们与3.1节的 MC 和 TD 方法共同构成了表格强化学习的完整工具箱。这些方法的共同点是通过"估计值函数 → 从值函数推导策略"的间接路径来寻找最优策略。下一节将介绍一种截然不同的思路——策略梯度方法：直接参数化策略 $\pi_\theta$，通过梯度上升优化策略的期望回报。
