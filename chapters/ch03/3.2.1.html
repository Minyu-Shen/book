3.2.1 SARSA

从预测到控制：为什么需要 $Q$ 值

3.1节介绍的 MC 和 TD 方法解决的是预测问题：给定策略 $\pi$，估计其状态值函数 $V^\pi(s)$。然而，我们最终的目标是控制问题：找到最优策略 $\pi^*$。如果只有 $V(s)$，要改善策略就必须知道转移概率 $P(s' \mid s, a)$，因为需要计算每个动作的期望回报来比较优劣：

$$\pi'(s) = \arg\max_a \left[\sum_{s'} P(s' \mid s, a)\bigl[R(s,a) + \gamma\,V^\pi(s')\bigr]\right]$$

这又回到了"依赖模型"的老路。解决方案是直接学习动作值函数 $Q(s, a)$。有了 $Q$ 值，策略改善只需 $\pi'(s) = \arg\max_a Q(s, a)$，完全不需要转移概率。

SARSA 的核心思想

3.1.2节的 TD(0) 通过单步自举更新 $V$ 值：

$$V(S_t) \leftarrow V(S_t) + \alpha\bigl[\underbrace{R_t + \gamma\,V(S_{t+1})}_{\text{TD目标}} - V(S_t)\bigr]$$

能否用同样的思路更新 $Q$ 值？回忆2.1.3节的贝尔曼期望方程(2.1-10)，它对 $Q$ 成立：

$$Q^\pi(s,a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s, a) \sum_{a'} \pi(a' \mid s')\, Q^\pi(s', a')$$

注意方程右端的两个求和。$\sum_{s'} P(s' \mid s,a)$ 需要转移概率模型，3.1.2节已经演示了如何用一次实际转移的样本 $S_{t+1}$ 来替代它。而 $\sum_{a'} \pi(a' \mid s')$ 需要显式知道策略 $\pi$ 在每个状态下选各个动作的概率。

SARSA 的巧妙之处在于：它不需要显式写出 $\pi(a'|s')$。策略隐含在 $Q$ 值表中：哪个动作的 $Q$ 值最大，就选哪个（后文会加入少量随机探索）。因此不必计算 $\sum_{a'} \pi(a'|s') Q(s',a')$，只需让智能体在 $S_{t+1}$ 按当前 $Q$ 值表选出一个动作 $A_{t+1}$，用 $Q(S_{t+1}, A_{t+1})$ 这一个样本来近似那个求和。这样，贝尔曼方程右端的两个求和都被单次采样替代：

SARSA 更新规则
$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[\, \underbrace{R_t + \gamma \, Q(S_{t+1}, A_{t+1})}_{\text{TD目标}} - Q(S_t, A_t) \,\right] \tag{3.2-1}$$

每次更新需要五个量：当前状态 $S_t$、当前动作 $A_t$、即时奖励 $R_t$、下一状态 $S_{t+1}$、下一动作 $A_{t+1}$。取首字母，即为算法名称 SARSA。

SARSA 就是策略迭代

回忆2.2.1节的策略迭代：先评估当前策略（求出 $Q^\pi$），再改善策略（对每个状态取 $\arg\max_a Q^\pi(s,a)$），如此交替直到收敛。SARSA 做的是同样的事，只是把评估和改善融合在了每一步中：

  1. 评估：用公式(3.2-1)更新 $Q$ 值（TD 学习，每步执行一次）
  2. 改善：用更新后的 $Q$ 值表选择下一个动作 $A_{t+1} = \arg\max_a Q(S_{t+1}, a)$

每次 SARSA 更新改变了 $Q$ 值，$\arg\max$ 就可能指向不同的动作，策略也随之改变。这种评估与改善持续交替的框架称为广义策略迭代（Generalized Policy Iteration, GPI）。

但纯贪心改善有一个严重问题。考虑网约车例子：

假设司机在郊区 $b$，所有 $Q$ 值初始为0。第一单碰巧留在了郊区（概率0.8），获得车费 $R = 1$，于是 $Q(b, \text{接单})$ 被更新为 $0 + 0.1 \times 1 = 0.1$，而 $Q(b, \text{空驶去市中心})$ 仍为0。如果采用纯贪心策略，司机此后每次到郊区都会选接单（因为 $0.1 > 0$），永远不会尝试空驶去市中心。

然而从2.2节的策略迭代结果我们知道，郊区的最优策略恰恰是空驶去市中心！纯贪心策略被一次低质量的早期经验锁死，错过了更好的选择。

2.2节的策略迭代不存在这个问题，因为它通过求解线性方程组来精确评估策略，能算出每个动作的真实价值。而 SARSA 从经验中学习，没被尝试过的动作的 $Q$ 值无法更新。这就是经典的探索-利用困境（exploration-exploitation dilemma）：智能体既要利用已有知识选择好的动作，又需要偶尔探索未知的动作以免遗漏更优方案。

$\varepsilon$-greedy 探索策略

最常用的解决方案是将纯贪心稍加修改：以 $1-\varepsilon$ 的概率选择 $\arg\max_a Q(s,a)$（利用），以 $\varepsilon$ 的概率随机选择一个动作（探索）。这就是 $\varepsilon$-greedy 策略：

$\varepsilon$-greedy 策略
$$\pi(a \mid s) = \begin{cases} 1 - \varepsilon + \dfrac{\varepsilon}{|\mathcal{A}(s)|}, & \text{if } a = \arg\max_{a'} Q(s, a') \\[4pt] \dfrac{\varepsilon}{|\mathcal{A}(s)|}, & \text{otherwise} \end{cases} \tag{3.2-2}$$

当 $\varepsilon = 0.1$ 且有2个动作时，最优动作被选中的概率是 $0.95$，其余动作被选中的概率是 $0.05$。这保证了每个动作都有正概率被选中，从而避免上述被锁死的困境。

将 $\varepsilon$-greedy 代入 SARSA 的框架，算法的完整循环变为：用 $\varepsilon$-greedy 选动作 → 执行并观察奖励 → 更新 $Q$ 值 → 用更新后的 $Q$ 值表再次 $\varepsilon$-greedy 选动作 → ...

注意，在这个循环中，用来生成数据的策略（$\varepsilon$-greedy）和 SARSA 正在评估的策略是同一个。这种性质有一个专门的名称：

On-policy 与 off-policy

如果算法用来生成数据（选择动作、与环境交互）的策略，和它正在评估、改善的策略是同一个，就称为 on-policy。反之，如果两者可以不同，就称为 off-policy。

SARSA 是 on-policy 算法。3.2.2节将介绍的 Q-learning 是 off-policy 算法。

On-policy 的一个直接后果是：TD 目标 $R_t + \gamma\,Q(S_{t+1}, A_{t+1})$ 中的 $A_{t+1}$ 是 $\varepsilon$-greedy 实际选出的动作，包括偶尔的随机探索。如果碰巧选了一个差的探索动作，这个"坏选择"的后果会被反映在 TD 目标中，拉低当前 $(S_t, A_t)$ 的 $Q$ 值。因此 SARSA 学到的 $Q$ 值不是最优策略的，而是 $\varepsilon$-greedy 策略自身的。

SARSA 算法

SARSA 算法
  1. 初始化 $Q(s, a)$ 为任意值（如全0），对所有 $s \in \mathcal{S}$，$a \in \mathcal{A}(s)$
  2. 对每个回合（episode）：
    1. 初始化状态 $S_0$
    2. 根据 $\varepsilon$-greedy 从 $Q$ 中选择动作 $A_0$
    3. 对 $t = 0, 1, 2, \ldots$（直到回合结束）：
      1. 执行动作 $A_t$，观察奖励 $R_t$ 和下一状态 $S_{t+1}$
      2. 根据 $\varepsilon$-greedy 从 $Q$ 中选择 $A_{t+1}$
      3. $Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha\,[R_t + \gamma\,Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)]$
      4. $S_t \leftarrow S_{t+1}$，$A_t \leftarrow A_{t+1}$
注意步骤 (ii)：$A_{t+1}$ 由 $\varepsilon$-greedy 选出后，既用于计算 TD 目标（步骤 iii，评估），又作为下一步实际执行的动作（步骤 iv，改善）。评估和改善共用同一个策略，这正是 on-policy 和 GPI 的体现。

例子：网约车司机的SARSA学习

继续使用网约车MDP（3个状态、6个状态-动作对），用SARSA算法从零开始学习 $Q$ 值。设 $\gamma = 0.9$，$\alpha = 0.1$，$\varepsilon = 0.1$。每步更新按公式(3.2-1)进行：

$$Q(\underbrace{S_t}_{\text{当前状态}},\, \underbrace{A_t}_{\text{当前动作}}) \;\leftarrow\; Q(S_t, A_t) \;+\; \underbrace{\alpha}_{0.1} \Big[\, \underbrace{R_t}_{\text{即时奖励}} + \underbrace{\gamma}_{0.9}\, Q(\underbrace{S_{t+1}}_{\text{下一状态}},\, \underbrace{A_{t+1}}_{\text{下一动作}}) - Q(S_t, A_t) \,\Big]$$

其中 $S_{t+1}$ 和 $R_t$ 是执行 $A_t$ 后从环境采样得到的（乘客去哪、车费多少由概率决定），$A_{t+1}$ 是 $\varepsilon$-greedy 根据当前 $Q$ 值表选出的。点击下方按钮逐步观察每一步的更新。

import numpy as np
gamma, alpha, epsilon = 0.9, 0.1, 0.1
T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}
def step(s, action):
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum: return s_next, reward
    return s_next, reward

def eps_greedy(s, Q):
    acts = list(T[s].keys())
    if np.random.random() < epsilon:
        return np.random.choice(acts)
    return max(acts, key=lambda a: Q[s][a])

Q = {s: {a: 0.0 for a in T[s]} for s in T}
for ep in range(10000):
    s = ['c', 'b', 'a'][ep % 3]
    a = eps_greedy(s, Q)              # 选初始动作
    for _ in range(200):
        s2, R = step(s, a)
        a2 = eps_greedy(s2, Q)        # 选下一动作 A'（on-policy）
        Q[s][a] += alpha * (R + gamma * Q[s2][a2] - Q[s][a])   # SARSA 更新
        s, a = s2, a2                 # S ← S', A ← A'

for s in T:
    for a in T[s]:
        print(f'Q({s}, {a}) = {Q[s][a]:.1f}')

收敛性

定理（SARSA 收敛性，Singh et al., 2000）

对于有限状态和动作的 MDP，如果满足：

  1. 每个状态-动作对 $(s,a)$ 被访问无穷多次；
  2. 学习率满足 Robbins-Monro 条件：$\sum_t \alpha_t(s,a) = \infty$，$\sum_t \alpha_t^2(s,a) < \infty$；

则 SARSA 的 $Q$ 值以概率1收敛到当前策略 $\pi$ 的真实动作值函数 $Q^\pi$。

直觉上，SARSA 更新可以写成

$$Q_{t+1}(s,a) = (1-\alpha_t)\,Q_t(s,a) + \alpha_t\bigl[R_t + \gamma\,Q_t(S_{t+1}, A_{t+1})\bigr]$$

这是一个随机逼近过程。TD 目标 $R_t + \gamma\,Q_t(S_{t+1}, A_{t+1})$ 的期望（对转移和 $A_{t+1}$ 取期望）恰好等于 $Q$ 的贝尔曼期望方程的右端，因此 SARSA 在期望意义上沿正确方向更新。Robbins-Monro 条件保证学习率既足够大（不会过早停止学习），又足够快地衰减（噪声最终被平均掉）。

SARSA 收敛到什么？

定理中收敛的目标是 $Q^\pi$，即当前 $\varepsilon$-greedy 策略本身的 $Q$ 值。当 $\varepsilon$ 固定不变时，$Q^{\pi_\varepsilon} \neq Q^*$，因为策略以 $\varepsilon$ 的概率执行随机动作，期望回报比最优策略低。这正是交互演示中 SARSA 的收敛值始终低于 $Q^*$ 列的原因。

如果让 $\varepsilon$ 随训练推进逐渐递减至0（如 $\varepsilon_t = 1/t$），同时保证每个状态-动作对仍被访问无穷多次，这种策略称为 GLIE（Greedy in the Limit with Infinite Exploration）。在 GLIE 下，同时让学习率 $\alpha$ 也递减以满足 Robbins-Monro 条件，SARSA 收敛到 $Q^*$。

固定 $\varepsilon$ 训练，$\varepsilon = 0$ 部署？

实践中一个常见做法是：用固定 $\varepsilon$ 训练至收敛，然后在部署时关闭探索（$\varepsilon = 0$），用纯贪心策略 $\arg\max_a Q^{\pi_\varepsilon}(s,a)$ 行动。这样得到的策略是最优的吗？

在网约车例子中，答案碰巧是肯定的：虽然 $Q^{\pi_\varepsilon}$ 的数值低于 $Q^*$，但每个状态下两个动作的大小关系没有改变，$\arg\max$ 排序一致。然而这并非总是成立。关键在于 $Q^{\pi_\varepsilon}$ 评估的是"执行 $a$ 之后继续用 $\varepsilon$-greedy 行动"的回报：如果某个动作本身是最优的，但后续的 $\varepsilon$ 探索会在某些路径上带来灾难性后果，$Q^{\pi_\varepsilon}$ 就会对该动作给出偏低的评价，导致排序翻转。

一个经典的例子是 Sutton & Barto 的悬崖边走（cliff walking）问题。如下图所示，智能体在一个 $4 \times 12$ 的网格中从左下角 S 走到右下角 G，每一步可以上下左右移动，每步奖励 $-1$。底部中间一排是悬崖（灰色），踩上去奖励 $-100$ 并被送回起点 S。

图 3.2-2：悬崖边走问题。最优路径沿悬崖边走（蓝色虚线），但 $\varepsilon$-greedy 在悬崖边有概率随机跌落。

最优路径沿悬崖边走（蓝色虚线），仅需13步。但 $\varepsilon$-greedy 在悬崖边的每一步都有概率随机向下跌落，触发 $-100$ 惩罚。SARSA 的 $Q^{\pi_\varepsilon}$ 会把悬崖边路径的价值评得很低（因为它评估的是"用 $\varepsilon$-greedy 走这条路"的回报，包含了跌落的风险），转而让远离悬崖的绕路（红色实线）排名更高。部署时切换到 $\varepsilon = 0$，跌落风险消失，绕路就不再是最优了。

网约车例子没有这个问题，因为探索的后果是温和的（最差不过多花几块钱油费），不存在类似"跌落悬崖"的不对称惩罚。如果需要保证收敛到 $Q^*$ 和最优策略，应使用递减的 $\varepsilon$。

选读：SARSA 收敛性的证明思路

SARSA 的收敛性证明基于随机逼近理论（stochastic approximation）。核心思路如下。

定义 TD 误差 $\delta_t = R_t + \gamma\,Q_t(S_{t+1}, A_{t+1}) - Q_t(S_t, A_t)$，SARSA 更新可写为

$$Q_{t+1}(s,a) = Q_t(s,a) + \alpha_t(s,a)\,\delta_t\,\mathbf{1}\{S_t=s,A_t=a\}$$

要证明 $Q_t \to Q^\pi$，需要验证该过程满足 Robbins-Monro 定理的条件：

  1. 期望方向正确。取条件期望 $\mathbb{E}[\delta_t \mid Q_t, S_t=s, A_t=a]$，利用 $A_{t+1} \sim \pi(\cdot \mid S_{t+1})$ 和 $S_{t+1} \sim P(\cdot \mid s, a)$，可以验证 $$\mathbb{E}[\delta_t \mid Q_t, S_t\!=\!s, A_t\!=\!a] = \bigl(\mathcal{T}^\pi Q_t\bigr)(s,a) - Q_t(s,a)$$ 其中 $\mathcal{T}^\pi$ 是贝尔曼期望算子。由于 $Q^\pi$ 是 $\mathcal{T}^\pi$ 的不动点，当 $Q_t = Q^\pi$ 时期望为0，更新在均值意义上指向不动点。
  2. 收缩性。贝尔曼期望算子 $\mathcal{T}^\pi$ 是关于 $\|\cdot\|_\infty$ 的 $\gamma$-压缩映射，即 $\|\mathcal{T}^\pi Q_1 - \mathcal{T}^\pi Q_2\|_\infty \leq \gamma\|Q_1 - Q_2\|_\infty$。这保证了不动点的唯一性以及迭代的全局稳定性。
  3. 噪声条件。$\delta_t$ 与其条件期望之差 $w_t = \delta_t - \mathbb{E}[\delta_t \mid Q_t, S_t, A_t]$ 是鞅差序列，且方差有界。结合 Robbins-Monro 条件 $\sum \alpha_t^2 < \infty$，噪声的累积效应收敛到0。

以上三个条件结合，由 Jaakkola et al. (1994) 的随机逼近定理可得 $Q_t \to Q^\pi$（a.s.）。完整证明见 Singh et al., Convergence Results for Single-Step On-Policy Reinforcement-Learning Algorithms, Machine Learning, 2000。
