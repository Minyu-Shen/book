3.1.4 TD($\lambda$)与资格迹

前两节分别介绍了 MC 和 TD(0)。MC 使用完整回报 $G_t$（无偏但高方差），TD(0) 使用一步自举目标 $R_t + \gamma V(S_{t+1})$（低方差但有偏）。这两个端点之间是否存在一个连续的过渡？本节将展示：通过引入参数 $\lambda \in [0, 1]$，我们可以在 MC 和 TD(0) 之间自由滑动，获得更灵活的偏差-方差权衡。

$n$ 步回报：从一步到多步

TD(0) 只看一步就自举：用 $R_t + \gamma V(S_{t+1})$ 替代完整回报。一个自然的推广是多看几步再自举。定义 $n$ 步回报：

$n$ 步回报
$$G_t^{(n)} = R_t + \gamma R_{t+1} + \cdots + \gamma^{n-1} R_{t+n-1} + \gamma^n V(S_{t+n}) \tag{3.1-10}$$

前 $n$ 步使用实际奖励，第 $n+1$ 步开始用当前估计值 $V(S_{t+n})$ 替代。当 $n=1$ 时就是 TD(0) 的目标 $R_t + \gamma V(S_{t+1})$；当 $n \to \infty$（或 $n$ 等于到回合结束的步数）时就是 MC 的目标 $G_t$。

$n$ 步回报给出了一族更新目标，形成从 TD(0) 到 MC 的离散谱：

$n$更新目标 $G_t^{(n)}$使用的真实奖励自举位置
$1$$R_t + \gamma V(S_{t+1})$1步$S_{t+1}$
$2$$R_t + \gamma R_{t+1} + \gamma^2 V(S_{t+2})$2步$S_{t+2}$
$3$$R_t + \gamma R_{t+1} + \gamma^2 R_{t+2} + \gamma^3 V(S_{t+3})$3步$S_{t+3}$
$\vdots$$\vdots$$\vdots$$\vdots$
$\infty$$G_t$（完整回报）全部不自举

例子：网约车司机的 $n$ 步回报

假设司机从市中心 $c$ 出发，按策略 $\pi_1$（总是接单）运营了5步，产生如下轨迹：

$$c \xrightarrow{R_0=5} c \xrightarrow{R_1=8} b \xrightarrow{R_2=1} b \xrightarrow{R_3=1} b \xrightarrow{R_4=8} a$$

设 $\gamma = 0.9$，当前的值函数估计为 $V(c) = 40$，$V(b) = 35$，$V(a) = 50$。对时刻 $t=0$ 的状态 $c$，不同 $n$ 对应的回报为：

$n=1$（TD(0)）
$G_0^{(1)} = R_0 + \gamma V(S_1) = 5 + 0.9 \times 40 = 41.0$
$n=2$
$G_0^{(2)} = R_0 + \gamma R_1 + \gamma^2 V(S_2) = 5 + 0.9 \times 8 + 0.81 \times 35 = 40.55$
$n=3$
$G_0^{(3)} = R_0 + \gamma R_1 + \gamma^2 R_2 + \gamma^3 V(S_3) = 5 + 7.2 + 0.81 + 0.729 \times 35 = 38.525$
$n=5$（用完所有已知奖励）
$G_0^{(5)} = 5 + 0.9 \times 8 + 0.81 \times 1 + 0.729 \times 1 + 0.6561 \times 8 + 0.9^5 \times 50 = 47.79$

$n$ 越大，回报中包含的真实信息越多，对值函数估计的依赖越少。但 $n$ 越大，涉及的随机转移也越多，方差也越高。选择哪个 $n$ 最好，取决于具体问题。能否不做这个选择，而是把所有 $n$ 步回报综合利用起来？

$\lambda$-回报：所有 $n$ 步回报的加权平均

TD($\lambda$) 的做法是：不选择某一个特定的 $n$，而是对所有 $n$ 步回报取加权平均。权重按几何级数递减：$n$ 步回报的权重为 $(1-\lambda)\lambda^{n-1}$，其中 $\lambda \in [0, 1]$。

$\lambda$-回报
$$G_t^\lambda = (1-\lambda) \sum_{n=1}^{\infty} \lambda^{n-1}\, G_t^{(n)} \tag{3.1-11}$$

系数 $(1-\lambda)\lambda^{n-1}$ 构成一个几何分布：$n=1$ 的权重为 $(1-\lambda)$，$n=2$ 的权重为 $(1-\lambda)\lambda$，$n=3$ 的权重为 $(1-\lambda)\lambda^2$，……所有权重之和为1。

$\lambda$ 控制了长程信息与短程信息之间的混合比例：

  • $\lambda = 0$ 时，所有权重集中在 $n=1$，$G_t^0 = G_t^{(1)} = R_t + \gamma V(S_{t+1})$，退化为 TD(0)
  • $\lambda = 1$ 时，权重均匀地延伸到无穷远，$G_t^1 = G_t$，退化为 MC
  • $0 < \lambda < 1$ 时，短步回报占更大权重，长步回报的权重指数衰减，在偏差和方差之间取得折中

图 3.1-5   $\lambda$-回报中各 $n$ 步回报的权重分布。$\lambda$ 越小，短步回报的权重越集中；$\lambda$ 越大，权重分布越平坦，长步回报获得更多权重。

有了 $\lambda$-回报，值函数的更新规则就是：

$$V(S_t) \leftarrow V(S_t) + \alpha\left[G_t^\lambda - V(S_t)\right] \tag{3.1-12}$$

这称为 $\lambda$-回报的前向视角（forward view）：站在时刻 $t$，向前看所有未来的奖励和状态，计算加权平均。前向视角在概念上清晰，但在实现上存在困难——要计算 $G_t^\lambda$，需要知道从 $t$ 之后所有步的奖励和状态，只有在回合结束后才能完成计算。这与 TD 的"每步更新"优势相矛盾。

解决这个矛盾的工具是资格迹（eligibility trace），它提供了一个等价的后向视角（backward view），使得每一步都可以在线更新所有状态的值函数。

资格迹：后向视角

资格迹 $e_t(s)$ 为每个状态 $s$ 维护一个实数，记录该状态"最近被访问的程度"。每走一步，所有状态的资格迹按 $\gamma\lambda$ 衰减，而刚被访问的状态的迹增加1：

累积资格迹（accumulating trace）
$$e_t(s) = \begin{cases} \gamma\lambda\, e_{t-1}(s) + 1, & \text{if } s = S_t \\ \gamma\lambda\, e_{t-1}(s), & \text{if } s \neq S_t \end{cases} \tag{3.1-13}$$

初始化 $e_{-1}(s) = 0$，对所有 $s$。乘积 $\gamma\lambda$ 控制迹的衰减速率。

资格迹的直觉是一种短期记忆：它标记了哪些状态对当前的 TD 误差"负有责任"。刚访问过的状态迹值高，很久以前访问的状态迹值接近零。

例子：网约车司机的资格迹变化

设 $\gamma = 0.9$，$\lambda = 0.8$，乘积 $\gamma\lambda = 0.72$。司机的轨迹为 $c \to c \to b \to b \to b \to a$。三个状态的资格迹变化如下：

时刻访问状态$e_t(c)$$e_t(b)$$e_t(a)$
$t=0$$c$$0 + 1 = 1$$0$$0$
$t=1$$c$$0.72 \times 1 + 1 = 1.72$$0$$0$
$t=2$$b$$0.72 \times 1.72 = 1.24$$0 + 1 = 1$$0$
$t=3$$b$$0.72 \times 1.24 = 0.89$$0.72 \times 1 + 1 = 1.72$$0$
$t=4$$b$$0.72 \times 0.89 = 0.64$$0.72 \times 1.72 + 1 = 2.24$$0$
$t=5$$a$$0.72 \times 0.64 = 0.46$$0.72 \times 2.24 = 1.61$$0 + 1 = 1$

可以看到，$c$ 的资格迹在 $t=1$ 时因被再次访问而增大到 1.72，之后逐步衰减。$b$ 被连续访问三次，迹值不断积累到2.24。到 $t=5$ 时，虽然司机已经到了机场，但 $c$ 和 $b$ 的迹值仍然不为零，意味着当前的 TD 误差仍会回溯更新这些之前访问过的状态。

TD($\lambda$) 算法

有了资格迹，TD($\lambda$) 的每步更新变得简洁：计算当前时刻的 TD 误差 $\delta_t$，然后用它乘以每个状态的资格迹来更新所有状态的值函数。

TD($\lambda$) 算法
  1. 初始化 $V(s)$ 为任意值，对所有 $s \in \mathcal{S}$
  2. 对每个回合（episode）：
    1. 初始化 $e(s) = 0$，对所有 $s$
    2. 初始化状态 $S_0$
    3. 对回合中的每一步 $t = 0, 1, 2, \ldots$：
      1. 执行策略 $\pi$ 选择的动作，观察 $R_t$ 和 $S_{t+1}$
      2. $\delta_t \leftarrow R_t + \gamma\,V(S_{t+1}) - V(S_t)$
      3. $e(S_t) \leftarrow e(S_t) + 1$
      4. 对所有状态 $s$：
        • $V(s) \leftarrow V(s) + \alpha\,\delta_t\,e(s)$
        • $e(s) \leftarrow \gamma\lambda\,e(s)$
步骤 (iii) 先增加当前状态的迹，步骤 (iv) 中 $\delta_t\,e(s)$ 完成更新后再衰减所有迹。在表格方法中，"对所有状态 $s$"只需遍历迹值非零的状态。

TD($\lambda$) 的后向视角可以这样理解：每一步产生一个 TD 误差 $\delta_t$，这个误差不仅更新当前状态 $S_t$，还回溯更新之前访问过的状态。更新量 $\alpha\,\delta_t\,e_t(s)$ 与资格迹成正比——刚访问过的状态更新多，很久以前访问的状态更新少。这就像给所有"有资格"分享功劳（或承担责任）的状态分配信用。

前向视角与后向视角的等价性

前向视角（式 $\text{(3.1-12)}$ 用 $\lambda$-回报更新）和后向视角（用资格迹逐步更新）在离线设置下是等价的：一个回合结束后，每个状态的累计更新量相同。直觉上，前向视角问的是"这个状态的未来回报是多少"，后向视角问的是"当前的误差应该归因给过去的哪些状态"。两种视角殊途同归。

选读：前向视角与后向视角等价性的证明思路

离线等价性

考虑一个长度为 $T$ 步的回合。在离线设置中（即值函数在回合结束后才统一更新），前向视角对状态 $s$ 的总更新量为：

$$\Delta V^{\text{forward}}(s) = \alpha \sum_{t:\, S_t = s} \left[G_t^\lambda - V(S_t)\right]$$

后向视角对状态 $s$ 的总更新量为：

$$\Delta V^{\text{backward}}(s) = \alpha \sum_{t=0}^{T-1} \delta_t\, e_t(s)$$

要证明二者相等，关键步骤是将 $\lambda$-回报 $G_t^\lambda$ 展开为 TD 误差 $\delta_t, \delta_{t+1}, \ldots$ 的加权和。利用 $G_t^{(n)}$ 的递推关系，可以证明：

$$G_t^\lambda - V(S_t) = \sum_{k=0}^{T-1-t} (\gamma\lambda)^k\, \delta_{t+k}$$

将此代入前向视角的更新量，交换求和顺序（从"对每个被访问的 $t$ 求和"变为"对每个时刻 $k$ 的 $\delta_k$ 求和"），就能得到后向视角的形式。资格迹 $e_t(s)$ 恰好起到了在交换求和顺序后正确累积权重的作用。

严格证明见 Sutton & Barto (2018) 第12章。注意在线设置（每步立即更新 $V$）中两个视角存在微小差异，但实践中可以忽略。

两个端点：TD(0) 和 TD(1)

$\lambda$ 的两个极端情况值得特别说明：

$\lambda = 0$：资格迹在每步衰减因子为 $\gamma \times 0 = 0$，只有当前被访问的状态有非零迹值。因此每一步只更新当前状态 $S_t$，更新量为 $\alpha\,\delta_t$，与 TD(0) 完全相同。

$\lambda = 1$：衰减因子为 $\gamma$，迹值衰减较慢。在回合结束时，可以证明每个状态的累计更新量等于 MC 更新。因此 TD(1) 等价于每次访问蒙特卡洛方法（every-visit MC），但 TD(1) 可以在线逐步更新，而不必等到回合结束。

统一框架：TD($\lambda$) 通过一个参数 $\lambda \in [0, 1]$ 将 TD(0) 和 MC 连成一个连续统。
$\lambda = 0 \Rightarrow$ TD(0)（纯自举，低方差高偏差）
$\lambda = 1 \Rightarrow$ MC（纯采样，高方差无偏差）
$0 < \lambda < 1 \Rightarrow$ 中间地带，通常 $\lambda \in [0.8, 0.95]$ 效果较好

Python 实现

import numpy as np

gamma, alpha, lam = 0.9, 0.1, 0.8
# 网约车MDP: 策略π₁（总是接单）
T = {
    'c': {'c': (0.5, 5), 'b': (0.3, 8), 'a': (0.2, 12)},
    'b': {'c': (0.1, 5), 'b': (0.8, 1), 'a': (0.1, 8)},
    'a': {'c': (0.6, 14), 'b': (0.4, 10)}
}
states = ['c', 'b', 'a']

def step(s):
    """按策略π₁接单, 返回 (下一状态, 奖励)"""
    trans = T[s]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum:
            return s_next, reward
    return s_next, reward

V = {s: 0.0 for s in states}
for ep in range(3000):
    e = {s: 0.0 for s in states}       # 每回合重置资格迹
    s = states[ep % 3]
    for _ in range(200):
        s2, R = step(s)
        delta = R + gamma * V[s2] - V[s]   # TD误差
        e[s] += 1                           # 累积迹
        for si in states:
            V[si] += alpha * delta * e[si]  # 更新所有状态
            e[si] *= gamma * lam            # 衰减迹
        s = s2

for s in states:
    print(f'V({s}) = {V[s]:.1f}')

小结

方法更新目标参数设置偏差方差
TD(0)$R_t + \gamma V(S_{t+1})$$\lambda = 0$有偏低
TD($\lambda$)$G_t^\lambda$（$\lambda$-回报）$0 < \lambda < 1$有偏（偏差随 $\lambda$ 增大而减小）中等
MC$G_t$（完整回报）$\lambda = 1$无偏高

TD($\lambda$) 和资格迹统一了 MC 和 TD 的更新机制。在实践中，$\lambda$ 的最优值取决于问题的结构，通常需要通过实验调整。在深度强化学习中，类似的思想以广义优势估计（Generalized Advantage Estimation, GAE）的形式出现，用 $\lambda$-加权的 TD 误差来估计优势函数，这将在第五章 PPO 的训练实践中进一步讨论。
