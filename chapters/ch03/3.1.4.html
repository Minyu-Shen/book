3.1.4 TD($\lambda$)与资格迹

前两节分别介绍了 MC 和 TD(0)。MC 使用完整回报 $G_t$（无偏但高方差），TD(0) 使用一步自举目标 $R_t + \gamma V(S_{t+1})$（低方差但有偏）。这两个端点之间是否存在一个连续的过渡？本节将展示：通过引入参数 $\lambda \in [0, 1]$，我们可以在 MC 和 TD(0) 之间自由滑动，获得更灵活的偏差-方差权衡。

$n$ 步回报：从一步到多步

回忆一下：TD(0) 的更新目标是 $R_t + \gamma V(S_{t+1})$，只用了一步真实奖励就自举；MC 的更新目标是完整回报 $G_t = R_t + \gamma R_{t+1} + \gamma^2 R_{t+2} + \cdots$，用了所有真实奖励。一个自然的推广是：用前 $n$ 步的真实奖励，然后自举。这就是 $n$ 步回报：

$n$ 步回报
$$G_t^{(n)} = R_t + \gamma R_{t+1} + \cdots + \gamma^{n-1} R_{t+n-1} + \gamma^n V(S_{t+n}) \tag{3.1-10}$$

$n=1$ 时，$G_t^{(1)} = R_t + \gamma V(S_{t+1})$，就是 TD(0) 的更新目标。$n \to \infty$ 时（或 $n$ 等于到回合结束的步数），所有奖励都用真实值、不再自举，$G_t^{(\infty)} = G_t$，就是 MC 的更新目标。因此 $n$ 步回报是 TD(0) 和 MC 之间的中间地带。

$n$ 步回报给出了一族更新目标，形成从 TD(0) 到 MC 的离散谱：

$n$更新目标 $G_t^{(n)}$使用的真实奖励自举位置
$1$$R_t + \gamma V(S_{t+1})$1步$S_{t+1}$
$2$$R_t + \gamma R_{t+1} + \gamma^2 V(S_{t+2})$2步$S_{t+2}$
$3$$R_t + \gamma R_{t+1} + \gamma^2 R_{t+2} + \gamma^3 V(S_{t+3})$3步$S_{t+3}$
$\vdots$$\vdots$$\vdots$$\vdots$
$\infty$$G_t$（完整回报）全部不自举

例子：网约车司机的 $n$ 步回报

假设司机从市中心 $c$ 出发，按策略 $\pi_1$（总是接单）运营了5步，产生如下轨迹：

$$c \xrightarrow{R_0=5} c \xrightarrow{R_1=8} b \xrightarrow{R_2=1} b \xrightarrow{R_3=1} b \xrightarrow{R_4=8} a$$

设 $\gamma = 0.9$，当前的值函数估计为 $V(c) = 40$，$V(b) = 35$，$V(a) = 50$。对时刻 $t=0$ 的状态 $c$，不同 $n$ 对应的回报为：

$n=1$（TD(0)）
$G_0^{(1)} = R_0 + \gamma V(S_1) = 5 + 0.9 \times 40 = 41.0$
$n=2$
$G_0^{(2)} = R_0 + \gamma R_1 + \gamma^2 V(S_2) = 5 + 0.9 \times 8 + 0.81 \times 35 = 40.55$
$n=3$
$G_0^{(3)} = R_0 + \gamma R_1 + \gamma^2 R_2 + \gamma^3 V(S_3) = 5 + 7.2 + 0.81 + 0.729 \times 35 = 38.525$
$n=5$（用完所有已知奖励）
$G_0^{(5)} = 5 + 0.9 \times 8 + 0.81 \times 1 + 0.729 \times 1 + 0.6561 \times 8 + 0.9^5 \times 50 = 47.79$

$n$ 越大，回报中包含的真实信息越多，对值函数估计的依赖越少。但 $n$ 越大，涉及的随机转移也越多，方差也越高。选择哪个 $n$ 最好，取决于具体问题。能否不做这个选择，而是把所有 $n$ 步回报综合利用起来？

$\lambda$-回报：所有 $n$ 步回报的加权平均

上面的 $n$ 步回报给出了一族估计目标 $G_t^{(1)}, G_t^{(2)}, G_t^{(3)}, \ldots$，每一个都可以用来更新 $V(S_t)$。选用哪个 $n$ 就引入了一个离散的超参数。TD($\lambda$) 的做法是：不选择某一个特定的 $n$，而是同时使用所有可能的 $n$ 步回报，按照几何级数递减的权重将它们混合在一起。

$\lambda$-回报
$$G_t^\lambda = (1-\lambda) \sum_{n=1}^{\infty} \lambda^{n-1}\, G_t^{(n)} \tag{3.1-11}$$

记号约定：$G_t^{(n)}$ 上标带括号，表示固定 $n$ 步的回报（式(3.1-10)）；$G_t^\lambda$ 上标不带括号，表示用 $\lambda$ 对所有 $n$ 步回报做加权混合的结果。

这个公式对所有回溯深度 $n = 1, 2, 3, \ldots$ 的 $n$ 步回报做加权求和。求和从 $n=1$（即 $G_t^{(1)}$，TD(0) 的目标）一直到 $n = \infty$（即 $G_t^{(\infty)} = G_t$，MC 的目标），覆盖了所有可能的回溯深度。每个 $n$ 步回报的权重为：

  • $n=1$（TD(0) 目标）的权重为 $(1-\lambda) \cdot \lambda^0 = (1-\lambda)$
  • $n=2$（二步回报）的权重为 $(1-\lambda) \cdot \lambda^1 = (1-\lambda)\lambda$
  • $n=3$（三步回报）的权重为 $(1-\lambda) \cdot \lambda^2 = (1-\lambda)\lambda^2$
  • $\cdots$

权重按等比级数衰减。由求和公式 $(1-\lambda)\sum_{n=1}^{\infty}\lambda^{n-1} = 1$ 可知它们的总和恰好为 $1$，因此 $G_t^\lambda$ 是一个合法的加权平均。

注意 $\lambda$ 不是用来选择某个固定的 $n$ 的——$\lambda$-回报始终利用所有 $n$ 步回报的信息，只是通过 $\lambda$ 调节各项的权重分配。因此 TD($\lambda$) 只引入了一个连续的超参数 $\lambda \in [0, 1]$，取代了离散的 $n$ 的选择。

$\lambda$ 控制了长程信息与短程信息之间的混合比例：

  • $\lambda = 0$ 时，$n=1$ 的权重为 $(1-0) \cdot 0^0 = 1$（这里 $0^0 = 1$），$n \geq 2$ 的权重为 $(1-0) \cdot 0^{n-1} = 0$。所有权重集中在一步回报上，$G_t^0 = G_t^{(1)}$，退化为 TD(0)
  • $\lambda = 1$ 时，每一项的权重 $(1-1) \cdot 1^{n-1} = 0$，但 $\lambda$-回报的极限为完整回报 $G_t$，退化为 MC
  • $0 < \lambda < 1$ 时，短步回报占更大权重，长步回报的权重指数衰减，在偏差和方差之间取得折中

图 3.1-5   $\lambda$-回报中各 $n$ 步回报的权重分布（显示 $n=1$ 至 $n=10$，灰色虚线柱为剩余权重）。拖动滑块观察：$\lambda$ 越小，权重越集中在短步回报；$\lambda$ 越大，权重分布越平坦。$\lambda = 1$ 时所有有限项的权重趋于零，全部权重归入完整回报（MC）。

有了 $\lambda$-回报，值函数的更新规则就是：

$$V(S_t) \leftarrow V(S_t) + \alpha\left[G_t^\lambda - V(S_t)\right] \tag{3.1-12}$$

这称为前向视角（forward view）：站在时刻 $t$，向前看所有未来的奖励和状态，计算加权平均。

$\lambda$-回报与 TD 误差的关系

$\lambda$-回报的定义（式(3.1-11)）看起来需要对无穷多个 $n$ 步回报求和，计算似乎很复杂。但有一个简洁的等价形式，可以将 $\lambda$-回报完全用 TD 误差 $\delta_t = R_t + \gamma V(S_{t+1}) - V(S_t)$ 表达出来。这个恒等式非常重要：它不仅给出了 $\lambda$-回报的高效计算方法，也是目前最广泛使用的策略优化算法 PPO 中核心组件 GAE（广义优势估计，5.2节）的理论基础。

推导分两步。

第一步：$n$ 步误差可以写成 $n$ 个连续 TD 误差之和。

$n$ 步误差的 TD 误差分解
$$G_t^{(n)} - V(S_t) = \sum_{l=0}^{n-1} \gamma^l\, \delta_{t+l} \tag{3.1-13}$$

这可以通过望远镜求和（telescoping sum）验证。将每个 $\delta_{t+l} = R_{t+l} + \gamma V(S_{t+l+1}) - V(S_{t+l})$ 代入右边，相邻项的 $V$ 会逐项抵消（$+\gamma V(S_{t+1})$ 和 $-\gamma V(S_{t+1})$ 相消，$+\gamma^2 V(S_{t+2})$ 和 $-\gamma^2 V(S_{t+2})$ 相消，……），最终只剩下：

$$\sum_{l=0}^{n-1} \gamma^l\, \delta_{t+l} = -V(S_t) + R_t + \gamma R_{t+1} + \cdots + \gamma^{n-1} R_{t+n-1} + \gamma^n V(S_{t+n}) = G_t^{(n)} - V(S_t)$$

换言之，$n$ 步回报与当前估计值之间的差距，恰好等于沿途经历的 $n$ 个 TD 误差的折扣累加。

第二步：将式(3.1-13)代入 $\lambda$-回报的定义，对所有 $n$ 做加权平均。

$$G_t^\lambda - V(S_t) = (1-\lambda)\sum_{n=1}^{\infty}\lambda^{n-1}\left[\sum_{l=0}^{n-1}\gamma^l\,\delta_{t+l}\right]$$

交换求和顺序：对每个 $\delta_{t+l}$，它在所有 $n \geq l+1$ 的项中出现，权重之和为 $(1-\lambda)\sum_{n=l+1}^{\infty}\lambda^{n-1}\gamma^l = (\gamma\lambda)^l$。因此：

$\lambda$-回报的 TD 误差表示
$$G_t^\lambda - V(S_t) = \sum_{l=0}^{\infty} (\gamma\lambda)^l\, \delta_{t+l} \tag{3.1-14}$$

$\lambda$-回报与当前估计之差，等于所有未来 TD 误差的 $(\gamma\lambda)$-折扣累加。

这个恒等式意义重大：

  • 计算上：给定一条轨迹的所有 TD 误差 $\delta_0, \delta_1, \ldots$，可以直接用式(3.1-14)从后往前递推计算 $\lambda$-回报，无需逐一算出所有 $n$ 步回报再加权平均
  • 理论上：这个恒等式正是5.2节 PPO 中广义优势估计（GAE）的基础。GAE 的公式 $\hat{A}_t^{\text{GAE}} = \sum_{l=0}^{T-t}(\gamma\lambda)^l \delta_{t+l}$（式(5.2-12)）与式(3.1-14)形式完全相同——GAE 就是将 $\lambda$-回报的思想应用于优势函数估计

两个端点：TD(0) 和 TD(1)

$\lambda$ 的两个极端情况值得特别说明：

$\lambda = 0$：式(3.1-14)只保留 $l=0$ 一项，$G_t^0 - V(S_t) = \delta_t$，更新量为 $\alpha\,\delta_t$，与 TD(0) 完全相同。

$\lambda = 1$：衰减因子变为 $\gamma$，式(3.1-14)变为 $\sum_{l=0}^{\infty}\gamma^l\delta_{t+l} = G_t - V(S_t)$（由式(3.1-13)取 $n \to \infty$），更新目标就是完整回报 $G_t$，等价于蒙特卡洛方法。

统一框架：TD($\lambda$) 通过一个参数 $\lambda \in [0, 1]$ 将 TD(0) 和 MC 连成一个连续统。
$\lambda = 0 \Rightarrow$ TD(0)（纯自举，低方差高偏差）
$\lambda = 1 \Rightarrow$ MC（纯采样，高方差无偏差）
$0 < \lambda < 1 \Rightarrow$ 中间地带，通常 $\lambda \in [0.8, 0.95]$ 效果较好

Python 实现

利用式(3.1-14)，在每条轨迹上从后往前递推计算 $\lambda$-回报，然后更新值函数：

import numpy as np

gamma, alpha, lam = 0.9, 0.1, 0.8
# 网约车MDP: 策略π₁（总是接单）
T = {
    'c': {'c': (0.5, 5), 'b': (0.3, 8), 'a': (0.2, 12)},
    'b': {'c': (0.1, 5), 'b': (0.8, 1), 'a': (0.1, 8)},
    'a': {'c': (0.6, 14), 'b': (0.4, 10)}
}
states = ['c', 'b', 'a']

def step(s):
    trans = T[s]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum:
            return s_next, reward
    return s_next, reward

V = {s: 0.0 for s in states}
for ep in range(3000):
    # 收集一条轨迹
    traj = []
    s = states[ep % 3]
    for _ in range(200):
        s2, R = step(s)
        traj.append((s, R, s2))
        s = s2
    # 从后往前递推计算 λ-回报误差: G^λ_t - V(S_t) = δ_t + γλ(G^λ_{t+1} - V(S_{t+1}))
    gl_err = 0.0
    for s, R, s2 in reversed(traj):
        delta = R + gamma * V[s2] - V[s]
        gl_err = delta + gamma * lam * gl_err
        V[s] += alpha * gl_err

for s in states:
    print(f'V({s}) = {V[s]:.1f}')
选读：资格迹与 TD($\lambda$) 的在线实现

从批量计算到在线更新

上面的前向视角实现需要先收集完整条轨迹，再从后往前计算。在某些场景下，我们希望每走一步就立即更新值函数，不必等到回合结束。资格迹（eligibility trace）提供了这样一种在线机制。

资格迹 $e_t(s)$ 为每个状态 $s$ 维护一个实数，记录该状态"最近被访问的程度"。每走一步，所有状态的迹按 $\gamma\lambda$ 衰减，刚被访问的状态的迹增加1：

$$e_t(s) = \begin{cases} \gamma\lambda\, e_{t-1}(s) + 1, & \text{if } s = S_t \\ \gamma\lambda\, e_{t-1}(s), & \text{if } s \neq S_t \end{cases}$$

在每个回合开始时，所有迹值初始化为零。有了资格迹，TD($\lambda$) 的在线算法为：

TD($\lambda$) 在线算法
  1. 初始化 $V(s)$ 为任意值，对所有 $s \in \mathcal{S}$
  2. 对每个回合：
    1. 初始化 $e(s) = 0$，对所有 $s$；初始化状态 $S_0$
    2. 对回合中的每一步 $t = 0, 1, 2, \ldots$：
      1. 执行策略 $\pi$ 选择的动作，观察 $R_t$ 和 $S_{t+1}$
      2. $\delta_t \leftarrow R_t + \gamma\,V(S_{t+1}) - V(S_t)$
      3. $e(S_t) \leftarrow e(S_t) + 1$
      4. 对所有迹值非零的状态 $s$：
          $V(s) \leftarrow V(s) + \alpha\,\delta_t\,e(s)$
          $e(s) \leftarrow \gamma\lambda\,e(s)$
每一步的 TD 误差 $\delta_t$ 不仅更新当前状态，还按资格迹大小回溯更新之前访问过的状态——刚访问过的状态更新多，很久以前的状态更新少。

可以证明，在离线设置下（回合结束后统一更新），资格迹的后向视角与前向视角的累计更新量完全一致。两种视角殊途同归：前向视角问的是"这个状态的未来回报是多少"，后向视角问的是"当前的误差应该归因给过去的哪些状态"。严格证明见 Sutton & Barto (2018) 第12章。

为什么现代深度RL不直接用资格迹

在表格方法中，资格迹只需为每个状态维护一个标量。但在深度强化学习中，值函数由神经网络 $V_\theta$ 参数化，资格迹需要为网络的每个参数维护一个迹值——对于有数百万参数的网络，这带来巨大的内存和计算开销。

实践中更高效的做法是：先收集一批轨迹，在轨迹上用前向视角（式(3.1-14)的递推形式）批量计算 $\lambda$-回报或 GAE，再进行梯度更新。这也是 PPO 等主流算法的标准实现方式。

TD($\lambda$)$G_t^\lambda$（$\lambda$-回报）$0 < \lambda < 1$有偏（偏差随 $\lambda$ 增大而减小）中等
MC$G_t$（完整回报）$\lambda = 1$无偏高

本节的核心结果是 $\lambda$-回报的 TD 误差表示（式(3.1-14)）：$G_t^\lambda - V(S_t) = \sum_{l=0}^{\infty}(\gamma\lambda)^l\delta_{t+l}$。这个恒等式不仅给出了 $\lambda$-回报的高效计算方法，也是5.2节 PPO 中 GAE 的理论基础。

至此，3.1节完成了预测问题的讨论：给定策略 $\pi$，如何从经验中估计其值函数 $V^\pi$。但我们最终的目标不只是评估一个策略的好坏，而是找到最优策略。如果我们只有 $V(s)$，要比较不同动作的好坏，就必须计算 $\sum_{s'} P(s'|s,a)[R(s,a) + \gamma V(s')]$——这需要知道转移概率 $P$，又回到了依赖模型的老路。要在无模型的设定下选择动作，需要直接学习动作值函数 $Q(s,a)$：有了 $Q$ 值，智能体只需选 $\arg\max_a Q(s,a)$，无需知道任何转移概率。这正是下一节要解决的控制问题。
