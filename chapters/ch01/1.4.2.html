1.4.2 值迭代

策略迭代的每一轮都需要完整地评估当前策略，即求解一个 $|\mathcal{S}| \times |\mathcal{S}|$ 的线性方程组（复杂度 $O(|\mathcal{S}|^3)$）。当状态空间较大时，这个代价很高。一个自然的问题是：能否在策略评估还不精确的时候就提前进行改进？

极端情况下，如果我们只做一步评估就立刻改进，会怎样？

将评估压缩为一步

回顾策略评估的迭代公式（贝尔曼期望方程的迭代形式）：

$$v_{k+1}(s) = \sum_{a} \pi(a \mid s) \left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s,a) \, v_k(s') \right]$$

如果做完这一步备份（backup）就立即进行贪婪改进，等价于把上式中对动作的加权平均替换为取最大值：

$$v_{k+1}(s) = \max_{a \in \mathcal{A}(s)} \left[\, R(s, a) + \gamma \sum_{s' \in \mathcal{S}} P(s' \mid s, a) \, v_k(s') \,\right] \tag{1.4-8}$$

这正是贝尔曼最优方程(1.4-4)被当作迭代更新规则来使用。反复应用公式(1.4-8)，值函数将收敛到最优值函数 $V^*$，最终从 $V^*$ 提取出最优策略。这就是值迭代（Value Iteration）算法。

算法：值迭代（Value Iteration）

输入：MDP $\langle \mathcal{S}, \mathcal{A}, P, R, \gamma \rangle$，收敛阈值 $\theta$

  1. 初始化：$v_0(s) = 0$，对所有 $s \in \mathcal{S}$
  2. 重复以下步骤，直到 $\max_s |v_{k+1}(s) - v_k(s)| < \theta$：
    1. 对每个状态 $s$：$v_{k+1}(s) = \max_{a} \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, v_k(s') \right]$
  3. 输出最优策略：$\pi^*(s) = \arg\max_a \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, v_k(s') \right]$

注意：迭代过程中没有显式的策略，只有值函数在不断更新。策略仅在最后一步从收敛的值函数中提取。

为什么值迭代一定收敛？因为公式(1.4-8)定义的更新算子（贝尔曼最优算子）是一个 $\gamma$-压缩映射：无论从哪个初始值函数出发，每次更新都会让 $v_k$ 与真正的 $V^*$ 之间的距离至少缩小 $\gamma$ 倍。由压缩映射定理（Banach不动点定理），反复应用这个算子必然收敛到唯一的不动点 $V^*$。

基于 $Q$ 值的值迭代

上面的值迭代直接更新 $V^*$，最后再从 $V^*$ 提取策略。但提取策略时仍然需要模型：$\pi^*(s) = \arg\max_a [R(s,a) + \gamma \sum_{s'} P(s'|s,a)\,V^*(s')]$。如果我们直接迭代 $Q^*$，就可以跳过这一步，因为最优策略可以直接从 $Q^*$ 读出：$\pi^*(s) = \arg\max_a Q^*(s,a)$。代价是需要维护的变量从 $|\mathcal{S}|$ 个 $V$ 值增加到 $|\mathcal{S}| \times |\mathcal{A}|$ 个 $Q$ 值，存储和每轮计算量都相应增大。在本例中从 3 个值变为 6 个，差距不大；但当状态空间和动作空间都很大时，这个倍数会很显著。

类似于 $V^*$ 满足贝尔曼最优方程(1.4-4)，$Q^*$ 也满足对应的方程。将(1.4-4)中的 $V^*(s') = \max_{a'} Q^*(s',a')$ 代入公式(1.3-9)的 $Q$-$V$ 关系，得到：

$$\boxed{Q^*(s,a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\, \max_{a'} Q^*(s',a')} \tag{1.4-9}$$

对比 $Q$ 的贝尔曼期望方程(1.3-10)：期望方程对下一步动作按策略 $\pi$ 做加权平均 $\sum_{a'} \pi(a'|s') Q(s',a')$，最优方程则取最大值 $\max_{a'} Q^*(s',a')$。将(1.4-9)当作迭代规则使用，就得到基于 $Q$ 值的值迭代：

$$q_{k+1}(s,a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\, \max_{a'} q_k(s',a') \tag{1.4-10}$$

注意表中加粗的 $Q^*$ 值恰好与前面 $V^*$ 值迭代的收敛结果相同：$56.3$，$47.7$，$60.0$。这并非巧合，而是贝尔曼最优方程(1.4-4)的直接体现：$V^*(s) = \max_a Q^*(s,a)$。每个状态的 $V^*$ 就是该状态下所有动作的 $Q^*$ 值取最大。$Q^*$ 表比 $V^*$ 信息更丰富：它不仅告诉我们每个状态的最优价值，还告诉我们每个动作各自的价值，以及最优动作比次优动作好多少。

有了 $Q^*$，最优策略可以直接读出：每个状态选 $Q^*$ 值最大的动作，无需再访问模型。这一性质使得 $Q$ 值成为无模型控制算法的核心对象。1.6节的 SARSA 和 Q-learning 正是将公式(1.4-10)的思想从"已知模型的精确计算"推广到"从经验中采样学习"。

策略迭代与值迭代的比较

策略迭代值迭代
核心操作完整评估 + 贪婪改进一步备份取 max
每轮代价$O(|\mathcal{S}|^3)$（矩阵求逆）$O(|\mathcal{S}|^2 \cdot |\mathcal{A}|)$（遍历状态和动作）
外层迭代次数通常很少（本例仅2轮）通常较多（本例约50轮）
收敛结果相同的 $V^*$ 和 $\pi^*$
适用场景状态空间适中状态空间较大但动作较少

两种方法在本质上是同一个思想的两个极端：策略迭代在每次改进前做完整的策略评估，值迭代在每次改进前只做一步评估。介于两者之间的方案也是可行的，例如做若干步（而非完整的）评估后就改进，这被称为广义策略迭代（Generalized Policy Iteration）。

模型依赖的局限

策略迭代和值迭代有一个共同的前提：完全知道MDP的转移概率 $P(s'|s,a)$ 和奖励函数 $R(s,a)$。在网约车例子中，我们假设知道了乘客目的地的精确概率分布。但在许多实际管理问题中，这些信息是未知的或难以精确估计的：

  • 公交系统中，每个站点的乘客到达率随时间和天气变化
  • 库存管理中，需求的概率分布可能未知或不稳定
  • 网约车平台中，供需模式随时段和城市区域动态变化

更关键的是，当状态空间非常大时（如连续状态或组合爆炸），即使知道模型，遍历所有状态的动态规划也不可行。下一节将介绍不依赖模型的方法，它们通过与环境的交互来学习值函数，而无需事先知道 $P$ 和 $R$。

小结

1.4节完成了从策略评估到最优策略求解的闭环：

  1. 利用 $V^\pi$ 计算动作值函数 $Q^\pi(s,a)$（一步前瞻），通过贪婪选择获得更好的策略
  2. 策略迭代交替进行评估与改进，在有限步内收敛到最优策略 $\pi^*$
  3. 值迭代将评估压缩为一步，直接对值函数做贝尔曼最优更新，同样收敛到 $V^*$
  4. 值迭代也可以直接对 $Q$ 值进行（公式(1.4-10)），得到的 $Q^*$ 可以直接读出最优策略，无需访问模型
  5. 以上算法都需要已知模型（$P$ 和 $R$），当模型未知或状态空间过大时，需要无模型的学习方法
选读：为什么动态规划一定收敛

正文中声称策略评估和值迭代一定收敛。本节给出严格的数学证明。核心思路是证明贝尔曼算子是压缩映射，从而由 Banach 不动点定理保证迭代收敛到唯一不动点。

压缩映射

给定一个度量空间 $(X, d)$，算子 $\mathcal{T}: X \to X$ 称为 $\gamma$-压缩映射，如果存在常数 $\gamma \in [0,1)$ 使得对所有 $x_1, x_2 \in X$：

$$d(\mathcal{T}x_1,\; \mathcal{T}x_2) \leq \gamma \, d(x_1, x_2) \tag{1.4-11}$$

直觉上，对任意两个点施加算子 $\mathcal{T}$ 后，它们之间的距离至少缩小了 $\gamma$ 倍。注意 $\gamma$ 必须严格小于 1，否则算子只是非扩张的（nonexpansive），不保证收敛。

在我们的问题中，$X$ 是所有值函数的集合（从 $\mathcal{S}$ 到 $\mathbb{R}$ 的映射），度量取无穷范数：

$$d(v_1, v_2) = \|v_1 - v_2\|_\infty = \max_{s \in \mathcal{S}} |v_1(s) - v_2(s)|$$

Banach 不动点定理

定理（Banach 不动点定理）

设 $(X, d)$ 是非空的完备度量空间，$\mathcal{T}: X \to X$ 是 $\gamma$-压缩映射。则 $\mathcal{T}$ 在 $X$ 中有且仅有一个不动点 $x^*$（即 $\mathcal{T}(x^*) = x^*$）。从任意初始点 $x_0$ 出发，序列 $x_{k+1} = \mathcal{T}(x_k)$ 收敛到 $x^*$。

存在性：迭代序列是 Cauchy 序列

首先证明相邻两步之间的距离指数衰减。由压缩性质反复应用可得（数学归纳法）：

$$d(x_{k+1}, x_k) = d(\mathcal{T}x_k, \mathcal{T}x_{k-1}) \leq \gamma \, d(x_k, x_{k-1}) \leq \cdots \leq \gamma^k \, d(x_1, x_0)$$

对任意 $m > n$，利用三角不等式：

$$d(x_m, x_n) \leq \sum_{i=n}^{m-1} d(x_{i+1}, x_i) \leq \sum_{i=n}^{m-1} \gamma^i \, d(x_1, x_0) \leq \frac{\gamma^n}{1 - \gamma} \, d(x_1, x_0) \tag{1.4-12}$$

当 $n \to \infty$ 时，右端趋于 0。因此 $\{x_k\}$ 是 Cauchy 序列，在完备度量空间中必收敛到某个极限 $x^*$。

还需验证 $x^*$ 确实是不动点。由于压缩映射是连续的（Lipschitz 连续），可以对 $x_{k+1} = \mathcal{T}(x_k)$ 两端取极限：

$$x^* = \lim_{k \to \infty} x_{k+1} = \lim_{k \to \infty} \mathcal{T}(x_k) = \mathcal{T}\!\left(\lim_{k \to \infty} x_k\right) = \mathcal{T}(x^*)$$
唯一性：反证法

假设存在两个不动点 $x_1^*$ 和 $x_2^*$，则：

$$d(x_1^*, x_2^*) = d(\mathcal{T}x_1^*, \mathcal{T}x_2^*) \leq \gamma \, d(x_1^*, x_2^*)$$

由于 $\gamma < 1$，上式仅在 $d(x_1^*, x_2^*) = 0$ 时成立，即 $x_1^* = x_2^*$。

贝尔曼最优算子是压缩映射

定义贝尔曼最优算子 $\mathcal{T}$ 为：$(\mathcal{T}v)(s) = \max_{a} [R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, v(s')]$。下面证明它在 $\|\cdot\|_\infty$ 下是 $\gamma$-压缩映射。

证明 $\|\mathcal{T}v_1 - \mathcal{T}v_2\|_\infty \leq \gamma \|v_1 - v_2\|_\infty$
$$\|\mathcal{T}v_1 - \mathcal{T}v_2\|_\infty = \max_{s} \left|\mathcal{T}v_1(s) - \mathcal{T}v_2(s)\right| \tag{1.4-13}$$

展开算子定义，需要处理两个 max 的差。关键不等式是：对任意函数 $f,g$，$|\max_a f(a) - \max_a g(a)| \leq \max_a |f(a) - g(a)|$。

（证明这个不等式：设 $a^* = \arg\max_a f(a)$，则 $\max_a f(a) - \max_a g(a) \leq f(a^*) - g(a^*) \leq |f(a^*) - g(a^*)| \leq \max_a |f(a) - g(a)|$。对称地，$\max_a g(a) - \max_a f(a)$ 也满足同样的上界。两者取绝对值即得。）

应用这个不等式，$R(s,a)$ 项消去：

$$\leq \max_s \max_a \left| \gamma \sum_{s'} P(s'|s,a) \big(v_1(s') - v_2(s')\big) \right|$$

由三角不等式 $|\sum_i c_i x_i| \leq \sum_i c_i |x_i|$（其中 $c_i \geq 0$），以及概率求和 $\sum_{s'} P(s'|s,a) = 1$，凸组合不超过最大值：

$$\leq \gamma \max_s \max_a \sum_{s'} P(s'|s,a) |v_1(s') - v_2(s')| \leq \gamma \max_{s'} |v_1(s') - v_2(s')| = \gamma \|v_1 - v_2\|_\infty$$

因此贝尔曼最优算子 $\mathcal{T}$ 是 $\gamma$-压缩映射。由 Banach 不动点定理，值迭代 $v_{k+1} = \mathcal{T}v_k$ 从任意初始值函数出发，必收敛到唯一不动点 $V^*$（即贝尔曼最优方程的解）。

贝尔曼期望算子也是压缩映射

定义贝尔曼期望算子 $\mathcal{T}_\pi$ 为：$(\mathcal{T}_\pi v)(s) = \sum_a \pi(a|s) [R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, v(s')]$。证明方法类似，且更简单（因为不涉及 max 运算）。

证明 $\|\mathcal{T}_\pi v_1 - \mathcal{T}_\pi v_2\|_\infty \leq \gamma \|v_1 - v_2\|_\infty$

展开算子并消去 $R(s,a)$ 项：

$$\|\mathcal{T}_\pi v_1 - \mathcal{T}_\pi v_2\|_\infty = \max_s \left| \sum_a \pi(a|s) \, \gamma \sum_{s'} P(s'|s,a) \big(v_1(s') - v_2(s')\big) \right| \tag{1.4-14}$$

对策略概率 $\pi(a|s)$ 和转移概率 $P(s'|s,a)$ 逐层放缩，凸组合不超过最大值：

$$\leq \gamma \max_s \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \, |v_1(s') - v_2(s')| \leq \gamma \max_{s'} |v_1(s') - v_2(s')| = \gamma \|v_1 - v_2\|_\infty$$

最后一步用到 $\sum_a \pi(a|s) \sum_{s'} P(s'|s,a) = 1$（概率归一化）。

因此策略评估的迭代 $v_{k+1} = \mathcal{T}_\pi v_k$ 也保证收敛到唯一不动点 $V^\pi$。这解释了为什么1.3.3节中直接求解线性方程组得到的 $V^\pi$ 与迭代计算的结果一致。
