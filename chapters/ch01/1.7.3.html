1.7.3 Actor-Critic框架

从广义策略迭代的视角

回顾1.4节的策略迭代算法：它交替进行两个步骤，策略评估（给定策略，计算值函数）和策略改进（根据值函数，更新策略）。事实上，本章介绍的所有方法都可以看作这一广义策略迭代（Generalized Policy Iteration, GPI）框架的不同实例。区别仅在于评估和改进各自使用什么手段：

算法评估（Critic）改进（Actor）
策略迭代（1.4）精确求解 $V^\pi$贪心 $\arg\max$
SARSA（1.6.1）TD估计 $Q$$\varepsilon$-greedy
Q-learning（1.6.2）TD估计 $Q^*$$\varepsilon$-greedy（行为策略）
REINFORCE（1.7.2）MC回报 $G_t$（隐式）梯度上升
Actor-CriticTD估计 $V^\pi$梯度上升

从表中可以看出，Actor-Critic结合了两方面的优势：它像TD方法一样用自举（bootstrapping）来评估值函数，避免了蒙特卡洛方法等到回合结束才能更新的缺点（1.5.2节已分析了这一点）；同时它像REINFORCE一样直接通过梯度上升优化参数化策略，保留了策略梯度方法的灵活性。

基线与优势函数

1.7.2节的REINFORCE算法使用蒙特卡洛回报 $G_t$ 作为策略梯度的估计信号。正如1.5.2节在预测问题中所分析的，MC回报的方差很高：即使从同一个状态出发，不同轨迹的 $G_t$ 可能相差甚远。方差大意味着梯度估计噪声大，策略参数的更新方向不稳定，收敛速度慢。

一个有效的降方差技巧是引入基线（baseline）。策略梯度定理告诉我们，从 $G_t$ 中减去任何仅依赖于状态 $S_t$ 的函数 $b(S_t)$，都不会改变梯度的期望值：

$$\nabla_\theta J(\theta) = \mathbb{E}_\pi\left[\sum_{t=0}^{T-1} \big(G_t - b(S_t)\big) \,\nabla_\theta \ln \pi_\theta(A_t \mid S_t)\right] \tag{1.7-5}$$
为什么减去基线不改变梯度期望？（选读）

对于任意仅依赖 $S_t$ 的基线 $b(S_t)$，考虑被减去的项：

$$\mathbb{E}_\pi\!\left[b(S_t)\,\nabla_\theta \ln \pi_\theta(A_t \mid S_t)\right] = \sum_s d^\pi(s)\, b(s) \sum_a \nabla_\theta \pi_\theta(a\mid s)$$

由于 $\sum_a \pi_\theta(a \mid s) = 1$ 对所有 $\theta$ 成立，对 $\theta$ 求导得到 $\sum_a \nabla_\theta \pi_\theta(a \mid s) = 0$。因此，减去的这一项期望为零，不影响梯度的期望方向。

虽然减去 $b(S_t)$ 不改变期望，但它会改变方差。最优的基线（使方差最小的那个）接近于状态值函数 $V^\pi(S_t)$。直觉上，$V^\pi(S_t)$ 代表从状态 $S_t$ 出发的平均回报；用 $G_t - V^\pi(S_t)$ 作为梯度信号，衡量的是这次具体经历比平均水平好了多少（或差了多少），而非绝对回报的大小。

优势函数（Advantage Function）

定义动作 $a$ 在状态 $s$ 下的优势为：

$$A^\pi(s, a) = Q^\pi(s, a) - V^\pi(s)$$

$A^\pi(s,a) > 0$ 表示动作 $a$ 优于当前策略的平均表现，$A^\pi(s,a) < 0$ 则相反。

在实践中，我们并不需要同时维护 $Q^\pi$ 和 $V^\pi$ 来计算优势函数。注意到 $Q^\pi(S_t, A_t)$ 的一个无偏样本就是 $R_t + \gamma V^\pi(S_{t+1})$（这正是1.5.2节TD目标的含义），因此：

$$A^\pi(S_t, A_t) \approx R_t + \gamma\, V^\pi(S_{t+1}) - V^\pi(S_t) = \delta_t \tag{1.7-6}$$

其中 $\delta_t$ 正是1.5.2节介绍的TD误差。这个联系非常自然：TD误差衡量的是实际获得的即时奖励加上后继状态的估值与当前状态估值之差，它恰好是优势函数的一步采样估计。当 $\delta_t > 0$，说明这一步的表现好于预期，应当增加选择该动作的概率；反之则降低。

Actor-Critic算法

有了上面的分析，Actor-Critic的更新规则就非常清晰了。算法同时维护两组参数：

Critic（评论家）

维护状态值函数 $V(s)$，用TD(0)更新：

$$V(S_t) \leftarrow V(S_t) + \alpha_w \cdot \delta_t$$

与1.5.2节的TD(0)完全相同。

Actor（演员）

维护策略参数 $\theta$，用策略梯度更新：

$$\theta \leftarrow \theta + \alpha_\theta \cdot \delta_t \cdot \nabla_\theta \ln \pi_\theta(A_t \mid S_t)$$

用 $\delta_t$ 替代REINFORCE中的 $G_t$。

其中TD误差为：

$$\delta_t = R_t + \gamma\, V(S_{t+1}) - V(S_t) \tag{1.7-7}$$
Actor-Critic 算法
  1. 初始化 $V(s) = 0$，对所有 $s \in \mathcal{S}$；初始化策略参数 $\theta$（如全0）
  2. 对每个回合（episode）：
    1. 初始化状态 $S$
    2. 对回合中的每一步：
      1. 根据 $\pi_\theta(\cdot \mid S)$ 采样动作 $A$
      2. 执行动作 $A$，观察奖励 $R$ 和下一状态 $S'$
      3. 计算TD误差：$\delta \leftarrow R + \gamma\, V(S') - V(S)$
      4. Critic更新：$V(S) \leftarrow V(S) + \alpha_w \cdot \delta$
      5. Actor更新：$\theta \leftarrow \theta + \alpha_\theta \cdot \delta \cdot \nabla_\theta \ln \pi_\theta(A \mid S)$
      6. $S \leftarrow S'$
与REINFORCE不同，Actor-Critic每步都更新（像TD一样），无需等到回合结束。Critic的更新与1.5.2节的TD(0)完全相同；Actor使用TD误差 $\delta_t$ 替代MC回报 $G_t$，方差显著降低。

例子：网约车司机的Actor-Critic

继续使用网约车MDP（$\gamma = 0.9$），用Actor-Critic从零开始学习。策略采用softmax参数化（与1.7.2节相同）：对每个状态 $s$ 和动作 $a$，维护参数 $\theta_{s,a}$，策略概率为

$$\pi_\theta(a \mid s) = \frac{\exp(\theta_{s,a})}{\sum_{a'} \exp(\theta_{s,a'})}$$

对于softmax参数化，得分函数的梯度为

$$\nabla_{\theta_{s,a}} \ln \pi_\theta(a \mid s) = \mathbf{1}[a' = a] - \pi_\theta(a' \mid s)$$

即对被选中的动作 $a$，梯度分量为 $1 - \pi_\theta(a \mid s)$；对其他动作 $a' \neq a$，梯度分量为 $-\pi_\theta(a' \mid s)$。下面的代码实现了完整的Actor-Critic训练过程：

import numpy as np
gamma = 0.9
alpha_w, alpha_theta = 0.01, 0.01

T = {
    'c': {'接单': {'c':(.5,5),'b':(.3,8),'a':(.2,12)}, '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5),'b':(.8,1),'a':(.1,8)}, '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14),'b':(.4,10)}, '空驶去市中心': {'c':(1,-5)}}
}
states = ['c', 'b', 'a']
actions = {s: list(T[s].keys()) for s in states}

def step(s, action):
    trans = T[s][action]
    r, cum = np.random.random(), 0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum: return s_next, reward
    return s_next, reward

# Critic: V table
V = {s: 0.0 for s in states}
# Actor: softmax parameters
theta = {s: {a: 0.0 for a in actions[s]} for s in states}

def policy_probs(s):
    acts = actions[s]
    logits = [theta[s][a] for a in acts]
    max_l = max(logits)
    exps = [np.exp(l - max_l) for l in logits]
    total = sum(exps)
    return {a: e/total for a, e in zip(acts, exps)}

def sample_action(s):
    probs = policy_probs(s)
    acts, ps = zip(*probs.items())
    return acts[np.random.choice(len(acts), p=ps)]

for ep in range(5000):
    s = np.random.choice(states)
    for _ in range(200):
        a = sample_action(s)
        s_next, r = step(s, a)
        # TD error
        delta = r + gamma * V[s_next] - V[s]
        # Critic update
        V[s] += alpha_w * delta
        # Actor update
        probs = policy_probs(s)
        for act in actions[s]:
            if act == a:
                theta[s][act] += alpha_theta * delta * (1 - probs[act])
            else:
                theta[s][act] -= alpha_theta * delta * probs[act]
        s = s_next

for s in states:
    probs = policy_probs(s)
    best = max(probs, key=probs.get)
    print(f"  {s}: {best} (π={probs[best]:.2f}), V={V[s]:.1f}")

运行结果：

Actor-Critic得到的最优策略：市中心→接单，郊区→空驶去市中心，机场→接单
与1.4节动态规划和1.6.2节Q-learning得到的结果一致。

小结与展望

至此，第一章介绍的所有算法可以统一到下面的表格中：

算法需要模型？预测/控制值函数/策略更新频率
DP（1.4）是控制值函数—
MC（1.5.1）否预测值函数每条轨迹
TD（1.5.2）否预测值函数每步
SARSA（1.6.1）否控制值函数（$Q$）每步
Q-learning（1.6.2）否控制值函数（$Q$）每步
REINFORCE（1.7.2）否控制策略每条轨迹
Actor-Critic（1.7.3）否控制策略+值函数每步

至此，我们用一个只有3个状态的网约车例子，完整地展示了强化学习的核心算法族谱。这些方法共同的局限是：它们都基于表格表示，状态和动作必须是有限且可枚举的。当状态空间很大或连续时（如有成百上千个区域的网约车调度、连续变化的库存水平），表格方法不再可行。解决方案是用函数近似（如神经网络）来表示值函数或策略。第二章将介绍深度学习基础，第三章将把本章的表格算法升级为深度强化学习：DQN（Q-learning + 神经网络）、PPO（策略梯度 + 神经网络）、SAC（Actor-Critic + 神经网络）。
