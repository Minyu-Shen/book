1.3 马尔可夫决策过程

马尔可夫决策过程（Markov Decision Process, MDP）是强化学习的数学基础。但在直接给出它的数学定义之前，让我们先从一个直观的问题出发——一个你在运筹学课上可能见过的经典最短路问题。通过这个例子，我们将自然地引出MDP中的每一个核心概念。

从一个最短路问题出发

假设一位旅行者要从城市 A 出发，最终到达城市 G。途中必须依次经过若干中间城市，每一步只能前进到下一层的城市，不能后退。城市之间的道路长度各不相同，旅行者的目标是找到一条总路程最短的路线。

我们将所有城市按照地理位置分成若干层，旅行者在每一层都需要选择下一步去往哪个城市。这就构成了一个多阶段决策问题：每一层是一个决策阶段，每个城市是一种状态，选择下一个城市就是做出一个决策。

多阶段决策过程的核心概念

通过上面的最短路问题，我们已经直观地感受了"分阶段做决策"的过程。现在，让我们用这个例子来正式定义多阶段决策过程中的每一个核心概念。这些概念将直接对应到马尔可夫决策过程的数学框架中。

阶段（Stage）
对应MDP中的时间步 Time Step

将问题的过程分解为若干个相互联系的阶段，以便按一定的次序去求解。在最短路问题中，旅行者从 A 到 G 需要经过6个阶段，用变量 $k = 1, 2, \ldots, 6$ 来标记。

在管理科学问题中，阶段通常对应时间的推进：每个月的库存决策、每分钟的调度决策、每个路口的行驶决策等。

状态（State）
State, $s_k$

状态描述了每个阶段开始时，系统所处的客观条件。在最短路问题中，状态就是旅行者当前所在的城市。例如，第3阶段的状态集合为 $S_3 = \{C_1, C_2, C_3, C_4\}$。

状态必须具备一个关键性质——无后效性：当前状态一旦确定，未来的演变就不再受更早历史的影响。在最短路中，旅行者到达 $C_2$ 后，从 $C_2$ 到 G 的最短路程只取决于"当前在 $C_2$"，与旅行者是经由 $B_1$ 还是 $B_2$ 到达 $C_2$ 的无关。

决策 / 动作（Action）
Action, $u_k(s_k)$ 或 $a$

在某个阶段处于某个状态时，决策者可以做出不同的选择，从而影响下一阶段的状态。在最短路中，决策就是"选择下一个城市"。例如，在第2阶段处于状态 $B_1$ 时，允许的决策集合为 $U_2(B_1) = \{C_1, C_2, C_3\}$。

注意：可选的决策通常依赖于当前状态。从 $B_1$ 可以到 $C_1, C_2, C_3$，但从 $B_2$ 只能到 $C_2, C_3, C_4$。这在管理科学问题中非常普遍——可用的资源、当前的约束条件都会限制你能做的选择。

状态转移（State Transition）
Transition, $s_{k+1} = T_k(s_k, u_k)$

状态转移描述了"在当前状态 $s_k$ 下做出决策 $u_k$ 后，系统转移到下一个状态 $s_{k+1}$"的过程。在最短路中，状态转移是确定性的：从 $B_1$ 选择去 $C_2$，就一定到达 $C_2$，即 $s_3 = T_2(B_1, C_2) = C_2$。

但在很多现实问题中，状态转移是随机的——我们稍后会讨论这一点，这正是MDP区别于确定性动态规划的关键所在。

阶段收益 / 即时奖励（Immediate Reward）
Reward, $r_k(s_k, u_k)$

每做一次决策，都会产生一个即时的效果。在最短路中，这个效果就是两个城市之间的路程（代价）。例如从 $A$ 到 $B_1$ 的代价为 $r_1(A, B_1) = 5$。

整个过程的总效果就是各阶段收益之和：$V = \sum_{k=1}^{n} r_k(s_k, u_k)$。在最短路中我们要最小化总路程，在其他问题中可能要最大化总利润。

策略（Policy）
Policy, $\pi$

策略是一套完整的决策方案：它规定了在每个阶段、每个可能的状态下应该做什么决策。在最短路中，一个策略就是一个完整的路线规划，例如 $\{A \to B_1, B_1 \to C_2, C_2 \to D_1, D_1 \to E_2, E_2 \to F_2, F_2 \to G\}$。

我们的目标是从所有可能的策略中，找到使总效果最优的那个——最优策略。

值函数（Value Function）
Value Function, $f_k(s_k)$

值函数 $f_k(s_k)$ 表示从第 $k$ 阶段的状态 $s_k$ 出发，采用最优策略所能获得的最优总效果。在最短路中，$f_k(s_k)$ 就是"从城市 $s_k$ 到终点 G 的最短距离"。

在上面的交互演示中，你已经看到了值函数是如何通过逆向归纳逐步计算出来的。这正是动态规划的核心思想，也是后续所有强化学习算法的出发点。

当道路变得不确定：从确定性到随机性

在上面的最短路问题中，旅行者选择从 $B_1$ 去 $C_2$，就一定能到达 $C_2$——状态转移是完全确定的。但在现实的管理科学问题中，决策的结果往往带有不确定性：

  • 物流配送：安排卡车走某条路线，但可能因交通拥堵而延误，实际到达时间是随机的。
  • 库存管理：决定补货1000件，但下个月的需求是随机的，可能卖出800件也可能卖出1200件。
  • 网约车调度：派车去某个区域接单，但乘客是否会取消、途中是否有新订单，都是不确定的。

让我们修改最短路的例子来体现这种随机性。假设由于天气和路况的影响，旅行者选择前往某个城市时，有一定概率会被"分流"到相邻的城市：

对比：确定性转移 vs. 随机性转移

确定性转移

从 $B_1$ 选择去 $C_2$：

选择去 $C_2$，就一定到达 $C_2$。

随机性转移

从 $B_1$ 选择去 $C_2$：

选择去 $C_2$，70%概率到达 $C_2$，20%概率被分流到 $C_1$，10%概率被分流到 $C_3$。

一旦状态转移带有随机性，我们就不能简单地计算"走这条路的距离是多少"，而是需要考虑期望代价——在所有可能的随机结果上取平均。这就是从确定性动态规划走向马尔可夫决策过程的关键一步。

这个"随机性"的引入带来了一个根本性的变化：状态转移不再是一个确定的函数 $s_{k+1} = T_k(s_k, u_k)$，而是由一个概率分布描述：

$$P(s_{k+1} \mid s_k, u_k) = \text{在状态 } s_k \text{ 下采取决策 } u_k \text{ 后，转移到状态 } s_{k+1} \text{ 的概率}$$

当我们把确定性的状态转移替换为概率性的状态转移，同时保留"无后效性"（即马尔可夫性质），我们就得到了马尔可夫决策过程。

马尔可夫决策过程的正式定义

一个马尔可夫决策过程由以下五个要素构成，通常记为 $\langle \mathcal{S}, \mathcal{A}, P, R, \gamma \rangle$：

符号 名称 含义 最短路中的对应
$\mathcal{S}$ 状态空间 所有可能状态的集合 所有城市 $\{A, B_1, B_2, \ldots, G\}$
$\mathcal{A}$ 动作空间 所有可能动作的集合；$\mathcal{A}(s)$ 为状态 $s$ 下可选的动作 可前往的下一层城市
$P$ 状态转移概率 $P(s' \mid s, a)$：在状态 $s$ 执行动作 $a$ 后转移到 $s'$ 的概率 确定性情形下概率为1；随机情形下为各种可能的概率分布
$R$ 奖励函数 $R(s, a)$ 或 $R(s, a, s')$：执行动作获得的即时奖励 两城市间的路程（取负即为奖励）
$\gamma$ 折扣因子 $\gamma \in [0, 1]$，衡量未来奖励相对于当前奖励的重要程度 最短路中 $\gamma = 1$（每段路程同等重要）

在引入折扣因子后，智能体的目标是找到一个策略 $\pi$，使得累计折扣奖励的期望最大化：

$$\max_\pi \; \mathbb{E}_\pi \left[ \sum_{t=0}^{\infty} \gamma^t \, R(s_t, a_t) \right]$$

其中 $\gamma$ 的作用是让智能体更重视近期的奖励。当 $\gamma = 1$ 时，未来的奖励与当前一样重要，这就退化为我们最短路中的情形。当 $\gamma < 1$ 时，越远的未来越不重要——这在很多管理决策中是合理的，因为未来的收益存在不确定性和时间价值。

折扣因子的管理学直觉：一元钱在今天的价值高于一年后的一元钱，这就是金融中的"时间价值"。折扣因子 $\gamma$ 扮演的正是类似角色：$\gamma = 0.99$ 意味着一年后的一元钱在今天只值0.99元。在无限时间步的问题中，$\gamma < 1$ 还保证了累计奖励的有界性。

马尔可夫性质：未来只取决于现在

马尔可夫决策过程之所以叫"马尔可夫"，是因为它要求状态转移满足马尔可夫性质（Markov Property），也就是动态规划中所说的"无后效性"：

$$P(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \ldots, s_0, a_0) = P(s_{t+1} \mid s_t, a_t)$$

用通俗的话说：给定当前状态和动作，未来的状态只与"现在"有关，与"过去"无关。

回到最短路问题，如果旅行者当前在城市 $D_2$，那么从 $D_2$ 到 $G$ 的最短路程只取决于"当前在 $D_2$"这个事实，而不取决于旅行者是从 $C_1 \to D_2$、$C_2 \to D_2$、$C_3 \to D_2$ 还是 $C_4 \to D_2$ 到达这里的。这正是我们能够使用动态规划的前提条件。

实践提示：在将管理科学问题建模为MDP时，检验马尔可夫性质是否成立非常重要。如果当前状态无法完整描述未来的演变（例如，决策还取决于过去三天的需求历史），那么需要将这些历史信息也纳入状态定义中，以满足马尔可夫性质。我们将在第四章"管理科学问题的建模范式"中详细讨论这一点。

从动态规划到MDP：概念对照

如果你已经学过运筹学中的动态规划，下面这张对照表可以帮助你快速建立两套术语之间的映射：

动态规划术语 MDP / 强化学习术语 核心区别
阶段 $k$ 时间步 $t$ 仅符号不同
状态变量 $s_k$ 状态 $s_t$ 含义一致
决策变量 $u_k$ 动作 $a_t$ 含义一致
状态转移方程 $T_k(s_k, u_k)$ 转移概率 $P(s' \mid s, a)$ DP通常确定性；MDP允许随机
阶段收益 $r_k(s_k, u_k)$ 即时奖励 $R(s, a)$ 含义一致，MDP中常加折扣
策略 $p_{1,n}$ 策略 $\pi$ DP的策略是序列；RL的策略是映射 $s \to a$
最优值函数 $f_k(s_k)$ 最优值函数 $V^*(s)$ 含义一致
最优性原理 贝尔曼最优方程 本质相同，MDP版本包含期望

可以看到，MDP本质上就是一个带随机性的多阶段决策过程。如果你把转移概率退化为确定性（概率为1），把折扣因子设为1，MDP就变成了运筹学中经典的动态规划问题。

在下一节中，我们将介绍如何求解MDP——当我们完全知道转移概率和奖励函数时，可以使用值迭代和策略迭代这两种经典的动态规划算法来找到最优策略。
