1.5.1 蒙特卡洛方法

1.4节的策略迭代和值迭代能精确求解最优策略，但它们有一个强假设：完全知道MDP的转移概率 $P(s'|s,a)$ 和奖励函数 $R(s,a)$。在网约车例子中，这意味着必须事先知道每个区域接单后乘客去各个方向的精确概率。在实际管理问题中，这些概率往往是未知的，或者状态空间太大导致无法枚举所有转移。

不依赖模型的方法提供了一条不同的路径：智能体不需要知道环境的转移概率，而是通过与环境的反复交互来学习。蒙特卡洛（Monte Carlo, MC）方法是其中最直观的一种：让智能体按照策略 $\pi$ 运行若干条轨迹，观察每条轨迹的累计回报，然后用回报的均值来估计值函数。这就是"跑很多次，取平均"的思想。

蒙特卡洛策略评估

回顾值函数的定义：$V^\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$，其中 $G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$ 是从时刻 $t$ 开始的累计折扣回报。这是一个期望值，而蒙特卡洛方法的思路就是用样本均值来近似它：让智能体按照策略 $\pi$ 运行很多条轨迹，对于每次访问状态 $s$ 的时刻，计算从该时刻起的回报 $G_t$，然后取平均。

$$V^\pi(s) \approx \frac{1}{N(s)} \sum_{t:\, S_t = s} G_t \tag{1.5-1}$$

其中 $N(s)$ 是状态 $s$ 被访问的总次数。由大数定律，当 $N(s) \to \infty$ 时，这个估计收敛到真实值 $V^\pi(s)$。

一条轨迹中，同一个状态 $s$ 可能被多次访问。如何处理？这引出了两种变体。

首次访问MC与每次访问MC

首次访问MC（First-visit MC）：在每条轨迹中，只对状态 $s$ 的第一次出现计算回报 $G_t$ 并更新 $V(s)$，忽略后续的访问。

每次访问MC（Every-visit MC）：在每条轨迹中，对状态 $s$ 的每一次出现都计算回报 $G_t$ 并更新 $V(s)$。

两种方法都收敛到 $V^\pi(s)$。首次访问MC的分析更简单：每条轨迹贡献一个独立的 $G_t$ 样本，无偏性直接由大数定律保证。每次访问MC的各样本之间存在相关性（同一条轨迹内），但利用了更多数据。本节主要使用首次访问MC。

MC能处理无限期MDP吗？

回报 $G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$ 的定义是一个无穷级数，这正是因为无限期（infinite-horizon）MDP中任务永远不终止。那MC是否就无法使用了？

答案是：只要 $\gamma < 1$，MC方法完全可以处理无限期MDP。原因在于，远处的奖励被 $\gamma^k$ 压缩得越来越小。在实践中，我们将轨迹截断到 $H$ 步，截断误差不超过 $\gamma^H \cdot R_{\max} / (1-\gamma)$。例如 $\gamma=0.9$ 时，截断到 $H=100$ 步的误差仅为 $0.9^{100} \approx 2.7 \times 10^{-5}$ 乘以一个常数，几乎可以忽略。

管理科学中的应用：库存管理、网约车调度、公交运营等管理问题通常被建模为无限期MDP，这并非因为问题"本质上"是无限的，而是因为无限期模型避免了指定终止时刻的人为性，使决策规则不依赖于剩余时间。MC方法通过截断轨迹同样适用，只需保证截断长度 $H$ 足够大（一般取 $H \gg 1/(1-\gamma)$）。我们的网约车例子就采用了这种建模方式。

增量更新与学习率

公式(1.5-1)的样本均值需要保存所有历史回报。在实际实现中，我们使用增量更新的形式，每收到一个新的回报 $G_t$，只需一步即可更新估计值：

$$V(S_t) \leftarrow V(S_t) + \alpha\left[G_t - V(S_t)\right] \tag{1.5-2}$$

这里 $\alpha$ 称为学习率（step size）。方括号中的 $G_t - V(S_t)$ 是"预测误差"：实际回报与当前估计之间的偏差。每次更新就是朝着消除偏差的方向走一步。

  • 当 $\alpha = 1/N(s)$（$N(s)$ 为状态 $s$ 被访问的次数）时，增量更新与样本均值完全等价（推导见选读部分）
  • 当 $\alpha$ 取固定的小常数（如0.01）时，近期的回报获得更大权重，适合跟踪非平稳环境。在强化学习中，由于策略在不断改进，固定学习率反而更常用

MC的工作方式：沿一条轨迹采样到底

MC方法的核心特征可以用下面的备份图（backup diagram）来理解。与动态规划（DP）的全宽度备份不同，MC只沿着一条实际采样的轨迹一路向下，用这条轨迹的完整回报来更新值函数估计。

图 1.5-1   MC备份图。MC沿一条实际采样的轨迹（蓝色实线）从 $S_t$ 一直走到截断点，用这条轨迹的完整折扣回报 $G_t$ 来更新 $V(S_t)$。灰色虚线是可能但未被采样的转移方向。

这张图揭示了MC方法的关键特征：它只使用一条采样轨迹（不像DP那样对所有可能的下一状态求期望），并且沿着这条轨迹一直走到底才计算回报。这意味着更新一次 $V(S_t)$ 需要等待整条轨迹完成。是否有可能不走到底、每走一步就更新？这正是下一节时序差分学习将要解决的问题。

首次访问MC算法

算法：首次访问蒙特卡洛策略评估（First-visit MC Prediction）

输入：待评估的策略 $\pi$，折扣因子 $\gamma$

  1. 初始化：对所有状态 $s$，令 $V(s) = 0$，访问计数 $N(s) = 0$
  2. 重复以下步骤（每次生成一条轨迹）：
    1. 按策略 $\pi$ 生成一条完整轨迹：$S_0, A_0, R_1, S_1, A_1, R_2, \ldots, S_{T-1}, A_{T-1}, R_T$
    2. 从轨迹末端向前计算回报：$G \leftarrow 0$；对 $t = T\!-\!1, T\!-\!2, \ldots, 0$：$G \leftarrow \gamma\, G + R_{t+1}$
    3. 对轨迹中每个首次出现的状态 $S_t$ 更新：
      • $N(S_t) \leftarrow N(S_t) + 1$
      • $V(S_t) \leftarrow V(S_t) + \dfrac{1}{N(S_t)}\left[G_t - V(S_t)\right]$

注：步骤 2b 从后向前扫描，利用递推关系 $G_t = R_{t+1} + \gamma\, G_{t+1}$ 一次遍历即可计算所有时刻的回报，时间复杂度 $O(T)$。此算法直接适用于有终止条件的片段式任务。对于无限期MDP的处理方式，见下面的例子。

例子：用MC估计网约车司机的值函数

我们用首次访问MC来评估1.3节定义的策略 $\pi_1$（所有状态都选择接单）。代码的整体流程如下：

  1. 用MDP的转移概率构建仿真环境（第五章将详细讨论仿真器的搭建）
  2. 让司机按照 $\pi_1$ 运营 $H=200$ 步，记录完整轨迹
  3. 从后向前扫描，计算轨迹中每个时刻的回报 $G_t$
  4. 仅对前 $T_{\text{eval}}=100$ 步内首次访问的状态更新 $V$。后100步是冗余段，只提供奖励信息，不用于更新任何状态

为什么需要冗余段？因为无限期MDP中轨迹必须在某处截断，而截断点附近的回报估计会丢失大量信息。例如 $t=190$ 时只剩10步，$\gamma^{10} \approx 0.35$，即35%的真实回报被截断掉了。通过设置 $T_{\text{eval}} = 100$，$H = 200$，保证评估窗口内最晚的时刻（$t=99$）仍有101步后续奖励，截断误差 $\gamma^{101} \approx 0$。冗余段的长度只需满足 $H - T_{\text{eval}} \gg 1/(1-\gamma)$。

import numpy as np
np.random.seed(42)

gamma = 0.9
H_total = 200   # 轨迹总长度
H_eval  = 100   # 评估窗口：只对前100步中首次访问的状态更新V
                 # 后100步是冗余段（padding），保证回报的截断误差可忽略

# MDP定义：T[s][action] = {next_state: (prob, reward), ...}
T = {
    'c': {'接单': {'c':(.5,5), 'b':(.3,8), 'a':(.2,12)},
          '空驶去机场': {'a':(1,-4)}},
    'b': {'接单': {'c':(.1,5), 'b':(.8,1), 'a':(.1,8)},
          '空驶去市中心': {'c':(1,-3)}},
    'a': {'接单': {'c':(.6,14), 'b':(.4,10)},
          '空驶去市中心': {'c':(1,-5)}}
}

def step(s, action):
    """根据转移概率采样下一状态和奖励"""
    trans = T[s][action]
    r, cum = np.random.random(), 0.0
    for s_next, (p, reward) in trans.items():
        cum += p
        if r < cum:
            return s_next, reward
    return s_next, reward

def generate_episode(policy, start_state, H):
    """仿真H步，返回轨迹 [(S_t, R_{t+1}), ...]"""
    episode = []
    s = start_state
    for _ in range(H):
        a = policy[s]
        s_next, reward = step(s, a)
        episode.append((s, reward))
        s = s_next
    return episode

def compute_returns(episode, gamma):
    """从后向前计算每个时刻的回报 G_t"""
    returns = [0.0] * len(episode)
    G = 0.0
    for t in range(len(episode) - 1, -1, -1):
        G = episode[t][1] + gamma * G
        returns[t] = G
    return returns

# 策略π₁：所有状态都选择"接单"
policy_pi1 = {'c': '接单', 'b': '接单', 'a': '接单'}

# 首次访问MC
V = {'c': 0.0, 'b': 0.0, 'a': 0.0}
N = {'c': 0,   'b': 0,   'a': 0}

for ep in range(3000):
    start = np.random.choice(['c', 'b', 'a'])
    episode = generate_episode(policy_pi1, start, H=H_total)
    returns = compute_returns(episode, gamma)

    visited = set()
    for t in range(H_eval):           # 只遍历评估窗口
        s = episode[t][0]
        if s not in visited:          # 首次访问
            visited.add(s)
            N[s] += 1
            V[s] += (1.0 / N[s]) * (returns[t] - V[s])

print("首次访问MC估计（3000条轨迹）：")
for s in ['c', 'b', 'a']:
    true_v = {'c': 52.3, 'b': 42.3, 'a': 55.9}
    print(f"  V({s}) = {V[s]:.1f}  (理论值: {true_v[s]})")
# 期望输出: V(c)≈52.3, V(b)≈42.3, V(a)≈55.9

运行上述代码，MC估计值将接近理论值 $V^{\pi_1}(c)=52.3$，$V^{\pi_1}(b)=42.3$，$V^{\pi_1}(a)=55.9$。增加轨迹数量可以进一步提高精度。

MC方法的特点与局限

蒙特卡洛方法有几个显著的优点：

  • 无偏估计：$G_t$ 是 $V^\pi(S_t)$ 的无偏估计，不依赖于任何对值函数的近似
  • 不需要模型：只需要与环境交互产生的轨迹数据，不需要知道转移概率
  • 概念简单：跑轨迹、算回报、取平均，整个流程非常直观

但MC方法也有明显的局限：

  • 更新延迟高：回报 $G_t$ 的计算需要等到一条轨迹（或截断片段）结束，不能在每一步后立即更新。轨迹越长，等待越久。
  • 方差较高：$G_t$ 包含了从时刻 $t$ 到轨迹结束的所有随机转移和奖励，每一步的随机性都会累积。轨迹越长，$G_t$ 的方差越大，需要更多轨迹才能得到精确的估计。

这两个局限自然引出一个问题：能否不等到轨迹结束，而是每走一步就更新值函数？下一节将介绍的时序差分（TD）学习正是解决这个问题的方法。

选读：增量均值公式的推导与 Robbins-Monro 条件

增量均值公式

公式(1.5-2)中，当 $\alpha = 1/N(s)$ 时，增量更新等价于样本均值。下面给出推导。

设前 $n$ 个样本的均值为 $q_n = \frac{1}{n}\sum_{i=1}^{n} f(x_i)$。将求和拆成前 $n-1$ 项与第 $n$ 项：

$$\begin{aligned} q_n &= \frac{1}{n}\sum_{i=1}^{n} f(x_i) = \frac{1}{n}\left[(n-1)\,q_{n-1} + f(x_n)\right] \\[6pt] &= q_{n-1} + \frac{1}{n}\left[f(x_n) - q_{n-1}\right] \tag{1.5-3} \end{aligned}$$

等价地写成加权平均：$q_n = (1 - \frac{1}{n})\,q_{n-1} + \frac{1}{n}\,f(x_n)$。每来一个新样本，只需一次加法和一次乘法就能更新估计值，无需保存任何历史数据。

Robbins-Monro 条件

将步长 $1/n$ 推广为一般的步长序列 $\alpha_n$，得到更灵活的更新格式：

$$q_n = q_{n-1} + \alpha_n\left[f(x_n) - q_{n-1}\right] \tag{1.5-4}$$

这就是随机逼近理论中 Robbins-Monro 算法的基本形式。要保证公式(1.5-4)收敛到真实期望值 $\mathbb{E}[f(X)]$，步长序列 $\{\alpha_n\}$ 需要同时满足两个条件：

  1. $\displaystyle\sum_{n=1}^{\infty} \alpha_n = \infty$，保证估计值能到达任意远的目标
  2. $\displaystyle\sum_{n=1}^{\infty} \alpha_n^2 < \infty$，保证更新的方差逐渐趋于零

$\alpha_n = 1/n$ 同时满足这两个条件（调和级数发散，而 $\sum 1/n^2 = \pi^2/6$ 收敛）。固定步长 $\alpha_n = \alpha$（常数）只满足第一个条件，不会精确收敛到期望值，但它对近期样本赋予更大的权重，能够跟踪非平稳的目标，因此在强化学习中反而更常用。
