9.2 公交串车控制

公交线路运营中最棘手的问题之一是串车（bus bunching）：原本应当均匀分布在线路上的公交车逐渐聚集成簇，导致部分乘客长时间等车，另一部分乘客却看到两三辆车接踵而至。这一现象的根源是一个正反馈循环，而强化学习为打破这一循环提供了新的思路。本节将介绍如何将公交串车控制建模为 MDP，重点讨论两个建模难点：异步决策和图神经网络的引入。

公交串车现象

考虑一条环形公交线路上均匀分布的 $N$ 辆公交车。在理想情况下，相邻车辆之间的车头时距（headway）相等，乘客的平均等待时间最短。然而，这种均匀状态是不稳定的。假设某辆车因红灯、乘客上下车缓慢等原因略有延误，一个自我强化的恶性循环就会启动。

设第 $i$ 辆车与前车的车头时距为 $h_i$。当 $h_i$ 增大时，站台上积累的等候乘客更多，上下车时间（dwell time）更长，进而使 $h_i$ 继续增大。同时，紧随其后的第 $i+1$ 辆车发现站台上几乎没有乘客（前车刚走），停站时间很短，速度加快，$h_{i+1}$ 不断缩小。最终，第 $i+1$ 辆车追上第 $i$ 辆车，形成串车。

这一正反馈机制可以简单表述为：

$$h_i \uparrow \;\Longrightarrow\; \text{站台等候人数} \uparrow \;\Longrightarrow\; \text{停站时间} \uparrow \;\Longrightarrow\; h_i \uparrow\uparrow \tag{9.2-1}$$

图 9.2-1 展示了这一过程。上半部分是均匀分布的理想状态，下半部分是串车发生后的状态。

图 9.2-1　公交串车现象。上方为理想的均匀分布状态，车头时距 $h$ 接近目标值 $h^*$；下方为串车状态，车辆两两聚集，间距极不均匀。黄色圆点表示站台等候的乘客。

串车的后果是双重的。对乘客而言，等待时间的方差急剧增大：在大间距区间内等车的乘客等待时间远超平均值，而串车区间内的乘客虽然等待时间短，但车辆几乎是空驶。对运营方而言，串车意味着运力浪费，因为两辆紧挨的车实际上只发挥了一辆车的作用。

控制手段

运营者可以采取多种手段来干预串车。最常见的三种是：

  • 站点驻留（holding）：让到站的公交车在站点额外等待一段时间再出发，以拉开与后车的距离。这是最常用的控制手段，因为它不影响已上车乘客的目的地。
  • 跳站（skip-stopping）：让公交车跳过某些站点不停靠，以加速追赶目标车头时距。代价是被跳过站点的乘客需要等待下一辆车。
  • 速度调节（speed adjustment）：在站间区段调整车速。比驻留更平滑，但受限于交通条件和安全约束。

本节重点讨论站点驻留策略。传统做法是基于阈值的规则，例如：若实际车头时距 $h_i$ 超过目标时距 $h^*$ 的 $1.5$ 倍，则驻留 $\delta$ 秒。这种规则简单易实现，但有两个根本缺陷。

第一，阈值规则只关注局部信息，即当前车辆与前车的时距，而忽略了线路上其他车辆的状态。控制一辆车的行为会级联地影响所有后续车辆，局部规则无法协调全局。第二，固定阈值缺乏适应性，无法针对不同的客流模式和线路条件做出灵活调整。

MDP 建模

将公交串车控制建模为 MDP，需要明确五个要素。设线路上有 $N$ 辆公交车，编号为 $1, 2, \ldots, N$，共 $M$ 个站点。

MDP 五要素
  • 状态：当某辆车 $i$ 到达站点时，系统状态包括所有 $N$ 辆车的位置（当前所在站点或站间位置）、车头时距向量 $\mathbf{h} = (h_1, \ldots, h_N)$、各车的车载乘客数，以及当前站点的等候乘客数。
  • 动作：到站车辆 $i$ 的驻留时间 $a_i \in [0, a_{\max}]$。可以将连续动作离散化为 $\{0, 10, 20, \ldots, 60\}$ 秒。
  • 转移：驻留 $a_i$ 秒后，车辆 $i$ 出发驶向下一站。站间行驶时间受随机交通条件影响，到达下一站的时间为 $\tau_i + a_i + d_i + t_{\text{travel}}$，其中 $d_i$ 为上下客时间，$t_{\text{travel}}$ 为随机行驶时间。
  • 奖励：负的车头时距方差（鼓励均匀分布）。具体地，每次决策时的即时奖励为：
$$R = -\sum_{i=1}^{N} (h_i - \bar{h})^2 \tag{9.2-2}$$

其中 $\bar{h} = \frac{1}{N}\sum_{i=1}^{N} h_i$ 是当前平均车头时距。这个奖励函数的含义清晰：车头时距越均匀，奖励越高（越接近零）。

另一种常见的奖励设计是直接使用乘客等待时间：

$$R = -\sum_{j \in \mathcal{S}} w_j \tag{9.2-3}$$

其中 $\mathcal{S}$ 是所有站点的集合，$w_j$ 是站点 $j$ 的乘客平均等待时间。两种奖励高度相关，因为均匀的车头时距自然对应较短的乘客等待时间。公式 (9.2-2) 在实现上更方便，因为只需要车辆位置信息就能计算。

异步决策建模

公交串车控制的一个核心建模难点在于决策时机不同步。在 6.4 节中，我们讨论了事件驱动与时间驱动两种决策模式。公交控制天然适合事件驱动模式：每当一辆车到达某个站点，就需要为它决定驻留时间。不同车辆到达站点的时刻各不相同，因此决策是异步发生的。

这与许多标准多智能体 RL 设定不同。在交通信号控制、棋盘游戏等场景中，所有智能体在同一时刻做出决策。而在公交控制中，任何一个时刻通常只有一辆车需要做决策。

同步决策

所有智能体在相同时间步 $t = 0, 1, 2, \ldots$ 同时选择动作。状态转移后，所有智能体同时观测新状态。
适用场景：交通信号控制、多人棋盘游戏。

异步决策

不同智能体在不同时刻做出决策，由各自的事件触发。MDP 的时间步 $k$ 对应第 $k$ 次决策事件，而非物理时钟。
适用场景：公交控制、电梯调度、分布式任务分配。

为了在标准 RL 框架中处理异步决策，我们采用事件驱动的时间步定义。设 $e_k$ 表示第 $k$ 个决策事件，对应某辆车 $i_k$ 到达某个站点 $m_k$。MDP 的时间步 $k = 0, 1, 2, \ldots$ 对应的是这些事件的序列，而非均匀的物理时间。在事件 $e_k$ 发生时：

  1. 观测全局状态 $S_k$，包含所有 $N$ 辆车的位置和车头时距。
  2. 为到站车辆 $i_k$ 选择驻留时间 $a_k$。
  3. 仿真推进到下一个事件 $e_{k+1}$（即下一辆车到达某站点），得到新状态 $S_{k+1}$ 和奖励 $R_{k+1}$。

关键观察是：虽然每次只控制一辆车，但状态 $S_k$ 包含所有车辆的信息。这使得策略可以根据全局态势做出决策，例如当前车与前后车的时距不均衡时采取不同的驻留策略。

与 6.4 节的联系   6.4 节指出，事件驱动模式下相邻时间步的物理间隔不等距，这会影响折扣因子 $\gamma$ 的含义。在公交控制中，一种处理方式是使用基于物理时间的折扣 $\gamma^{\Delta t_k}$，其中 $\Delta t_k$ 是事件 $e_k$ 与 $e_{k+1}$ 之间的物理时间间隔。另一种更简单的做法是将一个运营周期（如早高峰 2 小时）定义为一个回合，在回合内不折扣（$\gamma = 1$），直接最小化累积的车头时距方差。

图神经网络建模

确定了 MDP 的状态和动作后，下一个问题是如何设计策略网络。一个简单的做法是将所有车辆的状态拼接成一个长向量，输入全连接网络（MLP）。这种做法有两个问题：输入维度随车辆数 $N$ 线性增长，使得网络难以迁移到不同规模的车队；而且全连接网络无法捕捉相邻车辆之间的空间关系。

公交线路上的车辆具有天然的图结构：每辆车是一个节点，相邻车辆之间有边。这正是 4.4 节介绍的图神经网络（GNN）的用武之地。

图的构建

将线路上的 $N$ 辆车构造为一个有向图 $\mathcal{G} = (\mathcal{V}, \mathcal{E})$。节点集 $\mathcal{V} = \{1, 2, \ldots, N\}$ 对应每辆车，边集 $\mathcal{E}$ 连接相邻车辆。对于环形线路，每辆车 $i$ 的邻居为前车 $i-1$ 和后车 $i+1$（模 $N$）。

每个节点 $i$ 的特征向量为：

$$\mathbf{x}_i = \big[\, p_i,\; h_i^{\text{front}},\; h_i^{\text{rear}},\; l_i,\; q_i,\; \mathbb{1}[i = i_k] \,\big]^\top \tag{9.2-4}$$

其中 $p_i$ 是车辆 $i$ 在线路上的归一化位置（$0$ 到 $1$），$h_i^{\text{front}}$ 和 $h_i^{\text{rear}}$ 分别是与前车和后车的车头时距，$l_i$ 是车载乘客数，$q_i$ 是最近经过站点的等候人数，$\mathbb{1}[i = i_k]$ 是指示变量，标记当前需要做决策的车辆。

消息传递机制

GNN 通过多轮消息传递（message passing）让每个节点聚合邻居的信息。第 $l$ 层的更新规则为：

$$\mathbf{h}_i^{(l+1)} = \sigma\!\left( \mathbf{W}^{(l)} \mathbf{h}_i^{(l)} + \sum_{j \in \mathcal{N}(i)} \mathbf{U}^{(l)} \mathbf{h}_j^{(l)} + \mathbf{b}^{(l)} \right) \tag{9.2-5}$$

其中 $\mathbf{h}_i^{(0)} = \mathbf{x}_i$ 是节点 $i$ 的初始特征，$\mathcal{N}(i) = \{i-1, i+1\}$ 是车辆 $i$ 的邻居集合，$\mathbf{W}^{(l)}$ 和 $\mathbf{U}^{(l)}$ 分别是自连接和邻居连接的可学习权重矩阵，$\sigma$ 是激活函数（如 ReLU）。

经过 $L$ 层消息传递后，需要做决策的车辆 $i_k$ 的节点表示 $\mathbf{h}_{i_k}^{(L)}$ 已经融合了邻居（甚至更远车辆）的信息。最终，将 $\mathbf{h}_{i_k}^{(L)}$ 输入一个输出头来得到动作：

$$a_k = f_{\text{out}}\!\left(\mathbf{h}_{i_k}^{(L)}\right) \tag{9.2-6}$$

对于离散动作空间，$f_{\text{out}}$ 输出各驻留时间的概率分布；对于连续动作空间，$f_{\text{out}}$ 输出驻留时间的均值和方差。

图 9.2-2 展示了这一 GNN 策略网络的结构。

图 9.2-2　基于 GNN 的策略网络结构。每辆公交车为一个节点，经过两层消息传递后，决策车辆（红色节点，Bus 3）的表示融合了邻居信息，最终输出驻留时间。紫色虚线表示邻居节点向决策节点传递的消息。

GNN 的优势

与全连接网络相比，GNN 策略有三个显著优势：

  • 参数共享。公式 (9.2-5) 中的权重 $\mathbf{W}^{(l)}$ 和 $\mathbf{U}^{(l)}$ 在所有节点间共享。无论线路上有 6 辆车还是 20 辆车，模型参数量不变。
  • 规模泛化。训练时使用 $N = 8$ 辆车的场景，部署时可以直接应用于 $N = 12$ 辆车的线路，因为 GNN 不依赖于固定的输入维度。
  • 归纳偏置。消息传递机制天然编码了公交控制的核心结构：一辆车的最优驻留决策主要取决于与相邻车辆的关系，而不是某辆远处车辆的状态。多层 GNN 可以将信息传播范围逐步扩大，$L$ 层 GNN 让每个节点看到 $L$ 跳以内的邻居信息。
为什么不用全连接网络

假设一个全连接策略网络接受所有 $N$ 辆车的特征拼接作为输入。当 $N$ 从 $8$ 变为 $12$ 时，输入维度从 $8 \times 6 = 48$ 变为 $12 \times 6 = 72$，网络结构必须重新设计和训练。更根本的问题是，全连接网络对输入顺序敏感：将车辆 $1$ 和车辆 $2$ 的特征交换位置会产生不同的输出，但在公交控制中，重要的是车辆之间的相对关系，而非绝对排列顺序。GNN 通过邻居聚合自然地捕捉这种相对关系。

训练流程

将上述 MDP 建模和 GNN 策略网络结合起来，整体训练流程如下。我们使用 PPO 算法（5.2 节），因为它在连续和离散动作空间中都有稳定的表现。

算法 9.2-1　事件驱动的公交控制 RL 训练

输入：公交仿真器 $\text{Env}$，GNN 策略网络 $\pi_\theta$，价值网络 $V_\phi$，回合数 $E$

  1. for 回合 $e = 1, 2, \ldots, E$ do
  2.  初始化仿真器，$N$ 辆车均匀分布在线路上
  3.  $\mathcal{D} \leftarrow \emptyset$ // 经验缓冲区
  4.  while 运营周期未结束 do
  5.   仿真推进至下一个到站事件 $e_k$：车辆 $i_k$ 到达站点 $m_k$
  6.   构建图 $\mathcal{G}_k$，计算所有节点特征 $\{\mathbf{x}_i\}_{i=1}^N$
  7.   通过 GNN 得到 $a_k \sim \pi_\theta(\cdot \mid \mathcal{G}_k, i_k)$ // 仅为到站车辆选择动作
  8.   执行驻留 $a_k$ 秒，观测奖励 $R_{k+1}$ 和下一状态
  9.   $\mathcal{D} \leftarrow \mathcal{D} \cup \{(S_k, a_k, R_{k+1}, S_{k+1})\}$
  10.  end while
  11.  用 $\mathcal{D}$ 中的数据，按 PPO 更新 $\theta$ 和 $\phi$
  12. end for

算法的核心在第 5-9 行。与标准 RL 训练循环不同的是，仿真器不是按固定时间步推进，而是跳跃到下一个到站事件。两个事件之间可能间隔几十秒到几分钟，在此期间仿真器内部按小时间步推进车辆移动和乘客到达，但不触发 RL 决策。这种事件驱动的仿真方式显著减少了决策步数，提高了训练效率。

  • 无控制 vs. 阈值规则。即使是简单的阈值规则，也能将车头时距标准差降低约 47%。这说明串车问题的核心不在于缺乏复杂算法，而在于是否进行干预。
  • 阈值规则 vs. RL 策略。RL 策略（无论 MLP 还是 GNN）进一步将标准差降低了 37%--44%。改善主要来自对全局状态的利用：RL 策略能够根据前后车的距离动态调整驻留时间，而非套用固定规则。
  • MLP vs. GNN。在相同车队规模下，GNN 的性能优势不算突出（$55$ vs. $62$）。GNN 的核心优势在最后一列：当车队规模变化时，GNN 策略可以零迁移地应用，而 MLP 需要重新训练。
关于数值的说明   上表中的数值来自一个简化的仿真场景，用于说明不同策略的相对表现趋势，而非某一特定城市线路的绝对性能。实际效果受线路长度、站点间距、客流模式、交通信号等因素影响，需要针对具体场景进行仿真验证。

小结

  1. 公交串车源于正反馈循环：延误导致更多乘客积压，进一步加剧延误。站点驻留是最常用的控制手段。
  2. 公交控制的 MDP 建模需要处理异步决策：不同车辆在不同时刻到达站点。事件驱动的时间步定义（6.4 节）自然地适配这一特点，每个时间步对应一次到站事件。
  3. 线路上车辆的天然图结构使得 GNN 成为理想的策略网络架构。GNN 通过消息传递机制（公式 (9.2-5)）让每辆车融合邻居信息，且参数量不随车队规模增长。
  4. GNN 策略的核心优势在于规模泛化：在一种车队规模下训练的策略可以直接部署到不同规模的线路上，这在实际运营中具有重要价值。
