10.3 离线强化学习

第五章介绍的 DQN、PPO、SAC 等深度强化学习算法都有一个共同的前提：智能体可以在训练过程中持续与环境交互，每执行一步动作就立即观察到反馈，再据此更新策略。这种模式称为在线强化学习（online RL）。然而，在许多管理科学场景中，这个前提并不成立。本节介绍一种不同的范式：离线强化学习（offline RL），也称为批量强化学习（batch RL），它允许智能体仅从一份固定的历史数据集中学习策略，无需与环境做任何额外交互。

为什么管理科学需要离线强化学习

在线强化学习的核心循环是"执行动作 → 观察反馈 → 更新策略 → 执行新动作"。这个循环在仿真环境中运行得很好，但当我们考虑真实的管理系统时，问题立刻显现。

  • 在线试错的代价不可接受。一家航空公司不可能让一个初始随机的定价策略在真实航班上进行数千次探索性定价。一次定价失误可能导致数百万元的收入损失，或者严重损害品牌信誉。
  • 安全约束不容违反。医院的手术排程系统不允许"探索"一下把急诊手术延后会怎样。交通信号控制系统的随机探索可能导致严重的交通拥堵甚至事故。
  • 反馈周期长。供应链优化中，一个补货决策的效果可能在几周后才能完整显现。如果依赖在线交互，训练一个有效策略可能需要数年的运营时间。

与此同时，管理科学中一个常被忽视的优势是：许多组织已经积累了大量的历史运营数据。

  • 网约车平台记录了每一次派单决策、乘客等待时间、司机轨迹；
  • 零售商的 ERP 系统保存了每天的库存水平、补货数量、实际需求和缺货记录；
  • 医院的 HIS 系统中有多年的手术排程、床位分配、患者等待时间数据；
  • 航空公司的收益管理系统记录了每一次定价决策和对应的上座率。

这些数据就是一份份完整的"状态-动作-奖励-下一状态"转移记录。离线强化学习的目标就是：仅从这些历史数据中学出比当前运营策略更好的决策策略，而不需要在真实系统上做任何在线试验。

问题设定

离线强化学习的设定非常简洁。假设某个管理系统在过去一段时间内按照某个行为策略（behavior policy）$\beta(a \mid s)$ 运行，产生了一批历史数据。这个行为策略可以是人类专家的操作规则、某个启发式算法，或者多个不同决策者的混合行为。我们将这些数据收集为一个静态数据集：

$$\mathcal{D} = \big\{(s_i, a_i, r_i, s_i')\big\}_{i=1}^{N} \tag{10.3-1}$$

其中每条记录 $(s_i, a_i, r_i, s_i')$ 表示：在状态 $s_i$ 下执行了动作 $a_i$，获得了奖励 $r_i$，环境转移到了下一状态 $s_i'$。

离线强化学习的目标

给定一个固定的数据集 $\mathcal{D}$（在学习过程中不再增长），找到一个策略 $\pi$ 使得期望累积折扣回报 $J(\pi) = \mathbb{E}_\pi[\sum_{t=0}^{\infty} \gamma^t R_t]$ 最大化。学习过程中不允许与环境做任何额外的交互。

这个设定与第五章的在线算法最大的区别在于：DQN 可以不断地 $\varepsilon$-greedy 采样新数据并存入经验回放缓冲区；SAC 每一步都向缓冲区存入新的转移。而在离线设定下，缓冲区就是 $\mathcal{D}$，容量固定、永远不会有新数据加入。

行为策略不需要已知。我们只需要数据集 $\mathcal{D}$ 本身，不需要知道 $\beta(a \mid s)$ 的显式形式。实际中，$\beta$ 可能是多个调度员在不同时期的混合操作记录，根本不存在一个统一的数学表达。

分布偏移：离线学习的核心困难

初看之下，离线 RL 似乎很简单：3.1 节的 DQN 本来就从经验回放缓冲区中采样来训练，如果我们把整个历史数据集当作一个"预填充好的缓冲区"，直接运行 DQN 的训练循环不就行了吗？

答案是：不行。直接将在线算法应用于离线数据几乎总是会失败，问题的根源在于分布偏移（distribution shift），也称为外推误差（extrapolation error）。

问题的数学描述

考虑 DQN 的 TD 目标（公式 3.1-5）：

$$y = r + \gamma \max_{a'} Q(s', a'; \theta^-)$$

关键在于 $\max_{a'}$ 这一步。它会选择 $Q$ 网络认为在 $s'$ 下最好的动作 $a^* = \arg\max_{a'} Q(s', a'; \theta^-)$。但这个 $a^*$ 很可能是数据集 $\mathcal{D}$ 中从未出现过的动作。

在线 RL 中，这不是问题：如果 $a^*$ 的 Q 值被高估了，智能体会在后续的交互中真正执行 $a^*$，获得真实奖励，从而纠正这个估计偏差。但在离线设定下，智能体永远不会去执行 $a^*$，因此没有任何数据可以纠正这个高估。

外推误差的恶性循环

在离线数据中，Q 网络对未见过的动作的估计本质上是外推（extrapolation），而非内插（interpolation）。神经网络在训练分布之外的外推没有任何保证——可能严重偏高，也可能严重偏低。当 $\max$ 运算持续选出被高估的动作时，这些虚假的高 Q 值通过贝尔曼方程向前传播，导致整个 Q 函数的估计越来越不可靠。这就是所谓的外推误差累积。

一个具体的例子

假设我们有一个库存管理系统的历史数据。数据中记录了调度员的补货决策，补货量通常在 $[15, 30]$ 之间。现在用 DQN 在这份数据上训练 Q 网络。

训练初期，Q 网络对补货量 $a = 45$（数据中从未出现）的 Q 值估计是随机初始化的值。由于没有训练样本对这个 (状态, 动作) 组合施加约束，$Q(s, 45; \theta)$ 的值纯粹由网络的随机权重和泛化特性决定。假如恰好 $Q(s, 45; \theta)$ 比数据中出现的任何动作的 Q 值都高，$\max$ 运算就会选择 $a' = 45$ 作为 TD 目标中的最优动作。

这个虚假的高 Q 值随后被用来更新其他状态的 Q 估计，进一步传播到更多的状态-动作对。几轮更新后，Q 网络可能对一系列从未在数据中出现的动作都给出了极高的 Q 值。最终学到的策略会选择这些"虚高"的动作，在实际部署时表现很差。

图 10.3-1　分布偏移的示意。蓝色区域是数据覆盖的动作范围，蓝色实线是 Q 网络在该范围内的估计（接近真实值 $Q^*$）。红色实线是 Q 网络在数据范围之外的外推，由于缺乏训练数据的约束，Q 值可能远高于真实值。$\arg\max$ 运算选中了外推区域中被严重高估的动作。

行为克隆：最简单的基线

在介绍专门的离线 RL 算法之前，先看一个最朴素的方案：直接模仿数据中的行为策略，完全不做值函数估计。这就是行为克隆（Behavior Cloning, BC）。

行为克隆将离线 RL 问题简化为一个标准的监督学习问题：用数据集 $\mathcal{D}$ 中的 $(s_i, a_i)$ 对作为训练样本，学一个策略网络 $\pi_\theta(a \mid s)$ 去拟合行为策略的动作选择。

对于离散动作空间，行为克隆的损失函数是交叉熵：

$$L_{\text{BC}}(\theta) = -\frac{1}{N}\sum_{i=1}^{N} \log \pi_\theta(a_i \mid s_i) \tag{10.3-2}$$

对于连续动作空间（如高斯策略），损失函数是负对数似然或均方误差：

$$L_{\text{BC}}(\theta) = \frac{1}{N}\sum_{i=1}^{N} \big\|a_i - \mu_\theta(s_i)\big\|^2 \tag{10.3-3}$$

行为克隆完全不涉及贝尔曼方程或 TD 学习，因此不受外推误差的影响。它的实现极其简单，训练也非常稳定。

行为克隆的适用条件。如果数据集中的行为策略 $\beta$ 本身已经相当好（例如经验丰富的调度员的操作记录），行为克隆可以有效地将人类专家的知识"蒸馏"到一个神经网络中，使其能够自动执行类似的决策。在一些管理科学应用中，行为克隆就足以提供有价值的自动化决策。

行为克隆的根本局限也很明显：它的性能无法超越数据中的行为策略。学到的策略充其量是行为策略的一个近似复制，如果行为策略本身是次优的，行为克隆也只能学到一个次优策略。离线 RL 的真正价值在于：利用数据中不同决策者的经验，"拼接"出一个比任何单一决策者都更好的策略，而这需要进行值函数估计和策略优化。

方法目标能否超越行为策略外推误差风险
行为克隆模仿 $\beta$不能无
在线 RL（DQN/SAC）直接应用最大化回报理论上能，实际常失败严重
离线 RL（CQL 等）保守地最大化回报能受控

保守Q学习（Conservative Q-Learning）

既然分布偏移的本质是 Q 值在数据范围之外被高估，一个自然的思路是：在训练 Q 网络时，显式地压低那些不在数据中的动作的 Q 值。这就是 Kumar 等人（2020）提出的保守 Q 学习（Conservative Q-Learning, CQL）的核心思想。

核心思想：学习 Q 值的下界

CQL 在标准的 TD 损失之上添加了一个正则化项，它惩罚 Q 网络对当前策略所选动作的 Q 值，同时鼓励 Q 网络对数据中实际出现的动作给出较高的 Q 值。

具体而言，CQL 的损失函数由两部分组成：

$$L_{\text{CQL}}(\theta) = \underbrace{\alpha_{\text{cql}} \left(\mathbb{E}_{s \sim \mathcal{D}}\!\left[\log \sum_{a} \exp\big(Q_\theta(s, a)\big)\right] - \mathbb{E}_{(s,a) \sim \mathcal{D}}\!\left[Q_\theta(s, a)\right]\right)}_{\text{CQL 正则化项}} + \underbrace{\frac{1}{2}\,\mathbb{E}_{(s,a,r,s') \sim \mathcal{D}}\!\left[\big(Q_\theta(s, a) - y\big)^2\right]}_{\text{标准 TD 损失}} \tag{10.3-4}$$

其中 $y = r + \gamma \max_{a'} Q_{\theta^-}(s', a')$ 是通常的 TD 目标，$\alpha_{\text{cql}} > 0$ 控制保守程度。

正则化项的直觉

CQL 正则化项的第一部分 $\log \sum_{a} \exp(Q_\theta(s, a))$ 是所有动作 Q 值的"软最大值"（log-sum-exp），它是 $\max_a Q_\theta(s, a)$ 的平滑上界。最小化这一项会压低所有动作的 Q 值，尤其是那些 Q 值偏高的动作。

第二部分 $\mathbb{E}_{(s,a) \sim \mathcal{D}}[Q_\theta(s, a)]$ 是数据集中实际出现的 (状态, 动作) 对的平均 Q 值。最大化这一项（因为整体是最小化，所以它前面有负号）会拉高数据中动作的 Q 值。

两者结合的净效果是：

  • 数据中出现过的动作：Q 值基本保持准确（被拉高的力和被压低的力大致平衡）；
  • 数据中未出现的动作：Q 值被系统性地压低（只有压低的力，没有拉高的力）。
CQL 学习的是 Q 值的下界

可以证明（Kumar et al., 2020），在一定条件下，CQL 学到的 Q 函数满足：

$$\mathbb{E}_\pi\big[Q_\theta(s, a)\big] \leq Q^\pi(s, a) \tag{10.3-5}$$

即对于学到的策略 $\pi$ 选出的动作，CQL 的 Q 值估计是真实 Q 值的下界。这意味着 CQL 对策略性能的评估是保守的（pessimistic）：如果 CQL 认为某个策略好，那它实际上至少和 CQL 评估的一样好，甚至更好。这种保守性正是离线 RL 所需要的安全保障。

连续动作空间中的 CQL

公式(10.3-4)中的 $\log \sum_a \exp(Q(s,a))$ 要求对所有动作求和，这对离散动作空间是可行的，但对连续动作空间则需要用采样来近似。实践中常用的做法是：从当前策略 $\pi_\theta$ 和均匀分布中各采样一批动作，用这些样本来近似 log-sum-exp 项。具体而言，采样 $M$ 个动作 $\{a^{(j)}\}_{j=1}^M$（一半来自当前策略，一半来自均匀分布），计算：

$$\log \sum_{a} \exp\big(Q_\theta(s, a)\big) \approx \log \frac{1}{M}\sum_{j=1}^{M} \exp\big(Q_\theta(s, a^{(j)})\big) \tag{10.3-6}$$

CQL 算法

CQL 可以建立在 DQN 或 SAC 的架构之上，唯一的改动是在损失函数中添加正则化项。以基于 SAC 框架的 CQL 为例（这是连续动作空间中最常用的组合），完整算法如下：

算法：CQL（基于 SAC 框架）
  1. 初始化策略网络 $\pi_\phi$、Q 网络 $Q_{\theta_1}, Q_{\theta_2}$、目标网络 $Q_{\bar{\theta}_1}, Q_{\bar{\theta}_2}$
  2. 加载离线数据集 $\mathcal{D}$
  3. 对每个训练步：
    1. 从 $\mathcal{D}$ 中随机采样小批量 $\{(s, a, r, s')\}$
    2. 对每个 Q 网络 $i = 1, 2$，计算 CQL 损失： $$L_i = \alpha_{\text{cql}} \left(\mathbb{E}_s\Big[\log \sum_{a'} \exp Q_{\theta_i}(s, a')\Big] - \mathbb{E}_{(s,a)}[Q_{\theta_i}(s, a)]\right) + \frac{1}{2}\,\mathbb{E}\big[(Q_{\theta_i}(s,a) - y)^2\big]$$ 其中 $y = r + \gamma \big(\min_i Q_{\bar{\theta}_i}(s', \tilde{a}') - \alpha \log \pi_\phi(\tilde{a}'|s')\big)$，$\tilde{a}' \sim \pi_\phi(\cdot|s')$
    3. 梯度下降更新 $\theta_1, \theta_2$
    4. 更新策略网络 $\phi$（与 SAC 相同，最大化期望软 Q 值）
    5. 软更新目标网络：$\bar{\theta}_i \leftarrow \tau \theta_i + (1-\tau)\bar{\theta}_i$
与 SAC 的唯一区别在于步骤 (b)：Q 网络的损失中多了 CQL 正则化项。超参数 $\alpha_{\text{cql}}$ 通常取 $1.0$ 到 $5.0$，控制保守程度。

可以看到，CQL 并非一个全新的算法框架，而是一个可以"插入"任何基于值函数的 RL 算法的正则化技术。在 DQN 上加 CQL 正则化就得到 CQL-DQN；在 SAC 上加 CQL 正则化就得到 CQL-SAC。

CQL = 在线 RL 算法 + 保守正则化项
核心改动：在 Q 网络的损失函数中，压低数据之外动作的 Q 值，确保 Q 值估计是下界。
选读：其他离线 RL 方法

CQL 是通过正则化 Q 值来实现保守估计的。除此之外，还有几种不同思路的离线 RL 方法：

  • BCQ（Batch-Constrained Q-learning，Fujimoto et al., 2019）：不直接压低 Q 值，而是限制策略只能选择"与数据中的动作相似"的动作。具体做法是训练一个生成模型（如 VAE）来拟合数据中的动作分布 $\beta(a \mid s)$，然后只在这个分布的支撑集上做 $\max$。
  • IQL（Implicit Q-Learning，Kostrikov et al., 2022）：完全避免在训练中查询数据外的动作。它使用分位数回归来学习 Q 值，使得 TD 目标中只涉及数据集中的动作，从根本上消除了外推误差。
  • Decision Transformer（Chen et al., 2021）：将 RL 问题完全转化为序列建模问题。把轨迹 $(R, s_0, a_0, R-r_0, s_1, a_1, \ldots)$ 当作一个序列，用 Transformer（第 2.3 节）来建模。在推理时，给定一个期望的回报值，Transformer 自回归地生成动作序列。这种方法完全不需要贝尔曼方程或 TD 学习。

在管理科学应用中，CQL 和 IQL 是目前最常用的选择。CQL 的优势在于实现简单（在现有 DQN/SAC 代码上只需添加几行正则化），效果也比较稳定。

例子：从历史调度数据学习派单策略

考虑一个简化的网约车派单场景，延续第一章的三区域设定（市中心 $c$、郊区 $b$、机场 $a$）。假设某平台过去一年的运营数据记录了调度员在不同状态下的派单决策。

数据来源

平台在过去一年使用了一个基于规则的派单策略 $\beta$：

  • 当司机在市中心 $c$ 时：$80\%$ 的概率接单，$20\%$ 的概率空驶去机场
  • 当司机在郊区 $b$ 时：$50\%$ 的概率接单，$50\%$ 的概率空驶去市中心
  • 当司机在机场 $a$ 时：$90\%$ 的概率接单，$10\%$ 的概率空驶去市中心

这个策略是人类调度员根据经验设定的，并非最优。数据集中一共记录了 $N = 50000$ 条转移。

我们的目标是从这 50000 条历史记录中学出一个更好的派单策略，而不需要在真实平台上做任何在线实验。

行为克隆的结果。行为克隆会学到与 $\beta$ 非常接近的策略，在郊区仍然有 $50\%$ 的概率接单。但回顾第一章的分析，我们知道在郊区接单是低效的（接单奖励低且大概率留在郊区），最优策略应该在郊区空驶去市中心。行为克隆无法发现这一点，因为它只是模仿数据中的动作。

直接用 DQN 的结果。如果将 50000 条数据放入经验回放缓冲区，直接运行 DQN 训练循环（但不再采集新数据），Q 网络很可能会对数据中较少出现的 (状态, 动作) 组合给出不可靠的 Q 值。例如，在郊区空驶去机场这个动作在数据中几乎没有出现（$\beta$ 中没有这个选项），DQN 可能对它给出虚假的高 Q 值，导致学到一个荒谬的策略。

CQL 的结果。CQL 会保守地估计 Q 值：对数据中频繁出现的 (状态, 动作) 对给出较准确的 Q 值，对罕见的组合给出偏低的 Q 值。由于数据中确实包含了一些"在郊区空驶去市中心"的转移（$\beta$ 中有 $50\%$ 的概率执行这个动作），CQL 能够可靠地估计这个动作的 Q 值，并发现它优于在郊区接单。最终学到的策略会在郊区更倾向于空驶去市中心，接近最优策略，同时不会对数据中未出现的动作（如在郊区空驶去机场）给出虚假的高评估。

数据覆盖的重要性。注意 CQL 能够发现"在郊区空驶去市中心优于接单"，是因为数据中确实包含了这两种动作的样本。如果行为策略 $\beta$ 在郊区 $100\%$ 接单、从未空驶，那么任何离线 RL 方法都无法可靠地评估空驶动作的价值。离线 RL 的性能取决于数据的覆盖度：数据中覆盖的 (状态, 动作) 对越多样，学到的策略越可能超越行为策略。

离线强化学习的实践建议

数据质量比算法选择更重要

在离线 RL 中，数据集的质量和覆盖度对最终性能的影响远大于算法的选择。以下几点在准备数据时需要特别注意：

  • 动作多样性。数据中应包含足够多样的动作选择。如果所有历史记录都来自同一个固定策略，离线 RL 能学到的信息非常有限。理想情况是数据来自多个不同的决策者或不同时期的策略（例如，不同调度员的操作记录，或策略在不同阶段的调整记录）。
  • 状态覆盖。数据应覆盖目标任务可能遇到的各种状态。如果训练数据只包含正常工作日的记录，但部署时需要处理节假日的特殊需求，学到的策略在节假日可能表现很差。
  • 奖励信号的准确性。在构建离线数据集时，需要确保奖励的计算方式与目标一致。如果历史数据中只记录了收入但没有记录客户满意度，而目标函数需要考虑满意度，就需要在预处理阶段补全这些信息。

离线评估的困难

在线 RL 中，评估一个策略的好坏很简单：让它在环境中跑几个回合，计算平均回报。但在离线设定下，我们无法让策略与环境交互来评估。这个问题称为离策略评估（off-policy evaluation, OPE），它本身就是一个活跃的研究领域。

实践中常用的评估方式有两种：

  • 仿真器评估。如果有一个可靠的仿真器（第七章的内容），可以在仿真器中运行离线学到的策略来评估性能。这是最可靠的方式。
  • 重要性采样评估。利用数据中的轨迹和行为策略 $\beta$ 的已知概率，通过重要性权重来估计新策略 $\pi$ 的期望回报。这种方法的方差通常很大，尤其当 $\pi$ 和 $\beta$ 差异较大时。
不要过度信任 Q 值。在离线 RL 中，Q 网络估计的值（即使使用了 CQL 的保守估计）与策略的真实性能之间可能存在较大差距。不要仅因为 Q 值看起来很高就认为策略一定好。如果条件允许，务必在仿真器中或通过小规模在线实验来验证策略的实际表现。

离线预训练 + 在线微调

在许多管理科学场景中，最实用的方案不是纯粹的离线 RL，而是离线预训练加在线微调（offline-to-online）的两阶段方法：

  1. 离线阶段：在历史数据上用 CQL 等算法训练一个初始策略。这个策略虽然可能不是最优的，但已经是一个合理的起点，远好于随机初始化。
  2. 在线阶段：将离线学到的策略部署到仿真环境（或在安全约束下部署到真实环境），用 SAC 或 PPO 进行少量的在线微调。由于起点已经很好，在线阶段所需的探索量大大减少，训练效率显著提高。

这种两阶段方法结合了离线 RL 的安全性（初始策略不是随机的）和在线 RL 的适应性（可以根据实际环境反馈继续优化），是将 RL 应用于实际管理系统时最推荐的路径。
